这道题是力扣 2421 题“好路径的数目”，属于高频困难题，常出现在模拟赛的中后位置。

### 1. 考察算法类型

**并查集 (DSU) + 贪心/排序**。

核心难点在于路径上的所有节点值必须 $\le$ 端点值。这暗示我们需要按节点值**从小到大**的顺序动态地将节点和边加入图中。

### 2. 最优解思路

- **按值处理**：将节点按数值从小到大分组。当我们处理值为 $v$ 的节点时，图中只存在数值 $\le v$ 的节点和连接它们的边。
    
- **动态连通**：对于值为 $v$ 的每个节点，将其与其所有数值 $\le v$ 的邻居合并（使用并查集）。
    
- **组合计数**：在当前连通状态下，如果一个连通分量中有 $k$ 个值为 $v$ 的节点，那么这 $k$ 个点之间两两构成的路径（共 $\binom{k}{2} = \frac{k(k-1)}{2}$ 条）都满足“路径中间节点 $\le v$”的条件。
    
- **结果汇总**：总路径数 = 节点总数 $n$（每个单节点路径） + 各阶段产生的组合路径数。
    

### 3. Python 3.8.6 代码实现
```Python
from typing import List
import collections

class Solution:
    def numberOfGoodPaths(self, vals: List[int], edges: List[List[int]]) -> int:
        n = len(vals)
        # 建邻接表
        adj = [[] for _ in range(n)]
        for u, v in edges:
            adj[u].append(v)
            adj[v].append(u)
            
        # 按节点值对索引进行分组
        val_to_nodes = collections.defaultdict(list)
        for i, v in enumerate(vals):
            val_to_nodes[v].append(i)
            #{ 1: [0, 3], 2: [2], 3: [1, 4] } 类似
            
        # 并查集：使用路径压缩的迭代实现，适配深树
        parent = list(range(n))
        def find(i):
            root = i
            while parent[root] != root:
                root = parent[root] # 直到找到他是他自己 这个时候就找到了根节点
            while parent[i] != root:
                parent[i], i = root, parent[i]  # i跳到旧父亲 
            return root

        ans = n # 初始包含 n 个单节点路径
        # 按节点值从小到大遍历（关键：保证路径中间节点值不大于端点）
        for v in sorted(val_to_nodes.keys()):
            for u in val_to_nodes[v]:
                for neighbor in adj[u]:
                    # 贪心策略：只连接已经处理过（值较小）或当前处理的节点
                    if vals[neighbor] <= v:
                        root_u, root_nb = find(u), find(neighbor)
                        if root_u != root_nb:
                            parent[root_u] = root_nb
            
            # 统计当前值 v 在各个连通分量中的分布
            group_count = collections.defaultdict(int)
            for u in val_to_nodes[v]:
                group_count[find(u)] += 1
            
            # 对于每个分量，若有 k 个值为 v 的点，新增 k*(k-1)//2 条路径
            for k in group_count.values():
                ans += k * (k - 1) // 2
                
        return ans
```

### 复杂度分析

- **时间复杂度**：$O(n \log n + n \alpha(n))$。其中 $n \log n$ 主要是对不同节点值排序的开销，$\alpha(n)$ 是并查集的操作复杂度。
    
- **空间复杂度**：$O(n)$。用于存储邻接表、并查集数组以及节点分组。
