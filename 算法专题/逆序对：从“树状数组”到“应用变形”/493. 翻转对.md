这道题目是“逆序对”问题的进阶变体。在算法竞赛中，这类涉及区间统计且带有比较条件的问题，通常有两类标准解法：**分治（归并排序变形）或树状数组（Fenwick Tree）**。

### 题目分析

**考察算法类型**：**分治算法 (Divide and Conquer)**，具体为 **归并排序 (Merge Sort)** 的扩展应用。

- **标准逆序对**：$i < j$ 且 $nums[i] > nums[j]$。
    
- **本题重要翻转对**：$i < j$ 且 $nums[i] > 2 \cdot nums[j]$。
    

### 最优解思路

1. **分治策略**：将数组递归分为左右两半，直到子数组长度为 1。
    
2. **跨区间统计（核心）**：
    
    - 在合并两个**已经有序**的左子数组 $L$ 和右子数组 $R$ 之前，先统计满足条件的对数。
        
    - **双指针优化**：遍历左数组中的 $i$，在右数组中寻找满足 $nums[i] > 2 \cdot nums[j]$ 的最大下标 $j$。由于 $L$ 和 $R$ 均有序，当 $i$ 向右移动（数值变大）时，$j$ 也只会向右移动，无需回溯。这使得统计过程的时间复杂度为 $O(n)$。
        
3. **排序归并**：统计完成后，执行标准的归并排序步骤，确保上一层递归能继续使用双指针。
    

**复杂度分析**：

- **时间复杂度**：$O(N \log N)$。
    
- **空间复杂度**：$O(N)$，递归栈及归并所需的辅助空间。
    

---

### Python 3.8.6 代码实现

```Python
class Solution:
    def reversePairs(self, nums: List[int]) -> int:
        def solve(l, r):
            if l >= r: return 0
            mid = (l + r) // 2
            # 1. 递归处理左右子区间
            cnt = solve(l, mid) + solve(mid + 1, r)
            
            # 2. 统计跨越左右区间的翻转对 (此时左右区间已各自有序)
            # 使用双指针维护，j 不随 i 循环重置
            j = mid + 1
            for i in range(l, mid + 1):
                while j <= r and nums[i] > 2 * nums[j]:
                    j += 1
                cnt += (j - (mid + 1))
            
            # 3. 归并排序，确保父级递归可见有序数组
            # Python 的 sorted (Timsort) 在处理部分有序数组时极快
            nums[l:r+1] = sorted(nums[l:r+1])
            return cnt
        
        return solve(0, len(nums) - 1)
```

### 关键点拨

- **为什么先统计再归并？** 如果先归并（排序），则原有的索引顺序 $i < j$ 就会被破坏，无法区分哪些数来自左区间，哪些来自右区间。
    
- **性能考量**：虽然手动实现 `merge` 函数能达到严格的 $O(N \log N)$，但在 Python 3.8 中，直接使用内置的 `sorted()`（底层为 C 实现的 Timsort）配合切片赋值，在 LeetCode 的 50,000 数据量下通常比手写 merge 循环更快。

# 树状数组

当然可以！**树状数组（Binary Indexed Tree, BIT）** 是解决这类“动态维护前缀和”问题的另一把利刃。

虽然归并排序在 Python 中因为有内置 C 实现的 `sorted()` 而跑得飞快，但树状数组逻辑更统一，尤其在处理**动态修改+区间查询**时是首选。

---

### 1. 树状数组思路解析

在归并排序中，我们利用的是“有序性”；而在树状数组中，我们利用的是 **“排名（Rank）”**。

1. **离散化（Discretization）**：
    
    由于 $nums[i]$ 的值可能非常大（32位整数），我们无法直接开一个那么大的数组。但题目只关心数字之间的**大小关系**。我们需要统计的是 $nums[i] > 2 \times nums[j]$，所以我们将数组中所有的 $x$ 和 $2x$ 全部收集起来，排序去重，给它们编上“排名”。
    
2. **动态维护**：
    
    我们从左到右遍历数组。对于当前的数字 $x$：
    
    - 先去树状数组里查询：**“已经出现的数字中，有多少个排名比 $2x$ 的排名还要大？”**
        
    - 将查询结果累加到答案。
        
    - 把当前的 $x$ 插入到树状数组中（即对应排名的计数 +1）。
        

---

### 2. Python 3.8.6 代码实现

根据你的要求 ，代码保持精简且符合 LeetCode 模式：

```Python
from typing import List

class Solution:
    def reversePairs(self, nums: List[int]) -> int:
        if not nums: return 0
        
        # 1. 离散化：收集所有可能涉及的数值（x 和 2x）
        # 这样可以保证在查询 rank[2*x] 时一定能找到对应的索引
        all_vals = sorted(list(set(nums + [2 * x for x in nums])))
        rank = {v: i + 1 for i, v in enumerate(all_vals)}
        
        m = len(all_vals)
        bit = [0] * (m + 1)
        
        # 树状数组标准操作
        def update(i: int):
            while i <= m:
                bit[i] += 1
                i += i & -i
        
        def query(i: int) -> int:
            s = 0
            while i > 0:
                s += bit[i]
                i -= i & -i
            return s
        
        res = 0
        for i, x in enumerate(nums):
            # i 是当前已插入树中的数字总数
            # query(rank[2*x]) 是当前已插入的数字中 <= 2*x 的数量
            # 两者相减即为 > 2*x 的数量
            res += i - query(rank[2 * x])
            # 更新树状数组：当前数字 x 出现次数 +1
            update(rank[x])
            
        return res
```

---

### 3. 为什么这样能行？

- **时间复杂度**：离散化 $O(N \log N)$，遍历数组 $N$ 次，每次查询和更新树状数组都是 $O(\log N)$。总复杂度 $O(N \log N)$。
    
- **关键点**：在遍历到 $nums[j]$ 时，树状数组里存储的全部是满足 $i < j$ 条件的 $nums[i]$。我们通过 `i - query(rank[2*x])` 瞬间算出了有多少个旧数字压过了当前的“两倍倍率”。
    

### 4. 两种解法的博弈

| **维度**        | **归并排序 (Merge Sort)**   | **树状数组 (BIT)**       |
| ------------- | ----------------------- | -------------------- |
| **直观程度**      | 稍微抽象（递归）                | 较直观（类似排队计数）          |
| **Python 性能** | **极快**（利用 Timsort 内置优化） | 较慢（纯 Python 循环和函数调用） |
| **适用场景**      | 静态数组、离线统计               | 动态修改、在线查询            |

在蓝桥杯中，如果题目没有限制必须在线处理，**归并排序**往往是 Python 选手的首选，因为它能最大限度压榨内置函数的性能。但掌握树状数组能让你在遇到更复杂的区间更新题目时游刃有余。
