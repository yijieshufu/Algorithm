这是一道非常经典的**并查集（Union-Find）** 与**数论（质因数分解）** 结合的题目。

### 题目分析

题目允许交换的条件是 $gcd(nums[i], nums[j]) > 1$。这意味着，如果两个数有**共同的质因数**，它们就可以直接交换。

由于交换具有**传递性**：

1. 如果 $A$ 与 $B$ 可交换，且 $B$ 与 $C$ 可交换，那么 $A$ 与 $C$ 也可以通过 $B$ 作为中转进行交换。
    
2. 结论：属于同一个“连通分量”的所有数字，可以在它们占据的这些位置上进行**任意排列**。
    

**核心思路：**

- **判断连通性**：我们将每个数字与其所有的质因数建立连接。例如，$15$ 会与 $3$ 和 $5$ 连通。如果另一个数 $9$ 与 $3$ 连通，那么 $15$ 和 $9$ 就在同一个分量里。
    
- **利用并查集**：遍历数组中的每个数，将其与它的所有质因数进行 `union` 操作。
    
- **最终校验**：将原数组排序得到 `sorted_nums`。遍历索引 $i$，如果 $nums[i] \neq sorted\_nums[i]$，则检查这两个数在并查集中是否属于同一个根。如果不属于，说明无法通过交换到达目标位置，返回 `False`。
    

---

### 算法实现

为了效率，我们先用 **埃氏筛（Sieve）** 预处理出每个数的最小质因数（SPF），这样分解质因数的时间复杂度可以降到 $O(\log (\max(nums)))$。

### Python 3.8.6 代码实现
```Python
class Solution:
    def gcdSort(self, nums: List[int]) -> bool:
        max_n = max(nums) + 1
        parent = list(range(max_n))
        
        def find(i):
            if parent[i] == i: return i
            parent[i] = find(parent[i])
            return parent[i]

        def union(i, j):
            root_i, root_j = find(i), find(j)
            if root_i != root_j: parent[root_i] = root_j

        # 1. 埃氏筛预处理最小质因数 (SPF)
        # `spf[x]` 存储的是能够整除数字 x 的 最小质数
        spf = list(range(max_n)) #令每个数的最小质因数都是它自己
        for i in range(2, int(max_n**0.5) + 1):
            if spf[i] == i:
                for j in range(i*i, max_n, i):
                    if spf[j] == j: spf[j] = i

        # 2. 将每个数字与其质因数合并
        for x in nums:
            temp = x
            while temp > 1:
                union(x, spf[temp])
                temp //= spf[temp]

        # 3. 排序并校验位置合法性
        sorted_nums = sorted(nums)
        for i in range(len(nums)):
            if nums[i] != sorted_nums[i]:
                if find(nums[i]) != find(sorted_nums[i]):
                    return False
        return True
```

### 复杂度分析

- **时间复杂度**：$O(M \log \log M + N \log M + N \log N)$。
    
    - $M$ 为数组中的最大值（$10^5$），预处理 SPF 耗时 $O(M \log \log M)$。
        
    - 每个数分解质因数并合并耗时 $O(N \log M)$。
        
    - 排序耗时 $O(N \log N)$。
        
- **空间复杂度**：$O(M)$，主要用于存储并查集的父节点数组和 SPF 数组。
    

这种方法在处理 $3 \times 10^4$ 数据量时非常高效，完全符合 LeetCode 和算法竞赛的时间限制。
