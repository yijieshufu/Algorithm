这道题目是数论中的经典题型，考察的是**质数筛法**以及**区间扫描**。

### 考察算法：数论（埃氏筛 / 线性筛）+ 贪心扫描

1. **预处理质数**：由于 $right$ 的范围是 $10^6$，我们可以使用 **埃氏筛（Sieve of Eratosthenes）** 快速求出 $[0, right]$ 范围内的所有质数。其时间复杂度为 $O(N \log \log N)$，在 $10^6$ 数据量下表现非常优秀。
    
2. **区间过滤**：从筛好的结果中，提取出落在 $[left, right]$ 区间内的所有质数，存入列表。
    
3. **寻找最小间距**：遍历质数列表，计算相邻两个质数的差值。
    
    - **贪心优化**：如果在遍历过程中发现差值为 **1**（只有 $[2, 3]$）或 **2**（孪生质数），由于这是质数间可能存在的最小差值，我们可以直接返回结果，不必继续向后搜索。
        

---

### LeetCode 提交代码 (Python 3.8.6)
```Python
from typing import List

class Solution:
    def closestPrimes(self, left: int, right: int) -> List[int]:
        # 1. 埃氏筛优化预处理
        is_p = [True] * (right + 1)
        is_p[0] = is_p[1] = False
        for i in range(2, int(right**0.5) + 1):
            if is_p[i]:
                # Python 切片赋值优化，比 for 循环更快
                is_p[i*i : right+1 : i] = [False] * len(range(i*i, right+1, i))
        
        # 2. 提取区间内的质数
        primes = [i for i in range(left, right + 1) if is_p[i]]
        
        if len(primes) < 2:
            return [-1, -1]
        
        # 3. 线性扫描找最小差值
        ans = [-1, -1]
        min_diff = float('inf') #正无穷大 -inf 负无穷大
        
        for i in range(len(primes) - 1):
            diff = primes[i+1] - primes[i]
            if diff < min_diff:
                min_diff = diff
                ans = [primes[i], primes[i+1]]
            # 剪枝：如果差值已经是 1 或 2，则不可能有更小的了
            if diff <= 2: 
                break
                
        return ans
```

### 关键点解析

- **复杂度**：时间复杂度约为 $O(R \log \log R)$（筛法）+ $O(\pi(R))$（扫描，$\pi(R)$ 为质数个数），空间复杂度 $O(R)$。这对于 $10^6$ 的数据量来说是秒杀级别的。
    
- **Python 优化**：在筛法中使用了 `is_p[i*i : right+1 : i] = [False] * ...`，这种切片操作是 Python 底层 C 实现的，速度远快于普通循环。
    
- **提前退出**：利用了质数间距的特性，一旦找到差值为 2 的质数对即可停下，这在处理大范围数据时能节省不少时间。
