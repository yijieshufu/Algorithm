### 题目分析

1. **连通性判定**：如果 $gcd(a, b) > 1$，则 $a$ 和 $b$ 连通。这意味着只要两个数共享**至少一个质因数**，它们就在同一个组件里。
    
2. **暴力法的局限**：数组长度为 $2 \times 10^4$，如果两两计算 $gcd$，复杂度为 $O(N^2 \log M)$，在 LeetCode 上会超时。
    
3. **最优解法（并查集 + 质因数分解）**：
    
    - 我们不直接在数字之间连边。
        
    - 我们让每个数字与其所有的**质因数**连边。
        
    - 例如：数字 $6$ 与质数 $2$ 和 $3$ 连通；数字 $15$ 与质数 $3$ 和 $5$ 连通。通过质数 $3$，数字 $6$ 和 $15$ 自动进入同一个连通块。
        
4. **计数**：最后遍历一遍数组，利用 `find` 函数找到每个数字所属的“根”，统计哪个“根”下的数字最多。
    

---

### 算法步骤

1. **预处理 SPF (Smallest Prime Factor)**：用线性筛或埃氏筛预处理出 $1 \dots 10^5$ 每个数的最小质因数，优化质因数分解的速度。
    
2. **并查集维护**：初始化一个能够容纳到 $10^5$ 的并查集。
    
3. **合并逻辑**：对于数组中的每个数 $x$，求出它的所有质因数 $p_1, p_2, \dots$，执行 `union(x, p1)`, `union(x, p2)` 等操作。
    
4. **统计最大值**：使用哈希表（或数组）记录 `find(x)` 出现的频率，返回最高频率。
    

---

### Python 3.8.6 代码实现

```Python
class Solution:
    def largestComponentSize(self, nums: List[int]) -> int:
        max_v = max(nums)
        parent = list(range(max_v + 1))
        
        def find(i):
            if parent[i] == i: return i
            parent[i] = find(parent[i])
            return parent[i]

        def union(i, j):
            root_i, root_j = find(i), find(j)
            if root_i != root_j: parent[root_i] = root_j

        # 1. 埃氏筛预处理最小质因数 (SPF)
        spf = list(range(max_v + 1))
        for i in range(2, int(max_v**0.5) + 1):
            if spf[i] == i:
                for j in range(i*i, max_v + 1, i):
                    if spf[j] == j: spf[j] = i

        # 2. 数字与其质因数合并
        for x in nums:
            temp = x
            while temp > 1:
                p = spf[temp]
                union(x, p)
                while temp % p == 0: # 彻底除尽该质因数
                    temp //= p

        # 3. 统计每个连通块中 nums 元素的个数
        from collections import Counter
        counts = Counter(find(x) for x in nums)
        return max(counts.values())
```

### 关键点解析

- **去重除法**：在 `while temp % p == 0` 这一步，我们把同一个质因数彻底除尽。虽然不写这一步也能对（因为 `union` 重复执行没影响），但写了能减少循环次数。
    
- **并查集规模**：并查集的大小要开到数组中的最大值 `max_v`。
    
- **时间复杂度**：预处理 $O(M \log \log M)$，质因数分解 $O(N \log M)$，最后统计 $O(N \alpha(M))$，总效率极高。
