LeetCode 1510（石子游戏 IV）是博弈论中的一个经典动态规划问题。与之前的 Nim 游戏或除数博弈不同，由于每次可选的石子数是**平方数**（1, 4, 9, ...），其规律并不像简单的取模那样直观，因此需要通过**动态规划（DP）或记忆化搜索**来求解。

---

## 题目分析

### 1. 核心逻辑：寻找必败态

博弈论 DP 的核心思想是：**只要当前玩家能一步走到让对手必败的状态，那么当前玩家就是必胜的。**

- **定义状态**：设 $dp[i]$ 为布尔值，表示当剩下 $i$ 个石子时，先手玩家是否必胜。
    
- **基础情况**：$dp[0] = \text{False}$。因为剩下 0 个石子时，玩家无法进行任何操作（无法找到满足 $k^2 \le 0$ 的正整数 $k$），根据规则该玩家输掉游戏。
    
- **状态转移方程**：对于数量为 $i$ 的石子，我们需要尝试所有可能的平方数 $k^2$（其中 $k^2 \le i$）：
    
    - 如果存在某一个 $k$，使得 $dp[i - k^2] = \text{False}$（即对手拿到 $i - k^2$ 个石子时会输），那么 $dp[i] = \text{True}$。
        
    - 如果对于所有可能的 $k$，$dp[i - k^2]$ 都是 $\text{True}$，说明无论你怎么走，留给对手的都是必胜态，那么 $dp[i] = \text{False}$。
        

---

## 复杂度分析

|**维度**|**复杂度**|**说明**|
|---|---|---|
|**时间复杂度**|$O(n\sqrt{n})$|外层循环 $n$ 次，内层尝试平方数，次数约为 $\sqrt{i}$。|
|**空间复杂度**|$O(n)$|需要一个长度为 $n+1$ 的 DP 数组。|

---

## LeetCode 提交代码 (Python 3)

为了保证执行效率，我们在内层循环中直接判断 $dp$ 状态，一旦找到必败态（False）就立即中断内层循环（`break`）。
```Python
class Solution:
    def winnerSquareGame(self, n: int) -> bool:
        # dp[i] 表示剩下 i 个石子时，先手是否必胜
        dp = [False] * (n + 1)
        
        # 从 1 个石子开始推导到 n
        for i in range(1, n + 1):
            # 尝试所有可能的平方数操作 k*k
            # 只要能让对手面对一个必败态 (dp[i - k*k] == False)，我就能赢
            for k in range(1, int(i**0.5) + 1):
                if not dp[i - k*k]:
                    dp[i] = True
                    break
                    
        return dp[n]
```

### 优化小贴士

在 Python 中，如果 $n$ 非常大（例如接近 $10^5$），上述代码的运行效率已经足够通过 LeetCode。如果追求极致速度，可以预先计算出所有的平方数存入列表，减少循环中开方运算的开销。
