这道题是博弈论中非常经典且具有欺骗性的题目。表面上看需要复杂的动态规划，但通过**归纳法**寻找“必败态”后，你会发现它其实是一个纯粹的数学结论。

---

## 1. [[题目分析：找规律与归纳]]

我们按照蓝桥杯常用的“打表法”来推导前几个数字：

- **$n = 1$**：先手无法找到满足 $0 < x < 1$ 的 $x$，**先手必败**。
    
- **$n = 2$**：先手选 $x = 1$，剩下 $n = 1$ 给后手。后手面对 $1$ 必败，故**先手必胜**。
    
- **$n = 3$**：先手只能选 $x = 1$（$3$ 的唯一约数），剩下 $n = 2$ 给后手。根据上一步可知，面对 $2$ 的人必胜，故**先手必败**。
    
- **$n = 4$**：先手选 $x = 1$，剩下 $n = 3$ 给后手。后手面对 $3$ 必败，故**先手必胜**。
    
- **$n = 5$**：先手只能选 $x = 1$（$5$ 是质数），剩下 $n = 4$ 给后手。后手面对 $4$ 必胜，故**先手必败**。
    

### 核心逻辑总结

1. **偶数是先手的护身符**：如果你拿到的是偶数，你总可以减去 $1$（$1$ 是所有整数的约数），让对方拿到奇数。
    
2. **奇数是先手的陷阱**：如果你拿到的是奇数，你的所有约数必然也都是奇数。奇数减去奇数，结果一定是偶数。也就是说，你只能把偶数拱手让给对方。
    
3. **终局**：由于数字不断减小，最终拿到 $1$ 的人（奇数）会输。
    

---

## 2. 算法结论

- **类型**：博弈论 / 数论
    
- **最优解思路**：直接判断 $n$ 的奇偶性。
    
- **时间复杂度**：$O(1)$
    
- **空间复杂度**：$O(1)$

## 3. Python 可提交代码

按照 LeetCode 的标准类格式编写：



```Python
class Solution:
    def divisorGame(self, n: int) -> bool:
        # 只要起始数字是偶数，Alice 就可以通过策略锁定胜局
        return n % 2 == 0
```

如果是为了蓝桥杯练习，在 IDLE 中快速验证的写法：

```Python
import sys
# 蓝桥杯风格：直接读取输入并判断

    print(int(line.strip()) % 2 == 0)
```

### 教练笔记：为什么要掌握 DP 解法？

虽然这题有 $O(1)$ 的数学解，但蓝桥杯中很多变型题（比如可选的 $x$ 有额外限制）是无法直接找规律的。那时候就需要用到**动态规划（DP）**。

**DP 思路**：

$dp[i]$ 表示当前数字为 $i$ 时，先手是否必胜。

$dp[i] = \text{any}(\text{not } dp[i - x])$，其中 $x$ 是 $i$ 的约数。

即：只要能给对手留下一个“必败态”，我就是“必胜态”。
