### 模运算与逆元 (Modular Arithmetic & Inverse)

#### 1. 为什么不能直接做除法？

在算法题中，答案通常很大，题目会要求结果对 $10^9+7$ 取模。

- **加、减、乘** 都可以直接取模：
    
    - $(A + B) \pmod M = (A\%M + B\%M) \pmod M$
        
    - $(A \times B) \pmod M = (A\%M \times B\%M) \pmod M$
        
- **除法不行！** * 例如：$(10 / 2) \pmod 3 = 5 \pmod 3 = 2$。
    
    - 但如果先取模：$(10\%3) / (2\%3) = 1 / 2 = 0.5$。**结果完全不对！**
        

因此，在计算组合数 $C(n, k) = \frac{n!}{k!(n-k)!}$ 时，我们不能直接除以阶乘，必须引入 **“逆元”**。
#### 2. 什么是逆元 (Modular Inverse)？

在模 $M$ 的世界里，**逆元** 的作用相当于“倒数”。

如果一个数 $b$ 满足 $b \times x \equiv 1 \pmod M$，那么 $x$ 就是 $b$ 的逆元，记作 $b^{-1}$。

> **核心结论：** 在模运算中，**除以一个数 $b$ 等于乘以 $b$ 的逆元**。
>
> $$
> \frac{a}{b} \pmod M = (a \times b^{-1}) \pmod M
> $$

#### 3. 如何求逆元？（代码中的黑科技）

代码里用了两种方法来求逆元，这都是你需要掌握的基本功：

##### 方法 A：费马小定理 (快速幂)

如果 $M$ 是质数（比如 $10^9+7$），那么 $b$ 的逆元就是 $b^{M-2} \pmod M$。

这就是为什么代码里有 `pow(fact[i], mod - 2, mod)` 这种写法。

##### 方法 B：扩展欧几里得算法 (Extended GCD)

当你提供的代码里出现 `extended_gcd(a, b)` 时，它是在通过解方程 $ax + by = \gcd(a, b)$ 来寻找逆元。

| **方法**     | **优点**    | **使用场景**              |
| ---------- | --------- | --------------------- |
| **费马小定理**  | 代码极短，逻辑简单 | 模数 $M$ 必须是**质数**      |
| **扩展欧几里得** | 适用范围更广    | 只要 $b$ 与 $M$ **互质**即可 |
#### 4. 动手练习：计算组合数

有了逆元，我们就能写出代码里的**组合数**公式了：

### 离散化（等级化）

#### 1. 为什么要学习离散化？

想象一下，题目给你的权重 $R_i$ 可能是 **1**，也可能是 **1,000,000,000 (十亿)**。

- 如果你试图创建一个长度为十亿的数组来存储这些数据，内存会直接崩溃。
    
- 但题目规则说：你只能跳到“下一个更大权重”的行。
    

这意味着：**具体的数值是多少并不重要，重要的是谁比谁大，以及谁是紧邻的“下一级”。**

> **核心思想：** 离散化就是把分布稀疏、数值巨大的数据，映射到一组连续且紧凑的整数（0, 1, 2...）上，同时保持它们原有的**大小顺序**。

#### 2. 形象化比喻：神秘的大厦

假设你在一座神秘大厦里，楼层的编号不是 1, 2, 3，而是按照业主的资产编号：

- 1楼业主资产：100元
    
- 2楼业主资产：5,000元
    
- 3楼业主资产：10,000,000元
    

虽然资产数额天差地别，但对于物业管家来说，它们只是**第1层、第2层和第3层**。

- 你从资产 100 元的房间出来，下一站只能去 5,000 元的房间。
    
- 你不需要管中间差了 4,900 元，你只需要知道 5,000 是 100 的“下一级”。
#### 3. 离散化的三步走（代码实现逻辑）

我们要把原始权重数组 `rr = [40, 10, 20, 20]` 离散化。

##### 第一步：去重并排序

找出所有出现过的数值，排好队。

- `unique_weights = sorted(list(set([40, 10, 20, 20])))`
    
- 结果：`[10, 20, 40]`
##### 第二步：建立“档案柜”（字典映射）

给每个数值发一个“等级证”。

- 10 $\rightarrow$ 等级 0
    
- 20 $\rightarrow$ 等级 1
    
- 40 $\rightarrow$ 等级 2
##### 第三步：替换原始数据

把原数组里的数值全部换成等级。

- `[40, 10, 20, 20]` $\rightarrow$ **`[2, 0, 1, 1]`**
#### 4. 在本题代码中的体现

代码中这两部分就是在做离散化：

1. **`rrfl` (Row Rank Fast Lookup)**：这就是我们的**等级映射表**。它告诉你原始第 $i$ 行现在处于等级几。
    
2. **`dup_rows` (Duplicate Stats)**：这是**楼层档案**。它记录了每一层（等级）里住了多少人（行数）。

```python
# 离散化核心代码片段
for idx, it in enumerate(sorted(list(enumerate(rr)), key=lambda x: x[1])):
    if it[1] != last:
        last = it[1]
        dup_rows.append([last, 1]) # 发现新等级，记录权重和初始人数 1
    else:
        dup_rows[-1][1] += 1       # 权重相同，当前等级人数 +1
    rrfl[it[0]] = len(dup_rows) - 1 # 给原索引分配等级编号
```

##### 一、 核心变量的“职业”

- **`rr`**: 原始的成绩单（比如：**[4, 2, 2, 1]**）。
    
- **`it`**: 正在处理的某个学生。`it[0]` 是他的**原始座位号**，`it[1]` 是他的**成绩（权重）**。
    
- **`dup_rows`**: 等级档案。记录每一级权重是多少，这一级有多少人。
    
- **`rrfl`**: 新座位表。输入**原始座位号**，输出**现在的等级**。

##### 二、 拆解这行“最长”的循环

`for idx, it in enumerate(sorted(list(enumerate(rr)), key=lambda x: x[1])):`

这行其实是在做**排序前的贴标签**：

1. **`enumerate(rr)`**: 给每个权重打上标签，变成 `(原始索引, 权重)`。
    
    - 原始：**[4, 2, 2, 1]** $\rightarrow$ 变成：`[(0, 4), (1, 2), (2, 2), (3, 1)]`。
        
2. **`sorted(..., key=lambda x: x[1])`**: 按照权重（也就是 `x[1]`）从小到大排队。
    
    - 排序后：`[(3, 1), (1, 2), (2, 2), (0, 4)]`。这就是我们循环遍历的顺序。
    

##### 三、 逐行逻辑模拟（以 [4, 2, 2, 1] 为例）

| **循环轮次**  | **当前 it (原位, 权重)**  | **逻辑判断与操作**                            | **状态更新结果**                                                              |
| --------- | ------------------- | -------------------------------------- | ----------------------------------------------------------------------- |
| **第 1 轮** | **(3, 1)**          | **1 != -1**（新等级）。创建等级 **0**。           | `dup_rows = [[1, 1]]`<br><br>  <br><br>**rrfl[3] = 0**                  |
| **第 2 轮** | **(1, 2)**          | **2 != 1**（新等级）。创建等级 **1**。            | `dup_rows = [[1, 1], [2, 1]]`<br><br>  <br><br>**rrfl[1] = 1**          |
| **第 3 轮** | **(2, 2)**          | **2 == 2**（旧等级）。在等级 **1** 里人数 **+1**。  | `dup_rows = [[1, 1], [2, 2]]`<br><br>  <br><br>**rrfl[2] = 1**          |
| **第 4 轮** | **(0, 4)**          | **4 != 2**（新等级）。创建等级 **2**。            | `dup_rows = [[1, 1], [2, 2], [4, 1]]`<br><br>  <br><br>**rrfl[0] = 2**  |

##### 四、 为什么要这样写？（它的妙处）

这段代码一次性解决了三个问题：

1. **去重**：通过 `if it[1] != last`，不管有多少个权重相同的行，它们都只会被归为一个 `dup_rows` 里的等级。
    
2. **映射**：`rrfl[it[0]] = len(dup_rows) - 1`。这是最关键的一步，它让程序记住了：**“那个本来在第 3 位的 1，现在是等级 0”**。
    
3. **统计**：`dup_rows[-1][1] += 1` 统计了每一层“有多少个可选行”。在算路径时，如果你跳到这一层，就意味着你有这么多选择。
    

##### 五、 总结它的“工作成果”

运行完这段代码后，如果你问程序：

- **“原始第 2 行在哪个等级？”** $\rightarrow$ 查 `rrfl[2]`，得到 **1**。
    
- **“等级 1 有多少个可选行？”** $\rightarrow$ 查 `dup_rows[1][1]`，得到 **2**。
    

#### 5. 为什么离散化后就能算方案数了？

一旦我们有了等级，计算就变得清晰了：

- **步数** = 终点等级 - 起点等级。
    
- **每一步的选择** = 该等级对应的行数（存储在 `dup_rows` 中）。
    

### 倍增思想与 ST 表（Sparse Table）

在解决了“逆元”和“离散化”之后，我们面临一个巨大的挑战：如果有 $10^5$ 次询问，每次都要计算一段很长的等级跨度里所有权重数量的乘积，该怎么办？

如果用 `for` 循环一个一个乘，时间复杂度是 $O(N \times T)$，这在 1 秒的运行限制下必死无疑。 于是，**倍增思想 (Binary Lifting)** 和 **ST 表 (Sparse Table)** 登场了。

#### 1. 什么是倍增思想？

倍增的核心在于：**任何一个整数都可以拆解为 2 的幂之和。**

比如我们要跳 13 层楼：$13 = 8 + 4 + 1$ ($2^3 + 2^2 + 2^0$)。

与其一层一层爬，不如我们提前准备好“跳 1 层”、“跳 2 层”、“跳 4 层”、“跳 8 层”的技能。 这样无论你想跳多少层，只需要像拼积木一样，跳几次就能到达目的地。
#### 2. ST 表：预处理所有“跳跃”的结果

为了实现这种快速跳跃，我们需要一个二维数组 `prod_rows[j][i]`。

- **它的定义**：从第 $j$ 个等级开始，连续 $2^i$ 个等级的权重数量乘积。
    
- **状态转移（如何填表）**：
    
    要计算跳 $2^i$ 步的结果，等于“先跳 $2^{i-1}$ 步，再跳 $2^{i-1}$ 步”的结果相乘。

    $$


prod\_rows[j][i] = prod\_rows[j][i-1] \times prod\_rows[j + 2^{i-1}][i-1]

$$
    

> **为什么要设列数为 18？**
> 
> 因为 $2^{17} = 131,072$，大于题目给出的最大数据量 $10^5$。 18 列（0 到 17）足以覆盖所有的跳转可能。

#### 3. 如何进行区间查询 (`cal_prod`)？

当你需要计算从等级 $begin$ 开始，长度为 $count$ 的区间乘积时，我们利用 $count$ 的二进制位：

```python
def cal_prod(begin, count, arr):
    if count <= 0: return 1
    ans = 1; base = 0
    while count > 0:
        if count & 1: # 如果当前二进制位是 1
            ans = ans * arr[begin][base] % mod # 加上这一段 2^base 长度的积
            begin += 1 << base # 起点向后移动
        count >>= 1 # 处理下一个二进制位
        base += 1
    return ans
```

这个函数让原本需要 $O(N)$ 的查询变成了 $O(\log N)$。 对于 $10^5$ 的数据，$\log N$ 大约只有 17 次循环，速度极快。
#### 4. 为什么本题要用 ST 表？

在棋盘移动中，每跳一个权重等级，你就多了一组“行”或“列”的选择。

- 行方向：从等级 $A$ 到等级 $B$，中间每一级的选择数都要乘起来。
    
- 列方向：同理。
    
- **ST 表的作用**：就是帮你把这些成千上万个“选择数”瞬间乘好。
#### 本小节总结

- **倍增**：化整为零，利用 $2^k$ 的组合快速覆盖任意距离。
    
- **ST 表**：通过 $O(N \log N)$ 的预处理，换取 $O(\log N)$ 的查询速度。
    
- **应用**：在本题中负责计算“跨越多个权重等级时，格子选择方案的累乘积”。
    

### 组合数学 —— 行与列的华尔兹

欢迎来到最后一课！现在我们已经修好了“行电梯”和“列电梯”，也能快速算出每一层有多少种选法了。最后的难题是：**如何把行和列的移动完美地揉在一起？**

#### 1. 独立性：左手画方，右手画圆

这道题最精妙的地方在于：**行移动和列移动是互不干扰的**。

- 无论你什么时候决定“向上跳一层（行移动）”，它只取决于行权重 $R$。
    
- 无论你什么时候决定“向右跳一层（列移动）”，它只取决于列权重 $C$。
    

小蓝在棋盘上的每一次移动，要么是改变行，要么是改变列。 这就像是在跳一场华尔兹，你可以先迈左脚（行），也可以先迈右脚（列），但最终迈出的总步数是固定的。

#### 2. 总步数的秘密

假设从起点到终点：

- **行方向**：需要跨越 $X$ 个等级（$X = rrfl[r2] - rrfl[r1]$）。
    
- **列方向**：需要跨越 $Y$ 个等级（$Y = crfl[c2] - crfl[c1]$）。
    

那么，从小蓝出发到抵达终点，他**一共必须且只能走 $X + Y$ 步**。

这 $X+Y$ 步中，有 $X$ 步必须是“行移动”，$Y$ 步必须是“列移动”。
#### 3. 排列组合：给指令排队

我们可以把这 $X+Y$ 步想象成一个指令序列。比如 $X=2, Y=1$，总步数是 3。可能的指令序列有：

1. **行、行、列**
    
2. **行、列、行**
    
3. **列、行、行**
    

这不就是数学里的组合问题吗？从 $X+Y$ 个位置里，选出 $X$ 个位置放“行移动”指令，剩下的自动就是“列移动”。

**计算公式就是：**
$$

C(X+Y, X) = \frac{(X+Y)!}{X! \times Y!}

$$

#### 4. 最终的大一统公式

现在，我们把前三课学到的所有知识像拼图一样拼起来：

1. **行路径方案数 ($V$)**：利用 ST 表算出中间每一级行权重的选择乘积。
    
2. **列路径方案数 ($W$)**：利用 ST 表算出中间每一级列权重的选择乘积。
    
3. **行走路径排列数 ($C$)**：计算组合数 $C(X+Y, X)$。
    

**最终答案 = $V \times W \times C \pmod{10^9+7}$**

> **为什么要乘组合数 $C$？**
> 
> 因为 $V$ 和 $W$ 只告诉了你在“确定了先走哪一步”的情况下每一跳有多少种选法。 而组合数 $C$ 告诉了你“行移动”和“列移动”一共有多少种不同的先后交替方式。
    


#### 5. 逻辑闭环总结

这就是这道题的全部奥秘：

- **离散化**：把复杂的权重变成简单的“步数”。
    
- **ST 表**：解决“每一步有多少种格子选法”的快速累乘。
    
- **组合数学**：解决“行移动和列移动如何交替”的排队问题。
    
- **逆元**：确保在模运算的世界里，除法不会出错。
