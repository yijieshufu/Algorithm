## 代码

```python
import sys

# 提高递归深度限制
sys.setrecursionlimit(300000)

def solve():
    # 快速读取所有输入
    input_data = sys.stdin.read().split() # 得到是字符
    if not input_data:
        return
    
    n = int(input_data[0]) # 节点的数量
    s = int(input_data[1]) - 1 # 根节点
    
    # 节点生命力 a_i
    arr = list(map(int, input_data[2:n+2]))
    
    # 建立邻接表
    adj = [[] for _ in range(n)] # 内存里准备了 $n$ 个“空抽屉”。
    ptr = n + 2 # 指针 (Pointer)
    for _ in range(n - 1):   # 有且仅有 $n-1$ 条边
        u = int(input_data[ptr]) - 1 
        v = int(input_data[ptr+1]) - 1
        adj[u].append(v)
        adj[v].append(u)
        ptr += 2
        
    max_a = max(arr)   # 生命力数值的最大值
    bit = [0] * (max_a + 1)

    # 树状数组基本操作
    def update(x, val):
        while x <= max_a:
            bit[x] += val
            x += x & (-x)

    def query(x):   # 求前缀和
        res = 0
        while x > 0:
            res += bit[x]
            x -= x & (-x)
        return res

    def get_range(l, r): # 求区间和
        if l > r: return 0
        return query(r) - query(l - 1)

    total_ans = 0

    # 使用迭代 DFS 避免爆栈
    # 栈中存储 (当前节点, 父节点, 状态) 0: 进入, 1: 退出
    stack = [(s, -1, 0)]
    
    while stack:
        u, p, state = stack.pop()
        val_u = arr[u] # 
        
        if state == 0: # 入栈/初次访问
            # --- 核心逻辑：统计当前节点 u 作为后代时，满足条件的祖先数量 ---
            # 条件：a_ancestor > a_u 且 a_ancestor % a_u != 0
            
            # 1. 先算所有比 a_u 大的祖先数量
            count_larger = get_range(val_u + 1, max_a) # 统计所有比 $a_u$ 大的祖先  # max_a 活力值最大
            
            # 2. 减去其中是 a_u 倍数的祖先数量 (a_p = k * a_u)
            
            count_multiples = 0
            for multiple in range(val_u * 2, max_a + 1, val_u):
                # 检查树状数组中 multiple 这个值出现了几次
                count_multiples += (query(multiple) - query(multiple - 1))
            
            total_ans += (count_larger - count_multiples)
            
            # --- 将当前节点加入树状数组，供其子树查询 ---
            update(val_u, 1)
            
            # 先压入退出状态，再压入子节点
            stack.append((u, p, 1))
            for v in adj[u]:
                if v != p:
                    stack.append((v, u, 0))
        else:  # 出栈/回溯点
            # 回溯：将当前节点从树状数组中移除
            update(val_u, -1)

    print(total_ans)

if __name__ == "__main__":
    solve()
```

###  问题引入：一道CSP - S真题
题目：给定一个数组，支持两种操作：当op为1时，将从l到r之间的数都加上 x， 当op不为1时，查询从l到r之间的数据和，一共m 次操作， 1≤ n,m≤10⁵ ，你是不是这样写的？

```python
import sys

def solve():
    # 使用 fast I/O 提高 Python 读取速度
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    
    ptr = 0
    n = int(input_data[ptr]); ptr += 1
    m = int(input_data[ptr]); ptr += 1
    
    # 初始化数组（下标从 1 开始方便逻辑对应，增加一个 0 占位）
    a = [0] * (n + 2)
    d = [0] * (n + 2)
    pre = [0] * (n + 2)
    
    # 读入原始数组并构建差分数组
    for i in range(1, n + 1):
        a[i] = int(input_data[ptr]); ptr += 1
        d[i] = a[i] - a[i-1]
        
    for _ in range(m):
        op = int(input_data[ptr]); ptr += 1
        l = int(input_data[ptr]); ptr += 1
        r = int(input_data[ptr]); ptr += 1
        
        if op == 1:
            x = int(input_data[ptr]); ptr += 1
            # 区间加：差分操作 O(1)
            d[l] += x
            d[r + 1] -= x
        else:
            # 还原数组 a：O(n)
            for i in range(1, n + 1):
                a[i] = a[i - 1] + d[i]
            # 计算前缀和 pre：O(n)
            for i in range(1, n + 1):
                pre[i] = pre[i - 1] + a[i]
            # 输出结果
            print(pre[r] - pre[l - 1])
```

### 2树状数组（Fenwick Tree）

**高手们到底用了哪种方法呢？**

**简单来说，他们是用了一种数组，名字叫树状数组。**

树状数组，不是真正的“树”，而是一个巧妙设计的数组。这棵“树”是一种逻辑结构，它并非像二叉树那样用 left 和 right 指针显式连接，而是通过数组下标之间的二进制关系（核心是 lowbit 运算）隐含定义的。物理上，它又确实只是一个普通的数组 C[]，因此被称为树状数组。


据说Peter M. Fenwick，他在1994年发现，传统处理“动态前缀和”问题，往往会遇到瓶颈。

如果直接存储原数组，单点修改是O(1)，但区间求和需要遍历，是O(n)。

如果使用前缀和数组，区间求和是O(1)，但单点修改会导致其后所有前缀和都要更新，也是O(n)。

|**数据结构**|**修改操作**|**查询操作**|**适用场景**|
|---|---|---|---|
|**普通数组**|$O(1)$|$O(n)$|修改多，查询少|
|**前缀和数组**|$O(n)$|$O(1)$|查询多，修改少|
|**树状数组/线段树**|$O(\log n)$|$O(\log n)$|**修改和查询都很多** (本题的情况)|

他的灵感源于一个巧妙的观察：任何一个正整数都可以表示为若干个2的幂次方之和（比如7 = 4 + 2 + 1）。

那么，在求前 7 项和时，是否可以不用一个个加，一直加 7 次，而是通过某些已经计算好的 **“区间块”** 来快速求和呢？这些 **“区间块”** 正是提前预处理好的、特定区间内的元素之和。

将这种二进制的想法付诸实现，需要一个桥梁，这就是 **lowbit 操作 / lowbit 运算**。

`lowbit(x)` 的功能是求 $x$ 的二进制表示中最低位的 $1$ 所代表的数值。

例如，$6$ 的二进制是 $110$，最低位的 $1$ 代表 $2$，所以 $lowbit(6) = 2$。

这个简单的操作是整个结构的神经中枢。

基于 lowbit，就可以定义每个“区间块”（即树状数组 $C$ 中的元素 $C[i]$）所需要累加的原数组的范围：$C[i]$ 负责原数组 $A$ 中从 **$i - lowbit(i) + 1$  到  $i$** 这个区间内的元素之和。即 $C[i] = A[i] + ... + A[i - lowbit(i) + 1]$。

比如，当 $i=6$ 时，$lowbit(6)=2$，$6-2+1=5$，所以 $C[6] = A[6] + A[5]$。

下表展示了索引 $1$ 到 $8$ 的 $C$ 数组累加的区间范围，可以帮大家更直观地理解其规律：

到此，可能有些同学还看不到其中的树形逻辑关系。没关系，我可以先看操作。

当我们进行单点更新时，比如修改A[3]，就需要更新所有包含A[3]的祖先节点C[3]、C[4]、C[8]。观察上表也可以看到A[3]包含在这三个元素中。

那如何找到A[3]所属的节点呢？首先A[3]存在于C[3]，对于C[3]，它的父亲节点的下标是：i=i+lowbit(i);当更新了 C[i]：C[i]+=value（要修改的值） ，再找 i 的父亲，如此循环，直到 i 超出了数组范围就停止。

当我们进行前缀和查询时，求前 i 项和的算法是不断令 **i = i - lowbit(i)**，并累加路径上的 C[i]，直到 i 变为 0。比如求前6项和sum(6)，需要累加C[6]、C[4]（因为6-lowbit(6)=4），最后加上C[0]（4-lowbit(4)=0)（已越界，停止）。

> `i = i - lowbit(i)` 的导航原理
> 这个公式能帮你找到**“==上一个不重叠区间==”**的末尾。
   以**求前 6 项和（`sum(6)`）**为例：
	1. **第一步 ($i=6$)**：拿到 $C[6]$ 的值。它包含了 $A[5] + A[6]$。
	2. **找下一个块**：计算 $6 - lowbit(6) = 6 - 2 = 4$。**导航指向 4**。
	3. **第二步 ($i=4$)**：拿到 $C[4]$ 的值。它包含了 $A[1] + A[2] + A[3] + A[4]$。
	4. **找下一个块**：计算 $4 - lowbit(4) = 4 - 4 = 0$。**导航指向 0**。
	5. **停止**：当 $i=0$ 时，说明我们已经把从 1 到 6 的所有元素都收齐了，查询结束。


#### 实例解析：如何通过树状数组求 `sum(7)`？

求 `sum(7)` 的本质是将前 7 项的和拆解为若干个已预处理好的“区间块”之和。其核心公式是不断令 $i = i - lowbit(i)$。

##### 1. 查找路径（跳跃过程）

通过 `lowbit` 运算，我们可以找到组成 `sum(7)` 的所有关键节点：

- **第一步**：$i = 7$，累加 **$C[7]$**。
    
- **计算下一步**：$7 - lowbit(7) = 7 - 1 = 6$。
        
- **第二步**：$i = 6$，累加 **$C[6]$**。
    
- **计算下一步**：$6 - lowbit(6) = 6 - 2 = 4$。
        
- **第三步**：$i = 4$，累加 **$C[4]$**。
    
- **计算下一步**：$4 - lowbit(4) = 4 - 4 = 0$。
        
- **停止**：$i = 0$，查询结束。
    

##### 2. 逻辑还原（拆解为原数组 $A$）

根据每个 $C[i]$ 管辖的区间定义，我们可以将上述过程还原为对原数组 $A$ 的累加：

$$
\begin{aligned} sum(7) &= C[7] + C[6] + C[4] \\ &= \underbrace{(A[7])}_{C[7]} + \underbrace{(A[6] + A[5])}_{C[6]} + \underbrace{(A[4] + A[3] + A[2] + A[1])}_{C[4]} \end{aligned}
$$

##### 3. 为什么这样更快？

- **普通前缀和**：需要从 $A[1]$ 一直加到 $A[7]$，执行 **7 次** 加法。
    
- **树状数组**：只需将 $C[7], C[6], C[4]$ 三个块相加，执行 **3 次** 加法。
    
- **结论**：对于规模为 $N$ 的数组，查询次数仅为 $i$ 的二进制中 $1$ 的个数，复杂度为 $O(\log N)$。
    


大家可以看到，在执行单点更新时，从 C[i] 到 C[i + lowbit(i)] 的跳跃，就是在遍历这棵逻辑树上从当前节点到其父节点的路径。


同样，在执行前缀和查询时，操作 i = i - lowbit(i) 会使索引 向减小的方向移动。这一步的关键在于，它让我们跳转到前一个不重叠的、预计算好的“区间块”的起始位置，从而能够将其累加至总和中。


因为每个 C[i] 都是提前维护好的“局部和”，我们只需要根据 **i 的二进制结构**，用 `lowbit` 不断“跳”到关键节点，就能在 **O(log n)** 时间内完成查询！


这正是树状数组的精妙之处：  
👉 **把二进制的数学智慧，用在数据结构的设计上**，  
👉 实现“**以空间换时间，用位运算加速**”的极致优化。

```python
C = [0] * N
n = 0  # 实际使用的数组长度

    return x & (-x)

    单点修改：a[i] += x
```

### 经典应用场景

**动态前缀和** 边修改，边查询，完美替代暴力

```python
import sys

# 增加递归深度（虽然此算法使用的是循环，但良好的 Python 竞赛习惯建议包含）
sys.setrecursionlimit(200000)

def solve():
    # 使用快速读取
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    
    ptr = 0
    n = int(input_data[ptr]); ptr += 1
    m = int(input_data[ptr]); ptr += 1
    
    # 初始化两个树状数组
    # tree1 维护差分数组 d[i]
    # tree2 维护 i * d[i]
    tree1 = [0] * (n + 2)
    tree2 = [0] * (n + 2)

    def lowbit(x):
        return x & (-x)

    def update(i, val, tree):
        """通用更新函数"""
        while i <= n:
            tree[i] += val
            i += lowbit(i)

    def query(i, tree):
        """通用前缀查询函数"""
        res = 0
        while i > 0:
            res += tree[i]
            i -= lowbit(i)
        return res

    def range_add(l, r, x):
        """区间修改：在 [l, r] 内每个数加 x"""
        # 更新 tree1 (差分维护)
        update(l, x, tree1)
        update(r + 1, -x, tree1)
        # 更新 tree2 (i * d[i] 维护)
        update(l, x * l, tree2)
        update(r + 1, -x * (r + 1), tree2)

    def prefix_query(i):
        """查询 [1, i] 的前缀和：利用公式 i * sum(d[j]) - sum(j * d[j])"""
        return i * query(i, tree1) - query(i, tree2)

    def range_query(l, r):
        """区间查询：[l, r] 的区间和"""
        return prefix_query(r) - prefix_query(l - 1)

    # 初始化数组
    for i in range(1, n + 1):
        val = int(input_data[ptr]); ptr += 1
        range_add(i, i, val) # 通过区间加法初始化单点值

    # 处理操作
    results = []
    for _ in range(m):
        op = int(input_data[ptr]); ptr += 1
        l = int(input_data[ptr]); ptr += 1
        r = int(input_data[ptr]); ptr += 1
        
        if op == 1:
            x = int(input_data[ptr]); ptr += 1
            range_add(l, r, x)
        else:
            results.append(str(range_query(l, r)))

    # 一次性输出结果，提高效率
    sys.stdout.write('\n'.join(results) + '\n')

if __name__ == "__main__":
    solve()
```

## 迭代 DFS 与回溯 (Iterative DFS & Backtracking)

### 深度优先搜索（DFS）

#### **核心思想**

- **定义**：DFS是一种用于遍历或搜索树/图的算法，通过“一条路走到黑”的策略深入探索分支，直到无法继续时回溯。
    
- **递归机制**：依赖函数调用栈，每层递归代表深入一层，回溯时栈帧出栈。
    
- **图类型**：适用于显式图（邻接表/矩阵）和隐式图（状态空间，如排列组合、棋局）。
    

#### **基本要素与实现**

1. **当前状态**：通过参数传递（如节点编号、排列长度）。
    
2. **访问标记**：`vis`数组记录已访问状态，防止重复和死循环。
    
3. **递归边界**：终止条件（如排列完成、找到解）。
    
4. **状态转移**：遍历当前状态的下一状态（如邻居节点或生成新状态）。
    
5. **回溯处理**：撤销操作（如重置`vis`标记），尝试其他分支。
