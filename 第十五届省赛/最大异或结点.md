# 最大异或结点

## 代码实现

```python
import sys

# 设置递归深度防止深层树结构报错（虽然本题主要用迭代）
sys.setrecursionlimit(200000)

def solve():
    # 快速读入所有数据
    # **“一键”将整个输入流转化为一个按空白符（空格/换行）切分的字符串列表，无视行列结构，方便按顺序直接取值。**
    input_data = sys.stdin.read().split() 
    if not input_data:
        return
    
    N = int(input_data[0])  # 结点的总数 N
    X = [int(x) for x in input_data[1:N+1]] # 所有结点的权值 一共N个节点权重
    F = [int(f) for f in input_data[N+1:2*N+1]] # 结点 $i$ 的**父亲结点的编号**
    
    # 建立邻接表，存储每个结点的直接邻居（父节点和子节点）
    adj = [[] for _ in range(N)] # 创建了一个包含 $N$ 个空列表的列表
    for i in range(N):
        p = F[i] # 取出当前节点的父亲节点
        if p != -1:
            adj[i].append(p) # 结点 i 说：“我的邻居里有我的爸爸 p”
            adj[p].append(i) # 爸爸 p 说：“我的邻居里有我的孩子 i”
            
    # Trie 树参数：最多节点数约为 N * 31
    # L 存储左子节点 (bit 0), R 存储右子节点 (bit 1), C 存储经过该路径的节点数
    max_nodes = N * 32 
    L = [-1] * max_nodes
    R = [-1] * max_nodes 
    C = [0] * max_nodes
    free_node = 1  # 根节点为 0，下一个可用节点索引从 1 开始
    
    # 插入函数：将数值 val 插入 Trie 树
    def insert(val):
        nonlocal free_node # 使用外部作用域（`solve` 函数里）定义的 `free_node` 变量。
        u = 0 # 根节点
        C[u] += 1 # 根节点的计数加 1
        for i in range(30, -1, -1): # 从高位到低位遍历二进制位
            bit = (val >> i) & 1 # 提取 `val` 二进制下的第 `i` 位。
            if bit == 1:
                if R[u] == -1:# 如果向右（1的方向）没有路
                    R[u] = free_node # 给这条路分配一个新房间（节点编号）
                    free_node += 1 # 房间号用掉一个，递增
                u = R[u] # 顺着这条路往下走一步，更新当前位置 u
            else:
                if L[u] == -1:
                    L[u] = free_node
                    free_node += 1
                u = L[u]
            C[u] += 1 #每走进一个新的节点 `u`，就把该节点的计数加 1。

    # 删除函数：将数值 val 从 Trie 树中“逻辑删除”（计数减一）
    def remove(val):
        u = 0 # 1. 依然从根节点 (编号 0) 出发
        C[u] -= 1 # 2. 根节点的总计数减 1 (表示树里少了一个可用的数)
        for i in range(30, -1, -1): # 从最高位到最低位拆解这个要删除的数字
            bit = (val >> i) & 1 # 提取第 i 位是 0 还是 1
            if bit == 1: # 如果这一位是 1
                u = R[u] #顺着 1 的路径走 (进入右房间)
            else: # 如果这一位是 0
                u = L[u] # 顺着 0 的路径走 (进入左房间)
            C[u] -= 1 # 每走进一个房间，就把这个房间的“可用人数”减 1
            
    # 查询函数：寻找与 val 异或最大的结果
    # 从最高位开始，每一位都拼命往“相反”的方向走
    def query(val):
        u = 0 # 根节点（编号 0）出发。
        if C[u] == 0: return -1 # Trie 树为空（排除禁区后没剩结点）
        res = 0 # 最大异或结果
        for i in range(30, -1, -1):  # 第 30 位（最高位）开始向下检查到第 0 位
            bit = (val >> i) & 1 
            # 优先走与 bit 相反的路径以获得异或值 1
            if bit == 0: # 想要 1
                if R[u] != -1 and C[R[u]] > 0: # 看看右边(1的方向)有没有路，且有没有“活着的”数
                    res |= (1 << i) #成功！异或结果的这一位可以变成 1
                    u = R[u] #顺着这条发财路走下去
                else:
                    u = L[u] #没办法，1的那边没人，只能走 0 这边，这一位异或得 0
            else: # 想要 0
                if L[u] != -1 and C[L[u]] > 0:
                    res |= (1 << i)
                    u = L[u]
                else:
                    u = R[u]
        return res

    # 1. 初始化 Trie 树，装入所有结点的权值
    for v in X:
        insert(v)
        
    ans = 0
    # 2. 遍历每个结点作为 a，寻找不相邻的 b
    for i in range(N):
        # 暂时移除结点 i 自己
        remove(X[i])
        # 暂时移除结点 i 的所有邻居
        for neighbor in adj[i]:
            remove(X[neighbor])
            
        # 在剩下的结点中找最大异或值
        res = query(X[i])
        if res > ans:
            ans = res
            
        # 恢复现场：重新插入
        insert(X[i])
        for neighbor in adj[i]:
            insert(X[neighbor])
            
    # 输出全局最大值
    print(ans)

if __name__ == "__main__":
    solve()
```

## 解题思路分析

1. **基本模型**：如果没有“不直接相连”的限制，求 $N$ 个数中任意两个数异或的最大值，通常使用 **Trie 树（字典树/前缀树）**。将所有数字以二进制形式存入 Trie 树，复杂度为 $O(N \log (\max X_i))$。
    
2. **处理限制条件**：
    
    - 题目要求结点 $a, b$ 不直接相连。在树结构中，这意味着 $b$ 不能是 $a$ 的父节点，也不能是 $a$ 的子节点。
        
    - 对于任意结点 $i$，它的“禁区”包括：**它自己**、**它的父节点**、**它的所有子节点**。
        
    - 我们可以利用 Trie 树的**动态维护**特性：
        
        1. 先将所有结点的权值 $X_i$ 插入 Trie 树。
            
        2. 遍历每个结点 $i$：
            
            - 从 Trie 树中**临时删除** $X_i$ 及其所有相邻结点的权值。
                
            - 在 Trie 树中查询与 $X_i$ 异或值最大的数。
                
            - 将刚才删除的权值**重新插入** Trie 树，恢复现场。
                
3. **效率优化**：
    
    - 由于 Python 运行速度较慢，为了通过 $10^5$ 规模的数据，我们需要避免使用类和对象，改用**扁平化的数组**来模拟 Trie 树。
        
    - 使用 `sys.stdin.read` 进行快速输入。

## 关键点拨

1. **为什么逻辑删除有效？**
    
    在字典树中，物理删除节点比较麻烦。通过 `C` 数组记录每个节点被多少个数字经过。当 `C[u] == 0` 时，意味着该分支当前没有可用的数字。
    
2. **时间复杂度分析**：
    
    - 构建邻接表：$O(N)$。
        
    - 插入/删除/查询：每次操作都是 $O(31)$，即 $O(\log X_i)$。
        
    - 主循环：遍历 $N$ 个点，每个点的邻居总数加起来是 $2 \times (N-1)$。
        
    - **总复杂度**：约为 $O(N \times 31)$。在 Python 3 中处理 $10^5$ 的量级大约需要几秒钟，完全符合题目给出的 10s 限制。
        
3. **空间复杂度**：
    
    $N \times 31$ 个节点，每个点存储 3 个整数（L, R, C），大约消耗几十 MB 内存，远低于 512MB 的限制。

**如果你在本地运行测试用例：**

输入示例中的数据，程序会输出 `7`。因为 $X_3 \oplus X_4 = 3 \oplus 4 = 7$，且 3 和 4 并不是邻居。

## Trie 树

### 1. 什么是 Trie 树？（背景与起源）

想象你要存几个英文单词：`cat`, `car`, `dog`。

- 传统的存法：开个列表 `["cat", "car", "dog"]`。找单词时，得一个一个比对。
    
- **Trie 树存法**：把它们公共的前缀连在一起。
    
    - `cat` 和 `car` 都以 `c`-`a` 开头。在树里，它们共享前两条边。
        
    - 到了第三个字母，分叉了：一个指向 `t`，一个指向 `r`。

**它的核心思想是：利用字符串的公共前缀来减少查询时间，提高效率。**

### 2. 什么是 0/1 Trie 树？（本题的核心）

在计算机里，数字都是二进制（0 和 1）。所以，**0/1 Trie 树就是一棵专门存“由 0 和 1 组成的单词”的字典树。**

- **普通字典树**：每个节点可能有 26 个分叉（a-z）。
    
- **0/1 字典树**：每个节点只有 **2 个分叉**（左边代表 0，右边代表 1）。

#### 为什么它能解决最大异或（XOR）问题？

这是最神奇的地方。异或的规则是：**相同为 0，不同为 1**。

如果要让 $A \oplus B$ 最大，我们就希望它们的二进制位尽可能地**不一样**。

- **贪心策略**：当我们拿着数字 $A$ 在 Trie 树里找搭档时，如果我们这一位是 `0`，我们就拼命往 `1` 的分叉走；如果我们这一位是 `1`，我们就拼命往 `0` 的分叉走。
    
- **效率**：这种“对着干”的选择，让你只需要走 31 步（位深），就能在成千上万个数中找到那个能产生最大异或值的“真命天子”。

### 3. 深度理解：代码里的数组到底在干嘛？

你可能被 `L[u]`, `R[u]`, `C[u]` 搞懵了。其实它们就是**地图上的路标**。

#### 想象你在一座迷宫里：

- **`u` 是当前的房间号**：你现在站在编号为 `u` 的房间。
    
- **`L[u]` 是左边的门**：门上写着“0”。如果你推开这扇门，它通往的下一个房间编号是 `L[u]`。如果 `L[u] == -1`，说明这里没路，是堵墙。
    
- **`R[u]` 是右边的门**：门上写着“1”。它通往的房间编号是 `R[u]`。
    
- **`C[u]` 是房间里的计数器**：记录有**多少条路经过了这个房间**。

#### 插入操作 `insert(val)` 的逻辑：

你要把数字 `5`（假设二进制是 `101`）存进去：

1. **从房间 0 出发**（根节点）。
    
2. **第一位是 `1`**：你看 `R[0]`。
    
    - 没路？（`-1`）那就新盖一个房间（`free_node`），让 `R[0]` 指向它。
        
    - 有路？那就直接走进去。
        
3. **进入新房间**：把这个房间的计数器 `C` 加 1。
    
4. **重复**：处理下一位 `0`，再处理下一位 `1`。

### 4. 为什么要用 `C[u]`？（本题的关键限制）

题目说：**不能选直接相连的结点**。

这就是 `C[u]` 存在的意义。普通的 Trie 树只要把数存进去就行，但这题我们要 **“动态移除”**。

- 当我们要处理结点 $i$ 时，我们要避开它的邻居。
    
- 我们不真的把邻居从树里“删掉”（太麻烦了），我们只是把邻居经过的所有房间的计数器 `C[u]` 都 **减 1**。
    
- **查询时**：如果我这一位是 `0`，想往 `1` 走，我会先看一眼 `R[u]` 那个房间的 `C` 是不是大于 0。
    
    - 如果 `C[R[u]] > 0`：说明里面还有“活着的”数字（不是我要避开的邻居），大胆走！
        
    - 如果 `C[R[u]] == 0`：说明这个方向的所有数字现在都被“封印”了（全是邻居），我只能走另一边。
