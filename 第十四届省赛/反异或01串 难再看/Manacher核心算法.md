**Manacher 算法（马拉车算法）** 是一个非常巧妙的算法。它的核心目的只有一个：

**在线性时间 $O(N)$ 内，求出一个字符串中所有的回文子串（最长回文子串）。**

普通的中心扩展法（遍历每个位置往两边扩）需要 $O(N^2)$ 的时间，遇到长一点的字符串（如 $10^5$）就会超时。Manacher 能够快到 $O(N)$，是因为它利用了 **“利用已知的回文去推导未知的回文”** 这一思想。

下面我用通俗易懂的方式拆解它的三个核心步骤：

---

### 第一步：巧妙的预处理（解决奇偶问题）

我们在判断回文时，总是很烦恼“奇数长度”和“偶数长度”要分开处理（一个中心是字符，一个中心是缝隙）。

Manacher 的第一招就是：**把所有回文都变成奇数长度**。

**做法**：在字符之间（包括首尾）插入一个从未出现过的特殊字符（通常用 `#`）。

- 原串：`1 2 2 1` (偶数长度)
    
- 新串：`# 1 # 2 # 2 # 1 #` (长度变成 9，奇数)
    
    - 此时中心是中间的 `#`，半径是 5。
        
- 原串：`1 2 1` (奇数长度)
    
- 新串：`# 1 # 2 # 1 #` (长度变成 7，奇数)
    
    - 此时中心是 `2`，半径是 4。

**好处**：不管原串是啥，新串永远是奇数长度，我们只需要统一写一套逻辑。

---

### 第二步：核心概念——加速盒子（Box）

这是算法的灵魂。我们需要维护两个关键变量：

1. **`c` (Center)**：当前已经找到的、**右边界延伸得最远**的那个回文串的中心。
    
2. **`r` (Right)**：那个回文串的右边界（也就是它能覆盖到的最右端）。

我们把这个由 `c` 和 `r` 确定的范围称为一个 **“盒子”**。

当我们计算后面的位置 `i` 的回文半径 `p[i]` 时，如果 `i` 在这个“盒子”里面（`i < r`），我们就可以偷懒！

---

### 第三步：[镜像原理](镜像原理.md)（照镜子）

假设我们正在计算位置 `i` 的回文半径 `p[i]`。

因为 `i` 在以 `c` 为中心的“大回文串”里面，那么 `i` 关于 `c` 的左边，一定有一个**对称点（镜像点）**，我们叫它 `j` (代码中通常写作 `2*c - i`)。

**既然是大回文串，左边和右边是对称的。那么 `i` 的情况，很大程度上可以参考 `j` 的情况。**

这就是代码中那行神来之笔的含义：

```Python
if r > i:
    p[i] = min(p[2*c - i], r - i)
```

这里有两种情况：

**情况 A：镜像点的圈很小（在盒子里）**

如果 `j` 的回文范围很小，完全被包在这个大盒子内部。

- 那么根据对称性，`i` 的回文范围肯定和 `j` **一模一样**。
    
- 直接 `p[i] = p[j]`，计算结束！一步都不用扩！

**情况 B：镜像点的圈很大（超出了盒子）**

如果 `j` 的回文范围特别大，超出了大盒子的左边界。

- 对于 `i` 来说，因为大盒子的右边界是 `r`，我们只能保证在 `r` 以内的部分是对称的。
    
- 超出 `r` 的部分我们不知道（因为还没探索过）。
    
- 所以我们暂时先把 `p[i]` 设为 `r - i`（到边界的距离），然后再尝试往外扩。

**情况 C：`i` 在盒子外面**

- 那就没办法利用以前的信息了，老老实实从 1 开始往外扩。

---

### 代码逻辑对照

回头看你代码中的这一段，现在应该豁然开朗了：

```Python
for i in range(n):
    # 1. 尝试利用已有的“盒子”加速
    if r > i:
        # i 的镜像点是 2*c - i
        # min(镜像点的半径, 到右边界的距离)
        l = min(p[2*c-i], r-i)
    else:
        # 在盒子外面，只能从头开始
        l = 1
    
    # 2. 在已知的基础上，继续尝试往外扩（暴力扩）
    while i+l < n and i-l >= 0 and s[i+l] == s[i-l]:
        l += 1
    
    # 3. 如果我们发现了更靠右的边界，更新“盒子”
    # 这样后面的点就能利用这个新盒子来加速了
    if i+l > r:
        r = i + l
        c = i
        
    p[i] = l
```

### 总结

Manacher 之所以快，是因为它**不走回头路**。

虽然代码里有 `while` 循环，但是 `r` (右边界) 是一直在单调向右移动的。每个字符只会被 `r` 扫过一次。

所以复杂度是完美的 **$O(N)$**。

**对于这道题：**

我们需要 Manacher 帮我们快速找出字符串中**所有**的回文中心和半径，然后判断哪个回文串最能帮我们“省钱”（也就是覆盖的 `1` 最多）。
