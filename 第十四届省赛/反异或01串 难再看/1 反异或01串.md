### 第一部分：题目深度剖析

#### 1. 核心操作的秘密：`s ^ rev(s)`

题目的关键在于理解操作 $s' = s \oplus rev(s)$ 会生成什么样的字符串。

假设 $s$ 的长度为 $L$，第 $i$ 位字符是 $s[i]$（$0 \le i < L$）。

$rev(s)$ 的第 $i$ 位就是 $s[L-1-i]$。

异或操作后，$s'[i] = s[i] \oplus s[L-1-i]$。

根据异或的交换律，我们可以发现：

$$
s'[L-1-i] = s[L-1-i] \oplus s[i] = s'[i]
$$

**结论 1：** 这个操作生成的一定是一个 **[回文串](回文串.md)**。

[回文串的定义](回文串的定义.md)
#### 2. 中心的特殊限制

对于回文串，我们需要区分奇数长度和偶数长度：

- **偶数长度**：对称中心在两个字符之间，不需要考虑中心字符的值。
    
- **奇数长度**：存在一个中心字符。假设中心位置为 $mid$，根据异或性质：

    $$


s'[mid] = s[mid] \oplus s[mid] = 0

$$
    

**结论 2：** 如果生成的串是奇数长度，**它的中心字符必须是 '0'**。如果是偶数长度，则无限制。

#### 3. 贪心策略（如何最少添加）

我们的目标是构造出 $T$。

- 添加 `0`：免费（或者说不影响我们计算 `1` 的数量）。
    
- 添加 `1`：需要花费 1 个代价。
    
- 使用异或操作：可以一次性生成一对 `1`（例如 $s=10, rev=01 \to 11$）。生成这一对 `1`，我们在 $s$ 中只需要放入一个 `1`。
    

**[最省钱的策略](最省钱的策略.md)：**

我们希望 $T$ 中尽可能多的一段连续子串（Palindrome）是由异或操作生成的。

因为在这个回文子串中，**每有两个对称的 '1'，我们实际只需要花费 1 个代价**。

换句话说，如果在 $T$ 中找到一个符合条件的回文子串，其中包含 $K$ 个 '1'，那么我们通过操作生成它只需要 $K/2$ 个 '1'。

我们可以节省的代价 = $K/2$（向下取整，因为奇数个1中间那个如果是1是不能被生成的，但本题中心限制为0，所以回文串里的1总是成对出现，肯定是偶数个）。

**最终算法思路：**

1. 统计 $T$ 中所有的 '1' 的个数，记为 `total_ones`。
    
2. 在 $T$ 中找到一个**最长回文子串**，要求该回文子串如果长度为奇数，中心必须是 '0'。
    
3. 计算这个最佳回文子串里包含的 '1' 的个数，记为 `max_sub_ones`。
    
4. 答案 = `total_ones - (max_sub_ones / 2)`。

### 第二部分： 代码与详解

```Python
import sys

# 增加递归深度和读取速度（虽然本题是非递归，但这是Python刷题好习惯）
sys.setrecursionlimit(200000)
input = sys.stdin.readline

def solve():
    # 读取输入并去除首尾空白
    raw_s = input().strip()
    if not raw_s:
        print(0)
        return

    # 1. 构造 Manacher 需要的字符串
    # 比如输入 "01"，变成 "$#0#1#^" (加$和^是为了防止越界，虽然Python支持负索引，但这样更安全通用)
    t = '#'.join('^{}$'.format(raw_s))
    n = len(t)
    
    # p 数组存放回文半径
    p = [0] * n
    c = 0 # 当前最右回文串的中心
    r = 0 # 当前最右回文串的右边界
    
    # 2. 预处理前缀和，用于快速计算回文串里有多少个 '1'
    # pre[i] 表示 t[0]...t[i-1] 中 '1' 的个数
    pre = [0] * (n + 1)
    for i in range(n):
        pre[i+1] = pre[i] + (1 if t[i] == '1' else 0)
    
    max_ones_in_palindrome = 0
    
    # 3. Manacher 核心算法
    for i in range(1, n - 1):
        # 这里的写法是标准的 Manacher
        if r > i:
            p[i] = min(r - i, p[2 * c - i])
        else:
            p[i] = 1 # 如果不包含中心，半径初始为0；如果包含中心，半径初始为1。这里我们包含中心。
        
        # 尝试扩展
        while t[i + p[i]] == t[i - p[i]]:
            p[i] += 1
        
        # 更新最右边界
        if i + p[i] > r:
            c = i
            r = i + p[i]
        
        # --- 核心解题逻辑 ---
        
        # 只有当中心字符不是 '1' 时，这个回文串才可能是通过 s ^ rev(s) 生成的
        # t[i] == '#' 对应偶数长度回文，有效
        # t[i] == '0' 对应奇数长度回文且中心为0，有效
        # t[i] == '1' 对应奇数长度回文且中心为1，无效 (因为 x^x=0，中心不可能是1)
        if t[i] != '1':
            # 当前回文串在 t 中的半径是 p[i]（包含中心）
            # 实际回文范围是 [i - p[i] + 1, i + p[i] - 1]
            # 利用前缀和计算这段范围内的 '1' 的个数
            # Count = Pre[Right + 1] - Pre[Left]
            
            left_idx = i - p[i] + 1
            right_idx = i + p[i] - 1
            
            # 注意：pre数组大小是 n+1，pre[k] 存储的是前 k 个字符的和
            # 也就是 t[0]...t[k-1]。
            # 所以要查询 t[left_idx]...t[right_idx]，应该是 pre[right_idx + 1] - pre[left_idx]
            current_ones = pre[right_idx + 1] - pre[left_idx]
            
            if current_ones > max_ones_in_palindrome:
                max_ones_in_palindrome = current_ones

    # 4. 计算结果
    # 原始字符串中 '1' 的总数
    total_ones = raw_s.count('1')
    
    # 我们可以通过操作生成 max_ones_in_palindrome 个 1
    # 生成这些 1，我们只需要在操作前准备一半数量的 1
    # 节省的 1 的数量 = 既然这部分回文里的 1 可以成对生成，那我们实际上省了一半的工
    # 比如回文里有 4 个 1，我们只需要 2 个 1 就能生成出来，省了 2 个。
    saved = max_ones_in_palindrome // 2
    
    print(total_ones - saved)

if __name__ == '__main__':
    solve()
```

### 复杂度分析

- **时间复杂度**：$O(N)$。Manacher 算法是线性的，前缀和预处理也是线性的。这对于 $N \le 10^6$ 的数据量是必须的。
    
- **空间复杂度**：$O(N)$。用于存储变换后的字符串、半径数组 $P$ 和前缀和数组。
