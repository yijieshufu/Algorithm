为了让代码更加简洁且逻辑统一，我们可以将“满层”和“最后一层”的统计逻辑合并。核心思路是利用 **`min` 函数限制右边界**，确保统计的节点始终在总数 $n$ 之内。

### 一、 题目深度分析

在一个 $m$ 叉树中，节点编号从 1 开始。对于任意一个节点 $x$，它的[子节点区间](子节点区间的计算公式.md)是非常规律的：

- **左子节点边界**：$L = (x - 1) \cdot m + 2$
    
- **右子节点边界**：$R = x \cdot m + 1$

当我们要统计节点 $k$ 的子树大小时，本质上是在统计以 $k$ 为顶点的“等腰梯形”区域内在编号 $1 \dots n$ 范围内的节点个数。

**层级演进规律：**

- 第一层（子树根节点）：范围是 $[k, k]$
    
- 第二层：范围是 $[k \cdot m - m + 2, \;\; k \cdot m + 1]$
    
- 第三层：将上一层的左右边界分别作为 $x$ 代入上述公式，得到新的 $[L, R]$。

---

### 二、 简洁版思路

1. **初始化**：令当前层的左边界 `L = k`，右边界 `R = k`。
    
2. **迭代累加**：只要 `L <= n`（当前层还有节点在总范围内），就执行：
    
    - **有效节点计算**：当前层在 $n$ 以内的节点数等于 $\min(n, R) - L + 1$。如果这个值小于 0（说明 $L > n$），则该层及以下没有节点。
        
    - **边界跳跃**：更新 $L$ 和 $R$ 到下一层。
        
    - **停止条件**：当 $L > n$ 时，说明整棵子树已经统计完毕。

---

### 三、 精简版代码实现（带注解）

```Python
import sys

def solve():
    # 使用快速读取，适应 T=10^5 的规模
    input_data = sys.stdin.read().split()
    if not input_data: return
    
    T = int(input_data[0])
    pointer = 1
    results = []
    
    for _ in range(T):
        n = int(input_data[pointer])
        m = int(input_data[pointer+1])
        k = int(input_data[pointer+2])
        pointer += 3
        
        ans = 0
        left = k   # 子树当前层的左边界
        right = k  # 子树当前层的右边界
        
        # 统一处理逻辑：逐层向下，直到左边界超出总节点数 n
        while left <= n:
            # 统计当前层落在 [1, n] 范围内的节点个数
            # min(n, right) 确保不会统计超过总数 n 的虚无节点
            ans += min(n, right) - left + 1
            
            # 提前检查防止 left * m 在极大值情况下进行无意义运算
            if left > (n - 2) // m + 1:
                break
            
            # 更新到下一层的左右边界
            left = left * m - m + 2
            right = right * m + 1
            
        results.append(str(ans))
    
    # 批量输出，提高效率
    sys.stdout.write("\n".join(results) + "\n")

if __name__ == "__main__":
    solve()
```

### 四、 代码逻辑注解

- **`ans += min(n, right) - left + 1`**：
    
    这是代码的核心。无论当前层是满的（`right <= n`）还是最后残缺的一层（`right > n`），这个公式都能准确计算出实际存在的节点数。
    
- **`left > (n - 2) // m + 1`**：
    
    这行是一个优化。虽然 Python 支持无限位数的整数，但当 `left` 已经大到下一层肯定超过 `n` 时，手动中断循环可以节省 CPU 周期。它等价于判断 `left * m - m + 2 > n`。
    
- **时间复杂度**：
    
    对于每个 $T$，循环次数为 $O(\log_m n)$。当 $m=2$ 时约为 30 次，当 $m$ 很大时次数更少。总复杂度 $O(T \log_m n)$，非常高效。
