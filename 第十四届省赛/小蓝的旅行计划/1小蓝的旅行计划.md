这是一道非常经典的 **“带容量限制的资源调度”** 问题，通常出现在算法竞赛（如蓝桥杯、ACM）或大厂面试中。

你提供的代码使用了 **反悔贪心（Greedy with Regret）** 结合**双堆（Two Heaps）** 的高级策略。这是解决此类问题最优美、最高效的方法之一。

---

### 1. 题目深度分析

#### **题目核心冲突**

我们需要开完一段旅程，目标是**省钱**。但我们面临两个互斥的限制：

1. **想囤便宜油**：遇到便宜的加油站，理智告诉我们要多买。
    
2. **油箱有限制 ($m$)**：想囤却装不下。如果强行囤，就得把油箱里已有的油倒掉。
    

#### **普通贪心的死胡同**

普通贪心策略通常是：“如果下一站更便宜，我就只加刚好够去下一站的油；如果下一站贵，我就把油箱加满。”

这种策略在简单的加油站问题（无加油量限制 $L_i$）是有效的。但本题每个站有 **$L_i$ (限量)**，导致普通贪心失效——因为你可能为了等下一站的便宜油，结果到了下一站发现它的存量不够你跑后面路程的，导致无解或被迫在更贵的站加油。

#### **正解：反悔贪心 (Buy All, Refund Expensive)**

这个算法的逻辑是 **“事后诸葛亮”**，或者叫 **“全买全退流”**。

**核心哲学：**

1. **全买入**：只要经过加油站，不管油箱满没满，先把这个站所有的限量油 $L_i$ **全买下来**（假装油箱无限大）。
    
2. **吃粗粮（优先烧便宜油）**：开车消耗油时，我们假设消耗的是手里**最便宜**的那批油。
    
    - _为什么？_ 因为我们要把**贵的油**留在账本上。万一后面油箱爆了，我们可以把贵的油退掉（反悔），这样能省更多钱。
        
3. **退奢侈品（溢出退最贵）**：每当我们的“虚拟油箱”超过真实容量 $m$ 时，说明买多了。此时强行退货，退掉手里**最贵**的那批油，直到油箱恢复到 $m$。
    

通过这种“先买爆，再优胜劣汰”的方式，我们自动保留了这一路走来性价比最高的油组合。

---

### 2. 代码逻辑架构

为了实现上述策略，我们需要维护同一批油的两种属性，因此使用了**双堆**：

- **Min-Heap (小根堆)**：用于**开车消耗**。我们需要快速找到**最便宜**的油来“烧掉”。
    
- **Max-Heap (大根堆)**：用于**溢出退款**。我们需要快速找到**最贵**的油来“退货”。
    
- **Stock (字典/哈希表)**：作为**“唯一真理”**，记录每站的油实际还剩多少。配合堆实现**懒惰删除**。

### 3. 详细代码注解

```Python

import sys
import heapq

# 设置递归深度（虽然本题非递归，但算法题常用配置）
sys.setrecursionlimit(200000)

def solve():
    # === 1. 极速输入读取 ===
    # 将所有输入读成一个流，避免大量的 IO 操作耗时
    # input_iter 是一个迭代器，每次 next() 都会吐出下一个整数
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    input_iter = map(int, input_data)

    try:
        num_stations = next(input_iter) # n: 站点数
        capacity = next(input_iter)     # m: 油箱容量
    except StopIteration:
        return

    # === 2. 核心数据结构 ===
    
    # stock: 库存账本。
    # Key=站点ID, Value=该站点买的油还剩多少升。
    # 这是由于堆无法直接修改内部元素的数值，所以用这个外部字典来同步状态。
    stock = {} 

    # min_heap: (价格, ID)。存入的格式
    # 作用：开车时，优先消耗便宜的油。  
    min_heap = []
    
    # max_heap: (-价格, ID)。存负数是为了模拟大根堆。
    # 作用：油箱爆了时，优先退掉贵的油。
    max_heap = []

    # === 辅助函数：懒惰删除 ===
    # 作用：从堆顶取数据时，如果发现库存(stock)已经是0，说明这批油
    # 已经在另一个堆里被操作完了（被烧掉或被退掉了），由于堆无法实时同步删除，
    # 此时遇到它就直接丢弃，直到找到一个还有库存的有效批次。
    def get_valid_batch(heap):
        # 只要堆顶元素的库存为0，就弹出丢弃
        while stock[heap[0][1]] == 0:
            heapq.heappop(heap)
        return heap[0] # 返回有效的堆顶 (价格, ID)

    # === 3. 初始状态 ===
    # 题目隐含条件：一开始油箱是满的。
    # 我们把这初始的 m 升油看作是 "ID为-1" 的站点提供的，价格为 0 元。
    stock[-1] = capacity
    # 用堆来记录 价格 和站点
    heapq.heappush(min_heap, (0, -1))
    heapq.heappush(max_heap, (0, -1))

    total_cost = 0           # 记录总花费
    current_fuel = capacity  # 记录当前油箱里的逻辑油量（可能暂时超过 m）

    # === 4. 遍历每一个站点 ===
    for i in range(num_stations):
        # 获取当前行数据
        distance = next(input_iter) # 到下一站的距离 (即本段路耗油)
        price = next(input_iter)    # 当前站油价
        limit = next(input_iter)    # 当前站限购量

        # ------ 阶段 A: 开车消耗 (Burn Phase) ------
        # 物理距离必须跑完，油必须扣
        current_fuel -= distance
        
        # 如果当前油量 < 0，说明把前面所有能买的油都买了也不够跑到这
        # 直接判定无解
        if current_fuel < 0:
            print("-1")
            return

        # 账目操作：逻辑上消耗哪批油？
        # 策略：Burn Cheapest (优先烧最便宜的)。
        # 为什么？因为我们要留着贵的油。万一等会油箱溢出了，我们可以退掉贵的换回钱。
        # 如果现在就把贵的烧了，钱就彻底花出去了。
        needed_to_burn = distance
        while needed_to_burn > 0:
            # 从小根堆找最便宜的
            p, idx = get_valid_batch(min_heap)
            
            # 这一批能提供多少？min(还得烧多少, 这批库存还剩多少)
            amount = min(needed_to_burn, stock[idx])
            
            # 执行消耗
            stock[idx] -= amount
            needed_to_burn -= amount
            # 注意：这里不需要 current_fuel 操作，因为上面已经 current_fuel -= distance 了
            # 这里只是在调整内部账本结构

        # ------ 阶段 B: 进站全买 (Buy Phase) ------
        # 策略：不管油箱满没满，先把这站的限购额度全部买入。
        # 这是一个“贪心”的假设，假设我们可能需要这些油。
        total_cost += price * limit
        current_fuel += limit
        stock[i] = limit # 记录库存
        
        # 将新买的油分别加入两个堆
        heapq.heappush(min_heap, (price, i))
        heapq.heappush(max_heap, (-price, i)) 

        # ------ 阶段 C: 溢出退款 (Refund Phase) ------
        # 策略：如果当前油量超过了油箱容量 m，必须倒掉一部分。
        # 倒掉谁？当然是倒掉手里最贵的那批油 (Refund Expensive)。
        # 相当于我们“反悔”了：刚才买多了，现在把最贵的退回去。
        while current_fuel > capacity:
            # 从大根堆找最贵的
            neg_p, idx = get_valid_batch(max_heap)
            real_price = -neg_p # 还原价格
            
            # 需要退多少？
            excess = current_fuel - capacity # 溢出量
            # 这批油能退多少？ min(溢出量, 这批库存量)
            refund_amount = min(excess, stock[idx])
            
            # 执行退款
            stock[idx] -= refund_amount
            current_fuel -= refund_amount
            total_cost -= refund_amount * real_price # 钱退回来

    # === 5. 终点清算 ===
    # 到达终点后，油箱里剩下的油都没用了。
    # 按照反悔贪心的逻辑，这些油应该被“退货”，因为我们只需要刚好跑完的油。
    # 剩下的油全部按原价退款，这样 total_cost 就是最小花费。
    while current_fuel > 0:
        # 找最贵的退（其实剩啥退啥都一样，但逻辑上保持一致）
        neg_p, idx = get_valid_batch(max_heap)
        
        refund_amount = min(current_fuel, stock[idx])
        
        stock[idx] -= refund_amount
        current_fuel -= refund_amount
        total_cost -= refund_amount * (-neg_p)

    # 输出最终答案
    print(total_cost)

if __name__ == '__main__':
    solve()
```

### 4. 关键点总结

1. **逻辑闭环**：
    
    - **消耗时找便宜的** = 为了保留贵的油作为“资产”。
        
    - **溢出时退贵的** = 抛售劣质资产（高价油），保留优质资产（低价油）。
        
    - 这两个操作结合，保证了油箱里永远留着**当前能获得的最便宜的 $m$ 升油**。
        
2. **懒惰删除 (Lazy Deletion)**：
    
    - 注意 `get_valid_batch` 函数。因为 Python 的 `heapq` 不支持删除堆中任意元素。
        
    - 所以我们不主动删除。当某批油在 `min_heap` 被烧光后，它在 `max_heap` 里仍然存在。
        
    - 当我们下次在 `max_heap` 访问到它时，检查 `stock` 发现它是 0，就直接扔掉。这种“用到时再删”的技巧叫懒惰删除。
        
3. **时间复杂度**：
    
    - 每个加油站的油最多进堆一次、出堆（被烧掉或被退掉）一次。
        
    - 堆操作是 $O(\log N)$。
        
    - 总时间复杂度 **$O(N \log N)$**。
        
    - 对于题目中 $N=2 \times 10^5$ 的规模，能在 0.5秒左右跑完，完美通过。
