这道题目是一道典型的**数论 + 哈希表（字典）** 应用题。它的核心在于如何高效地找到两个数，使它们有大于1的公因数（即不互质），并满足特定的索引最小化要求。

以下是题目的详细分析和代码的详细注解。

### 1. 题目分析

- **目标**：在序列 $A$ 中找到两个下标 $i$ 和 $j$（$1 \le i < j \le n$），满足 $\text{gcd}(A_i, A_j) > 1$。
    [
- **优化目标**：
    
    1. 优先让 $i$ 最小。
        
    2. 如果 $i$ 相同，让 $j$ 最小。
        
- **暴力做法的问题**：如果直接两两比较（双重循环），复杂度是 $O(n^2)$。当 $n=10^5$ 时，$n^2 = 10^{10}$，肯定会超时（Time Limit Exceeded）。
    
- **高效思路（倒排索引）**：
    
    - 两个数有公因数 $>1$，意味着它们至少共享一个**质因数**。
        
    - 我们可以分解每个数的质因数。
        
    - 建立一个映射（字典）：`质因数 -> [出现该质因数的所有下标列表]`。
        
    - 例如：$A=[6, 15, 10]$。
        
        - 6 (idx 1): 质因数 2, 3
            
        - 15 (idx 2): 质因数 3, 5
            
        - 10 (idx 3): 质因数 2, 5
            
        - 字典变成：
            
            - 2: [1, 3]
                
            - 3: [1, 2]
                
            - 5: [2, 3]
                
    - 因为我们是按顺序遍历数组的，所以字典里的下标列表天然是有序的。对于任意质因数，列表中前两个下标就是该质因数对应的最小 $(i, j)$ 组合。
        
    - 最后遍历字典中所有的 key，比较找出最小的 $(i, j)$。
        

---

### 2. 代码详细注解

你的代码逻辑完全符合上述的高效思路。下面是逐行详细解释：

```Python
# 读取 n，表示数字的个数
n = int(input())
# 读取 n 个整数，存入列表 a
a = list(map(int, input().split()))

# 初始化最终结果 ii 和 jj。
# 因为题目要求找最小的 i 和 j，所以初始化为一个比 n 大的数（题目中 n 最大 10^5），
# 充当“无穷大”的作用，方便后续做 min 比较。
ii = 100001
jj = 100001

# 定义一个字典，用于记录： 质因数 -> [拥有该质因数的下标列表]
# 例如：{2: [1, 3, 5], 3: [2, 4]} 表示第1,3,5个数都有因子2
zyz_to_index = {}

# --- 第一阶段：遍历每个数，进行质因数分解并记录下标 ---
for i in range(n):
    x = a[i]
    # 使用 set 是为了去重。例如数字 12 (2*2*3)，因子是 2 和 3，
    # 我们只需要记录它包含 2，不需要重复记录两次 2。
    zyz = set()
    
    # 试除法分解质因数：从 2 遍历到 sqrt(x)
    for j in range(2, int(x ** 0.5) + 1):
        # 如果 x 能被 j 整除，说明 j 是 x 的一个因数
        if x % j == 0:
            # 这里的 while 循环是为了把 x 中包含的 j 全部除尽
            # 这样能保证后续找到的因数都是“质数”
            # 例如 x=12, j=2: 12%2==0 -> x=6; 6%2==0 -> x=3. 此时 zyz 加入 2
            # 下一次循环 j=3, x=3, 3%3==0 -> x=1. zyz 加入 3
            # 如果没有这个 while，遇到 x=12 时，j=4 也会判定为因数，但 4 不是质数。
            # 不过因为外层从小到大遍历，合数肯定先被它的质因子除尽了，所以这里得到的 j 必定是质数。
            while x % j == 0:
                x //= j
                zyz.add(j) # 记录质因子
    
    # 如果试除完后 x 仍然大于 1，说明剩下的这个 x 本身就是一个质因子
    # 例如 x=10, 循环除以 2 后剩 5，5 > sqrt(10)，循环结束，剩下的 5 是质因子
    if x > 1:
        zyz.add(x)

    # --- 将当前数字的下标 (i+1) 记录到对应的质因数列表中 ---
    for yz in zyz:
        if yz not in zyz_to_index:
            # 如果该质因数第一次出现，创建一个新列表，存入当前下标
            # 题目要求下标从 1 开始，所以用 i + 1
            zyz_to_index[yz] = [i + 1]
        else:
            # 如果该质因数已存在，将当前下标追加到列表末尾
            zyz_to_index[yz].append(i + 1)

# --- 第二阶段：遍历字典，寻找满足条件的最小 (i, j) ---
for zyz, index in zyz_to_index.items():
    # 只有当一个质因数对应的下标列表长度 >= 2 时，
    # 说明至少有两个数共享这个质因数，这才是有效的公因数
    if len(index) >= 2:
        # index 列表是按从小到大生成的（因为外层 i 是 0 到 n-1 遍历）
        # 所以 index[0] 一定是当前质因数下最小的 i，index[1] 是最小的 j
        curr_i = index[0]
        curr_j = index[1]
        
        # 比较逻辑：寻找全局最小的 i，如果 i 相同则找最小的 j
        # 条件 1: index[0] < ii (发现了更小的 i)
        # 条件 2: index[0] == ii 且 index[1] < jj (i 一样，但发现了更小的 j)
        if curr_i < ii or (curr_i == ii and curr_j < jj):
            ii = curr_i
            jj = curr_j

# 输出最终结果
print(ii, jj)
```

### 3. 复杂度分析

- **时间复杂度**：
    
    - 分解质因数部分：对于每个数 $A_i$，试除法最坏情况是 $O(\sqrt{A_i})$。总共有 $n$ 个数，所以这部分是 $O(n\sqrt{A_{max}})$。
        
    - $A_{max} = 10^6$，$\sqrt{A_{max}} = 1000$。计算量大约是 $10^5 \times 1000 = 10^8$ 次运算。在 Python 中这属于稍微有点慢但通常能通过（1~2秒限时）的范围。
        
    - 遍历字典部分：字典的 Key 数量取决于所有数包含的不同质因数数量，远小于上述计算量，基本可以忽略。
        
- **空间复杂度**：
    
    - $O(n \times k)$，其中 $k$ 是平均每个数的不同质因子个数。对于 $10^6$ 以内的数，不同质因子个数很少（例如 $2 \cdot 3 \cdot 5 \cdot 7 \cdot 11 \cdot 13 \cdot 17 > 10^6$，最多 7 个）。空间完全够用。
        

### 4. 代码小贴士与潜在优化

你的代码通过逻辑是完全正确的。如果想要代码运行得更快（针对特别严苛的时间限制），可以使用**欧拉筛（线性筛）预处理最小质因子（SPF - Smallest Prime Factor）**。

- **当前方法**：对每个数都做一次 $O(\sqrt{X})$ 的试除。
    
- **优化方法**：
    
    1. 预处理 $10^6$ 以内的 SPF 数组（每个数的最小质因子）。
        
    2. 分解 $X$ 时，只需要不断做 $X = X / SPF[X]$，复杂度降为 $O(\log X)$。
        
    3. 这样总复杂度会降到 $O(n \log A_{max})$，速度会快非常多。
        

不过，对于这道题的规模（$A_i \le 10^6$），目前的**试除法**写法通常是可以 AC（通过）的。

**总结**：代码逻辑清晰，利用了“公因数必共享质因子”的原理，将 $O(n^2)$ 问题转化为了因数分解归类问题，是非常标准的解法。
