
# 有限制的多段求和 DP


## 有限制的多段求和

当题目满足以下特征时，可以套用该模板：

1. **阶段性**：任务分为 $n$ 个步骤（如 $n$ 条街、掷 $n$ 次骰子、分 $n$ 个盒子）。
    
2. **范围限制**：每一步选择的数值 $x$ 有明确范围（如 $x \in [low, high]$）。
    
3. **总和限制**：要求所有步骤选择的数值总和恰好等于 $k$（或者 $\le k$）。
    
4. **线性转移**：当前状态只取决于前一个状态的一个**连续区间**之和。


### 1. 最基础的模板：三层循环 (Naive DP)

所有这类问题的起点都是**背包思想**。假设有 $n$ 个阶段（如 $n$ 条街），每阶段可以选 $[1, m]$ 范围内的数，求总和为 $k$ 的方案数。

**状态定义：**

$f[i][j]$ 表示前 $i$ 个阶段，总和恰好为 $j$ 的方案数。

**转移方程：**

$$
f[i][j] = \sum_{x=1}^{m} f[i-1][j-x]
$$

这意味着：为了让前 $i$ 个阶段和为 $j$，我最后一步可以选 $1, 2, \dots$ 或 $m$。如果最后一步选了 $x$，那么前 $i-1$ 步的和必须是 $j-x$。

**基础代码模板：**
```Python
# 复杂度 O(n * k * m)
for i in range(1, n + 1):
    for j in range(k + 1):
        for x in range(1, m + 1): # 这一层循环是性能瓶颈
            if j >= x:
                f[i][j] = (f[i][j] + f[i-1][j-x]) % mod
```

---

### 2. 演变过程：为什么要变？

当 $n, k, m$ 的规模达到 $10^3 \sim 10^5$ 时，$O(n \cdot k \cdot m)$ 会让你直接 TLE（超时）。

观察上面的转移方程，你会发现：

$f[i][j] = f[i-1][j-1] + f[i-1][j-2] + \dots + f[i-1][j-m]$

这本质上是在求**前一阶段 $f[i-1]$ 数组中一段连续区间的和**。

既然是求区间和，我们自然想到：**前缀和优化**。

---

### 3. 如何变到你给出的那一步？

优化的核心逻辑如下：

1. **定义前缀和**：令 $s[i]$ 表示 $\sum_{t=0}^{i-1} f[i-1][t]$。
    
2. **区间求和**：那么 $\sum_{x=1}^{m} f[i-1][j-x]$ 就可以写成 $s[j] - s[j-m]$。
    
3. **降维（滚动数组）**：既然 $f[i]$ 只取决于 $f[i-1]$，我们可以只开一个一维数组 `f` 代表上一层，`nf` 代表当前层。


##  代码模板
```Python
# 优化后复杂度 O(n * k)
for _ in range(n):
    # 1. 预处理上一层的辅助前缀和数组 s
    s = [0] * (k + 2)
    for i in range(k + 1):
        s[i+1] = (s[i] + f[i]) % mod
    
    nf = [0] * (k + 1)
    for j in range(1, k + 1):
        # 2. 用 s[r+1] - s[l] 代替内层循环
        l = max(0, j - m)
        r = j - 1
        if l <= r:
            nf[j] = (s[r+1] - s[l] + mod) % mod
    f = nf
# 如果是求“恰好等于 k”：print(f[k]) 
# 如果是求“不超过 k”：print(sum(f) % mod) 
print(f[k])
```
