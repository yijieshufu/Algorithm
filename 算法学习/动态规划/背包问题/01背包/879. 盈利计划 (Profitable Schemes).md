# 题目分析

- **本质：** 带有“至少”限制的二维 0-1 背包计数问题。
    
- **逻辑：** 1. $dp[i][j]$ 表示使用 $i$ 个员工，获得**至少** $j$ 利润的方案数。
    
    2. 特殊处理：若当前工作利润 $p$ 大于目标利润 $j$，则它满足“至少为 $j$”的条件，应索引至 `dp[i-g][0]`。
    
    3. 结果：对所有满足员工数限制的方案求和。


```Python
class Solution:
    def profitableSchemes(self, n: int, minProfit: int, group: List[int], profit: List[int]) -> int:
        mod = 10**9 + 7
        # dp[i][j] 使用 i 人，至少获得 j 利润
        dp = [[0] * (minProfit + 1) for _ in range(n + 1)]
        dp[0][0] = 1
        for g, p in zip(group, profit):
            for i in range(n, g - 1, -1):
                for j in range(minProfit, -1, -1):
                    # 核心：“至少为 j” 逻辑，若 j-p < 0，视为从 0 开始
                    k = max(0, j - p)
                    dp[i][j] = (dp[i][j] + dp[i - g][k]) % mod
        # 返回所有使用员工数 <= n 且利润满足要求的方案总和
        res = 0
        for i in range(n + 1):
            res = (res + dp[i][minProfit]) % mod
        return res
```
