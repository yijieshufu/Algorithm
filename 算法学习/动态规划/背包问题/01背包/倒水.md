属于 **0/1 背包问题**的一个变种 ———分组背包

# 题目 条件分析
## 1. 题目条件深入分析
根据题目描述，对于每一位客人 $i$，倒水的毫升数 $x$ 与满意度之间的关系是一个**分段函数**：

- **区间 $[0, a_i)$：** 满意度为 $e_i$。为了节省水资源给其他客人，这一档的最优选是倒 **$0$** 毫升。
    
- **区间 $[a_i, c_i)$：** 满意度为 $b_i$。这一档的最优选是倒 **$a_i$** 毫升。
    
- **区间 $[c_i, \infty)$：** 满意度为 $d_i$。这一档的最优选是倒 **$c_i$** 毫升。
    

> **核心逻辑：** 既然每一段区间内的满意度是恒定的，那么在同一段内，倒水越少越划算。因此，对于每个客人，我们只有 **3 种选择**：
> 
> 1. 花 **$0$** 毫升水，获得 **$e_i$** 满意度。
>     
> 2. 花 **$a_i$** 毫升水，获得 **$b_i$** 满意度。
>     
> 3. 花 **$c_i$** 毫升水，获得 **$d_i$** 满意度。
> 状态转移方程

## 算法建模：分组背包
由于每位客人必须且只能从上述 3 种方案中选择一种（哪怕是倒 0 毫升也算一种选择），这完全符合**分组背包**的模型：

- **物品组：** $n$ 个客人即为 $n$ 个组。
    
- **组内物品：** 每组有 3 个物品，对应的（体积, 价值）分别为 $(0, e_i), (a_i, b_i), (c_i, d_i)$。
    
- **背包容量：** 总水量 $m$。
    
- **目标：** 在每组中选且仅选一个物品，使得体积之和 $\le m$，且价值之和最大。

#  代码
```Python
import sys
# 极大值，确保负数满意度也能正确转移
INF = 10**15
# 使用迭代器读取，效率最高
it = iter(sys.stdin.read().split())
n = int(next(it))
m = int(next(it))
# f[j] 表示当前刚好消耗 j 毫升水时的最大满意度
f = [-INF] * (m + 1)
f[0] = 0
for _ in range(n):
    a = int(next(it)); b = int(next(it))
    c = int(next(it)); d = int(next(it))
    e = int(next(it))
    # 当前客人的三种决策方案
    options = [(0, e), (a, b), (c, d)]
    # 分组背包：必须从上一轮的状态 f 转移到新状态 nf
    nf = [-INF] * (m + 1)
    for j in range(m + 1):
        if f[j] <= -INF: continue
        # 尝试每种方案
        for w, v in options:
            if j + w <= m:
                # 只有当新方案更好时才更新
                if f[j] + v > nf[j + w]:
                    nf[j + w] = f[j] + v
    f = nf
# 最终结果是所有可能消耗的水量中，满意度最大的那个
print(max(f))
```

# 优化 一维数组倒序遍历

### 1. 为什么可以优化？

在你的代码中，`new_f` 是为了防止“在同一个人的三个选项中选了多次”。

如果我们使用一维数组 `f` 并从 $m$ 倒着遍历到 $0$：

- 当我们计算 `f[j]` 时，我们需要用到 `f[j - 0]`、`f[j - a]` 和 `f[j - c]`。
    
- 因为是**倒序**，此时的 `f[j]`、`f[j - a]` 和 `f[j - c]` 都还没有被这一轮循环更新过，它们依然保存着**上一个客人**结算后的结果。
    
- 这样我们就省去了每一轮创建新数组 `new_f` 的内存开销和时间。

### 2. 优化后的代码

```Python
import sys

INF = 10**15

def solve():
    it = iter(sys.stdin.read().split())
    
	n = int(next(it))
	m = int(next(it))


    # f[j] 表示当前刚好消耗 j 毫升水时的最大满意度
    f = [-INF] * (m + 1)
    f[0] = 0
    
    for _ in range(n):
        a = int(next(it)); b = int(next(it))
        c = int(next(it)); d = int(next(it))
        e = int(next(it))
        
        # 倒序遍历水量 j
        for j in range(m, -1, -1):
            # 重要：在这一轮中，我们要从上一轮的 f[j] 转移出三种新可能
            # 方案1: 倒 0 毫升 -> 满意度变 f[j] + e
            # 方案2: 倒 a 毫升 -> 满意度变 f[j-a] + b (前提 j >= a)
            # 方案3: 倒 c 毫升 -> 满意度变 f[j-c] + d (前提 j >= c)
            
            # 先暂存上一轮的状态，因为 f[j] 会被立即更新
            prev_f_j = f[j]
            
            # 必须保证是从一个合法状态（非 -INF）转移过来
            # 方案1 (必须选，所以直接更新 f[j])
            f[j] = prev_f_j + e if prev_f_j > -INF else -INF
            
            # 方案2
            if j >= a and f[j-a] > -INF:
                if f[j-a] + b > f[j]:
                    f[j] = f[j-a] + b
            
            # 方案3
            if j >= c and f[j-c] > -INF:
                if f[j-c] + d > f[j]:
                    f[j] = f[j-c] + d

    print(max(f))

solve()
```
