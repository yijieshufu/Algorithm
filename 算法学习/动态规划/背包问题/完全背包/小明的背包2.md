
**完全背包问题**

---

### 代码思路分析
#### 2. 一维滚动数组优化

在竞赛中，为了压低空间复杂度，我们通常将第一维（物品维度）去掉：

- **状态定义**：`dp[j]` 表示容量为 $j$ 时的最大价值。
    
- **转移方程**：`dp[j] = max(dp[j], dp[j - w] + v)`
    
- **核心差异（遍历顺序）**：
    
    - **0/1 背包**：内层循环必须**逆序**（`V` 到 `w`），以防止同一个物品在同一层被重复计算。
        
    - **完全背包**：内层循环必须**顺序**（`w` 到 `V`）。当我们计算 `dp[j]` 时，`dp[j-w]` 已经是这一轮（第 $i$ 种物品）更新过的值了，这正好符合“可以多次重复选择同一物品”的逻辑。

---

## 一维优化版（推荐，空间效率最高）

这是完全背包的最优写法。

```Python
import sys

# 一次性读入所有数据并转换为整数列表
a = list(map(int, sys.stdin.read().split()))

if a:
    n, v = a[0], a[1]
    f = [0] * (v + 1)
    
    # 使用指针 k 遍历数据列表
    k = 2
    for _ in range(n):
        wi, vi = a[k], a[k + 1]
        k += 2
        # 完全背包：正序遍历
        for j in range(wi, v + 1):
            if f[j - wi] + vi > f[j]:
                f[j] = f[j - wi] + vi
    print(f[v])
```
