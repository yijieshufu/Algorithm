
# 算法笔记：环形 DP (状态机+破环成链) - 仓库打工

**Tags**: #Algorithm #DP #RingDP #StateMachine #Lanqiao #Python  
**Difficulty**: Hard (思维转化题)

---

## 1. 核心题意大白话翻译 (破题眼)

**题目原话**：“上交每段连续仓库的第一个仓库的工资”。

**打工人视角翻译**：
*   **新开工**（昨天没干活，今天干活）：老板剥削，第一天工资 $c_i$ 没收，实际收益为 $0$。
*   **连续干**（昨天干活了，今天接着干）：工资 $c_i$ 稳稳拿进自己口袋。

> **总结**：只有当 $i$ 和 $i-1$ **同时被选中**时，才能获得 $c_i$ 的收益。

---

## 2. 线性状态转移 (先假设是一条直街)

不管三七二十一，先用**一维滚动数组**定义两个状态：
*   `f0[j]`: 总共干了 $j$ 天，且**今天摸鱼 (不选)** 的最大存款。
*   `f1[j]`: 总共干了 $j$ 天，且**今天打工 (必选)** 的最大存款。

**状态转移方程** (每天都要做抉择，注意 `j` 必须**逆序遍历**)：

1.  **今天摸鱼 (`f0`)**：  
    既然今天不干活，肯定没钱拿。直接继承昨天的最高存款（管他昨天干没干活）。  
    `f0[j] = max(昨天摸鱼 f0[j], 昨天打工 f1[j])`

2.  **今天打工 (`f1`)**：消耗 1 个名额 (看 $j-1$)。
    *   若昨天摸鱼：今天算新开工，工资没收（拿 0 元）$\implies$ `f0[j-1] + 0`
    *   若昨天打工：今天算连续干，拿到工资 $c_i$ $\implies$ `f1[j-1] + c_i`
    
    取两者最大值：`f1[j] = max(f0[j-1], f1[j-1] + c_i)`

## 3. 破环成链大法 (蓝桥杯必杀技：平行宇宙)

因为 1 号仓库和 $n$ 号仓库首尾相连，算 1 号时不知道 $n$ 号的状态，直接卡死。

**解法：强行规定 1 号仓库的命运，跑两次 DP！**

### 关键：`-INF` 是封印，锁死不可能的状态

| 宇宙 | 设定 | `f0[0]` (0名额,不选) | `f1[1]` (1名额,选) | 最终答案 |
|:---|:---|:---|:---|:---|
| **A** | 1号不选 | `0` ✅ 合法起点 | `-INF` ❌ 不可能 | `max(f0[m], f1[m])` |
| **B** | 1号必选 | `-INF` ❌ 不可能 | `0` ✅ 合法起点(工资先欠着) | `max(f0[m], f1[m]+c[1])` |

**宇宙B详解**：
*   `f1[1]=0`：先假设1号是新开工，工资被没收（不是`c[1]`）
*   最后看$n$号状态：
    *   $n$号摸鱼(`f0[m]`)：1号确实是新开工，收益就是`f0[m]`
    *   $n$号打工(`f1[m]`)：首尾相连，1号不是新开工，补发工资$c_1$

**最终答案**：`max(宇宙A, 宇宙B)`

## 4. 极速数据读入：iter + next 模式

### 为什么用 iter + next？

| 传统 `input().split()`  | `iter + next` 模式         |
| :-------------------- | :----------------------- |
| 多次调用 `input()`，IO 开销大 | **一次性读取全部数据**，速度极快       |
| 代码分散，处理不规则输入麻烦        | **统一迭代器接口**，逻辑清晰         |
| 大数据量可能 TLE            | **竞赛标配**，轻松应对 $10^5+$ 数据 |

### 标准模板






### iter + next 进阶技巧

```python
# 处理不确定长度的输入
it = iter(sys.stdin.read().split())
while True:
    try:
        x = int(next(it))
        # 处理 x...
    except StopIteration:
        break  # 数据读完，优雅退出
```

## 5. 考场 Bug 自查清单 (Checklist)

| 检查项            | 正确做法                             | 错误后果                  |
| :------------- | :------------------------------- | :-------------------- |
| **非法状态初始化**    | 用 `-10**16`                      | 用 `0` 会和"新开工收益0"混淆，WA |
| **滚动数组遍历顺序**   | `for j in range(m, -1, -1)`      | 正序会导致今天状态覆盖昨天，WA      |
| **$m=0$ 边界特判** | 开头 `if m == 0: print(0); return` | `f1[1]` 数组越界，RE       |
| **iter 空输入处理** | 包 `try...except StopIteration`   | 无数据时崩溃，RE             |
| **宇宙B补发工资**    | 别忘了 `+ c[1]`                     | 1号工资漏算，WA             |

## 6. 一句话记忆口诀

> **"iter-next 读入快，环形破链跑两遍，逆序滚动负无穷，宇宙B里补工资"**

## 代码

```python
import sys

def solve():
    # ========== 极速读入模板 ==========
    it = iter(sys.stdin.read().split())  # 一次性读入，空格分割，生成迭代器
    try:
        n = int(next(it))                # 用 next() 逐个取数据
        m = int(next(it))
        # 下标从 1 开始，补 0 占位
        c = [0] + [int(next(it)) for _ in range(n)]
    except StopIteration:
        return  # 处理空输入
    
    # 特判：都不干活，赚 0 元
    if m == 0:
        print(0)
        return

    INF = 10**16  # 极小值代表"绝对不可能"
    
    # ========== 核心 DP 封装 ==========
    def run_dp(init_f0, init_f1):
        f0 = [-INF] * (m + 1)
        f1 = [-INF] * (m + 1)
        f0[0] = init_f0      # 0名额，不选第1个的状态
        if m >= 1:
            f1[1] = init_f1  # 1名额，选第1个的状态
            
        for i in range(2, n + 1):
            val = c[i]
            # 滚动数组：必须逆序！用昨天的状态推今天
            for j in range(m, -1, -1):
                # 今天摸鱼：继承昨天最高
                f0[j] = max(f0[j], f1[j])
                
                # 今天打工
                if j > 0:
                    f1[j] = max(f0[j-1], f1[j-1] + val)
                    
        return f0, f1

    # ========== 平行宇宙推演 ==========
    
    # 宇宙 A：1号不选（f0合法，f1非法）
    f0, f1 = run_dp(0, -INF)
    ans1 = max(f0[m], f1[m])

    # 宇宙 B：1号必选（f0非法，f1合法且工资欠着）
    f0, f1 = run_dp(-INF, 0)
    # n号不选：1号确实是新开工；n号选：首尾相连，补发c[1]
    ans2 = max(f0[m], f1[m] + c[1])

    print(max(ans1, ans2))

if __name__ == '__main__':
    solve()
```
