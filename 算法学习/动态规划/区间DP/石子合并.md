# 石子合并 完整笔记

---

## 题目条件

| 项目 | 内容 |
|:---|:---|
| 输入 | $n$ 堆石子排成一排，第 $i$ 堆有 $a_i$ 个 |
| 操作 | 每次只能合并**相邻**的两堆 |
| 代价 | 合并的花费 = 两堆石子总数 |
| 目标 | 经过 $n-1$ 次合并后，求**总最小花费** |
| 数据范围 | $1 \le n \le 200$，$1 \le a_i \le 10^5$ |

---

## 样例分析

```
输入：4
      4 5 9 4

一种合并方式：
  第1步：合并 [4,5] → 代价 9，变成 [9, 9, 4]
  第2步：合并 [9,4] → 代价 13，变成 [9, 13]
  第3步：合并 [9,13] → 代价 22
  总代价：9 + 13 + 22 = 44

输出：44
```

---

## 分析过程

### 第一步：为什么贪心不行？

```
反例：[1, 2, 3]

贪心策略（每次合最小的相邻对）：
  合并 [1,2] → 代价3，变成 [3,3]
  合并 [3,3] → 代价6
  总代价：9

另一种方式：
  合并 [2,3] → 代价5，变成 [1,5]
  合并 [1,5] → 代价6
  总代价：11

  
实际：需要枚举所有可能，贪心无法得到全局最优
```

**结论**：合并顺序影响最终结果，必须用动态规划枚举所有情况。

---

### 第二步：关键观察——最后一步

```
无论怎么合并，最后一步一定是这样：

    [i, i+1, ..., k]  和  [k+1, k+2, ..., j]
           ↓                    ↓
         已合并                已合并
           ↓                    ↓
          一堆    ←——合并——→    一堆
                    ↓
              [i, ..., j] 一大堆
```

**核心**：最后一步把某个 `k` 左边的区间和右边的区间合并。

---

### 第三步：状态设计

| 状态 | 含义 | 边界 |
|:---|:---|:---|
| `f[i][j]` | 将第 `i` 堆到第 `j` 堆合并成一堆的**最小代价** | `f[i][i] = 0` |

---

### 第四步：状态转移方程

```
f[i][j] = min{ f[i][k] + f[k+1][j] + sum(i,j) }  其中 i ≤ k < j
		           ↑        ↑           ↑
		         左区间   右区间    最后合并代价
```

**前缀和优化**：
```python
s[i] = a[1] + a[2] + ... + a[i]
sum(i,j) = s[j] - s[i-1]
```

**最终方程**：
$$f[i][j] = \min_{i \le k < j} \{f[i][k] + f[k+1][j]\} + s_j - s_{i-1}$$

---

### 第五步：计算顺序

```
必须保证：算大区间时，小区间已经算好了

正确顺序：按区间长度从小到大
  长度 = 2：f[1,2], f[2,3], f[3,4] ...
  长度 = 3：f[1,3], f[2,4], f[3,5] ...
  长度 = n：f[1,n] ← 答案
```

## 完整代码

```python
import sys

# 快速输入
input = sys.stdin.readline

def solve():
    n = int(input())
    # 下标从1开始，方便处理
    a = [0] + list(map(int, input().split()))
    
    # 前缀和：s[i] = a[1] + ... + a[i]
    s = [0] * (n + 1)
    for i in range(1, n + 1):
        s[i] = s[i - 1] + a[i]
    
    # dp[i][j]：合并区间[i,j]的最小代价
    # 初始化：dp[i][i] = 0（单堆无需合并），其他为无穷大
    dp = [[0] * (n + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for j in range(i + 1, n + 1):
            dp[i][j] = float('inf')
    
    # 区间DP：按长度从小到大枚举
    for length in range(2, n + 1):           # 区间长度
        for i in range(1, n - length + 2):   # 起点i
            j = i + length - 1                # 终点j
            # 枚举分割点k：i ≤ k < j
            for k in range(i, j):
                # 左区间[i,k] + 右区间[k+1,j] + 合并代价
                cost = dp[i][k] + dp[k + 1][j] + s[j] - s[i - 1]
                if cost < dp[i][j]:
                    dp[i][j] = cost
    
    print(dp[1][n])

if __name__ == "__main__":
    solve()
```

---

## 复杂度分析

| 项目 | 复杂度 | 说明 |
|:---|:---|:---|
| 状态数 | $O(n^2)$ | 所有区间 $[i,j]$ |
| 每个状态转移 | $O(n)$ | 枚举分割点 $k$ |
| 总时间 | $O(n^3)$ | $200^3 = 8 \times 10^6$，可过 |
| 空间 | $O(n^2)$ | dp数组 |

---

## 关键点总结

| 要点 | 说明 |
|:---|:---|
| **循环顺序** | 外层必须是**区间长度**，不是起点 |
| **前缀和** | 用 `s[j]-s[i-1]` O(1)算区间和 |
| **初始化** | `dp[i][i]=0`，其他为`inf` |
| **分割点** | `k` 从 `i` 到 `j-1`，左右区间是 `[i,k]` 和 `[k+1,j]` |

# 破环成链：复制一份
a = [0] + list(map(int, input().split()))
a = a + a[1:n+1]  # 长度变为2n

# 在2n长度上DP，答案取所有长度为n的区间最小值
ans = min(dp[i][i+n-1] for i in range(1, n+1))
print(ans)
