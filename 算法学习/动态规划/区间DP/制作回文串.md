
# 📝 算法笔记：最小代价回文串 (极致空间优化版)

## 📖 题目描述

给定长度为 $M$ 的字符串 $S$。通过**增加**或**删除**字符，将 $S$ 变为回文串。每种字符的操作代价不同，求最小总花费。

## 🧠 核心逻辑分析

### 1. 贪心简化：增删等价性 ⚖️

在回文匹配中，如果字符 $S[i]$ 无法配对，我们有两种选择：

- 在另一侧**添加**一个 $S[i]$。
    
- 将当前的 $S[i]$ **删除**。
    
    为了达到最小花费，我们预处理每个字符的代价为：
    
    $$w[c] = \min(\text{insert\_cost}(c), \text{delete\_cost}(c))$$
    

### 2. 状态定义与转移 动态规划

设 $f[i][j]$ 为子串 $S[i \dots j]$ 变成回文的最小代价。

- **若 $S[i] == S[j]$**：
    
    $$f[i][j] = f[i+1][j-1]$$
    
- **若 $S[i] \neq S[j]$**：
    
    $$f[i][j] = \min(f[i+1][j] + w[S[i]], f[i][j-1] + w[S[j]])$$
    

---

## 🚀 空间优化之路：从 $O(M^2)$ 到 $O(M)$

### 1. 滚动数组 (双行版)

利用 $i$ 逆序遍历，使用 `f` 存当前行，$pf$ 存上一行。

> **注意**：必须使用 `pf = f[:]` 进行**浅拷贝**，否则 `pf` 会随 `f` 同步变化，破坏状态。

### 2. 一维数组 (最终优化版) 🗜️

为了省去数组拷贝的开销，我们只用一个数组 `f`。

在计算新的 `f[j]` 时：

- **`f[j]` 的旧值**：代表“下方”依赖 $f[i+1][j]$。
    
- **`f[j-1]` 的新值**：代表“左侧”依赖 $f[i][j-1]$。
    
- **`pre` 变量**：代表“左下方”依赖 $f[i+1][j-1]$。我们在更新 `f[j-1]` 之前将其旧值存入 `pre`，供计算 `f[j]` 时使用。
    

---

## 💻 完整实现 (Python 3.7.3)

```Python
import sys

# 1. 使用迭代器高效处理输入
it = iter(sys.stdin.read().split())

n = int(next(it))
m = int(next(it))
s = next(it)

# 2. 预处理最小代价 w
w = [0] * 128
for _ in range(n):
    char = next(it)
    ins_v = int(next(it))
    del_v = int(next(it))
    w[ord(char)] = min(ins_v, del_v)

# 3. 准备数据 (下标从 1 开始)
a = [0] + [ord(c) for c in s]
f = [0] * (m + 1)

# 4. 区间 DP (一维数组优化)
for i in range(m - 1, 0, -1):
    pre = 0  # pre 存的是 f[i+1][j-1]，初始 i=j 时为 0
    for j in range(i + 1, m + 1):
        tmp = f[j]  # 备份“下方”值，它将是下一个循环的“左下方”
        
        if a[i] == a[j]:
            f[j] = pre  # 匹配：取左下方值
        else:
            # 不匹配：取 min(下方 + w[i], 左侧 + w[j])
            v1 = f[j] + w[a[i]]
            v2 = f[j-1] + w[a[j]]
            f[j] = v1 if v1 < v2 else v2
            
        pre = tmp  # 更新接力棒

print(f[m])
```

## 💡 总结

|**优化点**|**描述**|**收益**|
|---|---|---|
|**I/O 优化**|使用 `sys.stdin.read().split()` 配合 `iter`|提升大规模数据读取速度|
|**贪心思想**|插入与删除代价合并|简化状态转移逻辑|
|**一维 DP**|使用 `pre` 变量替代数组拷贝|内存占用极低，减少耗时操作|
