## 一、题目理解

### 场景
- 长度 $n$ 的木版，初始无色
- 每次操作：选**连续区间**涂成**同一种颜色**（后涂覆盖先涂）
- 目标：用最少的次数涂成目标字符串

### 关键洞察
> **后涂覆盖先涂** → 可以先用大区间打底，再用小区间覆盖细节

**示例**：`RGBGR`（最优3次）
```python
1. 涂 [1,5] 为 R  →  RRRRR
2. 涂 [2,4] 为 G  →  RGGGR  
3. 涂 [3,3] 为 B  →  RGBGR ✓
```

---

## 二、区间 DP 设计

### 状态
```python
f[i][j] = 将区间 [i, j] 涂成目标所需最少次数
```

### 转移方程

| 情况             | 方程                                    | 含义                   |
| :------------- | :------------------------------------ | :------------------- |
| `i == j`       | `f[i][i] = 1`                         | 单点必须涂1次              |
| `s[i] == s[j]` | `f[i][j] = min(f[i+1][j], f[i][j-1])` | 首尾同色，涂子区间时**顺带**涂另一端 |
| `s[i] != s[j]` | `f[i][j] = min(f[i][k] + f[k+1][j])`  | 首尾异色，必须**分割**成两段独立涂  |

---

## 三、"顺带涂" 核心原理

### 当 `s[i] == s[j]` 时

**关键**：涂子区间 `[i+1, j]` 的过程中，**必然有一笔是涂颜色 `s[j]` 的**（因为位置 `j` 最终要是这个颜色）。由于 `s[i]` 和 `s[j]` 相同，这一笔**往左多涂一格**就把位置 `i` 也覆盖了。

**示例**：`s = "ABA"`，`f[1][3]`

```python
子问题 f[2][3] = "BA"：
  第1笔：涂 [2,2] 为 'B'  → _ B _
  第2笔：涂 [2,3] 为 'A'  → _ B A  （这一笔颜色是 'A'）

因为 s[1]='A' 和 s[3]='A' 相同，把第2笔改成涂 [1,3]：
  第1笔：涂 [2,2] 为 'B'  → _ B _
  第2笔：涂 [1,3] 为 'A'  → A B A  ✓

次数不变 = 2次！
```

所以 `f[1][3] = f[2][3] = 2`，不需要 `+1`。

## 五、关键细节

| 要点       | 说明                                                   |
| :------- | :--------------------------------------------------- |
| **循环顺序** | 先长度 `l` 后左端点 `i`，确保子区间已计算                            |
| **下标处理** | `s = ' ' + s` 实现1-based，避免越界判断                       |
| **首尾相同** | 不是 `f[i+1][j-1]`！是 `f[i+1][j]` 或 `f[i][j-1]`（保留一端延伸） |
| **分割范围** | `k ∈ [i, j-1]`，确保左右子区间都非空                            |
