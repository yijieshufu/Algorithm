
## 题目与核心思路：汇总最终确定的逻辑

**题目：** 计算区间 $[l, r]$ 内包含子串 “2023” **或** “14” 的幸运数字个数。

**核心思路：**

1. **区间转化**：利用前缀和思想，计算 $[l, r]$ 的个数等价于求 $f(r) - f(l-1)$。
    
2. **数位 DP 状态设计**：
    
    - `p`：当前处理的数位（从高到低）。
        
    - `s2`：匹配 “2023” 的进度（0:无, 1:"2", 2:"20", 3:"202"）。
        
    - `s1`：匹配 “14” 的进度（0:无, 1:"1"）。
        
    - `ok`：布尔值，表示是否已经包含目标子串。一旦为 True，后续数位无论填什么该数都为幸运数字。
        
    - `lim`：数位上限限制开关。
        
    - `ld`：前导零开关，防止补位的 0 被误计入匹配逻辑。
        
3. **状态转移**：通过当前填入的数字 `i` 不断更新进度条 `s1` 和 `s2`，或直接激活 `ok` 状态。

## 我的盲点记录：重点记录互动环节中的疑问及解释

1. **ok 标志的权威性**：
    
    - _疑问_：在匹配完成 “2023” 时，是修改进度 `ns2` 还是 `ok`？
        
    - _解释_：虽然 `ns2` 可以继续变化，但 `ok` 是最终的“判定开关”。一旦满足条件，`ok` 置为 1，这个数字就永远贴上了“幸运”标签，简化了后续的逻辑判断。 🎯
        
2. **前导零的“隔离效应”**：
    
    - _疑问_：为什么前导零阶段不能进入匹配？
        
    - _解释_：如果前导零也参与匹配，那么像数字 `23`（在高位补零后）可能会因为前导零而错误触发 “2023” 里的 `0` 匹配。只有当填入第一个非零数字后，匹配才真正开始。 🚫0️⃣
        
3. **状态机转移的细节**：
    
    - _理解点_：理解了 `s2` 在填入 `0` 或 `2` 时，如何根据之前的进度进行“升级”或“重置”（例如 “202” 后面填 `0` 会变回进度 “20”）。

## 最终范本代码：一份最简洁且符合我习惯的最终代码

```Python
import sys
from functools import lru_cache

# 提高递归深度
sys.setrecursionlimit(2000)

def f(n):
    if n <= 0: return 0
    # 1-indexed 数组储存数位
    a = [0] + [int(x) for x in str(n)]
    m = len(a) - 1

    @lru_cache(None)
    def dp(p, s2, s1, ok, lim, ld):
        # 递归终点
        if p > m: return int(ok)
        
        res = 0
        up = a[p] if lim else 9
        for i in range(up + 1):
            nl = lim and (i == up)
            nld = ld and (i == 0)
            
            if nld:
                # 处于前导零状态，所有匹配进度保持 0
                res += dp(p + 1, 0, 0, False, nl, True)
            else:
                nok, ns2, ns1 = ok, 0, 0
                if ok:
                    nok = True
                elif (s2 == 3 and i == 3) or (s1 == 1 and i == 4):
                    # 触发匹配成功
                    nok = True
                else:
                    # 匹配 "2023"
                    if i == 2: ns2 = 3 if s2 == 2 else 1
                    elif i == 0: ns2 = 2 if s2 in [1, 3] else 0
                    # 匹配 "14"
                    if i == 1: ns1 = 1
                
                res += dp(p + 1, ns2, ns1, nok, nl, False)
        return res

    return dp(1, 0, 0, False, True, True)

# 使用 sys.stdin.read 和 iter 快速读入
raw = iter(sys.stdin.read().split())
l = int(next(raw))
r = int(next(raw))

print(f(r) - f(l - 1))
```

## 边界与坑点：总结该题容易出错的地方

- **$10^{12}$ 范围**：数据量极大，必须使用数位 DP + 记忆化搜索，严禁暴力枚举。 ⚡
    
- **区间减法**：求 $[l, r]$ 时，记得是 $f(r) - f(l-1)$，减错或漏减 $1$ 是常见错误。
    
- **s2 状态的重叠更新**：在处理 “2023” 匹配时，比如填入 `0`，要考虑它可能让进度从 `1`("2") 变成 `2`("20")，也可能让进度从 `3`("202") 回退到 `2`("20")。
    
- **递归清理**：由于有 $l$ 和 $r$ 两次调用，且 `a` 和 `m` 随 $n$ 改变，务必确保 `lru_cache` 能够正确区分不同 $n$ 下的计算结果（代码中将 `dp` 定义在 `f(n)` 内部已自动处理此问题）。
