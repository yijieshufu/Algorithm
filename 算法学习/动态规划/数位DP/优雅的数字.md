## 题目与核心思路：汇总最终确定的逻辑

- **题目目标**：在区间 $[L, R]$（最大达 $10^{18}$）中，统计十进制表示中非零数字个数不超过 3 个的正整数数量。
    
- **核心算法：数位 DP** 🧠
    
    - **前缀和转化**：将问题转化为 $\text{count}(R) - \text{count}(L-1)$。
        
    - **DFS 状态定义**：`dfs(i, j, limit)`。
        
        - `i`：剩余待填写的位数（从高位向低位处理）。
            
        - `j`：当前路径已使用的非零数字个数。
            
        - `limit`：当前位是否受到数字上界的限制。
            
    - **记忆化搜索**：使用 `dp[i][j]` 存储在不受限（`limit=False`）情况下，剩余 `i` 位且已用 `j` 个非零数的方案数。该表在多组测试数据间可以共用。

## 我的盲点记录：重点记录互动环节中的疑问及解释

- **前导零的处理**：在统计“非零数字个数”时，前导零对结果没有贡献（计入 0 个），且不影响数字的“优雅性”，因此本题不需要专门维护 `lead` 状态变量。
    
- **数字 0 的抵消逻辑**：`count(x)` 函数会将数字 `0` 计入（因为 0 个非零数字 $\le 3$）。但在计算前缀和差值时，`count(R)` 和 `count(L-1)` 都包含了 `0`，相减后自动抵消，结果即为 $[1, R]$ 区间内的数量。
    
- **记忆化共用的原理**：由于 `dp[i][j]` 记录的是剩余位数的填充方案，这与具体的数字上界无关。只要 `limit` 为 `False`，后续的搜索空间是完全一致的。

## 最终范本代码：最简洁且符合 Python 习惯的代码

```Python
import sys

# 提高递归深度
sys.setrecursionlimit(2000)

# 全局共用 dp 数组: dp[剩余位数][已用非零数个数]
dp = [[-1] * 4 for _ in range(25)]

def dfs(i, j, limit, a, n):
    if j > 3: return 0
    if i == 0: return 1
    
    # 只有在不受限时才读写 dp 数组
    if not limit and dp[i][j] != -1:
        return dp[i][j]
    
    res = 0
    # 确定当前位上界: limit 为 True 则受 a 限制，否则为 9
    up = a[n - i + 1] if limit else 9
    
    for d in range(up + 1):
        # 递归处理下一位: 剩余位数减 1，更新非零计数 j
        res += dfs(i - 1, j + (1 if d > 0 else 0), limit and (d == up), a, n)
    
    if not limit:
        dp[i][j] = res
    return res

def count(x):
    if x < 0: return 0
    s = str(x)
    n = len(s)
    # 1-indexed 数组 a，方便与位次对应
    a = [0] + [int(c) for c in s]
    return dfs(n, 0, True, a, n)

def solve():
    it = iter(sys.stdin.read().split())
    try:
        line = next(it)
        m = int(line)
        for _ in range(m):
            l = int(next(it))
            r = int(next(it))
            # 输出 count(R) - count(L-1)
            sys.stdout.write(str(count(r) - count(l - 1)) + '\n')
    except StopIteration:
        pass

solve()
```

## 边界与坑点：总结该题容易出错的地方

1. **递归深度**：Python 默认递归限制较小，处理 $10^{18}$ 级别的数位 DP 必须手动调大 `sys.setrecursionlimit`。
    
2. **上界限制逻辑**：`limit` 状态必须正确传递，只有当当前位受限且填入的数字等于 `up` 时，下一位才继续受限。
    
3. **下标偏移**：在使用 1-indexed 数组 `a` 时，要注意 `a[n-i+1]` 与当前处理位数 `i`（剩余位数）之间的转换逻辑。
    
4. **$L-1$ 风险**：题目给出的 $L$ 最小为 1，计算 $L-1$ 得到 0 是安全的，但要确保代码逻辑能正确处理 $x=0$ 的输入。
