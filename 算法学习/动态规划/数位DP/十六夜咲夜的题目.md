
## 题目与核心思路 🧩

**题目目标**：统计 $1 \sim N$ 范围内，十六进制表示下恰好包含 $k$ 个不同数字的个数。结果对 $10^9 + 7$ 取模。

**核心逻辑**：

由于 $N$ 的位数极长（$2 \times 10^5$），且十六进制有 16 种字符，直接带掩码的 DP 状态数高达 $2^{16} \times L$，在 Python 中会超时。我们采用**降维打击**：

1. **预处理转移表**：定义 `dp[i][j]` 为还剩 $i$ 位待填，当前已用 $j$ 种不同数字，最终能凑够 $k$ 种的方案数。
    
    - 填旧数字：$j \times dp[i-1][j]$
        
    - 填新数字：$(16-j) \times dp[i-1][j+1]$
        
2. **分段统计**：
    
    - **长度小于 $N$**：首位 15 种，后续 $L-1$ 位填空。
        
    - **长度等于 $N$**：逐位收缩。前缀保持一致，当前位填比 $N$ 小的数，后续位查表。
        
    - **检查本身**：如果 $N$ 的字符种类恰好为 $k$，答案加 1。

## 我的盲点记录 🧠

在互动过程中，我们重点攻克了以下几个关键点：

- **进制转换**：`int(s, 16)` 的 `16` 代表十六进制，它能把字符（如 `'E'`）精准映射到数值（`14`）。
    
- **DP 初始状态**：`dp[0][k] = 1` 是我们的“终点线”🏁。只有当最后剩余 0 位且凑够了 $k$ 种时，才算 1 种有效方案，否则之前所有的路径累加都会归零。
    
- **剪枝逻辑**：当当前前缀的数字种类已经超过了 $k$，后续无论怎么填，种类只会增加或保持不变，不可能再回到 $k$，因此必须直接 `break` 提前结束循环。✂️
    
- **位运算计数**：在 Python 3.7.3 环境下，使用 `bin(m).count('1')` 来统计位掩码中 1 的个数（即已用种类数），这是最稳健的替代方案。

## 最终范本代码 💻



```Python
import sys

# 按照 Python 3.7.3 习惯，不使用 class/main，使用 sys.stdin.read
it = iter(sys.stdin.read().split())
s = next(it)
k = int(next(it))

if k > 16:
    print(0)
    sys.exit()

mod = 10**9 + 7
n = len(s)

# 1-indexed 数组 a 存储 N 的每一位数值
a = [0] * (n + 1)
for i in range(1, n + 1):
    a[i] = int(s[i-1], 16)

# dp[i][j] 表示剩余 i 位，当前已用 j 种，达到 k 种的方案数
dp = [[0] * 18 for _ in range(n + 1)]
dp[0][k] = 1 
for i in range(1, n + 1):
    for j in range(17):
        # 选已用过的数字
        res = (j * dp[i-1][j]) % mod
        # 选没用过的数字
        if j + 1 <= 16:
            res = (res + (16 - j) * dp[i-1][j+1]) % mod
        dp[i][j] = res

ans = 0

# 1. 长度比 N 短的数
for i in range(1, n):
    ans = (ans + 15 * dp[i-1][1]) % mod

# 2. 长度等于 N 且数值小于 N 的数
m = 0 # 掩码记录前缀已用的数字
for i in range(1, n + 1):
    low = 1 if i == 1 else 0
    for d in range(low, a[i]):
        nm = m | (1 << d)
        c = bin(nm).count('1')
        ans = (ans + dp[n-i][c]) % mod
    
    m |= (1 << a[i])
    # 剪枝：如果当前种类已经超过 k，后面怎么填都不行
    if bin(m).count('1') > k:
        break
else:
    # 3. 检查 N 本身
    if bin(m).count('1') == k:
        ans = (ans + 1) % mod

print(ans % mod)
```

## 边界与坑点 ⚠️

- **前导零**：在统计短位数字和等长数字的第一位时，必须跳过数字 `0`。
    
- **K 的范围**：如果 $k > 16$，在十六进制下是永远无法达成的，应直接返回 `0`。
    
- **位运算溢出**：虽然 Python 处理大整数很强，但位掩码 `1 << 15` 在某些逻辑中需要确保理解其代表的是第 16 种状态。
    
- **模运算**：在每一处加法和乘法后都要及时 `% mod`，防止数值增长过快影响性能。
