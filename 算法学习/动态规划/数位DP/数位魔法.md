## 题目与核心思路：汇总最终确定的逻辑

**题目简述**：给定正整数 $n$（很大）和 $k$，求 $1 \sim n$ 中各位数字之和是 $k$ 的倍数的个数。

**核心逻辑**：

采用**数位 DP** 的思想，通过“填数”的方式从高位到低位构造数字。为了控制数字不超过 $n$，我们将状态分为三类：

1. **稳赢状态 (`dp`)**：之前某位填的数已经比 $n$ 对应位小了，之后每一位都可以自由填 $0 \sim 9$。
    
2. **分支状态**：当前位填入比 $n$ 对应位小的数字，使该分支从“贴合”转为“稳赢”。
    
3. **锁定状态 (`ts`)**：目前每一位都和 $n$ 一样。

---

## 我的盲点记录：重点记录及解释

- **关于 `ts` 的作用**：`ts` 记录的是那个“唯一直贴上界”的数的数字之和余数。它不参与 `dp` 数组的批量运算，而是作为逻辑 B 产生新“稳赢”分支的基准。
    
- **逻辑 B 与 C 的边界**：在处理当前位 $v$ 时，填入 $0 \sim v-1$ 的数会进入 `dp` 数组（逻辑 B），而填入 $v$ 则仅更新 `ts`（逻辑 C）。
    
- **为什么要减 1**：DP 过程会自然产生前缀全为 $0$ 的情况（如 `000`）。因为 $0 \pmod k = 0$，它会被计入合法答案，但题目要求范围是 $1 \sim n$，所以最后必须减去这个全 $0$ 的情况。

---

## 最终范本代码：Python 3.7.3

```Python
import sys

# 使用 sys.stdin.read 获取全部输入，配合 iter 和 next
s_in = sys.stdin.read().split()
it = iter(s_in)

s = next(it)
k = int(next(it))
MOD = 998244353

m = len(s)
# 1-indexed 数组 a 存储 n 的每一位
a = [0] + [int(c) for c in s]

# dp[j] 记录余数为 j 的“稳赢”方案数
dp = [0] * k
ts = 0

for i in range(1, m + 1):
    v = a[i]
    nf = [0] * k
    
    # 逻辑 A：已稳赢状态转移 (0-9 自由填)
    for j in range(k):
        if dp[j] == 0: continue
        for d in range(10):
            nj = (j + d) % k
            nf[nj] = (nf[nj] + dp[j]) % MOD
            
    # 逻辑 B：从锁定状态转为稳赢 (填 0 到 v-1)
    for d in range(v):
        nj = (ts + d) % k
        nf[nj] = (nf[nj] + 1) % MOD
        
    # 逻辑 C：维持锁定状态 (填 v)
    ts = (ts + v) % k
    dp = nf

# 最终结果：稳赢数 + 刚好等于 n 的数 - 全 0 的情况
ans = (dp[0] + (1 if ts == 0 else 0) - 1 + MOD) % MOD
print(ans)
```

## 边界与坑点：总结该题容易出错的地方

1. **$n$ 的规模**：$n$ 高达 $10^{1000}$，必须按字符串处理，不能转为整数。
    
2. **取模运算**：在执行 `-1` 操作时，为了防止 Python 在某些极端环境下对负数取模的实现差异（虽然 Python 默认支持），建议加上 `MOD` 再取模。
    
3. **时间复杂度**：本算法复杂度为 $O(m \cdot k \cdot 10)$。当 $m=1000, k=100$ 时，运算量约为 $10^6$，在 $2s$ 内可以通过。
    
4. **前导零**：本题不涉及数字长度限制或特定位要求，因此直接将“全 0”视为普通数字处理，最后减掉即可，简化了状态定义。
