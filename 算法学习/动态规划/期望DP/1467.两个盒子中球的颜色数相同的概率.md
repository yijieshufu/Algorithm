
# LeetCode 1467. 两个盒子中球的颜色数相同的概率

**Tags**: #LeetCode #Math #Combinatorics #Backtracking #Probability #Hard  
**Link**: [LeetCode 1467](https://leetcode.cn/problems/probability-of-a-two-boxes-having-the-same-number-of-distinct-balls/)

## 1. 题目核心分析

### 题目描述简述
给定 $2n$ 个球，共有 $k$ 种颜色，`balls[i]` 表示第 $i$ 种颜色球的数量。  
将所有球**随机打乱**，平分放入两个盒子（每个盒子 $n$ 个球）。  
求：**两个盒子中「不同颜色的数量」相等**的概率。

### 关键条件拆解
> [!INFO] 核心约束
> 1.  **球的总数守恒**：$\sum \text{balls} = 2n$。
> 2.  **平分条件**：盒子 A 球数 = $n$，盒子 B 球数 = $n$。
> 3.  **成功判定**：$Count(\text{Distinct Colors in A}) == Count(\text{Distinct Colors in B})$。
> 4.  **盒子差异性**：题目指出“两个盒子是不同的”，但由于我们计算的是概率 $P = \frac{\text{Valid}}{\text{Total}}$，只要分子分母对于“是否有序”的定义保持一致即可。通常**视所有球为独立个体**（即使颜色相同也打上编号）模型最简单。

---

## 2. 数学模型构建

### 视角选择：组合视角 (Combinations)
为了方便计算，我们假设**所有的球都是独一无二的**（例如给相同颜色的球编号 1, 2, ...）。

#### 分母：总样本空间 (Total Cases)
从总共 $2n$ 个球中，选出 $n$ 个球放入盒子 A（剩下的自动归入盒子 B）。

$$
\text{Total} = \binom{2n}{n}
$$

#### 分子：合法方案数 (Valid Cases)
我们需要遍历所有可能的**颜色分配方案**。  
假设对于第 $i$ 种颜色（总数 $C_i$），盒子 A 分到了 $a_i$ 个，盒子 B 分到了 $b_i$ 个（$b_i = C_i - a_i$）。

对于某一种具体的分配方案 $(a_0, a_1, \dots, a_{k-1})$，它对应的组合数贡献为：

$$
\text{Ways} = \binom{C_0}{a_0} \times \binom{C_1}{a_1} \times \dots \times \binom{C_{k-1}}{a_{k-1}}
$$

> **含义**：从第 0 种颜色的 $C_0$ 个球里选 $a_0$ 个给 A，从第 1 种颜色的 $C_1$ 个球里选 $a_1$ 个给 A……以此类推。

#### 合法性判断
一个分配方案需同时满足：
1.  **球数校验**：$\sum a_i = n$
2.  **颜色数校验**：$\sum_{i} [a_i > 0] = \sum_{i} [b_i > 0]$

## 3. 算法设计 (DFS/回溯)

由于颜色种类 `balls.length` 很小 ($1 \le k \le 8$)，我们可以使用 **DFS (深度优先搜索)** 暴力枚举每种颜色的分配情况。

### 状态定义
`dfs(index, count_A, distinct_A, distinct_B)`
*   `index`: 当前正在处理第几种颜色。
*   `count_A`: 盒子 A 目前已有的球总数。
*   `distinct_A`: 盒子 A 目前的颜色种类数。
*   `distinct_B`: 盒子 B 目前的颜色种类数。

> [!TIP] 剪枝优化
> 1. 如果 `count_A > n`，说明 A 拿多了，剪枝。
> 2. 可以计算 `count_B` （通过总处理球数 - `count_A`），若 `count_B > n`，说明 B 拿多了，剪枝。

### 转移方程
对于当前颜色 `i`，总数为 `total = balls[i]`。  
枚举盒子 A 拿取的数量 `k` ($0 \le k \le total$)：
1.  若 $k=0$：A 的颜色种数不变，B 的颜色种数 +1。
2.  若 $k=total$：A 的颜色种数 +1，B 的颜色种数不变。
3.  若 $0 < k < total$：A 和 B 的颜色种数都 +1。

**累加贡献**：

$$
\text{current\_ways} = \binom{balls[i]}{k} \times \text{dfs}(\dots)
$$

## 4. 代码实现 (Python)

```python
import math

class Solution:
    def getProbability(self, balls: List[int]) -> float:
        total_balls = sum(balls)
        n = total_balls // 2
        k_colors = len(balls)
        
        # 预计算总组合数（分母）
        total_cases = math.comb(total_balls, n)
        
        # DFS 状态: (当前颜色索引, A盒球数, A盒颜色数, B盒颜色数)
        # 返回值: 该状态下满足条件的组合数之和
        
        # 使用 memoization 优化（虽然本题数据规模较小，不用也可以）
        memo = {}

        def dfs(idx, count_a, distinct_a, distinct_b):
            if count_a > n: # 剪枝：A拿多了
                return 0
            
            # 同样可以判断 B 是否拿多了: 
            # 当前处理过的球总数 = sum(balls[:idx])
            # B当前的球数 = 当前处理过的球 - count_a
            # 如果 B当前的球数 > n，也可以剪枝，但这里不做复杂计算简化逻辑
            
            if idx == k_colors:
                # 只有当 A 刚好拿了 n 个球，且颜色种类相等时，才算有效
                return 1 if count_a == n and distinct_a == distinct_b else 0
            
            state = (idx, count_a, distinct_a, distinct_b)
            if state in memo:
                return memo[state]
            
            res = 0
            current_ball_count = balls[idx]
            
            # 枚举 A 拿取当前颜色的数量 x
            for x in range(current_ball_count + 1):
                # B 拿取的数量
                y = current_ball_count - x
                
                # 更新颜色种类计数
                new_distinct_a = distinct_a + (1 if x > 0 else 0)
                new_distinct_b = distinct_b + (1 if y > 0 else 0)
                
                # 计算当前分配的组合数贡献：C(total, x)
                ways = math.comb(current_ball_count, x)
                
                res += ways * dfs(idx + 1, count_a + x, new_distinct_a, new_distinct_b)
            
            memo[state] = res
            return res

        valid_cases = dfs(0, 0, 0, 0)
        return valid_cases / total_cases
```

## 5. 复杂度分析

*   **时间复杂度**: $O(P \times \prod balls[i])$，其中 $P$ 是状态参数的范围。由于 `balls` 长度最大 8，每个元素最大 6，状态空间非常有限，DFS 极快。
*   **空间复杂度**: $O(k \cdot n)$ 用于递归栈和 Memo 表。

## 6. 易错点笔记
1.  **分母计算错误**：容易尝试用全排列公式 $\frac{(2n)!}{\prod balls[i]!}$ 做分母，这样分子计算会非常麻烦（需要考虑顺序）。**推荐使用“组合选择”模型**，即视所有球不同，分母为 $\binom{2n}{n}$，分子乘上 $\binom{balls[i]}{x}$。
2.  **混淆球数与颜色数**：题目要求 `distinct colors` 相等，而不是球的数量相等（球的数量固定为 n）。
3.  **浮点数精度**：虽然题目要求误差 $10^{-5}$，但在中间计算过程尽量使用整数（组合数），最后一步再做除法。
