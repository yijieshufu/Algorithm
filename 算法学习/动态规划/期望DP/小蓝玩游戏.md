# 算法笔记：二维赠券收集问题 (期望 DP)

**Tags**: #Algorithm #DP #Expectation #Math #CouponCollector
**Difficulty**: Hard (期望线性性质与自环处理)

## 1. 题目背景
游戏有 $n$ 种职业和 $m$ 种阵营。每天随机获得一个英雄，属于某种职业的概率为 $1/n$，属于某种阵营的概率为 $1/m$（职业与阵营相互独立）。
**求**：集齐所有 $n$ 种职业和 $m$ 种阵营的期望天数。

## 2. 数学建模

### 状态定义
设 $f[i][j]$ 表示当前已拥有 **$i$ 种不同职业** 和 **$j$ 种不同阵营**，直到全部集齐所需的**额外期望天数**。
*   **起始状态**：$f[0][0]$ 即为最终答案。
*   **目标状态**：$f[n][m] = 0$（已经集齐，无需额外天数）。

### 状态转移分析
每一天都有四种可能的结果，影响下一状态的跳转：

| 情况 | 事件描述 | 概率 $P$ | 跳转状态 |
| :--- | :--- | :--- | :--- |
| **1** | 新职业 & 新阵营 | $P_1 = \frac{n-i}{n} \cdot \frac{m-j}{m}$ | $f[i+1][j+1]$ |
| **2** | 新职业 & 旧阵营 | $P_2 = \frac{n-i}{n} \cdot \frac{j}{m}$ | $f[i+1][j]$ |
| **3** | 旧职业 & 新阵营 | $P_3 = \frac{i}{n} \cdot \frac{m-j}{m}$ | $f[i][j+1]$ |
| **4** | 旧职业 & 旧阵营 | $P_4 = \frac{i}{n} \cdot \frac{j}{m}$ | $f[i][j]$ (原地不动) |

### 方程推导 (消除自环)
根据期望定义：

$$
f[i][j] = 1 + P_1 f[i+1][j+1] + P_2 f[i+1][j] + P_3 f[i][j+1] + P_4 f[i][j]
$$

将等式右边的 $f[i][j]$ 项移至左边（处理自环）：

$$
f[i][j](1 - P_4) = 1 + P_1 f[i+1][j+1] + P_2 f[i+1][j] + P_3 f[i][j+1]
$$

由于 $1 - P_4 = 1 - \frac{ij}{nm} = \frac{nm - ij}{nm}$，代入所有概率并同时乘以 $nm$，得到**计算公式**：

$$
f[i][j] = \frac{nm + (n-i)(m-j)f[i+1][j+1] + (n-i)j f[i+1][j] + i(m-j)f[i][j+1]}{nm - ij}
$$

---

## 3. 边界条件 (一维降维)

当某一维度（职业或阵营）已经收集满时，问题退化为标准的 **1D 赠券收集问题**。

1.  **职业已满 ($i=n$)**，只需收集阵营：

    $$


f[n][j] = f[n][j+1] + \frac{m}{m-j}

$$
2.  **阵营已满 ($j=m$)**，只需收集职业：
    
$$

f[i][m] = f[i+1][m] + \frac{n}{n-i}

$$

> [!TIP] 理解逻辑
> $\frac{m}{m-j}$ 实际上是“当前已集齐 $j$ 个，想抽中下一个新阵营”所需的期望天数（几何分布的期望）。

## 4. Python 竞赛级代码实现

```python
import sys

def solve():
    # 极速读取
    input_data = sys.stdin.read().split()
    if not input_data: return
    it = iter(input_data)
    
    n = int(next(it))
    m = int(next(it))
    
    # f[i][j] 初始化为 0
    # 维度 (n+1) * (m+1)
    f = [[0.0] * (m + 1) for _ in range(n + 1)]
    nm = n * m
    
    # 逆向推导：从 (n,m) 到 (0,0)
    for i in range(n, -1, -1):
        cur_row = f[i]
        next_row = f[i+1] if i < n else None
        ni = n - i
        
        for j in range(m, -1, -1):
            # 目标状态
            if i == n and j == m:
                continue
            
            mj = m - j
            
            if i == n:
                # 仅需集齐阵营：1D 模型
                cur_row[j] = cur_row[j+1] + m / mj
            elif j == m:
                # 仅需集齐职业：1D 模型
                cur_row[j] = next_row[j] + n / ni
            else:
                # 2D 模型公式推导结果
                # 分子
                numerator = (
                    nm + 
                    ni * mj * next_row[j+1] + 
                    ni * j * next_row[j] + 
                    i * mj * cur_row[j+1]
                )
                # 分母 (1 - P_stay) * nm
                denominator = nm - i * j
                cur_row[j] = numerator / denominator
                
    print(f"{f[0][0]:.4f}")

if __name__ == "__main__":
    solve()
```

## 5. 复杂度分析与笔记总结

### 复杂度
*   **时间**：$O(N \cdot M)$。在 $1000 \times 1000$ 规模下，操作量约 $10^6$，符合 1s~3s 的运行限制。
*   **空间**：$O(N \cdot M)$。双精度浮点数组占用约 $10^6 \times 8$ 字节 $\approx 8$ MB，远低于 256MB 限制。

### 关键点归纳
1.  **期望的自环处理**：当状态有概率留在原地时，必须移项消去等式右侧的 $f[i][j]$。
2.  **几何分布背景**：理解为什么边界是 $\frac{n}{n-i}$（概率为 $p$，期望为 $1/p$）。
3.  **计算顺序**：由于 $f[i][j]$ 依赖 $i+1$ 和 $j+1$，必须采用**逆序**（从大到小）遍历。
