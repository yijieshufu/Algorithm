# 🌲 树形 DP 必修课：蓝桥舞会 (Maximum Weight Independent Set)

## 1. 题目描述

**背景**：蓝桥公司有 $n$ 名员工，编号 $1 \sim n$。员工关系呈**树状结构**，董事长为根，父节点是子节点的直接上司。

**属性**：每个员工 $i$ 都有一个快乐指数 $a_i$。

**限制**：如果某个员工参加舞会，其**直接上司**和**直接下属**都不能参加。

**目标**：求舞会所有参会员工快乐指数总和的最大值。

---

## 2. 深度分析过程

### 核心模型

本题属于**树的最大权独立集问题**。在树上，每个节点只有“选”或“不选”两种决策，且决策受相邻节点（父与子）的约束。

### 状态定义

我们使用二维数组 $f[u][state]$ 表示以 $u$ 为根的子树能获得的最大价值：

- **$f[u][0]$**：节点 $u$ **不参加**舞会。
    
- **$f[u][1]$**：节点 $u$ **参加**舞会。

### 状态转移方程

假设 $v$ 是 $u$ 的所有子节点集合：

1. **若 $u$ 参加 ($f[u][1]$)**：
    
    - $u$ 必须加上自己的快乐值 $a[u]$。
        
    - 所有的下属 $v$ **绝对不能参加**。
        
    - **方程**：

        $$


f[u][1] = a[u] + \sum f[v][0]

$$
        
2. **若 $u$ 不参加 ($f[u][0]$)**：
    
    - $u$ 不贡献快乐值。
        
    - 对于每个下属 $v$，由于上司没去，下属**去或不去都可以**。我们取两者的最大值。
        
    - **方程**：
        
        
$$

f[u][0] = \sum \max(f[v][0], f[v][1])

$$

### 求解顺序

- **自底向上**：利用 DFS 递归到叶子节点，在回溯过程中计算父节点的状态。
    
- **根节点查找**：没有上司（父节点）的节点即为树根。最终答案为 $\max(f[root][0], f[root][1])$。

## 3. 竞赛级代码实现 (Python 3.7.3)

```Python
import sys

# 1. 扩充递归深度，防止长链树导致溢出
sys.setrecursionlimit(10**6)

def solve():
    # 2. 使用迭代器快速读取全部输入
    it = iter(sys.stdin.read().split())
    
    try:
        line = next(it)
    except StopIteration:
        return
        
    n = int(line)
    # 快乐指数 a，下标从 1 开始
    a = [0] + [int(next(it)) for _ in range(n)]
    
    # g 邻接表存储子节点，p 标记是否有父节点
    g = [[] for _ in range(n + 1)]
    p = [0] * (n + 1)
    
    # 读取 n-1 条关系 (u 是子, v 是父)
    for _ in range(n - 1):
        u, v = int(next(it)), int(next(it))
        g[v].append(u)
        p[u] = v
        
    # 3. 寻找董事长（根节点）
    r = 1
    for i in range(1, n + 1):
        if p[i] == 0:
            r = i
            break
            
    # f[u][0] 不选, f[u][1] 选
    f = [[0, 0] for _ in range(n + 1)]
    
    # 4. DFS 核心逻辑
    def dfs(u):
        f[u][1] = a[u] # 选 u 则基础分为 a[u]
        f[u][0] = 0    # 不选 u 则基础分为 0
        
        for v in g[u]:
            dfs(v) # 先求子节点
            # 转移：u 选则 v 必不选；u 不选则 v 选不选挑大的
            f[u][1] += f[v][0]
            f[u][0] += max(f[v][0], f[v][1])
            
    dfs(r)
    
    # 输出根节点两种决策的最优解
    print(max(f[r][0], f[r][1]))

if __name__ == "__main__":
    solve()
```

## 4. 总结与学习心得

### 核心知识点总结

| **维度**    | **内容**                   |
| --------- | ------------------------ |
| **算法类型**  | 树形动态规划 (Tree DP)         |
| **时间复杂度** | $O(n)$，每个节点和边仅访问一次       |
| **空间复杂度** | $O(n)$，取决于邻接表和状态数组       |
| **关键技巧**  | 后序遍历（回溯时更新）、快速 IO、递归深度设置 |

### 避坑指南

1. **递归死循环**：确保建的是单向边（父 $\to$ 子），或者是无向边但 DFS 时记录 `father` 防止回跳。
    
2. **输入格式**：部分题目给出的边可能是 `u v` 代表 `u` 是 `v` 的父，本题是 `v` 是 `u` 的父，读题要仔细。
    
3. **Python 特性**：在 $10^5$ 级别的数据下，`sys.setrecursionlimit` 是救命稻草。
