## ## 题目与核心思路：汇总最终确定的逻辑

**1. 目标函数定义**

我们要寻找一个最优节点 $i$ 作为新根，使得盈利 $f(i)$ 最大：

$$
f(i) = \text{TreeValue}(i) - \text{MoveCost}(i)
$$

- **TreeValue($i$)**: 根到所有节点路径长度的最大值 $\times k$。根据树的直径性质，这等价于 $\max(dist(i, a), dist(i, b)) \times k$，其中 $a, b$ 是树直径的两个端点。
    
- **MoveCost($i$)**: 从初始点 1 移动到点 $i$ 的成本，即 $dist(1, i) \times c$。

**2. 三遍 BFS 流程**

- **第一遍 BFS(1)**：从初始根 1 出发。
    
    - 得到所有点到 1 的距离 $d1[i]$（用于计算路费）。
        
    - 找到离 1 最远的点，它必然是直径端点之一，标记为 **$a$**。
        
- **第二遍 BFS(a)**：从端点 $a$ 出发。
    
    - 得到所有点到 $a$ 的距离 $da[i]$（用于计算树价值的一半）。
        
    - 找到离 $a$ 最远的点，它必然是直径的另一个端点，标记为 **$b$**。
        
- **第三遍 BFS(b)**：从端点 $b$ 出发。
    
    - 得到所有点到 $b$ 的距离 $db[i]$（用于计算树价值的另一半）。

**3. 线性遍历**

计算每个点 $i \in [1, n]$ 的利润：

$$
ans = \max_{i=1}^{n} \{ \max(da[i], db[i]) \times k - d1[i] \times c \}
$$

## ## 我的盲点记录：重点记录我在互动环节中提出的疑问及你的解释

- **盲点 1：三遍 BFS 的逻辑链条**
    
    - _疑问_：为什么不直接从 1 找直径，而要折腾三次？
        
    - _解释_：1 号点不一定在直径上。BFS(1) 的核心作用是把我们带到树的“边缘” $a$；一旦站到了边缘 $a$，BFS(a) 就能带我们横跨整棵树找到另一个边缘 $b$。这样我们就锁定了树的极限范围。
        
- **盲点 2：移动代价 $d1[i] \times c$ 的含义**
    
    - _疑问_：为什么要减去 $d1[i]$，它代表什么？
        
    - _解释_：$d1[i]$ 是从“老家”1 挪到“新家”$i$ 的最短步数。每挪一步都要花钱 $c$，所以它是你的**搬家路费**。利润等于卖树的钱减去搬家的路费，如果搬得太远，路费太贵，利润反而会缩水。

## ## 最终范本代码：一份最简洁且符合我习惯的最终代码

```Python
import sys
from collections import deque

# Python 3.7.3 环境，使用迭代器读入
it = iter(sys.stdin.read().split())

def solve():
    try:
        line = next(it)
        n = int(line)
        k = int(next(it))
        c = int(next(it))
    except StopIteration:
        return

    # 1-indexed 邻接表
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u = int(next(it))
        v = int(next(it))
        g[u].append(v)
        g[v].append(u)

    def bfs(s):
        # 1-indexed 距离数组，-1 表示未访问
        d = [-1] * (n + 1)
        d[s] = 0
        q = deque([s])
        while q:
            u = q.popleft()
            for v in g[u]:
                if d[v] == -1:
                    d[v] = d[u] + 1
                    q.append(v)
        return d

    # 1. 找第一个端点 a 并存下初始成本 d1
    d1 = bfs(1)
    a = 1
    for i in range(1, n + 1):
        if d1[i] > d1[a]: a = i
        
    # 2. 找第二个端点 b 并存下距离 da
    da = bfs(a)
    b = 1
    for i in range(1, n + 1):
        if da[i] > da[b]: b = i
        
    # 3. 存下到第二个端点的距离 db
    db = bfs(b)

    # 4. 遍历计算最大盈利
    ans = 0
    for i in range(1, n + 1):
        # 盈利 = 树价值 (max距离*k) - 路费 (d1距离*c)
        val = max(da[i], db[i]) * k - d1[i] * c
        if val > ans: ans = val
    print(ans)

# 执行测试用例
try:
    t_str = next(it)
    t = int(t_str)
    for _ in range(t):
        solve()
except StopIteration:
    pass
```

## ## 边界与坑点：总结该题容易出错的地方

1. **性能陷阱**：$n = 10^5$，必须使用 `collections.deque`。普通的 `list.pop(0)` 是 $O(n)$，会导致整体变成 $O(n^2)$ 从而超时。
    
2. **原地不动**：最优解可能是根本不移动（留在点 1）。代码中 `ans` 初始值为 0 且循环包含 $i=1$ 的情况（此时 $d1[1]=0$），完美兼容此边界。
    
3. **大数开销**：$k, c$ 可达 $10^9$，计算过程中会产生较大的整数，Python 3 自动处理长整型，无需担心溢出，但需确保计算公式顺序正确。
    
4. **树的直径**：记住结论——离树上任意点最远的点，一定是直径的端点。这是所有树上路径最大值问题的通用“钥匙”。
