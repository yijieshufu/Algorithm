
## 📝 算法笔记：帮派地位计算（树形结构与排序）

### 1. 题目核心描述

- **结构**：$n$ 个成员构成一棵**树**。
    
- **地位定义**：
    
    - **第一关键字**：下属人数（子树大小 $- 1$）。人数越多，排名越靠前（**降序**）。
        
    - **第二关键字**：序号。序号越小，排名越靠前（**升序**）。
        
- **目标**：求出特定序号 $m$ 的成员在全帮派的排名。

### 2. 解题思考过程

解决本题分为四个标准动作：

1. **建立关系网**：用邻接表存储“上级 $\rightarrow$ 下属”的指向，并记录每个人的**入度**。
    
2. **定位帮派老大**：入度为 0 的人就是整棵树的**根节点（Root）**。
    
3. **自底向上统计**：
    
    - 先用 **BFS（广度优先搜索）** 得到一个从根到叶子的层级序列。
        
    - **逆序遍历** 该序列，让小弟们层层向上汇报人数。这样可以避免 Python 递归过深导致的爆栈问题。
        
4. **双权值排序**：利用 Python 元组排序特性，将人数取负数实现降序，序号保持原样实现升序。

---

### 3. 关键代码解析

- **入度统计 (`deg[u] += 1`)**：
    
    标记谁有上级。最后 `deg` 为 0 的就是不需要向任何人汇报的老大。
    
- **BFS 序 (`q` 列表)**：
    
    利用 `while` 循环和指针 `i` 实现，将树结构“拍扁”成一个有序数组，保证父节点一定排在子节点前面。
    
- **逆序累加 (`sz[u] += sz[v]`)**：
    
    从 `q` 的末尾（叶子节点）开始往回走。当处理到 `u` 时，它所有的直接下属 `v` 已经领着各自的“徒子徒孙”统计完票数了，`u` 只需要把这些票数收上来即可。

---

### 4. 完整竞赛风格代码 (Python 3.7.3)

```Python
import sys

# 1. 使用 iter(next) 高效读入数据
it = iter(sys.stdin.read().split())
n = int(next(it))
m = int(next(it))

# 2. 变量初始化 (1-based 索引)
g = [[] for i in range(n + 1)] # 邻接表：上级 -> [下属1, 下属2]
deg = [0] * (n + 1)           # 入度：记录是否有上级

# 3. 建图
for k in range(n - 1):
    u = int(next(it)) # 下属
    v = int(next(it)) # 上级
    g[v].append(u)
    deg[u] += 1

# 4. 寻找根节点 (老大)
rt = 1
for i in range(1, n + 1):
    if deg[i] == 0:
        rt = i
        break

# 5. BFS 确定层级顺序
q = [rt]
i = 0
while i < len(q):
    u = q[i]
    i += 1
    for v in g[u]:
        q.append(v)

# 6. 逆序 BFS 序计算子树大小 sz
sz = [1] * (n + 1)
for i in range(n - 1, -1, -1):
    u = q[i]
    for v in g[u]:
        sz[u] += sz[v]

# 7. 组装数据并排序
# 排序规则：-sz[i] (人数越多，-sz越小，排越前), i (序号越小排越前)
a = []
for i in range(1, n + 1):
    a.append((-sz[i], i))

a.sort()

# 8. 输出小明 (m) 的排名
for i in range(n):
    if a[i][1] == m:
        print(i + 1)
        break
```

---

### 💡 避坑小贴士

- **数据规模**：$n=10^5$ 意味着 $O(n \log n)$ 的排序和 $O(n)$ 的遍历是安全的，但绝对要避免深度超过 1000 的递归。
    
- **下属人数 vs 子树大小**：题目要的是“下属人数”，即 $sz[i] - 1$。但在排序时，所有人的 $sz$ 都减 1 并不影响相对排名，所以直接用 `-sz[i]` 效果一样且更简洁。
