## 题目考察算法类型

本题考察 **动态规划 (Dynamic Programming, DP)**，属于经典的网格路径计数问题，但在基础的机器人路经（Robot Paths）上增加了 **“改变方向次数”** 以及 **“当前移动方向”** 两个状态维度。

---

## 最优解思路

1. **状态定义**：
    
    我们需要记录当前位置、已转弯次数和进入当前格子的方向。
    
    定义 $dp[i][j][p][d]$ 为到达坐标 $(i, j)$，且已经改变了 $p$ 次方向，且当前是从方向 $d$ 进入的方案数：
    
    - $d = 0$：表示上一步向右（Right）移动到达此处。
        
    - $d = 1$：表示上一步向下（Down）移动到达此处。
        
2. **转移方程**：
    
    - **向右走（To $dp[i][j][p][0]$）**：可以从左边 $(i, j-1)$ 走过来。
        
        - 若上一步也是向右走，转弯数不变：$dp[i][j][p][0] += dp[i][j-1][p][0]$
            
        - 若上一步是向下走，转弯数加 1：$dp[i][j][p][0] += dp[i][j-1][p-1][1]$
            
    - **向下走（To $dp[i][j][p][1]$）**：可以从上边 $(i-1, j)$ 走过来。
        
        - 若上一步也是向下走，转弯数不变：$dp[i][j][p][1] += dp[i-1][j][p][1]$
            
        - 若上一步是向右走，转弯数加 1：$dp[i][j][p][1] += dp[i-1][j][p-1][0]$
            
3. **边界处理**：
    
    - 起点 $(0, 0)$ 处不计入方向改变。
        
    - 初始的第一步（从 $(0,0)$ 到 $(0,1)$ 或 $(1,0)$）作为状态的起点，转弯次数设为 $0$。
        
4. **复杂度分析**：
    
    - **时间复杂度**：$O(n \times m \times k)$。在 $100 \times 100 \times 5$ 的规模下，计算量约 $1.2 \times 10^5$，Python 3.8 可在 1 秒内运行完毕。
        
    - **空间复杂度**：$O(n \times m \times k)$。

## Python 代码 (3.8.6)
```Python
import sys

# 1. 输入处理：使用 sys.stdin.read().split() 提高读取速度
d = sys.stdin.read().split()
if not d: exit()
n, m, k = map(int, d[:3])
g = d[3:]

# 2. 特判：起点即终点
if n == 1 and m == 1:
    print(1)
    exit()

# 3. 初始化：f[n][m][p][dir]
# p: 当前转弯次数, dir: 0 代表来自左边(向右走), 1 代表来自上边(向下走)
f = [[[[0] * 2 for _ in range(k + 1)] for _ in range(m)] for _ in range(n)]

# 4. 边界预处理：第一步不计入转向次数
if m > 1 and g[0][1] == '.':
    f[0][1][0][0] = 1
if n > 1 and g[1][0] == '.':
    f[1][0][0][1] = 1

# 5. 填表 (基础 DP 框架)
for i in range(n):
    for j in range(m):
        # 避开障碍物、起点以及已经预处理的第一步
        if g[i][j] == '#' or i + j <= 1:
            continue
        
        for p in range(k + 1):
            # --- 逻辑 A: 尝试向右走到达 (i, j) ---
            if j > 0:
                # 情况 1: 上一步也是向右 (不转弯)
                f[i][j][p][0] += f[i][j-1][p][0]
                # 情况 2: 上一步是向下 (转弯, p-1 -> p)
                if p > 0:
                    f[i][j][p][0] += f[i][j-1][p-1][1]
            
            # --- 逻辑 B: 尝试向下走到达 (i, j) ---
            if i > 0:
                # 情况 1: 上一步也是向下 (不转弯)
                f[i][j][p][1] += f[i-1][j][p][1]
                # 情况 2: 上一步是向右 (转弯, p-1 -> p)
                if p > 0:
                    f[i][j][p][1] += f[i-1][j][p-1][0]

# 6. 统计结果：终点处所有转向次数和方向的方案总和
ans = 0
for p in range(k + 1):
    ans += f[n-1][m-1][p][0] + f[n-1][m-1][p][1]
print(ans)
```


`dp[n-1][m-1]` 实际上指向的就是地图右下角的那个**终点格子**
