

## 题目与核心思路：分层图 BFS 🧭

本题本质上是一个**带约束的路径搜索问题**。核心在于“喷气背包”这一特殊状态。

- **状态定义**：使用 $(x, y, s)$ 表示状态，其中 $(x, y)$ 是当前坐标，$s \in \{0, 1\}$ 表示是否已使用过喷气背包。
    
- **逻辑拆解**：
    
    1. **普通移动**：若当前高度 $h_{x,y} > h_{nx,ny}$，状态 $s$ 保持不变，向 $(nx, ny, s)$ 转移。
        
    2. **特殊移动**（限一次）：若 $s=0$ 且 $h_{x,y} + k > h_{nx,ny}$，则消耗背包，向 $(nx, ny, 1)$ 转移。
        
- **算法选择**：使用 **BFS** 寻找连通性。由于每个状态只访问一次，时间复杂度为 $O(N \times M)$，空间复杂度为 $O(N \times M)$。

---

## 我的盲点记录：疑问与卡壳点 🧠

1. **背包使用时机**：最初讨论是在“进入格子”还是“迈向格子”时判定。最终确定在迈向下一个格子 $(nx, ny)$ 时，若当前高度不够，则尝试动用背包。
    
2. **背包持续性**：明确了喷气背包是“一次性原地升高”。一旦通过升高后的高度进入了下一个更低高度的格子，当前高度就重置为新格子的原始高度，不再享有 $+k$ 效果。
    
3. **递归风险**：意识到 $1000 \times 1000$ 的规模在 Python 中使用 DFS 极易触发 `RecursionError`，因此坚定了使用 `collections.deque` 实现 BFS 的方案。

## 最终范本代码：极简 Python 实现 🐍

```Python
import sys
from collections import deque

# I/O 初始化
input = sys.stdin.read().split()
it = iter(input)
n, m, k = int(next(it)), int(next(it)), int(next(it))
a, b, c, d = int(next(it)), int(next(it)), int(next(it)), int(next(it))

# 1-indexed 矩阵读入
h = [[0]*(m+1)] + [[0] + [int(next(it)) for _ in range(m)] for _ in range(n)]

# v[s][x][y] 表示状态 (x, y, s) 是否访问过
v = [[[0]*(m+1) for _ in range(n+1)] for _ in range(2)]
q = deque([(a, b, 0)])
v[0][a][b] = 1
res = "No"

while q:
    x, y, s = q.popleft()
    if x == c and y == d:
        res = "Yes"; break
    
    for dx, dy in [(0, 1), (0, -1), (1, 0), (-1, 0)]:
        nx, ny = x + dx, y + dy
        if 1 <= nx <= n and 1 <= ny <= m:
            # 1. 普通滑翔 (保持状态 s)
            if h[x][y] > h[nx][ny] and not v[s][nx][ny]:
                v[s][nx][ny] = 1
                q.append((nx, ny, s))
            # 2. 动用背包 (仅限从 0 变 1)
            if s == 0 and h[x][y] + k > h[nx][ny] and not v[1][nx][ny]:
                v[1][nx][ny] = 1
                q.append((nx, ny, 1))

print(res)
```

## 边界与坑点：考场细节 🚩

1. **高度判定**：注意题目要求是“低于当前高度”，所以必须是严格大于 $h_{curr} > h_{next}$，不能相等。
    
2. **起点即终点**：虽然题目一般会避开，但 BFS 的逻辑（先判终点再扩展）能自然处理起点等于终点的情况。
    
3. **内存管理**：$1000 \times 1000$ 的三维 `vis` 数组在 Python 中约占几十 MB 内存，完全符合 512M 的限制。
    
4. **方向数组**：注意在循环中检查 $nx, ny$ 的越界情况，尤其是在 1-indexed 下的范围是 `[1, n]` 和 `[1, m]`。
