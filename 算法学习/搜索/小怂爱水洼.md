
## 题目与核心思路

汇总最终确定的逻辑：

- **问题本质**：在 $N \times M$ 的网格图中，寻找所有由相邻（上下左右）非零权值点组成的**连通块**中，权值总和最大的一个。
    
- **核心算法**：使用 **广度优先搜索 (BFS)**。通过嵌套循环遍历网格，一旦发现未访问过的有水格子 ($a_{i,j} > 0$)，即启动 BFS 遍历整个大水洼。
    
- **关键优化**：采用 **“发现即清零”** 的策略。每当一个小水洼被加入队列或作为起点时，立即将其水量累加并置为 $0$，有效防止重复搜索和死循环，确保时间复杂度为 $O(N \times M)$。
    

## 我的盲点记录

重点记录互动环节中的疑问及解释：

- **移动方向的偏移量**：最初混淆了对角线移动。明确了“上下左右”对应的是 $(1, 0), (-1, 0), (0, 1), (0, -1)$，而 $(1, 1)$ 等坐标涉及斜向移动。
    
- **边界判断的简化**：通过给 1-indexed 数组四周填充一圈 $0$（Padding 🧱），可以省去繁琐的 `1 <= ni <= n` 越界检查。
    
- **入队时机的处理**：确认了在格子“入队前”就清零并累加水量，比在“出队后”处理更高效，能避免同一个格子被多次加入队列。
    

## 最终范本代码

一份最简洁且符合你习惯的最终代码：



```Python
import sys
from collections import deque

# 1. 初始化输入流与变量
it = iter(sys.stdin.read().split())
n, m = int(next(it)), int(next(it))

# 2. 构造 1-indexed 数组并补 0 垫层
a = [[0] * (m + 2) for _ in range(n + 2)]
for i in range(1, n + 1):
    for j in range(1, m + 1):
        a[i][j] = int(next(it))

ans = 0
d = [(0, 1), (0, -1), (1, 0), (-1, 0)]

# 3. 遍历网格寻找水洼起点
for i in range(1, n + 1):
    for j in range(1, m + 1):
        if a[i][j] > 0:
            curr = a[i][j]
            a[i][j] = 0 # 发现即清零
            q = deque([(i, j)])
            
            # BFS 探索整个连通块
            while q:
                r, c = q.popleft()
                for dr, dc in d:
                    ni, nj = r + dr, c + dc
                    if a[ni][nj] > 0:
                        curr += a[ni][nj]
                        a[ni][nj] = 0 # 入队前清零防止重复
                        q.append((ni, nj))
            
            # 更新全局最大值
            if curr > ans:
                ans = curr

print(ans)
```

## 边界与坑点

- **全干地情况**：如果所有 $a_{i,j}$ 均为 $0$，循环不会触发 BFS，`ans` 初始值应为 $0$，输出正确。
    
- **1-indexed 偏移**：读取数据时要注意 `range(1, n + 1)`，否则会漏掉最后一行或最后一列。
    
- **内存陷阱**：虽然 $100 \times 100$ 规模不大，但对于更大的图，使用 `collections.deque` 比普通 `list` 模拟队列效率更高（`popleft` 为 $O(1)$）。
    
- **题目陷阱**：题目提到“之前收集的水量会变 $0$”，这意味着我们只需要求出**单个**连通块的最大和，而不是多个连通块的累加。
