## ## 题目与核心思路

**题目背景**：给定长度为 $n$ 的字符串 $S$ 和 $T$，以及 $k$ 个操作。操作包括“某位加 $v$ 模 10”或“交换两位”。每个操作最多用一次，求能否使 $S=T$。

**核心逻辑**：

1. **数据范围**：$n \le 10, k \le 7$。$k$ 非常小，提示我们可以尝试**所有操作的全排列**。
    
2. **搜索策略**：使用 **DFS（深度优先搜索）**。每一层递归尝试从剩余的操作中选一个执行。
    
3. **状态记录**：使用 **Bitmask（位掩码）** 整数 `state` 记录操作的使用情况。
    
    - 检查第 $i$ 个操作是否用过：`state >> i & 1`。
        
    - 标记第 $i$ 个操作已使用：`state | (1 << i)`。
        
4. **回溯 (Backtracking)**：执行完递归后，必须将数组 $s$ **还原**到操作前的状态，以便尝试其他的操作顺序。

## ## 我的盲点记录

根据我们在互动中的讨论，以下是你重点关注的逻辑难点：

1. **下标映射 (1-indexed)**：由于你习惯使用 1-indexed 数组（`s = [0, ...]` ），题目给出的下标 $x, y$（从 0 开始）必须全部加 1。代码中通过 `op[1] += 1` 和 `op[2] += 1` 实现了这种“翻译”。
    
2. **加法回溯**：执行 `(s[x] + v) % 10` 后，还原时不能只减 $v$，要用 `(s[x] - v + 10) % 10` 来处理 Python 中可能的负数取模逻辑。
    
3. **位运算 `state | (1 << i)`**：这步相当于在清单上给第 $i$ 个任务“打钩”。`1 << i` 生成一个只有第 $i$ 位为 1 的数字，通过 `|`（按位或）将其合并进 `state`，代表该操作已执行。

## ## 最终范本代码

符合你的 Python 3.7.3 编码习惯：简洁变量名、`sys.stdin.read`、1-indexed。

```Python
import sys

# 1. 输入处理
input_data = sys.stdin.read().split()
it = iter(input_data)

n = int(next(it))
# 转换为 1-indexed 数字列表
s = [0] + [int(c) for c in next(it)]
t = [0] + [int(c) for c in next(it)]

k = int(next(it))
a = []
for _ in range(k):
    op = [int(next(it)), int(next(it)), int(next(it))]
    op[1] += 1 # 下标转换
    if op[0] == 2:
        op[2] += 1
    a.append(op)

# 2. DFS 核心逻辑
def dfs(state):
    # 只要当前相等就成功
    if s[1:] == t[1:]:
        return True
    
    for i in range(k):
        # 如果第 i 个操作没用过
        if not (state >> i & 1):
            op, x, yv = a[i]
            
            if op == 1:
                old = s[x]
                s[x] = (s[x] + yv) % 10
                if dfs(state | (1 << i)): return True
                s[x] = old # 回溯
            else:
                s[x], s[yv] = s[yv], s[x]
                if dfs(state | (1 << i)): return True
                s[x], s[yv] = s[yv], s[x] # 回溯
                
    return False

# 3. 输出结果
if dfs(0):
    print("Yes")
else:
    print("No")
```

## ## 边界与坑点

- **初始相等**：如果 $S$ 和 $T$ 初始就相等，即便 $k=0$ 也要输出 `Yes`。代码开头的 `if s[1:] == t[1:]` 保证了这一点。
    
- **操作可选性**：题目说“挑选出任意个操作”，这意味着递归深度不固定，只要中途匹配就立刻 `return True`。
    
- **操作顺序**：先交换再加值，和先加值再交换结果不同。DFS 通过尝试所有排列覆盖了所有情况。
