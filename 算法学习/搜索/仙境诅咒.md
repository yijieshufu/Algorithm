
## 题目与核心思路：汇总最终确定的逻辑 🗺️

- **问题本质**：这是一个**无向图的连通性问题**。如果两位修仙者之间的距离不超过 $D$，则他们之间存在一条可以传播诅咒的“边”。
    
- **数学建模**：
    
    - 将 $N$ 个修仙者视为节点。
        
    - 传播规则：若节点 $u$ 被感染且 $dist(u, v) \le D$，则节点 $v$ 也会被感染。
        
    - 判定技巧：为了避免浮点数开方运算的精度问题和计算开销，直接比较**距离的平方**：$(X_i - X_j)^2 + (Y_i - Y_j)^2 \le D^2$。
        
- **搜索策略**：从 1 号节点（妮妮）出发，执行 **BFS（广度优先搜索）**。使用一个标记数组记录每个节点是否被访问（感染）过，最终输出标记数组的状态。
    

## 我的盲点记录 📝

- **传染路径的抽象**：这类题目初看像几何题，但由于诅咒具有传递性（A 传 B，B 传 C），本质上是寻找从起点出发的**可达路径**。
    
- **效率考量**：对于 $N=1000$ 的规模，$O(N^2)$ 的暴力检查（即每个点出队时都扫描一遍其他所有点）总计算量约为 $10^6$，在 Python 3.7 的时间限制内是安全的。
    

## 最终范本代码 🐍

```Python
import sys

# 初始化输入流，适应简洁的编码习惯
it = iter(sys.stdin.read().split())

def solve():
    # 读取修仙者数量 N
    line = next(it, None)
    if line is None: return
    n = int(line)
    
    # 存储坐标，使用 1-indexed 数组
    x = [0.0] * (n + 1)
    y = [0.0] * (n + 1)
    for i in range(1, n + 1):
        x[i] = float(next(it))
        y[i] = float(next(it))
        
    # 读取传染距离 D 并计算平方
    d = float(next(it))
    d2 = d * d
    
    # st[i] 记录第 i 位修仙者是否被诅咒
    st = [0] * (n + 1)
    # BFS 队列，从 1 号修仙者开始
    q = [1]
    st[1] = 1
    
    # 队列指针 i 逐步推进
    i = 0
    while i < len(q):
        u = q[i]
        i += 1
        
        # 遍历所有修仙者，检查尚未被诅咒且在范围内的点
        for v in range(1, n + 1):
            if not st[v]:
                dist2 = (x[u] - x[v])**2 + (y[u] - y[v])**2
                if dist2 <= d2:
                    st[v] = 1
                    q.append(v)
    
    # 按照编号顺序输出结果
    for i in range(1, n + 1):
        print(st[i])

solve()
```

## 边界与坑点 ⚠️

1. **距离计算**：务必使用 $D^2$ 进行判定，减少开方运算的耗时。
    
2. **数据类型**：坐标 $X_i, Y_i$ 是实数，读取时必须转换为 `float`。
    
3. **1-indexed 对应**：由于题目明确指出第 $i$ 行代表第 $i$ 位修仙者，使用 1-indexed 数组可以直观地对应输出要求。
    
4. **初始点处理**：始终确保 1 号修仙者在初始状态下就被标记为已感染。
