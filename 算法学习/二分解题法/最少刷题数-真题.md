# 题目描述
给定 $N$ 个学生的刷题量 $A_i$，要求计算每名学生至少还要再刷多少道题，才能使得全班中“刷题比他多的人数”不超过“刷题比他少的人数”。  
题目要求：**“刷题比我多的人” $\le$ “刷题比我少的人”**。如果现在不满足，你就得加倍努力（多刷题），直到这个条件成立。  
**中间的人**：刷题数和 $X$ 刚好相等的同学（**这些人不参与统计**)  
**数据范围**：$N \le 100,000$，$A_i \le 100,000$。
# 分析
**中位数 + 二分查找 / 计数统计**
- **为什么适合**：题目要求的条件实质上是在寻找一个“动态的中位数”平衡点。
	- 刷题量是一个有序逻辑，通过排序后的中位数可以快速确定平衡基准。
	- 由于 $N$ 较大，需要 $O(N \log N)$ 或 $O(N)$ 的算法，排序配合二分查找或前缀和计数是最优选。
## 解题思路
- **确定中位数基准**：对 $A$ 进行排序得到 $S$。令中间位置的值 $mid\_val = S[N // 2]$。
- **核心逻辑：确定“安全分数” $T$**：  
    我们需要找到一个目标值 $T$，使得如果一名学生达到了 $T$，他就一定能满足条件。
    - 统计全班中：严格小于 $mid\_val$ 的人数 $L$，严格大于 $mid\_val$ 的人数 $R$。
    - **情况 A ($L > R$)**：说明中位数位置左侧的人很多，目标值 $T$ 设为 $mid\_val$ 即可。因为即使一名原本 $< mid\_val$ 的学生变成 $mid\_val$，左侧人数减 1 后依然 $\ge R$。
    - **情况 B ($L \le R$)**：说明右侧（强手）较多或相等。此时目标值 $T$ 必须设为 $mid\_val + 1$。这样原本的学生不仅能减少右侧的人数，还能把原本等于 $mid\_val$ 的人挤到“比自己少”的一侧。
- **遍历计算**：对于每个学生 $A_i$：
    - 如果他当前的 $R \le L$，直接输出 0。
    - 否则，输出 $\max(0, T - A_i)$。
# 代码

```python
import sys
from bisect import bisect_left,bisect_right
# 输入数据
it = iter(sys.stdin.read().split())
n = int(next(it))
a = [int(next(it)) for _ in range(n)]
# 重新排序
s=sorted(a)
# 计算 中位数的分布
mid_val = s[n//2]
cntl = bisect_left(s,mid_val)
cntr = n-bisect_right(s,mid_val)
# 确定中位数
if cntl > cntr:
  target = mid_val
else:
  target = mid_val+1
ans =[]
for x in a :
  l = bisect_left(s,x)
  r = n-bisect_right(s,x)
  if r <= l:
    ans.append(0)
  else:
    ans.append(max(0,target-x))
print(*ans)
```
