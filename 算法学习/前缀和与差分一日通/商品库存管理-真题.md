# 题目描述
**核心动作**：在 $n$ 个初始为 $0$ 的商品上执行 $m$ 次区间加 $1$ 操作，求：对于每一个操作 $i$，若不执行它，最终库存为 $0$ 的商品种数。  
**数据边界**：$n, m \le 3 \times 10^5$。必须实现 $O(n + m)$ 复杂度的算法，任何 $O(n \cdot m)$ 的暴力模拟均会 TLE。
# 分析
**算法/数据结构**：**差分数组 + 前缀和**。  
**为何适用**：区间批量修改（差分数组）与区间统计（前缀和）是处理此类“单点贡献”问题的标准组合。  
**通用骨架**：
```python
# 差分修改
diff[L] += 1; diff[R+1] -= 1
# 还原数组
for i in range(1, n+1): a[i] = a[i-1] + diff[i]
# 前缀和统计满足特定条件的元素个数
pre[i] = pre[i-1] + (1 if a[i] == target else 0)
```

**定级**：中等题（蓝桥杯省赛常见考点）。  
**核心难点**：理解“不执行操作 $i$ 后库存为 $0$”的转化逻辑：即原本库存就是 $0$ 的商品，加上原本库存为 $1$ 且该库存刚好由操作 $i$ 贡献的商品。

## 认知块解题推导（逻辑降维）
[**块 1：全局库存分布计算**]  
利用差分数组 `diff` 在 $O(m)$ 时间内记录所有操作，再通过前缀和还原出执行完所有 $m$ 个操作后，每个商品的实际库存 `count[i]`。  
[**块 2：零值基础与贡献转化**]  
统计全局 `base_zeros`（库存原本就为 $0$ 的商品数）。若操作 $i$ 覆盖区间 $[L, R]$，不执行该操作会导致该区间内所有商品的库存减 $1$。只有原本库存为 $1$ 的商品在减 $1$ 后会变成 $0$。  
[ **块 3：区间“1”统计** ]  
预处理一个前缀和数组 `pre1`，其中 `pre1[i]` 表示区间 $[1, i]$ 中库存刚好为 $1$ 的商品数量。那么操作 $i$ 撤销后新增的 $0$ 库存商品数即为 `pre1[R] - pre1[L-1]`。

## 代码

```python
import sys
# 块 1：高效输入流处理
it = iter(sys.stdin.read().split())
n = int(next(it))
m = int(next(it))
ops = []
diff = [0] * (n + 2)
# 记录操作并维护差分
for _ in range(m):
    l, r = int(next(it)), int(next(it))
    ops.append((l, r))
    diff[l] += 1
    diff[r + 1] -= 1
# 块 2：还原库存并统计基础 0
count = [0] * (n + 1) 
base_zeros = 0
curr = 0
for i in range(1, n + 1):
    curr += diff[i]
    count[i] = curr  # 在执行完所有 $m$ 个操作后，第 $i$ 种商品真实的库存总量
    if curr == 0: # 题目初始为0  只加0 统计原本为0 执不执行操作都为0 
        base_zeros += 1
# 块 3：预处理库存为 1 的前缀和
# 一个商品的 count[i] == 1，说明它只被唯一一个操作覆盖了
pre1 = [0] * (n + 1)
for i in range(1, n + 1): 
    pre1[i] = pre1[i - 1] + (1 if count[i] == 1 else 0)
# 块 4：计算每个操作被跳过的结果
ans = []
for l, r in ops:
    # 结果 = 原始 0 数量 + 该区间内库存为 1 的商品数量
    res = base_zeros + (pre1[r] - pre1[l - 1])
    ans.append(str(res))
sys.stdout.write('\n'.join(ans) + '\n')
```
