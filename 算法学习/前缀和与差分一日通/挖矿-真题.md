# 题目描述
小蓝在数轴上的 $0$ 点出发，要在总移动距离不超过 $m$ 的前提下，尽可能多地收集矿石。数轴上有 $n$ 个矿石，位置在 $a_i$。经过矿石点即收集，每个矿石只能收集一次。  
- **核心动作**：在数轴上从 $0$ 出发，移动总距离不超过 $m$，求最多能经过（覆盖）多少个矿洞点。
- **数据边界**：$n \le 10^5$（需 $O(n \log n)$ 或 $O(n)$ 算法），坐标及 $m \le 2 \times 10^6$。
# 分析
- **算法/数据结构**：**排序 + 双指针（滑动窗口）**。
- **为何适用**：最优路径必为 $[L, R]$ 区间覆盖。当左端点 $L$ 向远方延伸时，为了控制总路程，右端点 $R$ 必须向原点收缩，具备严格的**单调性**。
- **通用骨架**：
```python
    for i in range(len(A)):
        while j >= 0 and not check(A[i], B[j]):
            j -= 1
        ans = max(ans, i + j)
    ```
```

- **定级**：中等题（省一冲刺必备）。
- **核心难点**：理解“折返”逻辑。总路程 $dist = \min(2|L| + R, 2R + |L|)$，即左右两端点中较短的那一端走两次。

- **[块 1：路径本质]**：要收集坐标在 $[-L, R]$ 范围内的所有矿石（$L, R \ge 0$），你有两种走法：先左后右（$2L + R$）或先右后左（$2R + L$）。取其小者。
- **[块 2：贪心排序]**：矿石分布在正负轴。将负坐标取绝对值后从小到大排，正坐标从小到大排。我们要从两组里各选前 $i$ 个和前 $j$ 个。
- **[块 3：双指针联动]**：随着左侧选取的矿石变多（$L$ 增大），右侧能选的上限 $j$ 必然减小。通过 $j$ 递减的线性扫描，将 $O(n^2)$ 降至 $O(n)$。
