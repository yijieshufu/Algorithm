
## 题目与核心思路：幸运字符串

我们要寻找的是最长的前缀 $S[1 \dots L]$，使得它在 $S$ 的非起始位置（即 $i > 1$）再次出现。

1. **二分答案**：前缀长度 $L$ 具有单调性。若长度 $L$ 满足条件，则 $L-1$ 必满足；若 $L$ 不满足，则 $L+1$ 必不满足。搜索区间为 $[1, n-1]$，复杂度 $O(\log n)$。
    
2. **字符串哈希**：预处理前缀哈希数组 $h$ 和幂数组 $p$。利用公式 $Hash(i, i+L-1) = (h[i+L-1] - h[i-1] \times p[L]) \pmod M$ 在 $O(1)$ 时间内提取任意子串指纹。
    
3. **判定逻辑**：`check(mid)` 遍历 $i \in [2, n-mid+1]$，比较 $S[i \dots i+mid-1]$ 的哈希值是否等于前缀 $h[mid]$。
    
4. **复杂度分析**：时间复杂度 $O(n \log n)$，空间复杂度 $O(n)$。

---

## 我的盲点记录：

1. **区间提取公式**：类比十进制位移，提取子串哈希需要减去“左侧多余部分”并进行“权位对齐”，即 $h[r] - h[l-1] \cdot base^{len}$。
    
2. **循环边界推导**：通过“右端点 $i+mid-1 \le n$”推导出起始位置 $i$ 的最大值为 $n-mid+1$。
    
3. **冲突防护**：在 $N=2 \times 10^5$ 规模下，普通 $10^9$ 级别的单哈希容易冲突，需使用 $2^{61}-1$ 等大质数或双哈希。

## 最终范本代码：
```Python
import sys
# 极简 I/O 模式
input = sys.stdin.read().split()
it = iter(input)

n = int(next(it))
s = " " + next(it) # 补齐 0 位，实现 1-indexed

# 初始化大质数与进制
m = (1 << 61) - 1
b = 131
h = [0] * (n + 1)
p = [1] * (n + 1)

# 预处理哈希与幂数组
for i in range(1, n + 1):
    h[i] = (h[i-1] * b + ord(s[i])) % m
    p[i] = (p[i-1] * b) % m

def check(mid):
    if mid == 0: return True
    t = h[mid] # 目标前缀哈希
    # 遍历起始位 i 从 2 到 n-mid+1
    for i in range(2, n - mid + 2):
        now = (h[i + mid - 1] - h[i - 1] * p[mid]) % m
        if now == t: return True
    return False

# 二分搜索最大长度
res = 0
low, high = 1, n - 1
while low <= high:
    mid = (low + high) // 2
    if check(mid):
        res = mid
        low = mid + 1
    else:
        high = mid - 1

print(res)
```

---

## 边界与坑点：

- **1-indexed 偏移**：使用 `s = " " + s` 可以让字符索引与哈希数组 $h[i]$ 完全对齐，避免下标越界和逻辑混乱。
    
- **模数选择**：Python 处理大数有优势，建议使用 `(1 << 61) - 1` 作为模数，这几乎能免疫所有针对单哈希的哈希杀手（Hash Killer）测试用例。
    
- **二分上界**：`high` 应设为 $n-1$，因为前缀要“出现至少两次”，长度为 $n$ 的前缀（原串本身）不可能在后面再出现一个完整的自己。
    
- **空结果处理**：如果 $low$ 从 1 开始二分且不满足，`res` 默认值应为 0。
