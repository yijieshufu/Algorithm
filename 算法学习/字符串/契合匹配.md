
## 题目与核心思路 ⚙️

- **目标**：给定两个长度为 $n$ 的齿轮字符串 $S$ 和 $T$，判断它们在旋转后是否能完全契合（对应位置字母相同但大小写相反），并求出最少旋转位数。
    
- **核心逻辑**：
    
    1. **目标转换**：将 $S$ 使用 `swapcase()` 预处理。这样问题就退化为：判断 $T$ 是否是转换后的 $S$ 的一个循环移位。
        
    2. **破环成链**：利用 `(S'+S').find(T)` 的技巧，在 $O(n)$ 时间内判断 $T$ 是否为子串。
        
    3. **最小步数**：若匹配索引为 $p$，则旋转步数为 $\min(p, n-p)$。
        
- **复杂度**：时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 我的盲点记录 💡

- **内置方法应用**：最初对于如何简洁处理大小写反转不确定，学习了 Python 的 `.swapcase()` 这种极简方式。
    
- **环形匹配转化**：对于“循环移位”的判断逻辑起初较为模糊，通过“拼接字符串 $S'+S'$”的方法将其简化为了标准的子串查找问题。
    
- **步数计算关系**：明确了索引 $p$ 与 $n-p$ 分别代表顺时针和逆时针旋转的距离。

## 最终范本代码 💻

```Python
import sys
# 1. 极简 I/O 模式
input = sys.stdin.read().split(); it = iter(input)

# 2. 读取数据与 1-indexed 补齐
n = int(next(it))
s = ' ' + next(it).swapcase()  # 预处理 s 为理想契合目标串
t = ' ' + next(it)

# 3. 核心查找逻辑
# 截取有效内容进行双倍拼接
ss = s[1:] + s[1:]
p = ss.find(t[1:])

# 4. 结果处理
if p == -1:
    print("No")
else:
    # 取双向旋转的最小值
    res = min(p, n - p)
    print("Yes")
    print(res)
```

## 边界与坑点 ⚠️

- **效率瓶颈**：$n = 10^6$ 时，绝对不能使用两层 $for$ 循环。Python 的 `str.find()` 底层是高度优化的 C 代码，在本题中比手写 KMP 往往更简洁且够快。
    
- **双向旋转**：题目明确提到顺时针和逆时针均可，漏掉 `min(p, n-p)` 会导致部分测试点错误。
    
- **内存注意**：`ss = s[1:] + s[1:]` 会产生一个长度为 $2n$ 的新字符串。在内存限制极严（如 64MB）的情况下需要小心，但本题常规配置下完全没问题。
