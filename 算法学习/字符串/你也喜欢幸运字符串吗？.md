
## 题目与核心思路：幸运字符串统计 📝

**题目目标**：统计字符串 $S$ 的所有**前缀**（幸运字符串）在原字符串中作为**子串**出现的总次数。

**核心逻辑**：

1. **性质转换**：前缀 $S[1 \dots L]$ 在位置 $i$ 结束，等价于 $S[1 \dots L]$ 是 $S[1 \dots i]$ 的一个**后缀**。
2. **KMP 基础**：利用 KMP 的 `p` 数组（即 `next` 数组），`p[i]` 表示 $S[1 \dots i]$ 的最长相等前后缀长度。
3. **DP 递推**：设 $dp[i]$ 为以位置 $i$ 结尾的前缀数量。
    - $S[1 \dots i]$ 本身贡献 1 次。
    - $S[1 \dots p[i]]$ 以及它所包含的更短前缀也都在 $i$ 处结束。
    - 递推公式：$dp[i] = dp[p[i]] + 1$。
4. **复杂度分析**：
    - **时间复杂度**：构建 `p` 数组 $O(n)$，DP 遍历 $O(n)$，总计 $O(n)$。
    - **空间复杂度**：$O(n)$，用于存储 `p` 和 `dp` 数组。

---

## 我的盲点记录 🧠

- **KMP 回跳逻辑**：在匹配失败时，应回跳至前一个匹配状态，即 `j = p[j]`，而非 `j = p[i]`（当时尚未求出 $p[i]$）。
    
- **匹配更新**：当字符匹配成功 `s[i] == s[j+1]` 时，匹配长度应自增 `j += 1`，而非 `j += i`。
    
- **DP 意义理解**：初期对如何统计“所有前缀”感到困惑。经过引导意识到：利用 $p[i]$ 的跳跃属性，可以将以 $i$ 结尾的所有前缀看作一个链式结构，通过 $dp[p[i]]$ 一次性累加之前所有的合法前缀。

## 最终范本代码 🐍

```Python
import sys

# 极简 I/O 模式
input = sys.stdin.read().split()
it = iter(input)
n = int(next(it))
s = ' ' + next(it)  # 1-indexed 补齐

p = [0] * (n + 1)
dp = [0] * (n + 1)
j, res = 0, 0

for i in range(1, n + 1):
    if i > 1:
        # KMP 核心：j 回跳
        while j > 0 and s[i] != s[j + 1]: 
            j = p[j]
        # 匹配成功增加长度
        if s[i] == s[j + 1]: 
            j += 1
        p[i] = j
    
    # DP 状态转移：当前长前缀(1) + 隐藏的短前缀数量(dp[p[i]])
    dp[i] = dp[p[i]] + 1
    res += dp[i]

print(res)
```

## 边界与坑点 ⚠️

- **Python 3.7.3 效率**：$n = 2 \times 10^5$ 对 Python 来说计算压力较大。使用 `sys.stdin.read().split()` 配合 `iter` 是最高效的读入方式，避免多次调用 `input()`。
    
- **索引偏移**：使用 **1-indexed** 时，字符串补空格 `s = ' ' + s` 非常关键，否则在 `s[j+1]` 时容易出现 0 和 1 的逻辑混淆。
    
- **DP 初始化**：`dp[0]` 必须为 0。当 `p[i] = 0` 时，`dp[i] = dp[0] + 1 = 1`，保证了每个位置至少贡献其本身这个前缀。
    
- **内存限制**：虽然本题空间充足，但在某些严格题目中，若 $n$ 更大，需注意 `p` 和 `dp` 数组的内存占用。
