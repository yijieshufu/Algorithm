
### 1. 题目背景与模板

**题目描述**：给定长度为 $n$ 的字符串 $s$ 和长度为 $m$ 的字符串 $t$。判断能否从 $s$ 中选出两个长度为 $k$ 且**不相交**的子串，使得它们按原顺序拼接后的字符串包含 $t$。

**输入条件**：整数 $n, m, k$ ($2 \le m \le 2k \le n \le 10^4$)，以及字符串 $s$ 和 $t$。

**所选题型**：字符串匹配 / KMP 预处理与前后缀分解

**Sigma通用模板代码**：
```Python
import sys; input=sys.stdin.read().split(); it=iter(input)
# Minimal Vars: n, s, nxt, i, j, res
n = int(next(it))
s = " " + next(it) # 1-indexed padding
nxt = [0] * (n + 1)
# [TEMPLATE LOGIC: KMP Prefixes]
i = 2; j = 0
while i <= n:
    while j > 0 and s[i] != s[j + 1]: j = nxt[j]
    if s[i] == s[j + 1]: j += 1
    nxt[i] = j
    i += 1
```

### 2. 题目与核心思路

**核心思路**：

将目标串 $t$ 拆分为两部分：长度为 $i$ 的前缀 $t_{pre}$ 和长度为 $m-i$ 的后缀 $t_{suf}$。

如果要将 $t$ 放入两个互不重叠的、长度为 $k$ 的窗口（盒子 A 和盒子 B）中：

1. **正向 KMP**：预处理出 $t_{pre}$ 在 $s$ 中出现的最早结束位置 `min_e[i]`。为了保证左侧能放下盒子 A，必须满足 `min_e[i] >= k`。
    
2. **反向 KMP**：将 $s$ 和 $t$ 翻转，预处理出 $t_{suf}$ 在原串 $s$ 中出现的最晚起始位置 `max_s[m-i]`。为了保证右侧能放下盒子 B，必须满足 `max_s[m-i] <= n - k + 1`。
    
3. **合法性校验**：枚举所有可能的切割点 $i$（$1 \le i < m$），检查前缀和后缀长度是否都不超过 $k$。若满足，且盒子 A 的结束位置严格在盒子 B 的起始位置左侧（即 `min_e[i] < max_s[m-i]`），则说明可以成功拼接。
    

**时空复杂度**：

- **时间复杂度**：$O(n + m)$。正反两遍 KMP 匹配均为线性时间。更新 `min_e` 时的 `while` 回溯看似是嵌套循环，但由于每个前缀长度最多被赋值和跳跃一次（势能分析），总均摊复杂度仍为 $O(n + m)$。
    
- **空间复杂度**：$O(n + m)$。用于存储翻转字符串以及 `nxt`、`min_e`、`max_s` 数组。

### 3. 我的盲点记录

- **盲点 1：KMP `nxt` 数组的递推逻辑。**
    
    - _解答_：当字符失配时，不能从头开始，而是利用 `nxt` 数组寻找当前已匹配前缀的“最长相等前后缀”，即令 `j = nxt[j]` 实现跳跃，从而达到 $O(n)$ 的高效匹配。
        
- **盲点 2：KMP 匹配时，同一个结束位置可能对应多个长度的前缀。**
    
    - _解答_：当匹配到较长前缀时，其内部嵌套的较短前缀（通过 `nxt[cur]` 追踪）也同时在该位置匹配成功。代码中需要用 `while cur > 0: cur = nxt[cur]` 将这些较短前缀的最早结束位置也一并更新。
        
- **盲点 3：固定长度窗口的边界约束条件。**
    
    - _解答_：因为窗口长度固定为 $k$，如果前缀结束于 $r$，其起始位置为 $r-k+1 \ge 1 \Rightarrow r \ge k$。如果后缀起始于 $l$，其结束位置为 $l+k-1 \le n \Rightarrow l \le n-k+1$。互不重叠的条件则是 $r < l$。

### 4. 最终范本代码
```Python
import sys

# [MOD] 强制使用 Sigma 竞赛 I/O 规范
input = sys.stdin.read().split()
it = iter(input)

def solve():
    n = int(next(it))
    m = int(next(it))
    k = int(next(it))
    s = " " + next(it)
    t = " " + next(it)

    # [MOD] 特判：如果 m <= k 且 t 是 s 的子串，必然能找到两个满足条件的盒子
    if m <= k and t[1:] in s[1:]:
        print("Yes")
        return

    # [MOD] 正向 KMP：记录 t 的前缀最早结束位置 min_e
    min_e = [n + 1] * (m + 1)
    nxt = [0] * (m + 1)
    
    i, j = 2, 0
    while i <= m:
        while j > 0 and t[i] != t[j + 1]: j = nxt[j]
        if t[i] == t[j + 1]: j += 1
        nxt[i] = j
        i += 1
    
    i, j = 1, 0
    while i <= n:
        while j > 0 and (j == m or s[i] != t[j + 1]): j = nxt[j]
        if s[i] == t[j + 1]: j += 1
        # [MOD] 满足容纳长度为 k 的左盒子条件
        if i >= k:
            cur = j
            while cur > 0 and min_e[cur] > i:
                min_e[cur] = i
                cur = nxt[cur]
        i += 1

    # [MOD] 反向 KMP：记录 t 的后缀最晚开始位置 max_s
    tr = " " + t[1:][::-1]
    sr = " " + s[1:][::-1]
    max_s = [-1] * (m + 1)
    nxt_r = [0] * (m + 1)
    
    i, j = 2, 0
    while i <= m:
        while j > 0 and tr[i] != tr[j + 1]: j = nxt_r[j]
        if tr[i] == tr[j + 1]: j += 1
        nxt_r[i] = j
        i += 1
        
    i, j = 1, 0
    while i <= n:
        while j > 0 and (j == m or sr[i] != tr[j + 1]): j = nxt_r[j]
        if sr[i] == tr[j + 1]: j += 1
        # [MOD] 翻转坐标映射：右盒子的起始位置 l
        l = n - i + 1
        # [MOD] 满足容纳长度为 k 的右盒子条件
        if l <= n - k + 1:
            cur = j
            while cur > 0 and max_s[cur] < l:
                max_s[cur] = l
                cur = nxt_r[cur]
        i += 1

    # [MOD] 遍历所有合法的切分点
    i = 1
    while i < m:
        l1, l2 = i, m - i
        if l1 <= k and l2 <= k:
            # [MOD] 不相交约束：左盒结束 < 右盒开始
            if min_e[l1] < max_s[l2]:
                print("Yes")
                return
        i += 1

    print("No")

solve()
```

### 5. 变体题型简述

- **变体 1：允许窗口重叠**。去掉或修改 $r < l$ 的限制，只需判定两个窗口是否各自合法即可。
    
- **变体 2：求最多能拼接出多少个 $t$**。转变为贪心匹配问题，需要在 KMP 匹配过程中动态维护非重叠的匹配段数量。
    
- **变体 3：不固定子串长度，求最短的总拼接长度**。可以使用动态规划结合 KMP 状态机来求解，寻找覆盖 $t$ 所需的最小 $s$ 的子段和。
