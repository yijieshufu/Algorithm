# 题目描述
- **核心动作**：在 $n$ 个点 $m$ 条边的无权图中，给定 $Q$ 个查询 $(x_i, y_i)$，求从 $x_i$ 出发步数 $\le y_i$ 可达的星球数量的平均值（数学期望）。
- **数据边界**：$n \le 1000$，$m \le 5n$，$Q \le 50000$。$n$ 较小而 $Q$ 较大，提示**必须预处理**。
# 分析
- **算法/数据结构**：**全源 BFS + 前缀和预处理**。
- **为何适用**：边权均为 1，BFS 是求单源最短路的最优选择。$n$ 规模允许 $O(n(n+m))$ 的全源遍历，预处理后可 $O(1)$ 回答查询。

**通用骨架**：  
```python
# 1. 遍历每个起点进行 BFS
# 2. 统计各距离下的节点数量
# 3. 计算前缀和：pre[d] 表示距离 <= d 的节点总数
# 4. 期望 = 总和 / Q
```

- **定级**：基础题。
- **核心难点**：识别出 $Q$ 远大于 $n$，从而将计算重心转移到 $O(n^2)$ 的预处理，而非 $O(Q \times \text{BFS})$。

- **[块 1：图论预处理]**：利用邻接表存储 $m$ 条双向边。由于 $n=1000$，建立 $n \times n$ 的**距离分布矩阵** `f[i][d]`。
- **[块 2：全源距离统计]**：对每个星球执行一次 BFS。记录从该点出发，距离为 $d$ 的星球有多少个。
- **[块 3：前缀和转换]**：将 `f[i][d]` 转换为“距离 $\le d$”的星球总数。即 $f[i][d] = \sum_{j=0}^{d} (\text{距离为 } j \text{ 的点数})$。
- **[块 4：期望计算]**：对 $Q$ 个查询，直接累加 $f[x_i][min(y_i, n)]$，最后除以 $Q$。
# 笔记

## 代码

```python
import sys
# 读入数据
it = iter(sys.stdin.read().split())
n = int(next(it))
m = int(next(it))
q = int(next(it))
g = [[] for _ in range(n+1)]
for _ in range(m):
    u, v = int(next(it)), int(next(it))
    g[u].append(v)
    g[v].append(u)
# 全源 BFS 与前缀和预处理
f = [[0]*(n+1) for _ in range(n+1)]
for i in range(1,n+1):
    dist =[-1]*(n+1)
    dist[i] = 0
    que = [i]
    # DBS 算的是 当前节点到该节点的距离
    for u in que:
        for v in g[u]:
            if dist[v]==-1:
                dist[v]=dist[u]+1
                que.append(v)
    # i 走d步 的 有多少能到达的星球数量 
    for d in dist:
        if d != -1:
            f[i][d]+=1
    # 前缀和 计算 小于等于d的星球
    for d in range(1,n+1):
        f[i][d]+=f[i][d-1]
# 查询 计算期望
ans=0
for _ in range(q):
    x,y = int(next(it)),int(next(it))
    ans+=f[x][min(n,y)]
print(f"{ans/q:.2f}")
```
