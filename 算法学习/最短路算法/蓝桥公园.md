# 题目描述
**目标与边界**：在 $N \le 400$ 个点、$M$ 条边的带权无向图中，查询 $Q \le 10^3$ 对点之间的最短路径。  
**数据边界**：$N=400$ 是核心指标，隐含 $O(N^3)$ 复杂度上限；边权高达 $10^9$，需用 `float('inf')` 初始化。
# 分析
**算法/数据结构**：**Floyd-Warshall (全源最短路)**。  
**为何适用**：节点数 $N$ 较小（400），且存在大量随机起终点的查询（$Q=1000$）。Dijkstra 跑 $Q$ 次时间复杂度为 $O(Q \cdot M \log N)$，而 Floyd 一次性预处理只需 $O(N^3)$。  
**通用骨架**：
```python
# Floyd 核心三层循环（k 必须在外层）
for k in range(1, n + 1):
    for i in range(1, n + 1):
        for j in range(1, n + 1):
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

**定级**：基础题。 **核心难点**：对 $N=400$ 这一数字的敏感度。在 Python 中，$400^3 \approx 6.4 \times 10^7$，在 2 秒限制下较为紧凑，需注意循环效率（如：减少不必要的 `min` 调用）。

**[块 1：邻接矩阵初始化]**  
创建一个 $N \times N$ 的矩阵 `f`。初始化 `f[i][i] = 0`，其余为无穷大。读入边时，由于是无向图且可能存在重边，取 `f[u][v] = f[v][u] = min(f[u][v], w)`。

**[块 2：三层动态规划]**  
Floyd 本质是 DP：`f[i][j]` 表示只允许经过前 `k` 个点时 $i$ 到 $j$ 的最短路。最外层枚举中转点 `k`，内两层更新所有点对。

**[块 3：不可达判定与输出]**  
若 `f[st][ed]` 仍为无穷大，说明不连通，输出 -1。注意无穷大的判定要用一个足够大的数（如 `10^18`），避免加法溢出。
