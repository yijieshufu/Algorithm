# 题目描述
- **核心动作**：在 $n \times m$ 棋盘中，从 $(1,1)$ 格子开始触发染色，每个格子染色需 $t_{i,j}$ 秒，完成后触发四周邻居。求**全盘完成染色**的最早时刻（即最后一个格子完成染色的时间）。
- **数据边界**：$n, m \le 500$，格点总数 $2.5 \times 10^5$，染色时间 $t_{ij} \le 1000$。
# 分析
- **算法/数据结构**：**Dijkstra 最短路算法**。
- **为何适用**：格子的染色顺序具有明显的“**先到先触发**”特征。每个格子完成染色的时间 $F(i, j)$ 等于其最早被触发的时间（邻居完成时间）加上自身的染色耗时 $t_{i,j}$。这本质上是**求源点到所有点的最短路径**，权重落在目标节点上。
- **通用骨架**：
```python
while pq:
    d, u = heappop(pq)
    if d > dist[u]: continue
    for v in neighbors(u):
        if dist[u] + weight(v) < dist[v]:
            dist[v] = dist[u] + weight(v)
            heappush(pq, (dist[v], v))
```

- **定级**：基础题。
- **核心难点**：理解“触发机制”与“最短路”的等价性，并将网格抽象为带权图（边权即为目标格子的染色时间）。

- **[块 1：状态定义与初始化]**：定义 `dist[i][j]` 为格子 $(i, j)$ 完成染色的最早时刻。初始令所有 `dist = inf`，起始点 `dist[0][0] = t[0][0]`。

- **[块 2：最短路转移策略]**：从当前完成染色的格子 $(r, c)$ 向四周扩散，邻居 $(nr, nc)$ 的潜在完成时间为 `dist[r][c] + t[nr][nc]`。使用**优先队列维护最早完成**的格子。

- **[块 3：结果提取]**：当优先队列为空（所有可达点已处理），遍历 `dist` 矩阵寻找最大值。由于题目保证 $t_{ij} > 0$ 且棋盘连通，最大值即为最后染完的时间。
