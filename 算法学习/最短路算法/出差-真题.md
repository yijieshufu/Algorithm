# 题目描述
**目标**：在含点权的图中，求城市 $1$ 到 $N$ 的最短路径。总耗时 = 道路行驶时间 + 途中经过城市的隔离时间（注意：起点 $1$ 和终点 $N$ 的隔离时间不计入）。  
**数据边界**：$N \le 1000, M \le 10000$。典型的中小型规模单源最短路问题。
# 分析
**算法**：**Dijkstra (堆优化)**。  
**为何适用**：所有边权和点权均为正数（非负权图），且求单源最短路径。  
**通用骨架**（提取自模板）：
```python
import heapq
pq = [(0, s)] # (距离, 点)
while pq:
    d, u = heapq.heappop(pq)
    if d > dist[u]: continue
    for v, w in g[u]:
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            heapq.heappush(pq, (dist[v], v))
```

**定级**：基础题。 **核心难点**：**“点权化边权”** 的思维转换。需要通过逻辑预处理，**将城市的隔离时间合并到指向该城市的路径权重中**。

**[块 1：点权转化策略]** 隔离发生在“到达一个城市后”。若路径为 $1 \to 2 \to N$，总时间为：$(1,2)_{cost} + C_2 + (2,N)_{cost}$。 结论：从 $u$ 到 $v$ 的实际代价为 `edge_weight(u, v) + C_v`。 **特殊处理**：题目规定终点 $N$ 不计隔离时间，直接令 $C_N = 0$ 即可完美适配该逻辑。

**[块 2：构图与初始化]** 使用**邻接表存储转化后的有向边**。由于原图是**双向**的，转化后 $u \to v$ 的权值为 $w+C_v$，而 $v \to u$ 的权值为 $w+C_u$。

**[块 3：堆优化寻路]** 执行标准 Dijkstra 流程，维护一个 `dist` 数组记录从城市 $1$ 到各城市的最短时间。
