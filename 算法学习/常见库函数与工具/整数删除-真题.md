# 题目
- **核心动作**：在长度为 $N$ 的数列中，执行 $K$ 次“找最小值并删除，将其值加到左右邻居”的操作。若最小值不唯一，取最左边的。
- **数据边界**：$N, K \le 5 \times 10^5$，$A_i \le 10^8$。必须在 $O(N \log N)$ 复杂度内完成，普通 $O(NK)$ 模拟必 TLE。
# 分析
- **算法/数据结构**：**优先队列（堆）+ 双向链表（数组模拟）**。
- **为何适用**：需要频繁提取动态变化的最小值（堆的强项），且删除元素后需快速定位逻辑上的左右邻居（双向链表的强项）。
- **通用骨架（懒删除堆）**：
```python
while hq and count < K:
    val, idx = heappop(hq)
    if val != current_val[idx]: continue # 懒删除：值已过期，跳过
    # 执行逻辑...
```
- **定级**：**省一冲刺题 / 研究生组中等题**。
- **核心难点**：删除节点后，左右邻居会更新，导致堆中原本的“邻居值”失效。如何**低成本处理过期数据（懒删除）** 是关键

- **[块 1：逻辑邻居维护]**  
    使用两个数组 `L` 和 `R` 记录每个索引 $i$ 的左右邻居。删除 $i$ 时，让 $L[i]$ 的右边指向 $R[i]$，让 $R[i]$ 的左边指向 $L[i]$。这避免了物理删除数组元素的 $O(N)$ 开销。
- **[块 2：动态极值追踪]**  
    将 `(value, index)` 放入小根堆。因为 Python 的 `heapq` 不支持直接修改元素，我们采用 **“懒更新”策略**：当邻居加分后，直接把 `(新值, 索引)` 压入堆。弹出时，若堆中记录的值与当前数组 `V` 中的值不符，说明是“废弃数据”。
- **[块 3：邻居值累加]**  
    被选中的元素 $V[idx]$ 消失，其值分别加到 $V[L[idx]]$ 和 $V[R[idx]]$。加完后，这两个邻居因为“变强了”，需要作为新记录重新入堆。

## 代码

```python
import os
import sys
import heapq
it = iter(sys.stdin.read().split())
n = int(next(it))
k = int(next(it))
V =[int(next(it)) for _ in range(n)] # 最原始的值
L = [i - 1 for i in range(n)] # L[i] 存 i 的左邻居，0 的左邻居是 -1
R = [i + 1 for i in range(n)] # R[i] 存 i 的右邻居
R[n - 1] = -1
hq = []
for i in range(n):
  heapq.heappush(hq, (V[i], i)) # 存入小根堆
deleted = [False] * n 
ops = 0
while ops<k and hq:
  val, idx = heapq.heappop(hq) # 当前最小的值
  if deleted[idx] or val!=V[idx]: # 假删除 不是原来值 不看
    continue
  deleted[idx]=True
  ops+=1
  left_idx = L[idx] # 左邻居
  right_idx = R[idx] # 右邻居
  if left_idx != -1:
    V[left_idx] += val #值分给左边
    R[left_idx] = right_idx # 更新右邻居
    heapq.heappush(hq,(V[left_idx],left_idx)) # 将分了钱的新左邻居添加进去 加入新纸
  if right_idx != -1: # 同理
    V[right_idx] += val
    L[right_idx] = left_idx
    heapq.heappush(hq,(V[right_idx],right_idx))
ans = (str(V[i]) for i in range(n) if not deleted[i])
sys.stdout.write(" ".join(ans)+"\n")
```
