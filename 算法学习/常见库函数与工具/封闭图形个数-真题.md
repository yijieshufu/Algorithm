# 题目
题目重新定义了数字的“大小”逻辑，我们需要严格遵循以下优先级：
- **第一优先级（封闭图形数）**：统计每个数字包含的“圈”的总数，总数越少，数字越小。
- **第二优先级（数值大小）**：如果两个数字的“圈”一样多，则比较它们本身的数值，数值越小，数字越小。
- **单个数字的“权值”对照表**：

| **封闭图形数量** | **对应的数字成员**   |
| ---------- | ------------- |
| **0 个**    | 1, 2, 3, 5, 7 |
| **1 个**    | 0, 4, 6, 9    |
| **2 个**    | 8             |
> **注意：** 圈的个数是**累加**的。例如数字 $68$：$6$ 有 1 个，$8$ 有 2 个，总计 3 个。
# 分析
- **映射效率**：不要用 `if-else` 或 `switch` 去判断每个位，直接用长度为 10 的列表（哈希查找思想）下标索引，速度最快。
- **大数量级 I/O**：$n$ 高达 $2 \times 10^5$，如果用传统的 `input()` 循环读取，Python 极大概率会超时（TLE）。必须使用 `sys.stdin.read().split()` 配合迭代器一次性读入。
- **排序稳定性**：Python 的 `sort()` 是 Timsort（稳定的），且原生**支持元组**比较。我们构造 `(总圈数, 原始数值)` 这种元组，排序时它会自动先比第一项，第一项同再比第二项。

# 代码

```python
import sys
it = iter(sys.stdin.read().split())
n = int(next(it))
nums=[next(it) for _ in range(n)]
M = [1, 0, 0, 0, 1, 0, 1, 0, 2, 1]
res= []
for s in nums:
  sum =0
  for char in s:
    sum+=M[int(char)]
  items=(sum,int(s),s)
  res.append(items)
res.sort()
ans = []
for results in res:
  ans.append(results[2])
print(*ans)
```
