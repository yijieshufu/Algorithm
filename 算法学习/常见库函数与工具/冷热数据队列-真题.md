# 题目描述
- **核心动作**：模拟“冷热双队列”缓存机制。根据数据页 $p$ 是否在热队 $q_1$ 或冷队 $q_2$ 中，执行插入、移动或级联淘汰。
- **数据边界**：$n_1, n_2 \le 10^4$，$m \le 10^5$。必须实现 $O(1)$ 或 $O(\log N)$ 的单次操作，Python 需用哈希表优化。  
操作：
1. 若 $p$ 不在队列 $q$ 中（既不在 $q_1$ 中，也不在 $q_2$ 中），则加载数据页 $p$，并插入到 $q_2$ 的首部。 
2. 若 $p$ 已经在队列 $q$ 中，则将 $p$ 移动至 $q_1$ 首部。 
3. 当 $q_1$ 或 $q_2$ 队列容量不足时，会将其尾部的数据页淘汰出去。
4. 当 $q_1$ 已满但 $q_2$ 未满时，从 $q_1$ 中淘汰出的数据页会移动到 $q_2$ 首部。 
# 分析
- **算法/数据结构**：**LRU 变种（双双向链表 + 哈希表）**。
- **为何适用**：题目涉及大量的“移动至首部”、“淘汰尾部”及“是否存在”的操作，这正是 `OrderedDict` 或 `LinkedHashMap` 的核心应用场景。
- **通用骨架**：
```python
from collections import OrderedDict
d = OrderedDict()
d.move_to_end(k, last=False) # 移到队首
d.popitem(last=True)         # 弹出队尾
```

- **定级**：基础题（纯模拟）。
- **核心难点**：在于**规则 4 的级联逻辑**：从 $q_1$ 淘汰出的数据只有在 $q_2$ 未满时才进入 $q_2$ 首部，否则直接消失

- **[块 1：p 不在缓存中]** (Rule 1 & 3)：插入 $q_2$ 首部。若 $q_2$ 溢出（$> n_2$），则直接弹出 $q_2$ 尾部。
- **[块 2：p 已在缓存中]** (Rule 2)：无论原在 $q_1$ 还是 $q_2$，统一提升至 $q_1$ 首部（若在 $q_2$ 需先删除）。
- **块 3：$q_1$ 的溢出流向** (Rule 4)：当 $q_1$ 长度超过 $n_1$ 时，弹出 $q_1$ 尾部元素。检查 $q_2$ 当前长度，若 $< n_2$，则将被弹出的元素移至 $q_2$ 首部。

看到 LRU/缓存模拟必用 `OrderedDict`；严格区分 $q_1$ 溢出后的“去向条件”（$q_2$ 满与不满的情况）。

## 代码

```python
import sys
from collections import OrderedDict

# 输入流快速读取
it = iter(sys.stdin.read().split())
n1 = int(next(it))
n2 = int(next(it))
m = int(next(it))
q1 = OrderedDict()
q2 = OrderedDict()
for _ in range(m):
    p = next(it)
    if p in q1: # 已经在 q1，只需移到 q1 首部 [块 2]
        q1.move_to_end(p, last=False)
    elif p in q2: # 在 q2，提升到 q1 首部 [块 2]
        del q2[p]
        q1[p] = True # 无意义的占号
        q1.move_to_end(p, last=False)
        # 此时 q1 长度可能 +1，需检查溢出 [块 3]
        if len(q1) > n1:
            out_p, _ = q1.popitem(last=True)
            if len(q2) < n2: # Rule 4: q2 未满才接收
                q2[out_p] = True
                q2.move_to_end(out_p, last=False)
    else: # 全都不在，插入 q2 首部 [块 1]
        q2[p] = True
        q2.move_to_end(p, last=False)
        # 检查 q2 是否溢出 [块 1]
        if len(q2) > n2:
            q2.popitem(last=True)
# 格式化输出
print(*(q1.keys()))
print(*(q2.keys()))
```
