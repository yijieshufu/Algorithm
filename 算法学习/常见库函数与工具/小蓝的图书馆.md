# 题目描述
- **核心动作**：在 $n$ 次操作中，动态维护“作者—书籍数量”的映射。`add` 时增加计数，`find` 时查询计数。
- **数据边界**：$n \le 1000$，字符串长度 $\le 10$。数据量极小，无需考虑高级优化，哈希表（Dict）即可秒杀。
# 分析
- **算法/数据结构**：**哈希表（Python Dictionary）**。
- **为何适用**：题目本质是**频率统计**（Frequency Counting），哈希表支持 $O(1)$ 的插入与查询。
- **通用骨架**：
```python
d = {}
d[key] = d.get(key, 0) + val # 统计/累加
res = d.get(key, 0)         # 查询
```
- **定级**：基础题。
- **核心难点**：理解“书的数量”含义。即便书名相同，只要执行了 `add` 就要计数；书名在统计时其实是干扰信息。

- **[块 1：状态定义]**：使用字典 `d` 存储 `author: count`，记录每个作者拥有的书籍总数。
- **[块 2：核心逻辑]**：
    - `add`: 读取 `bookname` 和 `author`，仅将 `d[author]` 加 1（书名不影响计数）。
    - `find`: 读取 `author`，输出 `d[author]` 的当前值。
- **[块 3：边界处理]**：若 `find` 的作者从未出现，需输出 `0`（使用 `dict.get(au, 0)` 优雅解决）。

```python
import sys
# 块 3：输入流初始化
it = iter(sys.stdin.read().split())
n = int(next(it))
# 块 1：定义状态容器
d = {}
for _ in range(n):
    op = next(it)
    if op == 'add':
        # 块 2：核心更新（忽略无用的 bookname）
        _book = next(it)
        au = next(it)
        d[au] = d.get(au, 0) + 1
    else:
        # 块 2：核心查询
        au = next(it)
        print(d.get(au, 0))
```
