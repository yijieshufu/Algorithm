# 题目描述
- **核心动作**：在一个动态集合中执行 $q$ 次“插入一个数”或“查询一个数是否存在”。
- **数据边界**：$q \le 10^5$（操作多），$x \le 10^9$（数值大，不可直接开数组，必须用哈希）。
# 分析
- **算法/数据结构**：**哈希表 (Hash Set)**。
- **为何适用**：题目本质是成员判定问题，Python 的 `set` 底层由高性能哈希表实现，插入与查询的平均时间复杂度均为 $O(1)$。
```python
res = set()
res.add(val)      # 插入
if val in res:    # 查询
```
- **定级**：基础题。
- **核心难点**：处理 $10^5$ 级的数据量时，必须使用 `sys.stdin.read` 快速输入，否则 Python 原生 `input()` 易超时。
- **[块 1：极速 I/O 准备]** 使用迭代器一次性读取所有输入，避免循环内多次调用 I/O 函数。
- **[块 2：容器初始化]** 建立空集合 `s = set()`。
- **[块 3：逻辑分支处理]** 遍历 $q$ 次操作，根据操作符 `I` 或 `Q` 执行 `add` 或 `in` 判断。
