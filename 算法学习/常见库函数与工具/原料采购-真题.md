# 题目描述
- **核心动作**：在一轴线上选择若干采购点，使采购总量达到 $m$。总花费 = $\sum(\text{物品单价} \times \text{数量}) + \text{最远采购点距离} \times o$。求最小总花费。
- **数据边界**：$n \le 10^5$，$m \le 10^9$（数量巨大，需按“块”处理），距离 $c_i$ 各不相同。
# 分析
- **算法/数据结构**：**贪心 + 大根堆（优先队列）**。
- **为何适用**：当我们决定跑 $c_k$ 这么远时，最优策略一定是：**在 $c_k$ 范围内的所有店里，挑最便宜的买**。随着距离 $c$ 增加，可选的货变多了，但路费也贵了。我们需要在每一个新店处，尝试用新店的廉价货替换手里已有的高价货。
- **通用骨架**：
    1. 按距离对采购点排序。
    2. 用大根堆维护当前“最省钱的 $m$ 个原料”。
    3. 每到一个新点，由于距离增加，计算一次当前成本并更新最小值。

- **定级**：中等题（省赛/国赛高频考点）。
- **核心难点**：$m$ 很大，不能按单位重量入堆，必须维护 `[价格, 数量]` 二元组进行**批量扣减**。

- **[块 1：排序预处理]**  
    必须按距离 $c_i$ 从小到大排序。因为路费是随距离单调递增的，只有按顺序走，才能动态决定“走到哪停最划算”。
- **[块 2：大根堆的“去粗取精”]**  
    每经过一个点，把该店所有库存存入大根堆。如果当前总货量超过了 $m$，就从堆顶弹出**最贵**的货扔掉。这样保证了在当前距离下，我们的物料费是最低的。
- **[块 3：分批扣除技巧]**  
    当需要扔掉多余的货时，如果堆顶那批货的数量大于超出的量，仅扣除数量而不全弹出；否则直接整批弹出。这保证了程序不会因为 $m$ 太大而超时。

**“多跑路，是为了换更便宜的货。”**  
碰到这种“总费用 = 采购费 + 距离费”的题，直接**按距离排序**，用**大根堆**维护一个容量为 $m$ 的“最廉价备选库”。每推进一步，就用新出的低价货踢走库里的高价货！
