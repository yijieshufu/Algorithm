## 算法笔记：埃拉托斯特尼筛法（埃氏筛）

### 1. 它解决了什么痛点？

如果我们要找 $1$ 到 $n$ 之间所有的质数（素数），最朴素的办法是：对每个数字都进行一次“质数判定”（看它能不能被 $2$ 到 $\sqrt{n}$ 之间的数整除）。

- **痛点：** 这种“一个一个查”的方法效率很低。就像你在操场上找哪些同学没带红领巾，你得跑到每个同学面前去问一遍，非常耗时。
    

### 2. 核心思路：类比“连坐法”

埃氏筛的思路非常巧妙，它不是“寻找质数”，而是 **“排除合数”**。

- **生活类比：** 想象你在一个班级里，所有人初始都贴着“好人”标签。
    
    1. 从 2 号同学开始，2 号是好人，但他所有的**倍数**（4, 6, 8...）都被判定为“坏人”。
        
    2. 找到下一个还没被贴“坏人”标签的（比如 3 号），3 号是好人，但他所有的**倍数**（6, 9, 12...）也要贴上“坏人”标签。
        
    3. 以此类推，最后剩下没被贴“坏人”标签的，全是好人（质数）。

### 代码
```python
prime = []
is_prime = [True] * N

def Era(n):
    is_prime[0] = is_prime[1] = False
    for i in range(2, n + 1):
        if is_prime[i]:
            prime.append(i)
            if i * i > n:
                continue
            for j in range(i * i, n + 1, i):
                is_prime[j] = False
```


##  算法笔记：整数分解 (Integer Factorization)

### 1. 它解决了什么痛点？

在数学和算法竞赛中，我们经常需要知道一个数字的“本质属性”。

- **痛点：** 拿到一个很大的数字（比如 84），我们很难一眼看出它是由哪些基础数字构成的，这导致我们很难计算它的约数个数、最大公约数或者进行复杂的数论变换。
    
- **解决：** 整数分解就像是一台 **“粉碎机”**，能把任何一个合数拆解到不能再拆为止，剩下的全部是质数。
    

### 2. 核心思路：数字的“乐高模型”

**算术基本定理**告诉我们：每一个大于 1 的整数，都可以唯一地拆成一堆质数的乘积。

- **生活类比：**
    
    - 质数就是 **“化学元素”**（如氢、氧），不可再分。
        
    - 合数就是 **“化学物质”**（如水 $H_2O$）。
        
    - 整数分解的过程，就是把物质拆回元素的过程。例如 12 可以拆成 $2 \times 2 \times 3$，即 $2^2 \times 3^1$。无论你怎么拆，最后得到的“零件”种类和数量一定是固定的。
        
- **核心步骤（试除法）：**
    
    1. 从最小的质数 2 开始试除。
        
    2. 如果能整除，就一直除，并记录除掉的次数（指数）。
        
    3. 除不动了，就换下一个质数（3, 5, 7...）。
        
    4. **优化技巧：** 我们只需要试除到 $\sqrt{n}$ 即可。如果除完剩下的数字大于 1，那它本身就是一个质数。

## 组合数 (Combination Numbers) —— 选东西的“花样”

**它解决了什么痛点？**

当你想从 $n$ 个不同的苹果里挑出 $k$ 个带走，但**不在乎拿的先后顺序**时，组合数 $C(n, k)$ 就能直接告诉你一共有多少种拿法。

**核心思路：**

- **计算公式**：
    
    $$C(n, k) = \frac{n!}{k!(n-k)!}$$
    
    其中 $n!$ 是阶乘，表示把 $n$ 个数全部排成一排的方案数。
    
- **杨辉三角（递推法）**：
    
    - 这是省赛中最常用的技巧。它的逻辑是：对于第 $n$ 个苹果，你要么**选它**（剩下的要在 $n-1$ 个里选 $k-1$ 个），要么**不选它**（剩下的要在 $n-1$ 个里选 $k$ 个）。
        
    - 公式：$C(n, k) = C(n-1, k-1) + C(n-1, k)$。


## 快速幂

### 1. 解决了什么痛点？

- **普通幂**：算 $a^n$ 要乘 $n$ 次，如果要算 $2^{10^9}$，电脑得跑好几秒，比赛直接超时。
    
- **快速幂**：利用“翻倍”的逻辑，把时间从 **$O(n)$** 降到 **$O(\log n)$**。算 $10^9$ 次方，只需要约 30 次运算。
    

---

### 2. 核心思路：折纸与二进制

**大白话类比：**

如果你想让一张纸叠出 64 层厚度：

- **笨办法**：拿 64 张纸一张一张往上叠（乘法 63 次）。
    
- **聪明办法**：对折 6 次（$2 \to 4 \to 8 \to 16 \to 32 \to 64$）。
    

**算法逻辑：**

把指数 $n$ 看成二进制。比如 $n=13$（二进制 `1101`），其实就是 $8 + 4 + 1$。

我们只需要维护一个不断“自乘翻倍”的底数（$a^1 \to a^2 \to a^4 \to a^8$），碰到二进制位是 1 的时候，把它乘进结果里就行了。

### 代码

```python
def quick_pow(a, n):
    ans = 1
    while n > 0:
        if n & 1:  # 如果该二进制位存在
            ans = ans * a % MOD
        a = a * a % MOD
        n >>= 1  # n除以2,判断下一个二进制位
    return ans
```
