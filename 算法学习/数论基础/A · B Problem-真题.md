# 题目描述

设有两个二维向量 $\vec{A}(X_A, Y_A), \vec{B}(X_B, Y_B)$。给定 $L$，求 $(X_A, Y_A), (X_B, Y_B)$ 有多少种不同的取值，使得：
1. $X_A, Y_A, X_B, Y_B$ 均为**正整数**；
2. $\vec{A} \cdot \vec{B} \le L$，其中 $\vec{A} \cdot \vec{B}$ 表示 $\vec{A}, \vec{B}$ 的内积，即 $X_A \cdot X_B + Y_A \cdot Y_B \le L$。
### 输入格式
输入的每一行包含一个正整数 $L$，表示题目描述中的限制条件。
# 分析过程
## 为什么要求约数？

回到公式：$X_A \cdot X_B + Y_A \cdot Y_B \le L$。  
我们把 $X_A \cdot X_B$ 抽象成 **礼物 A 的价格 $s$**。  
假设现在礼物 A 的价格 **$s$ 固定是 6 元**。题目问的是 $(X_A, X_B)$ 有多少种取值。  
这不就是在问：**哪两个正整数相乘等于 6？**  
我们列举一下：
- $1 \times 6 = 6$
- $2 \times 3 = 6$
- $3 \times 2 = 6$
- $6 \times 1 = 6$  
你看，这里的 $1, 2, 3, 6$ 恰好就是 **6 的所有约数**。  
所以：**“价格为 $s$ 的礼物 A 有多少种组合方案” = “$s$ 这个数字有多少个约数”**。  
如果不求约数，我们就得用两层循环去枚举 $X_A$ 和 $X_B$，再加上礼物 B 的 $Y_A$ 和 $Y_B$，四层循环会让计算机直接罢工。求约数是为了把 $X_A$ 和 $X_B$ 这两个变量 **“打包”** 成一个变量 $s$。

---

## 💡向量内积组合计数

### 1. 核心矛盾（痛点）
直接枚举四个变量 $(X_A, Y_A, X_B, Y_B)$ 的复杂度是 $O(L^2)$ 甚至更高，面对 $L=10^6$ 的数据量会超时。
### 2. 破题思路：化繁为简（变量打包）
我们将复杂的内积公式 $X_A X_B + Y_A Y_B \le L$ 拆解为两个独立的部分：
- **部分 S**：$X_A \times X_B = s$
- **部分 T**：$Y_A \times Y_B = t$
- **约束条件**：$s + t \le L$
### 3. 关键逻辑链条
1. **约数即方案**：对于每一个确定的价格 $s$，能凑出它的 $(X_A, X_B)$ 组合数，本质就是 $s$ 的**约数个数**，记作 $d(s)$。
    
2. **前缀和提速**：
    - 我们需要计算当礼物 A 选定 $s$ 后，礼物 B 可以选哪些价格。
    - 礼物 B 的价格 $t$ 范围是 $[1, L-s]$。
    - 总方案数是 $d(1) + d(2) + \dots + d(L-s)$。
    - 为了不重复计算这串加法，我们用**前缀和** $G(n)$ 提前存好结果，查表即得。
3. **计算终式**：

    $$


\text{Total Ans} = \sum_{s=1}^{L-1} (d(s) \times G(L-s))

$$
    

### 4. 执行流程（三步走）
- **第一步（倍数法）**：用“发传单”的思路， $O(L \log L)$ 速度求出 $1 \dots L$ 所有数的约数个数 $d[i]$。
- **第二步（求前缀）**：一叠即成，算出 $G[i]$ 数组。
- **第三步（求和）**：一层循环遍历 $s$，利用 $d[s] \times G[L-s]$ 算出结果。

# 代码

```python 
import os
import sys
it = iter(sys.stdin.read().split())
L = int(next(it))
if L < 2:print(0)
else:
  # 计算约数的个数
  d = [0]*(L+1)
  for i in range(1,L+1):
    for j in range(i,L+1,i):
      d[j]+=1
  # 计算约数前缀的个数
  G = [0]*(L+1)
  for i in range(1,L+1):
    G[i]=G[i-1]+d[i]
  ans = 0
  for s in range(1,L):
    ans += d[s] * G[L - s]
  print(ans)
```
