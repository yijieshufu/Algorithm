# 题目
小明需要将给定的 $n$ 个上半部分口诀 $a_i$ 和 $m$ 个下半部分口诀 $b_j$ 组合成完整口诀 $S = a_i + b_j$。

**有效魔法判定条件：**

1. $S \le n + m$
    
2. $S$ 为质数
    

**目标：** 计算一共能组合出多少种**不同**的有效魔法（即不同的 $S$ 值的数量）。

# 分析过程
### 第一步：去重（去粗取精）

虽然 $a$ 组有 $20,000$ 个口诀，但里面可能有很多重复的数字（比如有 100 个数字都是“5”）。

- **类比**：我们要凑重量，手里有 100 个 5 克的砝码和 1 个 5 克的砝码是一样的。
    
- **操作**：用 Python 的 `set()` 集合功能，只保留不重复的数字。
    

### 第二步：质数筛（准备好“质数检测仪”）

我们需要频繁判断一个和 $S$ 是不是质数。不要临时去算，要提前准备好一张“质数表”。

- **核心算法**：**埃氏筛**（Sieve of Eratosthenes）。
    
- **做法**：先假设 2 到 40,000 全是质数，然后从 2 开始，把 2 的倍数全部划掉，再把 3 的倍数划掉……剩下的就是质数。
    

### 第三步：位运算（终极加速器）

这是本题最精妙的地方。我们要找“所有可能的和 $S$”，而且要快。

- **核心逻辑**：利用计算机处理二进制的特性。
    
- **类比（尺子平移法）**：
    
    想象有一把无限长的**尺子 B**，如果在 $b$ 组里有数字 1、3、5，我们就在尺子的第 1、3、5 厘米处画个标记。
    
    现在看 $a$ 组，如果 $a$ 组里有个数字 2，我们就把整把**尺子 B** 向右移动 2 厘米。
    
    移动后，原来 1、3、5 的标记就变成了 3、5、7。这些新位置就是 $a_i + b_j$ 可能产生的和。
    

在 Python 中，一个超大的整数（BigInt）本质上就是一串二进制位。`1 << x` 就是在第 $x$ 位放一个“标记”。通过 `|`（按位或）和 `<<`（左移），我们可以一次性处理成千上万个组合。

# 代码

```python
import os
import sys
it = iter(sys.stdin.read().split())
n = int(next(it))
m = int(next(it))
a=[]
b=[]
for _ in range(n):
  a.append(int(next(it)))
for _ in range(m):
  b.append(int(next(it)))
a_set=set(a)
b_set=set(b)
limit = 40000
is_prime = [True] * (limit + 1)
is_prime[0]=is_prime[1] = False # 1 不是质数
for i in range(2,int(limit**0.5) + 1):
  if is_prime[i]:
    for j in range (i*i,limit+1,i):
      is_prime[j]=False
bits_b = 0
for x in b_set:
  bits_b |=(1 << x)
sum = 0
for x in a_set:
  sum |=(bits_b << x )
res = 0
for p in range(2,m+n+1):
  if is_prime[p]:
    if (sum>>p&1):
      res+=1
print(res)
```
