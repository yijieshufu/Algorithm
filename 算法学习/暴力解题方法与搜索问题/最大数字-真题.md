# 题目
- **核心动作**：在限定的 $A$ 次“+1（9变0）”和 $B$ 次“-1（0变9）”操作内，求正整数 $N$ 变换后能达到的最大值。
- **数据边界**：$N \le 10^{17}$（最多 18 位），$A, B \le 100$。
# 分析
- **算法/数据结构**：**记忆化搜索（DFS + Memoization）**。
- **为何适用**：每一位的选择会消耗 $A$ 或 $B$，进而影响后位的可选范围。这符合数位 DP 或记忆化搜索的特征。
- **通用骨架**：
```python
memo = {}
def dfs(idx, rem_a, rem_b):
    if idx == n: return 0
    state = (idx, rem_a, rem_b)
    if state in memo: return memo[state]
    # 决策逻辑...
    memo[state] = result
    return result
```
- **定级**：中等题（省赛常客）。
- **核心难点**：理解加法和减法在“变大”策略上的差异性——加法是“能加多少加多少”，减法则是“不成功（变 9）便成仁”。
- **[块 1：单步决策的分支]**  
    对于当前位数字 $d$，我们面临两个主要选择：
    1. **策略一（向下绕回 9）**：如果剩余 $B \ge d + 1$，可以将 $d$ 减到 $0$ 再减一次变成 $9$。这消耗 $d+1$ 次 $B$。
    2. **策略二（向上推向 9）**：优先用 $A$ 将 $d$ 推向 $9$。即便 $A$ 不够到 $9$，也要尽量消耗掉去增加数值（例如 $d=1, A=5$，变到 $6$ 总比保持 $1$ 大）。
- **[块 2：价值权重的叠加]**  
    由于是从最高位开始处理，当前位数字 $v$ 的实际贡献是 $v \times 10^{n-1-idx}$。我们递归地获取“后续位能提供的最大价值”，并与当前位贡献相加。
- **[块 3：状态压缩与记忆化]**  
    状态由 `(当前位置, 剩余A, 剩余B)` 唯一确定。状态总数约 $18 \times 100 \times 100 = 180,000$，在 Python 的内存和时间容忍范围内。

## 代码

```python
import sys

# 设置递归深度，防止处理 18 位数字时溢出
sys.setrecursionlimit(2000)
# [块 1: 核心 I/O 起手式] 
# 使用模板中推荐的极简高效输入
input_data = sys.stdin.read().split()
if not input_data:
    exit()
it = iter(input_data)

n_str = next(it)
a_limit = int(next(it))
b_limit = int(next(it))

nums = [int(c) for c in n_str]
n = len(nums)
memo = {}

def dfs(idx, a, b):
    # [块 3: 边界与初始化]
    if idx == n:
        return 0
    # 检查记忆化
    state = (idx, a, b)
    if state in memo:
        return memo[state]
    d = nums[idx]
    res = 0
    p10 = 10**(n - 1 - idx) # 当前位权重
    # [块 1 & 2: 核心推导 - 决策分支]
    # 方案 1: 尝试用减法 (B) 绕回 9
    if b >= d + 1:
        res = max(res, 9 * p10 + dfs(idx + 1, a, b - (d + 1)))
        
    # 方案 2: 尝试用加法 (A) 增加
    # 贪心：高位即使到不了 9，多加 1 也是赚的
    use_a = min(a, 9 - d)
    res = max(res, (d + use_a) * p10 + dfs(idx + 1, a - use_a, b))
    
    memo[state] = res
    return res

# 顺序执行并输出结果
print(dfs(0, a_limit, b_limit))
```
