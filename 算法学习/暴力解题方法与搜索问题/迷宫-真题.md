# 题目

- **核心动作**：在 $n \times n$ 的网格中，从所有格子均匀随机出发，求到达终点 $(n, n)$ 的最短步数期望值。
- **移动方式**：上下左右移动（代价 1）或使用 $m$ 个双向传送门（代价 1）。
- **数据边界**：$n, m \le 2000$。格点总数 $N^2 = 4 \times 10^6$。
# 分析
- **算法/数据结构**：**单源最短路 (BFS)**。
- **为何适用**：题目所有边权（网格移动与传送门）均为 $1$，且目标是求从一个终点到全图所有点的距离。
- **通用骨架**：
```python
# 逆向思维：从终点出发跑一次 BFS，距离即为所有点的最短步数
dist = [-1] * (N*N)
queue = [target]
while queue:
    curr = queue.pop(0) # 赛场建议用头指针模拟
    for neighbor in adj[curr]:
        if dist[neighbor] == -1:
            dist[neighbor] = dist[curr] + 1
            queue.append(neighbor)
```
- **定级**：中等题。
- **核心难点**：**Python 常数优化**。在 $4 \times 10^6$ 规模下，Python 必须规避函数调用、多维数组索引和频繁的边界逻辑判断，否则会超时（TLE）。

- **[块 1：一维平铺与边界哨兵]**  
    将二维坐标 $(r, c)$ 映射为 $r \times (n+2) + c$。在 $n \times n$ 外部多包围一圈“哨兵”节点并标记为已访问。这样 BFS 时只需判断 `dist[v] == -1`，无需每次检查坐标是否越界。
- **[块 2：双向传送门建图]**  
    由于传送门是双向的，且端点可能重复，使用**邻接表存储**。只有带有传送门的节点才触发传送逻辑，减少对普通节点的无效判断。
- **[块 3：极速 BFS 循环]**  
    使用大列表 `q` 和头指针 `head` 模拟队列，这是 Python 3.7.3 中最快的队列实现方式。
- **[块 4：期望值汇总]**  
    期望 $E = \frac{\sum \text{dist}}{n \times n}$。只需累加所有有效格子的距离即可。
