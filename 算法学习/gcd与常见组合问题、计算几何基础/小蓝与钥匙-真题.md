# 题目描述
**题目名称**：小蓝与钥匙
**核心背景**：
- 有 **28** 个孩子，每人对应一把专属房间钥匙。
- 将 28 把钥匙打乱，随机发给 28 个孩子（每人一把）。
- **目标**：计算有多少种方案，使得**恰好有 14 个孩子**拿到了自己的钥匙，而另外 **14 个孩子**拿到的都不是自己的钥匙。
    **约束条件**：
- 总方案数为 $28!$（这只是背景，不需要全部算出）。
- 必须是“恰好”14 人对，14 人错。
# 分析过程
### Step 1: 痛点与定位

- **痛点**：如果你只想到“选出 14 个人”，你会发现剩下的 14 个人怎么分配钥匙是个大难题——万一剩下的 14 个人里又有人拿对了呢？
    
- **题型定位**：**组合数学 + 错排问题 (Derangement)**。这是一类专门研究“如何让所有人都不在自己位置上”的经典模型。
    

### Step 2: 通用解法模板

这类“$N$ 个人里恰好 $K$ 个人对”的问题，标准思考模板如下：

1. **第一步（选对的）**：从 $N$ 个人中选出 $K$ 个人，给他们正确的钥匙。方案数：$C(N, K)$。
    
2. **第二步（选错的）**：剩下的 $N-K$ 个人，必须**全部拿错**。这就是典型的**错排问题**，记作 $D(N-K)$。
    
3. **最终答案**：$Ans = C(N, K) \times D(N-K)$。
    

**错排公式 $D(n)$ 的递推式**：

- $D(1) = 0$（一个人不可能拿错自己的钥匙）
    
- $D(2) = 1$（两个人只能互换钥匙才能都拿错）
    
- $D(n) = (n-1) \times [D(n-1) + D(n-2)]$
    

### Step 3: 大白话拆解

想象一下，你手里有 28 个小朋友和 28 把钥匙：

1. **挑出“幸运儿”**：
    
    你先在大屏幕上随机点名，选出 14 个小朋友。你温柔地把他们各自的钥匙发给他们。这一步就像从一篮子苹果里挑 14 个红的，有 $C(28, 14)$ 种挑法。
    
2. **制造“大混乱”**：
    
    现在剩下的 14 个小朋友眼巴巴地看着你。你的任务是：**把剩下的 14 把钥匙发下去，但绝对不能让任何一个人拿到属于自己的那一把！**
    
    - 如果只剩 1 个人，他肯定拿对，所以 $D(1)=0$。
        
    - 如果剩 2 个人（A 和 B），只有一种可能：A 拿 B 的，B 拿 A 的。所以 $D(2)=1$。
        
    - 当剩下 14 个人时，情况就很复杂了，但我们可以用上面那个神奇的递推公式像上楼梯一样算出来。


# 代码
```python
import os
import sys
n = 28
k = 14
fac = [1]*(n+1)
for i in range(1,n+1):
  fac[i]=i*fac[i-1]
comb = fac[n] // (fac[n-k] * fac[k])
d = [0]*(k+1)
if k>=1:d[1]=0
if k>=2:d[2]=1
for i in range(3,k+1):
  d[i] = (i-1)*(d[i-1]+d[i-2])
print(comb*d[k])
```
