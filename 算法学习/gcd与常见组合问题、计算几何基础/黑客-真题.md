# 题目描述

**场景描述**：

原本有一个 $n \times m$ 的矩阵，加上 $n$ 和 $m$ 这两个数，总共 $n \times m + 2$ 个正整数。现在顺序被黑客打乱了，只给你这一堆乱序的数字，问你：这些数字**最多**能构成多少种不同的原矩阵？

**判定标准**：

两个矩阵不同，要么是行列数 $(n, m)$ 不同，要么是矩阵里某个坑位的数字不同。

**约束条件**：

- 总数 $K = n \times m + 2$。
    
- 需要输出结果对 $10^9 + 7$ 取模。
# 分析过程
## 核心痛点与解题思路

当数据规模高达 **$5 \times 10^5$** 时，如果针对每一对可能的矩阵长宽 $(n, m)$ 都去暴力重算一次排列组合，一定会超时。

**破局点**：计算一个“假想的全局基准值”，然后利用数学公式，将复杂的 **“分母变化”** 转化为极简的 **“分子乘法”**，实现 $O(1)$ 时间内的动态修正。

---

### Step 1：明确我们在“数”什么？（题目规则拆解）

想象你手里有 **6** 个数字：`{2, 2, 1, 4, 3, 3}`。

题目要求：挑出两个数作为矩阵的“长”和“宽”，剩下的数全部填进矩阵里。

- **规则 A（面积守恒）**：长 $\times$ 宽 必须等于 剩下的数字个数。
    
    在当前例子中，总数 $K=6$，所以必须满足 $n \times m = 6 - 2 = 4$。
    
- **规则 B（就地取材）**：长和宽本身，必须从这 **6** 个数里挑。
    
    比如你挑了 `1` 和 `4` 当长宽，剩下的数字就是 `{2, 2, 3, 3}`。
    
- **规则 C（多样性）**：只要长宽组合不同，或者填入矩阵的数字顺序不同，就算作不同的矩阵。

**小白痛点**：如果你挑了 `(1, 4)`，剩下的 `{2, 2, 3, 3}` 能在 4 个格子里摆出多少种不同的花样？这就是我们要计算的。

---

### Step 2：多重集排列——消灭“双胞胎”的影响

如果剩下的数字是 `{A, B, C, D}`（全都不一样），那随便排，排列方式就是 $4! = 24$ 种。

但如果是 `{2, 2, 3, 3}`，两个 `2` 互换位置，矩阵看起来是一模一样的，这就“算重”了。

**数学公式（多重集排列）**：

$$
\text{排列数} = \frac{(\text{总个数})!}{(\text{重复数字1的个数})! \times (\text{重复数字2的个数})! \times \dots}
$$

对于 `{2, 2, 3, 3}`，排列数就是 $\frac{4!}{2! \times 2!} = \frac{24}{4} = 6$ 种。

---

### Step 3：核心神技——为什么“动态修正”能变除为乘？

**【面临的灾难】**

因为你要不断尝试不同的 $(n, m)$ 组合。拿走的数字不同，剩下的数字频率就不同，公式的分母每次都要重新计算，速度极慢！

**【第一步：制造一个假想的“基准大蛋糕” $W$】**

假设所有 $K$ 个数字出现的次数记为 $f(i)$。我们先算一个“如果在 $K-2$ 个位置里，数字一个都没被拿走”的假想基准值：

$$
W = \frac{(K-2)!}{\prod f(i)!}
$$

**【第二步：见证奇迹的时刻（详细推导）】**

假设数字 `2` 在总库里原本有 **3** 个（即 $f(2) = 3$）。

- 基准值 $W$ 的分母里，此时除以的是 **$3!$**（也就是 $3 \times 2 \times 1$）。
    
- 现在，你拿走了一个 `2` 当矩阵的长。剩下的 `2` 只剩 **2** 个了。
    
- 按照真实的排列规则，你本来应该除以的是 **$2!$**（也就是 $2 \times 1$）。

那么，如何把“除以 $3!$”修正为“除以 $2!$”呢？

我们在数学上知道：$3! = 3 \times 2!$。

所以：

$$
\frac{1}{2!} = \frac{3}{3!}
$$

看明白了吗？**原本你应该除以一个变小的阶乘，现在完全等同于：在除以大阶乘的基础上，直接乘回数字原本的个数！**

**【结论：点金之笔】**

不用重新算分母了！你拿走了谁，就在基准值 $W$ 的基础上，乘上谁**原本的个数**。

- **如果 $n \neq m$**（长方形）：
    
    你拿走了一个 $n$ 和一个 $m$。
    
    **当前方案数 = 基准值 $W \times f(n) \times f(m)$**
    
- **如果 $n = m$**（正方形）：
    
    你连续拿走了两个相同的 $n$。第一次拿走，乘 $f(n)$；第二次拿走时，这个数字只剩 $f(n)-1$ 个了，所以乘 $f(n)-1$。
    
    **当前方案数 = 基准值 $W \times f(n) \times (f(n)-1)$**

> [!TIP]
> 
> **基准值 $W$** 就像一个固定不动的大型分配模板，每当你换一对领队 $(n, m)$，你不需要重算模板，只需要根据这两个领队“原本的势力范围（数量）”，在这个模板上乘上对应的倍数即可。

---

### Step 4：乘法逆元——在取模世界里做除法

题目要求结果对 $10^9+7$ 取模。

在取模的世界里，不允许直接使用 `/` 号做除法。

- **规则**：想除以一个数 $X$，就等同于乘以 $X$ 在模意义下的“倒数”（即乘法逆元）。
    
- **操作**：我们通过费马小定理 `pow(X, mod-2, mod)` 预处理出所有的逆元存进 `invf` 数组里，这样计算基准值 $W$ 的除法时，就能一瞬间用乘法替代完成。

---

### Step 5：最终算法流程总结

1. **统计库房**：遍历输入的数据，数一数每个数字出现了几次，存进频次数组 $F$ 中。
    
2. **预处理工具箱**：算出 $1$ 到 $K$ 的所有阶乘，以及它们的阶乘逆元，备用。
    
3. **计算基准 $W$**：先算出假想状态下的分数 $W = \frac{(K-2)!}{\prod F[i]!}$。
    
4. **根号枚举找约数**：枚举能让 $n \times m = K-2$ 的所有组合（循环条件 `d * d <= K-2` 即可，极大节省时间）。
    
5. **秒算累加**：根据 Step 3 的“点金之笔”公式，直接用乘法算出每种 $(n, m)$ 组合的方案数，全部加起来，最后乘以 $W$ 即为通关答案。

# 代码

```python
import os
import sys
from collections import Counter
it = iter(sys.stdin.read().split())
mod =1000000007
# 1. 预处理阶乘与逆元
def prepare(limit):
  fac = [1]*(limit+1)
  inv = [1]*(limit+1)
  for i in range(1,limit+1):
    fac[i] = fac[i-1]*i % mod
  inv[limit] = pow(fac[limit],mod-2,mod)
  for i in range(limit-1,-1,-1):
    inv[i] = inv[i+1]*(i+1)%mod
  return fac,inv
K = int(next(it))
nums=[int(next(it)) for _ in range(K)]
counts= Counter(nums)
X = K - 2
fac,inv = prepare(K)
denom_inv = 1
for val in counts:
  denom_inv = denom_inv * inv[counts[val]] %mod
W = (fac[X]* denom_inv)% mod
total_contribution = 0
d = 1
while d * d <= X:
  if X % d == 0:
    y = X // d 
    if d == y :
      if counts[d]>=2:
        term = (counts[d]*(counts[d]-1))%mod
        total_contribution = (total_contribution +term) % mod
    else:
      if counts[d]>=1 and counts[y]>=1:
        term = (2*counts[d]*counts[y])%mod
        total_contribution = (total_contribution +term) % mod
  d+=1
print((W*total_contribution)%mod)
```
