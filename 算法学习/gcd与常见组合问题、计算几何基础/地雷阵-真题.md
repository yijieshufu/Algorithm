# 题目
### 📝 题目转述：小蓝的逃生游戏

**场景设定：**

在一个平面直角坐标系的第一象限里，埋了 $n$ 颗地雷。每颗地雷都有自己的位置 $(x_i, y_i)$ 和爆炸半径 $r_i$。如果小蓝踏入任何一颗雷的圆形触发范围（含边界），游戏就直接 GG。

**任务目标：**

小蓝从原点 $(0,0)$ 出发，他必须在第一象限内（即 $0^\circ$ 到 $90^\circ$ 之间）**随机选择一个固定的方向**，然后沿着这个方向头也不回地一直走下去。

**核心问题：**

如果在 $[0, \frac{\pi}{2}]$ 这个角度区间内均匀随机地选一个方向，请问小蓝**不触发任何地雷**并顺利通关的概率是多少？

**输入输出要求：**

- **输入**：第一行是地雷数 $n$；接下来 $n$ 行，每行三个整数 $x, y, r$。
    
- **输出**：一个实数，四舍五入保留 **3 位小数**。

**约束条件（划重点）：**

- $1 \le n \le 10^5$（数据量很大，$O(n^2)$ 算法会超时，必须 $O(n \log n)$ 或 $O(n)$）。
    
- $1 \le x_i, y_i \le 10^4$。
    
- $r_i < \min(x_i, y_i)$：这保证了地雷圆圈**不会覆盖原点**，游戏开始是安全的。

# 分析过程

这道题是蓝桥杯中典型的**计算几何**与**贪心算法（区间合并）** 的结合体。

---

### Step 1: 痛点与定位

- **核心痛点**：题目问的是“概率”，看似是概率题，实则是一个**几何覆盖问题**。我们需要知道在 $0^\circ$ 到 $90^\circ$（即 $0$ 到 $\frac{\pi}{2}$ 弧度）的视野里，有多少比例的角度被地雷“挡住”了。
    
- **题型定位**：
    
    1. **解析几何**：将圆转化为原点视角下的角度区间。
        
    2. **区间合并**：处理多个地雷扇区重叠的情况，防止重复计算。
        
- **复杂度分析**：地雷数 $n$ 高达 $10^5$，必须使用 $O(n \log n)$ 的算法，其中排序是性能瓶颈。

---

### Step 2: 通用解法模板

这类“视角覆盖”或“区间遮挡”问题的标准逻辑如下：

1. **坐标转换**：遍历每个障碍物，计算其相对于观测点（原点）的起始角度 $\alpha$ 和终止角度 $\beta$。
    
2. **区间清洗**：将所有区间修剪到目标观测范围（如 $[0, \frac{\pi}{2}]$）内。
    
3. **排序合并**：使用贪心策略进行区间合并。
    
    - 按区间左端点排序。
        
    - 维护当前合并区间的 `start` 和 `end`，若新区间与旧区间重叠，更新 `end`。
        
4. **结果产出**：$\text{概率} = \frac{\text{总范围} - \text{合并后的总长度}}{\text{总范围}}$。

---

### Step 3: 大白话拆解

#### 1. 把地雷看作“挡箭牌”

想象你站在坐标原点 $(0,0)$，向第一象限望去。每个圆形地雷就像一面圆形的盾牌挂在空中。如果你射出的子弹（行进方向）穿过了这面盾牌，你就输了。

#### 2. 计算盾牌遮住的角度

我们要计算盾牌边缘与原点形成的夹角。

- **圆心角**：圆心在 $(x,y)$，利用 `atan2(y, x)` 可以算出圆心相对于 $x$ 轴的角度 $\theta$。
    
- **张角的一半**：原点到圆心的距离为 $d = \sqrt{x^2+y^2}$。从原点向圆引切线，利用直角三角形关系可知，切线与圆心连线的夹角 $\Delta \theta$ 满足 $\sin(\Delta \theta) = \frac{r}{d}$，即 $\Delta \theta = \arcsin(\frac{r}{d})$。
    
- **危险区间**：这颗雷挡住的角度就是 $[\theta - \Delta \theta, \theta + \Delta \theta]$。

#### 3. 处理“影子”重叠

有的雷在你的左前方，有的在你的右前方，它们的“影子”（遮挡区间）可能会重合。如果直接把所有遮挡角度相加，就会算重。

- **例子**：雷 A 挡住了 $10^\circ \sim 30^\circ$，雷 B 挡住了 $20^\circ \sim 40^\circ$。它们一共只挡住了 $10^\circ \sim 40^\circ$（共 $30^\circ$），而不是 $20+20=40^\circ$。
    
- 这就是为什么要用到**区间合并**算法。

## 代码

```python
import os
import sys
import math

it = iter(sys.stdin.read().split())

n = int(next(it))
intervals = []

Limit_L=0;Limit_R=math.pi/2

for _ in range(n):
  x = int(next(it))
  y = int(next(it))
  r = int(next(it))

  dist = math.sqrt(x**2+y**2)
  mid_angle = math.atan2(y,x)
  delta = math.asin(r/dist)
  
  L=max(Limit_L,mid_angle-delta)
  R=min(Limit_R,mid_angle+delta)

  if L<R:
    intervals.append((L,R))

if not intervals:
        print(f"{1.000:.3f}")

intervals.sort()

merged_len = 0.0

cur_start, cur_end = intervals[0]

for i in range(1,len(intervals)):
  nxt_start, nxt_end = intervals[i]
  if nxt_start<=cur_end:
    cur_end = max(cur_end,nxt_end)
  else:
    merged_len +=(cur_end-cur_start)
    cur_start, cur_end = nxt_start, nxt_end


merged_len +=(cur_end-cur_start)
total_range = Limit_R - Limit_L
safe_prob = (total_range - merged_len) / total_range

# 保证结果不小于 0 (防止浮点误差)
print(f"{max(0.0, safe_prob):.3f}")
```
