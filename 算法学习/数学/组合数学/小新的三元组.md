
### 1. 题目背景与模板

- **题目描述**：小新有 $N^3$ 个三元组 $(i, j, k)$（$1 \le i, j, k \le N$）。按照如下规则排序：1. $(i+j+k)$ 较小的排在前面。2. 若和相同，则 $i$ 较小的排在前面。3. 若和与 $i$ 都相同，则 $j$ 较小的排在前面。求按此规则排在第 $K$ 小的三元组。
    
- **输入条件**：单行包含 2 个正整数 $N, K$ （$1 \le K \le N^3$）。
    
- **所选题型**：组合计数 + 容斥原理 + 逐位确定。

### 2. 题目与核心思路

- **逻辑总结**：本题采用“逐位确定”（贪心）的思想寻找字典序目标元素。第一步：按和 $S = i+j+k$ 从小到大枚举，利用插板法配合容斥原理（总数 - 至少1个变量越界 + 至少2个变量越界）快速算出和为 $S$ 且满足变量皆 $\le N$ 的三元组数量，进而锁定目标 $K$ 所在的具体和数值 $S$。第二步：在锁定的 $S$ 内，依序从小到大枚举 $i \in [1, N]$，利用 $j+k = S-i$ 以及隐含的 $1 \le j, k \le N$ 直接解出 $j$ 的合法区间界限 $[L, R]$。一旦 $K$ 落入当前区间长度内，直接推算出具体的 $j$ 和 $k$。
    
- **时空复杂度**：
    
    - **时间复杂度**：$O(N)$。遍历 $S$ 最多执行 $3N$ 次，固定 $S$ 后遍历 $i$ 最多执行 $N$ 次。每次循环内部均为 $O(1)$ 的数学运算，无嵌套。
        
    - **空间复杂度**：$O(1)$。仅需常数级别空间用于记录中间求和与索引状态。

### 3. 我的盲点记录

- **疑惑点**：不理解“插板法”和“容斥原理”是如何在本题结合起来计算特定约束解的。
    
- **解答**：可以把两个方法分工拆解来看：
    
    1. **插板法解决的是“下界”**：它用来算出“每个人最少拿1个”的总分法，即方程 $i+j+k=S \ (i,j,k \ge 1)$ 的正整数解，公式为 $C(S-1, 2)$。
        
    2. **容斥原理解决的是“上界”**：插板法没考虑到“每个人最多只能拿 $N$ 个”的限制。所以要用容斥原理修剪非法状态：用**总方案数** 减去 **非法情况（至少1个变量 $>N$）**，再加上被多减的 **（至少2个变量 $>N$）**。由于 $S \le 3N$，三个变量不可能同时严格 $>N$，因此不需要再减第三层。

### 4. 最终范本代码

```Python

import sys; input=sys.stdin.read().split(); it=iter(input);

# -> 剔除了预处理大阶乘的模板。因为本题只涉及 C(n, 2)，可以直接 O(1) 计算，避免多余开销
def C2(n):
    if n < 2: return 0
    return n * (n - 1) // 2

# -> 针对 3 个变量对 count_triplets 进行了特化改造，剥离了内部的 for 循环和 combinations 调用
def count_triplets(S, N):
    res = C2(S - 1) 
    res -= 3 * C2(S - N - 1)
    res += 3 * C2(S - 2 * N - 1)
    return res

# 核心算法：纯函数封装
def solve(n, k):
    target_s = -1
    for s in range(3, 3 * n + 1):
        cnt = count_triplets(s, n)
        if k <= cnt:
            target_s = s
            break
        k -= cnt # -> 如果 k 大于当前和的方案总数，则说明目标不在该组，减去当前组合数，继续找下一个和
        
    for i in range(1, n + 1):
        rem = target_s - i
        # -> 根据 1 <= j <= n 和 1 <= k <= n (即 k = rem - j >= 1)，直接计算出 j 动态缩紧后的有效区间
        min_j = max(1, rem - n) 
        max_j = min(n, rem - 1) 
        
        if min_j <= max_j:
            count_j = max_j - min_j + 1
            if k <= count_j:
                res_j = min_j + k - 1
                res_k = rem - res_j
                return f"{i} {res_j} {res_k}"
            k -= count_j # -> 若 k 依然大于该 i 前缀的合法解数量，继续推移并累减
    return ""

# 主流程：外部只负责 I/O 
if input:
    n = int(next(it))
    k = int(next(it))
    ans = solve(n, k)
    sys.stdout.write(ans + "\n")
```
