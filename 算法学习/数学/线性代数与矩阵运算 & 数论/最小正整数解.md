### 2. 题目与核心思路

**逻辑总结：**

本题核心在于理解**裴蜀定理（Bézout's identity）**：对于不全为 $0$ 的整数 $a, b$，方程 $ax + by = d$ 有整数解的充要条件是 $d$ 为 $\gcd(a, b)$ 的倍数。因此，$ax + by$ 能够构造出的**最小正整数**必然是 $\gcd(|a|, |b|)$。

**复杂度分析：**

- **时间复杂度：** $O(T \cdot \log(\min(|a|, |b|)))$。单次 GCD 复杂度为对数级，总计算量约 $10^5 \times 31$ 次运算，远低于 1s 限制。
    
- **空间复杂度：** $O(1)$。仅使用常数级额外变量。

---

### 3. 我的盲点记录

- **负数处理：** 初始可能疑惑 $a, b$ 为负数时怎么办。实际上，$x, y$ 可取任意整数，负号可以通过 $x, y$ 的正负切换来抵消，因此统一取 $|a|$ 和 $|b|$ 计算 GCD 即可。
    
- **定理直观化：** 明白为什么凑不出来比 GCD 更小的数（因为 $a$ 和 $b$ 都是 $\gcd$ 的倍数，它们的线性组合也一定是 $\gcd$ 的倍数）。

---

### 4. 最终范本代码

```Python
import sys; input=sys.stdin.read().split(); it=iter(input);

# 核心算法：欧几里得算法
def gcd(a, b):
    while b:
        a, b = b, a % b
    return a

# 核心算法：纯函数封装
def solve():
    # 按照规范不使用 it 以外的输入方式
    if not it: return
    t = int(next(it))
    for _ in range(t):
        # 变量命名极简，强制 1-indexed 逻辑
        v1 = abs(int(next(it))) # -> 取绝对值是因为 ax+by 的符号可由 x,y 任意控制
        v2 = abs(int(next(it))) # -> 同上，确保 gcd 结果为正
        
        # 核心逻辑：根据裴蜀定理直接输出 gcd
        res = gcd(v1, v2)
        print(res)

# 主流程：处理 I/O 并调用
if it:
    solve()
```
