# 题目描述
- **核心目标**：在 $n \times m$ 矩阵中，根据特定移动规则（同行/同列内“低向高处”或“高向低处”跨越）求每个起点能达到的最大高度的平均值。
- **数据边界**：$n \times m \leq 10^6$，要求算法复杂度必须为 $O(NM)$。
# 分析
- **算法/数据结构**：**并查集 (DSU) + 连通性分析**。
- **为何适用**：题目移动规则具有**对称性**：若 $A$ 可一步到 $B$，则 $B$ 必可一步回 $A$。因此，所有能互相到达的格子构成一个**连通分量**，每个格子的“最优策略”即到达该分量内的最大高度。

- **定级**：中等题（省一冲刺级别）。
- **核心难点**：发现移动规则的对称性，并将“全连接图”的联通问题转化为**高效的“相邻位置”判定**。

- **[块 1：对称性与连通分量]**  
    分析规则：向下/向右走需高度减小，向上/向左走需高度增加。这意味着在同一行或列中，若格子 $u$ 和 $v$ 满足 $(index\_v - index\_u) \times (a_v - a_u) < 0$，则它们连通。这证明了图是无向的。
- **[块 2：高效连通判定]**  
    对于一行，位置 $j$ 和 $j+1$ 是否属于同一个分量？若不存在“断裂点”（即左侧所有数都小于右侧所有数），则连通。断裂点判定条件：$\max(a_1 \dots a_j) \le \min(a_{j+1} \dots a_m)$。反之，若 $\max_{left} > \min_{right}$，则 $j$ 与 $j+1$ 必连通。
- **[块 3：维护分量最大值]**  
    使用并查集维护每个连通块。合并时，动态更新根节点的 `max_h`。
