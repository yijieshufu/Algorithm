# 题目描述
**核心动作**：在 $n$ 个长度为 $m$ 的**环形**字符串中，选取 $n-1$ 条边构建**最大生成树 (MST)**，边权为两环形串旋转后能达到的**最长公共子串长度**。  
**数据边界**：$n \le 200, m \le 50$。关键点在于环形处理与 MST 的高效结合。
# 分析
**算法/数据结构**：**Kruskal 算法 (贪心 + 并查集)**。  
**为何适用**：求 $n-1$ 条边使总权值最大是典型的 MST 问题。由于边权（公共子串长度）范围极小 ($[1, m]$)，可采用**从大到小枚举边权 $L$** 的策略，通过并查集维护连通性。

**定级**：中等题。 **核心难点**：直接计算所有点对的“**环形 LCS**”复杂度较高（$O(n^2 m^2)$），需转化思维，利用 $L$ 的范围优势进行反向枚举。
- **[块 1：环形变线性]**  
    通过 `s + s` 的操作，将环形旋转问题转化为在线性串中滑动窗口抓取长度为 $L$ 的子串。
- **[块 2：反向索引（字典分组）]**  
    不暴力比对两两字符串，而是通过字典 `d[substring] = [index1, index2, ...]`，快速找出哪些字符串拥有相同的“基因”（公共子串）。
- **[块 3：并查集贪心合并]**  
    从 $L = m$ 降序循环到 $1$。只要发现两个串有共同子串且尚未连通，就执行 `union` 并累加 $L$。因为是从大到小枚举，第一次让两个集合合并的 $L$ 一定是它们之间的最大边权。
