# 题目描述
- **核心目标**：求最小电池容量 $C$，使得图中任意两点连通，且路径上每条边的耗电量 $w \le C$。
- **数据边界**：$N, M \le 2 \times 10^5$，$w_i \le 10^9$。需 $O(M \log M)$ 或 $O(M \alpha(N))$ 算法。
# 分析
- **算法/数据结构**：**Kruskal 算法（最小生成树 MST）**。
- **为何适用**：题目要求最小化路径中的最大边权（瓶颈问题），这等价于求最小生成树中的最大边权。
- **通用骨架**：
```python
    edges.sort(key=lambda x: x[2]) # 按权值升序
    for u, v, w in edges:
        if find(u) != find(v):
            union(u, v)
            ans = w # 最后一次合并时的 w 即为答案
```

- **[块 1：并查集初始化与路径压缩]**  
    使用数组 `p` 维护连通性，`cnt` 记录当前连通块个数。初始时 `cnt = n`。
- **[块 2：贪心加边]**  
    将所有公路按电量 $w$ 从小到大排序。依次遍历，若边连接的两个城市不在同一连通块，则合并。
- **[块 3：动态更新答案与连通判定]**  
    记录每次合并时的边权 $ans$。当连通块个数 `cnt` 降为 1 时，说明全图已连通，当前的 $ans$ 即为最小所需容量。若遍历完所有边后 `cnt > 1`，输出 -1。

## 代码

```python
import sys
# 数据读入
it = iter(sys.stdin.read().split())
n = int(next(it))
m = int(next(it))
# 并查集
p = list(range(n + 1))
cnt = n # 连通块计数
ans = 0
# 路径压缩
def find(x):
    if x!= p[x]:
        p[x]= find(p[x])
    return p[x]
# 录入边
edges = []
for _ in range(m):
    u,v,w = int(next(it)),int(next(it)),int(next(it))
    edges.append((u,v,w))
edges.sort(key= lambda x:x[2])
for u,v,w in edges:
    root_u = find(u);root_v = find(v)
    if root_u!= root_v:
        p[root_u] = root_v
        ans = w 
        cnt -=1
        if cnt ==1 :break
print(ans if cnt ==1 else -1)
```
