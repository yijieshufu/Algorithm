# 题目描述
- **核心动作**：在 $N$ 个学生的动态关系网中，执行“建立好友关系”与“查询是否为好友”两类操作。
- **数据边界**：$N, M \le 2 \times 10^5$。必须使用 $O(M \log N)$ 或 $O(M \alpha(N))$ 复杂度的算法，Python 需注意递归深度与 I/O 效率。
# 分析
- **算法/数据结构**：**并查集 (Disjoint Set Union, DSU)**。
- **为何适用**：题目定义的“朋友的朋友也是朋友”**具备传递性**，本质是维护连通分量，并查集是处理动态连通性问题的标准模型。
- **通用骨架**：
```python
    p = list(range(n + 1))
    def find(x):
        if p[x] != x: p[x] = find(p[x])
        return p[x]
    def union(x, y):
        rootX, rootY = find(x), find(y)
        if rootX != rootY: p[rootX] = rootY
```
- **定级**：基础题。
- **核心难点**：$2 \times 10^5$ 的数据量下，Python 默认递归深度（1000）会触发 `RecursionError`，且大量 `input()` 会导致 TLE。

- **[块 1：高效 I/O 与环境预设]** 使用 `sys.stdin.read().split()` 一次性读入，并手动提升递归深度（或改用迭代版 `find`）。
- **[块 2：并查集核心逻辑]** 初始化 `p[i] = i`。实现带路径压缩的 `find` 函数，确保后续查询近乎 $O(1)$。
- **[块 3：操作分流处理]** 遍历 $M$ 次操作：若 $op=1$，执行 `union` 合并；若 $op=2$，判断 `find(x) == find(y)`。

## 代码

```python
import sys
sys.setrecursionlimit(300000)
it = iter(sys.stdin.read().split())
n,m= int(next(it)),int(next(it))
# 并查集
p =list(range(n+1))
def find(x):
    if x!=p[x]:
        p[x] = find(p[x])
    return p[x]
for _ in range(m):
    op = next(it)
    x = int(next(it))
    y = int(next(it))
    root_x,root_y=find(x),find(y)
    if op =='1':
        if root_x!=root_y:p[root_x]=root_y
    else:
      if root_x==root_y:
        sys.stdout.write("YES\n")
      else:
        sys.stdout.write("NO\n")
```
