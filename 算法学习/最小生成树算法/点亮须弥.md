# 题目描述
- **核心动作**：从 1 号点出发，通过边权为 $w$ 的路径不断扩展“已点亮”集合，求点亮所有 $n$ 个点的最小总耗时。
- **数据边界**：$n \le 10^5$，$m \le 10^6$，$w \le 10^3$。边数较多，属于中等规模图论题。
# 分析
- **算法/数据结构**：**Prim 算法（堆优化版）** 或 **Kruskal 算法**。
- **为何适用**：题目要求连接所有点且总权值最小，且具备“从一个点开始向外扩张”的特性，非常契合 Prim 算法。考虑到 Python 的运行效率，堆优化的 Prim 或 Kruskal 均可，但 Prim 在处理这种“起始点固定”的逻辑时更直观。
- **通用骨架**：  
    ```python
    # Prim 堆优化骨架
    pq = [(0, start_node)]
    while pq:
        w, u = heapq.heappop(pq)
        if vis[u]: continue
        vis[u] = True
        res += w
        for v, weight in g[u]:
            if not vis[v]: heapq.heappush(pq, (weight, v))
    ```
- **定级**：基础题。
- **核心难点**：在于将题目中“传送点点亮后可瞬移”这一背景抽象为 **MST 模型**。

- **[块 1：图的存储]**  
    由于 $m$ 高达 $10^6$，使用邻接表（`list` 嵌套 `tuple`）存储无向图。
- **[块 2：贪心扩展 (Prim)]**  
    维护一个优先队列 `pq`，存储当前“已点亮集合”到“未点亮集合”的所有可选路径。每次取出权值最小的边，将其连接的点并入集合。
- **[块 3：连通性判定]**  
    记录已点亮的节点数量 `cnt`。如果最终 `cnt < n`，说明存在无法到达的点，输出 -1。
