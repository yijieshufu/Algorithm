# 题目描述
学校有三种宿舍：2人寝（$a_2$）、3人寝（$a_3$）、4人寝（$a_4$）。食堂有两种桌子：4人桌（$b_4$）、6人桌（$b_6$）。 **规则**：同一个宿舍的人必须坐同一张桌子，不能拆分。  
**目标**：求食堂最多能同时容纳多少学生用餐。
# 分析

- **题目类型**：多维资源分配 / 变形多重背包问题
- **难度评级**：中等偏上（省赛拔高题，冲击省一的**必备题**）
- **数据规模**：$q \le 100$，$b_4 + b_6 \le a_2 + a_3 + a_4 \le 100$
- **核心痛点**：这道题极具迷惑性，很多人一上来就想用“纯贪心”去做（比如每次都把最大的寝室塞进最大的桌子）。但纯贪心会翻车！因为大桌子有多种组合（比如 6人桌可以坐 4+2，也可以坐 3+3），如果前面的桌子瞎贪心，可能会导致剩下的寝室和桌子没法完美匹配，白白浪费座位。这道题本质上是一个**变形的多维背包问题**，属于我们动态规划（DP）必杀技的范畴。

不要去背什么复杂的组合数学公式，我们把这道题想象成 **“高级版俄罗斯方块”** 。
1. **坑位与方块**：6人桌和4人桌是你要填满的“坑”，而 2人、3人、4人寝室是不同形状的“方块”。为了让最多的人坐下，我们要尝试所有最完美的“拼接方式”。
2. **定序策略（先大后小）**：6 人桌容量大、能接纳的组合多（比如 4+2, 3+3, 2+2+2 等），所以我们**必须优先把 6 人桌分配完**，然后再处理 4 人桌。这能极大减少决策的混乱度。
3. **不留余地的压榨（极限塞满）**：对于每一张桌子，我们穷举所有能把它**尽可能塞满**的策略。
    - **6 人桌黄金组合**：优先尝试 (4, 2)、(3, 3)、(2, 2, 2)，如果这些完美组合没有，再退而求其次尝试 (3, 2)、(4) 等。
    - **4 人桌黄金组合**：优先尝试 (4)、(2, 2)、(3)。
4. **记忆化搜索（小本本大法）**：由于有 $q = 100$ 组数据，如果每次都暴力傻算肯定超时。因为桌子和寝室的总数都不超过 100，状态空间非常小。我们搞一个全局的 `memo` 字典当“备忘录”，算过的状态直接抄答案，速度快到飞起！

# 代码
```python
import sys

# 增加递归深度，防止爆栈 (蓝桥杯基石操作)
sys.setrecursionlimit(5000)
# 全局备忘录：跨越 100 组查询共享状态，究极加速！
memo = {}
def solve(a2, a3, a4, b4, b6):
    # 终点站：如果所有学生都安排完了，或者桌子用光了，能增加的人数就是 0
    if (a2 == 0 and a3 == 0 and a4 == 0) or (b4 == 0 and b6 == 0):
        return 0
    state = (a2, a3, a4, b4, b6)
    if state in memo:
        return memo[state]
    ans = 0
    # 第一阶段：优先疯狂压榨 6 人桌
    if b6 > 0:
        if a4 >= 1:
            if a2 >= 1: ans = max(ans, 6 + solve(a2 - 1, a3, a4 - 1, b4, b6 - 1)) # 完美 4+2
            else: ans = max(ans, 4 + solve(a2, a3, a4 - 1, b4, b6 - 1))           # 只能坐 4
        if a3 >= 2:
            ans = max(ans, 6 + solve(a2, a3 - 2, a4, b4, b6 - 1))                 # 完美 3+3
        if a3 >= 1:
            if a2 >= 1: ans = max(ans, 5 + solve(a2 - 1, a3 - 1, a4, b4, b6 - 1)) # 凑合 3+2
            else: ans = max(ans, 3 + solve(a2, a3 - 1, a4, b4, b6 - 1))           # 只能坐 3
        if a2 >= 3:
            ans = max(ans, 6 + solve(a2 - 3, a3, a4, b4, b6 - 1))                 # 完美 2+2+2
        elif a2 == 2:
            ans = max(ans, 4 + solve(0, a3, a4, b4, b6 - 1))                      # 凑合 2+2
        elif a2 == 1:
            ans = max(ans, 2 + solve(0, a3, a4, b4, b6 - 1))                      # 只能坐 2
            
    # 第二阶段：6 人桌用完后，榨干 4 人桌
    else:
        if a4 >= 1:
            ans = max(ans, 4 + solve(a2, a3, a4 - 1, b4 - 1, 0))                  # 完美 4
        if a3 >= 1:
            ans = max(ans, 3 + solve(a2, a3 - 1, a4, b4 - 1, 0))                  # 凑合 3
        if a2 >= 2:
            ans = max(ans, 4 + solve(a2 - 2, a3, a4, b4 - 1, 0))                  # 完美 2+2
        elif a2 == 1:
            ans = max(ans, 2 + solve(a2 - 1, a3, a4, b4 - 1, 0))                  # 只能坐 2
            
    memo[state] = ans
    return ans

# --- 极简 I/O 起手式 ---

it = iter(sys.stdin.read().split())
q = int(next(it))
for _ in range(q):
    a2 = int(next(it))
    a3 = int(next(it))
    a4 = int(next(it))
    b4 = int(next(it))
    b6 = int(next(it))
    print(solve(a2, a3, a4, b4, b6))
```
