# 题目描述
我们将含有不少于 **6个6** 的数视为一个“好数”。
- **例子**：`666666`、`162636465666` 是好数；`12366666`（只有5个6）不是好数。
- **任务**：给定 $n$ 个正整数 $a_i$，你可以将这些数分成若干组拼接起来。
- **规则**：
    1. 每组内的数可以按任意顺序拼接。
    2. **重点**：每一组最多只能有 **3 个数**。
- **求解**：最多可以得到多少个好数？
# 代码
```python
import sys
it = iter(sys.stdin.read().split())
n = int(next(it))
f = [0]*7 # 每个数据的中6的个数
ans = 0 

# 记录6的个数
for _ in range(n):
	g = next(it).count("6")
	if g >=6:ans+=1
	else:f[g]+=1
	
# 5 1配对 # 5 只会在这里用到
m = min(f[5],f[1])
ans+=m
f[5]-=m 
f[1]-=m
f[4]+=f[5]

#  4 1 1 
m = min(f[4],f[1]//2) 
ans+=m
f[4]-=m
f[1]-=m*2
# 4 2  # 后面不用4了
m = min(f[4],f[2])
ans+=m
f[4]-=m
f[2]-=m
f[3]+=f[4]
# 3+2+1
m = min(f[3], f[2], f[1]) 
ans += m 
f[3] -= m 
f[2] -= m 
f[1] -= m
# 3+3
m =f[3]//2
ans+=m
f[3]-=2*m  
f[2]+=f[3] # 3还可以当作2用
# 2 2 2 
ans+=f[2]//3
print(ans)

```
# 解题 过程

**【核心思想】优先消灭废料 1 ＋ 资源逐层降级**  
**【前提规则】**最多 3 数组合，带 $\ge 6$ 个的直接 `ans+1`。  
**第一层：5 的专属任务**
- **动作**：`5 + 1` 配对。（用最强资源救最废的 1）
- **降级**：用不完的 5，直接当 4 用（加到 4 的个数里）。

**第二层：4 的清障任务**
- **动作**：优先 `4 + 1 + 1`（消耗废料的最后大好时机），次选 `4 + 2`。
- **降级**：用不完的 4，直接当 3 用。

**第三层：3 的大杂烩**
- **动作**：优先 `3 + 2 + 1`（榨干最后的 1），次选 `3 + 3`（自给自足）。
- **降级**：用不完的 3，直接当 2 用。

**第四层：2 的扫尾任务**
- **动作**：全体 `2 + 2 + 2` 抱团取暖，榨干最后价值。
