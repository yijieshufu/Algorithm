# 题目描述

想象你正在经营一家**士兵健身房**，你的目标是让 $n$ 名士兵全部“出师”。
### 1. 任务目标
每位士兵 $i$ 都有一个“出师标准”，即必须完成至少 $c_i$ 次训练。
### 2. 两种付费方式
你可以随时在这两种方式间切换，且次数不限：
- **零售模式（单人训）**：给第 $i$ 名士兵单独买 1 次训练，花费 $p_i$ 枚金币。
- **团购模式（全体训）**：一口气付 $S$ 枚金币，**所有**士兵（不管他需不需要，只要他还没出师）全部自动训练 1 次。
# 分析

本题是典型的**贪心决策与代价优化**问题。
- **难度等级**：中等。
- **高频程度**：省赛 **必备题**。这类题考察你是否能动态地在“集体方案”和“个人方案”之间切换。
## 逻辑转化（为什么这题不简单？）

这道题的难点在于**动态变化**：
1. **团购的性价比是递减的**：一开始大家都没出师，买一次团购能省很多钱；但随着团购次数增加，一部分士兵达到了 $c_i$ 已经“出师”了，剩下的士兵越来越少。
2. **决策点**：你需要判断，在什么时候剩下的士兵单买“零售票”的总价已经低于了团购价 $S$。一旦到了这个点，你就应该停止买团购，改买零售

# 代码

```python
import sys
it = iter(sys.stdin.read().split())
n = int(next(it))
s = int(next(it))
a = []
for _ in range(n):
    p = int(next(it)) # 这里的 p 是单次成本
    c = int(next(it)) # 这里的 c 是总需求次数
    a.append((c, p))
a.sort()

ans = 0
cur_k = 0
total_p = sum(x[1] for x in a)
idx = 0
while idx < n:
    c, p = a[idx]
    if total_p > s:
        # 团购逻辑：团购到让当前这个士兵满级
        if c > cur_k:
            ans += (c - cur_k) * s
            cur_k = c
        while idx < n and a[idx][0] <= cur_k:
            total_p -= a[idx][1]
            idx += 1
    else:
        if c > cur_k:
            ans += (c - cur_k) * p
        idx += 1 
print(ans)
```
