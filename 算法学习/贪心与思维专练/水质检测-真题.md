# 题目描述
在一个 $2 \times n$ 的格子里，已经有一些点（检测器 `#`）了。两个点只要上下或左右相邻就是连通的。你的任务是：**最少**往空格子 `.` 里填多少个新的 `#`，能让现有的所有 `#` 都连在一起。

- **关键约束**：$n$ 高达 $1,000,000$，意味着我们只能用 $O(n)$ 的时间复杂度，也就是只能扫一遍。
- **题型**：线性动态规划 / 贪心
- **难度**：⭐⭐⭐（中等，省赛高频题）
- **是否必会**：**是**。这种 $2 \times N$ 的窄条状网格问题，是**动态规划**的绝佳切入点，必须掌握。
# 代码
```python
import sys
it = iter(sys.stdin.read().split())
s1 = next(it)
s2 = next(it)
n = len(s1)
L,R=-1,-1 # 定位L，R L第一个，R最后一个
for i in range(n):
  if s1[i]=='#' or s2[i]=='#':
    if L ==-1:L=i
    R = i 

if L == -1:print(0);sys.exit()# 特殊 情况 无
f0 = 0 if s1[L] == '#' else 1 # 初始 新增数 f0，f1  如果f1的L列没有 要花费一个补充
f1 = 0 if s2[L] == '#' else 1
for i in range(L+1,R+1): # 开始走L+1 到R列：
  c0 = 1 if s1[i]=='.' else 0 # 两个 都是遇到 . 直接 安装
  c1 = 1 if s2[i]=='.' else 0
  p0,p1=f0,f1 # 预存上个状态
  f0 = min(p0+c0,p1+c1+c0) # 计算 f0，f1 到底当前的最小花费
  f1 = min(p1+c1,p0+c0+c1)
  if s1[i] == '#' and s2[i]=='#': # 如果 当前有 则取f0，f1的最小
    f0=f1=min(f0,f1)
print(min(f0,f1)) # 输出两个的最小
```

# 分析
这道题是典型的**线性动态规划（DP）** 在网格问题中的应用。由于网格只有 2 行，状态非常有限，这使得我们可以用 $O(n)$ 的复杂度通过一次扫描解决问题。
## 一、 核心逻辑推导：为什么能这么做？

### 1. 范围界定 及 L 到 R 的“剪枝”

题目要求让“所有检测器”连通。如果 **第 1 列 到 最后一列** 都没有检测器 直接 `sys.exit()`

- **做法：** 找到第一个出现 `#` 的列 `L` 和 最后一个出现 `#` 的列 `R`。
    
- **目的：** 我们的目标变成了：**从 `L` 列开始，建立一条连续的“路”，直到覆盖 `R` 列的所有已知检测器。**

### 2. 状态定义：2 行网格的特权

在 $2 \times n$ 的窄条中，如果我们从左向右推进，要保证连通性，到达第 $i$ 列时，我们关心的只有：  
	**是走 第一行 还是 第二行**

- `f0`: 到达第 $i$ 列且**保证第 $i$ 列第一行 $(0, i)$ 有检测器**时，最少需要增加的数量。
    
- `f1`: 到达第 $i$ 列且**保证第 $i$ 列第二行 $(1, i)$ 有检测器**时，最少需要增加的数量。

### 3. 状态转移：怎么跨到下一列？

假设我们已经知道了第 $i-1$ 列的最优解 `p0` 和 `p1`，现在要走到第 $i$ 列：

- **目标：站稳第一行 $(0, i)$**
    
    - 方案 A：从上一列的第一行 $(0, i-1)$ 横着走过来。花费：`p0 + (如果 (0, i) 是空的就加 1)`。
        
    - 方案 B：从上一列的第二行 $(1, i-1)$ 走过来。因为不能斜着走，必须先走到 $(1, i)$，再向上走到 $(0, i)$。花费：`p1 + (如果 (1, i) 空就加 1) + (如果 (0, i) 空就加 1)`。
        
    - **取两者最小值。**
        
- **目标：站稳第二行 $(1, i)$**
    
    - 逻辑同上，要么从 `p1` 横跳，要么从 `p0` 拐弯跳。

## 二、 代码逐行深度解析

```python
# 初始化：L列的处理
f0 = 0 if s1[L] == '#' else 1 
f1 = 0 if s2[L] == '#' else 1
```

- **思考：** 为什么 `L` 列要初始化？  
	因为 `L` 列是起点。如果 `L` 列的第一行本来是空的，但我们要从这里开始推 `f0`，就必须强制补一个检测器。

```python
for i in range(L+1, R+1):
  c0 = 1 if s1[i]=='.' else 0 # 如果是点，说明需要花费 1 个检测器来填补
  c1 = 1 if s2[i]=='.' else 0
  
  p0, p1 = f0, f1
  # 转移方程：横着走 vs 拐弯走
  f0 = min(p0 + c0,      # 路径：(0, i-1) -> (0, i)
           p1 + c1 + c0) # 路径：(1, i-1) -> (1, i) -> (0, i)
           
  f1 = min(p1 + c1,      # 路径：(1, i-1) -> (1, i)
           p0 + c0 + c1) # 路径：(0, i-1) -> (0, i) -> (1, i)
```

### 关键点：双 `#` 碰撞（同步机制）

```python
if s1[i] == '#' and s2[i] == '#':
    f0 = f1 = min(f0, f1)
```

- **为什么要这一步？** 如果当前列的两行**原本就有检测器**，那么这两行在这一列已经自动连通了！
    
    这代表“从第一行推过来的路径”和“从第二行推过来的路径”在此处**汇合**了。为了后面列的最优性，我们直接取两者的最小值，更新 `f0` 和 `f1`。这相当于一次“**状态压缩**”。
