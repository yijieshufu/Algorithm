# 题目描述
给定一个数组，通过“修改相邻两个数”或“修改单个数”的操作，以最少步数使其变成回文数组。
# 分析
- **难度等级**：中等
- **高频程度**：极高（回文构造 + 差分/贪心是省赛常客）

### 一、 核心定位：将“回文”抽象化

要让一个数组变成回文，本质是让对称位置的两个数相等，即 $a_i = a_{n-i+1}$。
- **第一步：锁定矛盾**。我们只关心对称位置的**差值**。定义 $D_i = a_i - a_{n-1-i}$（仅需计算前一半，$i < n//2$）。
- **第二步：明确目标**。题目要求的“回文”，等同于将所有 $D_i$ 变成 $0$。
### 二、 操作拆解：寻找“顺风车”机会
题目给了两种操作，转化到 $D$ 数组上就是：
1. **单点打击**：修改一个 $D_i$，代价为 $1$。
2. **双重打击**：同时修改相邻的 $D_i$ 和 $D_{i+1}$，代价也为 $1$。

**贪心逻辑**：既然“双重打击”和“单点打击”花销一样，我们当然优先用“双重打击”。
- **触发条件**：只有当 $D_i$ 和 $D_{i+1}$ **方向相同**（同为正或同为负）时，“双重打击”才能同时把两个数都向 $0$ 靠近。

### 三、 贪心决策：只为“差额”买单
遍历差值数组时，我们想象在填一排深浅不一的坑：
1. **另起炉灶**：如果当前这个坑 $D_i$ 和前一个坑 $D_{i-1}$ **方向相反**，或者前一个没坑，你必须支付全部代价：$ans += |D_i|$。
2. **搭顺风车**：如果方向相同，你在填前一个坑 $D_{i-1}$ 时，可以顺便把 $D_i$ 也填了：
    - 如果 $D_i$ 比 $D_{i-1}$ **浅**：填完前一个，这个已经顺便填平了，**代价为 0**。
    - 如果 $D_i$ 比 $D_{i-1}$ **深**：你只需要补齐多出来的那部分深度，**代价为 $|D_i| - |D_{i-1}|$**。
# 代码
```python
import os
import sys
it = iter(sys.stdin.read().split())
n = int(next(it))
a = [int(x) for x in it]
diffs = [a[i] - a[n-1-i] for i in range(n // 2)]
ans = 0
prev = 0
for x in diffs:
  if x * prev > 0:  # 方向相同，尝试搭顺风车
    # 只有当前坑更深，才需要额外付钱
    ans += max(0, abs(x) - abs(prev))
  else:             # 方向不同，之前的操作帮不上忙
    ans += abs(x)
  prev = x
print(ans)
```
