# 题目描述
- **核心动作**：在允许“单点修改”或“相邻修改”的前提下，以最少步数消除原数组与目标状态（回文）的对称差值。
- **数据边界** $N \le 10^5$：意味着必须在线性 $O(N)$ 时间复杂度内完成，不允许高维 DP。
# 代码

```python
import sys

# 认知块 1 & 2: 输入与对称差值预处理
it = iter(sys.stdin.read().split())
n = int(next(it))
a = [int(next(it)) for _ in range(n)]
# 仅计算前一半的差值即可
d = [a[i] - a[n-1-i] for i in range(n // 2)]

ans = 0
prev = 0

# 认知块 3: 贪心计算增量贡献
for x in d:
    if x * prev > 0: # 块 3: 方向相同，计算绝对值的正增量
        if abs(x) > abs(prev):
            ans += abs(x) - abs(prev)
    else:            # 块 3: 方向不同，必须重新起步
        ans += abs(x)
    prev = x

print(ans)
```

# 分析
- **算法/数据结构**：**差分贪心 / 增量贡献法**。
- **为何适用**：题目出现了“修改相邻两个数”，这是典型的“区间操作”简化版。凡是涉及“区间/相邻同时改变”且求“最少步数”的问题，优先考虑**差分或相邻状态的增量**。
- **通用骨架**：
```python
# 计算目标差值序列 d
# 遍历 d，比较当前项与前一项的关系
if 方向相同: 
    ans += max(0, 当前绝对值 - 前项绝对值)
else:
    ans += 当前绝对值
```

要让一个数组变成回文，本质是让对称位置的两个数相等，即 $a_i = a_{n-i+1}$。
- **第一步：锁定矛盾**。我们只关心对称位置的**差值**。定义 $D_i = a_i - a_{n-1-i}$（仅需计算前一半，$i < n//2$）。
- **第二步：明确目标**。题目要求的“回文”，等同于将所有 $D_i$ 变成 $0$。

题目给了两种操作，转化到 $D$ 数组上就是：
1. **单点打击**：修改一个 $D_i$，代价为 $1$。
2. **双重打击**：同时修改相邻的 $D_i$ 和 $D_{i+1}$，代价也为 $1$。

**贪心逻辑**：既然“双重打击”和“单点打击”花销一样，我们当然优先用“双重打击”。
- **触发条件**：只有当 $D_i$ 和 $D_{i+1}$ **方向相同**（同为正或同为负）时，“双重打击”才能同时把两个数都向 $0$ 靠近。

遍历差值数组时，我们想象在填一排深浅不一的坑：
1. **另起炉灶**：如果当前这个坑 $D_i$ 和前一个坑 $D_{i-1}$ **方向相反**，或者前一个没坑，你必须支付全部代价：$ans += |D_i|$。
2. **搭顺风车**：如果方向相同，你在填前一个坑 $D_{i-1}$ 时，可以顺便把 $D_i$ 也填了：
    - 如果 $D_i$ 比 $D_{i-1}$ **浅**：填完前一个，这个已经顺便填平了，**代价为 0**。
    - 如果 $D_i$ 比 $D_{i-1}$ **深**：你只需要补齐多出来的那部分深度，**代价为 $|D_i| - |D_{i-1}|$**。

以后只要看到题目提到 **“区间修改”**、**“相邻修改”**、**“最少操作次数”** 且目标是 **“全部相等/变为0”**，直接提取相邻元素的 **“差额”**。这种“顺风车”逻辑（即处理大需求时顺便覆盖小需求）是解决此类问题的唯一正解。
