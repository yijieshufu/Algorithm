
## 1. 题目描述与痛点分析

### 题目在说什么？

你有 $n$ 个水晶，每个都有个颜色（属性值）。给你一个区间 $[l, r]$，让你在这个区间里玩游戏：

- 找两个颜色一样的水晶，位置分别是 $i$ 和 $j$，得分就是它们之间的距离 $j - i$。
    
- 拿完分后，**这种颜色**的所有水晶就都消失了。
    
- 问：怎么拿分，最后总分最高？

### 它的痛点在哪里？

如果你在区间里有很多个红水晶，比如位置在 $2, 5, 8, 15$。

- 如果你选 $2$ 和 $5$，得分 $3$，然后红水晶全没了。
    
- 如果你选 $2$ 和 $15$，得分 $13$，红水晶也全没了。
    
- **显而易见**：为了得分最高，每种颜色我们都要选这个区间内**最左边**和**最右边**的那两个。

**痛点：** 不同的查询区间 $[l, r]$，每种颜色的“最左”和“最右”都在变。如果暴力去找，时间复杂度是 $O(Q \times N)$，在 $5 \times 10^4$ 的数据量下肯定会超时。

---

## 2. 题型判断

这道题属于 **“离线区间贡献查询”**。

- **离线（Offline）**：指我们不需要立刻回答每一个询问，可以把所有询问存起来，排个序，找个最舒服的顺序统一处理。
    
- **区间贡献**：我们需要统计区间内满足某种条件的数值总和。

---

## 3. 核心分析：木板拼凑法（化繁为简）

我们要算的得分是 $Position_{last} - Position_{first}$。

### 绝妙的转化

假设红色水晶在区间 $[l, r]$ 内的位置是 $p_1, p_2, p_3, p_4$。

我们要的是 $p_4 - p_1$。

数学上，我们可以把它拆开：

$$
p_4 - p_1 = (p_2 - p_1) + (p_3 - p_2) + (p_4 - p_3)
$$

你可以想象成：在相邻的同色水晶之间，钉上一块**木板**。

- **木板 1**：从 $p_1$ 到 $p_2$，长度 $L_1 = p_2 - p_1$
    
- **木板 2**：从 $p_2$ 到 $p_3$，长度 $L_2 = p_3 - p_2$
    
- **木板 3**：从 $p_3$ 到 $p_4$，长度 $L_3 = p_4 - p_3$

**关键逻辑（重点！）：**

只要一根木板的**起点**和**终点**都在查询区间 $[l, r]$ 内，这根木板就是“安全的”，它的长度就可以计入总分。

- 如果一根木板的起点小于 $l$，说明这根木板跨出了左边界，它的左端点那个水晶不在 $[l, r]$ 里，这根木板就不能要。

**结论：**

本题转化为：统计所有 **“起点 $\ge l$ 且 终点 $\le r$”** 的木板长度之和。

---

## 4. 算法套路：扫描线 + 树状数组

现在我们有两个限制条件：**起点 $\ge l$** 和 **终点 $\le r$**。处理两个条件很麻烦，我们“杀掉”一个：

1. **干掉终点限制**：把所有查询按 $r$（右边界）从小到大排序。
    
2. **扫描线**：我们的“扫描枪”从 $1$ 扫到 $n$。
    
3. **动态入库**：当扫描枪扫到位置 $i$ 时，如果这个颜色的水晶之前在 $pre$ 出现过，就产生了一根木板：**起点是 $pre$，终点是 $i$，长度是 $i - pre$**。
    
4. **树状数组记账**：我们在树状数组的 **$pre$ 位置**（木板的起点）加上这个长度。
    
5. **回答查询**：当扫描枪扫到 $r$ 时，对于所有右边界是 $r$ 的查询，因为我们只处理到了 $r$，所以柜子里所有木板的终点都肯定 $\le r$。此时，我们只需要用树状数组查一下 **$[l, r]$ 范围内的总和**（也就是起点 $\ge l$ 的木板）。

---

# 代码解析

```Python
ans = [0] * (q + 1)
last = {}   # 每种颜色上次出现位置
p = 1
for r, l, idx in queries:
    # 扫描到 r：同色成对 (last[c], p) 为一块木板，在起点 last[c] 处贡献长度
    while p <= r:
        c = a[p] 
        if c in last:
            update(last[c], p - last[c])
        last[c] = p
        p += 1
    # 起点在 [l, r] 内的木板长度和 = query(r) - query(l-1)
    ans[idx] = query(r) - query(l - 1)

```

这段代码解决的核心问题是：**如何快速判断哪些“木板”完整地落在了 $[L, R]$ 区间内？**

---

## 1. 扫描指针 `p`：不断右移的“施工线”

```Python
for r, l, idx in queries:
    while p <= r:
        ...
        p += 1
```

- **逻辑：** 因为查询是按 `r` 从小到大排好队的，我们的指针 `p` 就像一个修路工，从左往右走。
    
- **目的：** 当我们要处理右端点是 `r` 的查询时，施工员必须先跑到 `r` 这个位置，把一路上能看到的“木板”全部铺好。
    
- **结果：** 这一步保证了：**目前所有记在账本里的木板，它们的右端点（终点）都一定 $\le r$。**（已经满足了右边界限制）

---

## 2. `update`：把长度记在“起点”上

这是最神的一笔：
```Python
if c in last:
    update(last[c], p - last[c])
```

- **场景：** 施工员走到位置 `p`，发现这块水晶是红色的。他查了一下记事本（`last`），发现上一个红色水晶在位置 `last[c]`。
    
- **动作：** 他在 `last[c]` 和 `p` 之间钉了一块木板。
    
- **记账：** 他在地面上的 **`last[c]`（起点）** 这个位置，写下了这块木板的长度。

> **为什么要记在起点？**
> 
> 因为我们已经保证了木板的终点 $\le r$。现在唯一要担心的就是：这块木板的**起点**有没有掉出左边界 $l$？
> 
> 如果我们将长度记在起点，后续只需要统计 `起点 >= l` 的木板即可。

---

## 3. `query`：算账拿分

Python

```python
ans[idx] = query(r) - query(l - 1)
```

- **逻辑：** 树状数组的 `query(x)` 能快速算出 $1$ 到 $x$ 位置所有记下的数字之和。
    
- **算式：** `query(r) - query(l - 1)` 算出的其实是 **$[l, r]$ 范围内所有数字的和**。
    
- **含义：**
    
    - 我们现在手里所有的木板，终点都 $\le r$（前面 `while` 循环保证的）。
        
    - 我们只加起那些 **起点 $\ge l$** 的数字。
        
    - **终点 $\le r$** 且 **起点 $\ge l$** $\implies$ **整块木板都在 $[l, r]$ 里面！**

---

## 4. 模拟一遍（大白话版）

假设水晶颜色是：`红(1), 绿(2), 红(3), 红(5)`，查询区间 $[2, 5]$。

1. **施工到位置 1**：看到红，`last[红] = 1`。没木板。
    
2. **施工到位置 2**：看到绿，`last[绿] = 2`。没木板。
    
3. **施工到位置 3**：看到红。发现上个红在 1。
    
    - 钉木板：$[1, 3]$，长度 2。
        
    - **记账：在位置 1 写下数字 2。**
        
    - 更新 `last[红] = 3`。
        
4. **施工到位置 5**：看到红。发现上个红在 3。
    
    - 钉木板：$[3, 5]$，长度 2。
        
    - **记账：在位置 3 写下数字 2。**
        
    - 更新 `last[红] = 5`。
        
5. **开始算账 $[2, 5]$**：
    
    - 看地面位置 2 到 5 之间有哪些数字？
        
    - 位置 1 的数字（2）在圈外，不计入（因为它起点是 1，小于左边界 2，说明木板跨出去了）。
        
    - 位置 3 的数字（2）在圈内，计入！
        
    - **总分 = 2**。

**验算：** 在区间 $[2, 5]$ 内，红水晶在位置 3 和 5。得分应该是 $5 - 3 = 2$。完全正确！

---

### 总结这个逻辑的“狠招”：

- **排序**：杀掉了右边界的限制。
    
- **拆分**：把“长跨度”拆成“相邻短木板”，让它们可以像堆砖块一样求和。
    
- **记在起点**：利用树状数组的区间求和，顺手杀掉了左边界的限制。
