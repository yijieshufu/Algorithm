## 题目描述
### 1. 详细题目描述：就像“给一排存钱罐加硬币”

想象你有 **$N$ 个存钱罐**排成一排，每个罐子里最初都有一些硬币。

你会收到 **$Q$ 个指令**，指令只有两种：

1. **“区间修改”：** 老师说：“把第 $l$ 个到第 $r$ 个罐子，每个都额外放入 $k$ 枚硬币。”（$k$ 也可以是负数，代表取走硬币）。
    
2. **“区间查询”：** 老师问：“请算一算，从第 $l$ 个到第 $r$ 个罐子，里面一共有多少枚硬币？”

**关键难点：**

罐子和指令的数量都非常多（最高 $10^5$ 次操作）。如果你每次加硬币或算总数都一个一个去数（也就是用 `for` 循环），当罐子很多时，你的速度会慢得像蜗牛，程序会因为超时（TLE）而失败。

---

### 2. 判断题型：线段树 (Segment Tree) 的练兵场

这道题的官方名称叫：**“区间修改 + 区间求和”**。

在算法世界里，它属于 **高级数据结构** 题。最标准、最通用的解法是 **线段树（Segment Tree）**，并且需要配合一个叫 **“懒惰标记”（Lazy Tag）** 的核心技巧。

_注：虽然这题也可以用“树状数组”的进阶技巧来写，但线段树更直观，也更适合处理复杂的区间问题。_

---

### 3. 它解决了什么痛点？

如果不使用线段树，我们面临的痛点是 **“工作量巨大”**：

- **普通方法：** 每次修改区间 $[l, r]$，你都要跑去把这中间的每一个罐子都改一遍。如果区间很长，效率就是 $O(N)$。
    
- **线段树的优势：** 它把 $10^5$ 个罐子通过“分组管理”的方式（类似公司架构，一个组长管两个组员），把每次修改和查询的时间从 $O(N)$ 降到了 $O(\log N)$。
    
    - 简单说：普通方法数 10 万次，线段树只需要数 17 次左右。

---

### 4. 核心思路：懒政的智慧（Lazy Tag）

线段树的核心思路是 **“分而治之”** 和 **“不到万不得已不干活”**。

1. **分层管理：** 将整个区间不断二分。顶层管 $[1, N]$，下一层管 $[1, mid]$ 和 $[mid+1, N]$，直到最底层的单个罐子。这样形成了一棵树。
    
2. **懒惰标记（核心）：** 当你要给区间 $[l, r]$ 加硬币时，如果你发现某个“组长”管辖的范围 **正好落在** 你要修改的区间内，你就直接告诉这位组长：“你手下所有罐子都加 $k$ 个硬币”，然后在他背上贴个纸条（Lazy Tag），你就走人。
    
3. **标记下传：** 只有当你下次必须查询这个组长底下的某个具体罐子时，组长才会看一眼背上的纸条，把任务分配给底下的组员。

---

## 代码

```python
import sys
input = sys.stdin.read().split()
it = iter(input)

n, q = int(next(it)), int(next(it))
a = [0] * (n + 1)
for i in range(1, n + 1):
    a[i] = int(next(it))

tree = [0] * (4 * n + 1)
tag = [0] * (4 * n + 1)

def push_up(u):
    tree[u] = tree[u << 1] + tree[u << 1 | 1]

# 懒标记下传：把当前节点加的 k 分摊给左右子区间
def push_down(u, l, r):
    if tag[u]:
        mid = (l + r) // 2
        tag[u << 1] += tag[u]
        tree[u << 1] += tag[u] * (mid - l + 1)
        tag[u << 1 | 1] += tag[u]
        tree[u << 1 | 1] += tag[u] * (r - mid)
        tag[u] = 0

def build(u, l, r):
    if l == r:
        tree[u] = a[l]
        return
    mid = (l + r) // 2
    build(u << 1, l, mid)
    build(u << 1 | 1, mid + 1, r)
    push_up(u)

# 区间 [L,R] 加 k：被完全覆盖则打标记，否则下传后递归
def update(u, l, r, L, R, k):
    if L <= l and r <= R:
        tree[u] += k * (r - l + 1)
        tag[u] += k
        return
    push_down(u, l, r)
    mid = (l + r) // 2
    if L <= mid:
        update(u << 1, l, mid, L, R, k)
    if R > mid:
        update(u << 1 | 1, mid + 1, r, L, R, k)
    push_up(u)

# 区间 [L,R] 求和：被完全覆盖直接返回，否则下传后递归左右
def query(u, l, r, L, R):
    if L <= l and r <= R:
        return tree[u]
    push_down(u, l, r)
    mid = (l + r) // 2
    res = 0
    if L <= mid:
        res += query(u << 1, l, mid, L, R)
    if R > mid:
        res += query(u << 1 | 1, mid + 1, r, L, R)
    return res

build(1, 1, n)
for _ in range(q):
    op = next(it)
    if op == '1':
        l, r, k = int(next(it)), int(next(it)), int(next(it))
        update(1, 1, n, l, r, k)
    else:
        l, r = int(next(it)), int(next(it))
        print(query(1, 1, n, l, r))
```
