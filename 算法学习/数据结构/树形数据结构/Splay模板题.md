
## 1. 题目详细描述

**题目背景：**

给定一个长度为 $n$ 的正整数序列，初始状态下序列为自增序列：$1, 2, 3, \dots, n$。

**操作要求：**

你需要进行 $m$ 次操作。每次操作给定两个参数 $l$ 和 $r$（$1 \le l \le r \le n$），表示将当前序列中从第 $l$ 个位置到第 $r$ 个位置的这一段连续子序列进行**翻转**（例如：序列 `1 2 3 4 5`，翻转 `[2, 4]` 后变为 `1 4 3 2 5`）。

**输入格式：**

- 第一行：两个正整数 $n$ 和 $m$。
    
- 接下来 $m$ 行：每行两个正整数 $l, r$。
    

**输出格式：**

- 输出一行：包含 $n$ 个正整数，表示经过 $m$ 次翻转后的最终序列。

## 2. 题型判断

这道题属于 **“平衡树（Balanced BST）维护序列”** 题型，更具体地说是 **Splay Tree（伸展树）** 或 **Treap（树堆）** 的区间操作应用。

- **为什么不是普通的数组或线段树？**
    
    - **数组/向量：** 翻转操作需要 $O(r-l)$ 的时间，最坏情况下 $m$ 次操作会导致 $O(nm)$，即 $10^{10}$ 次运算，必然超时。
        
    - **普通线段树：** 线段树擅长维护区间和、区间最值，但很难直接处理物理结构上的“翻转”（即改变节点的先后顺序）。
        
- **核心特征：** 题目要求的不是修改数值，而是**修改位置**。这种动态维护序列物理顺序的操作，是平衡树的拿手好戏。


## 解题思路

## 1. 问题分析：为什么要用 Splay？

- **痛点：** 传统的数组如果要翻转区间 $[L, R]$，需要物理移动元素，最坏情况要处理 $O(n)$ 个数。如果操作 $m$ 次，总时间 $O(n \times m)$，在 $10^5$ 的数据量下直接超时。
    
- **算法选择：** Splay Tree（伸展树）。
    
- **核心优势：** 它能通过“旋转”操作，在 $O(\log n)$ 的时间内把任意一个连续区间“揪”出来变成一棵独立的子树，然后通过“懒标记”实现 $O(1)$ 的虚拟翻转。

## 2. 做题思路与标准过程

这道题的标准化过程可以总结为 **“建、找、转、贴、传”** 五部曲：

### 第一步：建（建树与哨兵）

- **目的：** 初始化平衡树，维护序列。
    
- **哨兵：** 在序列头尾增加 `0` 和 `n+1` 两个虚拟节点。这是为了保证当你翻转 $[1, n]$ 时，左边界的前一个和右边界的后一个依然存在。
    

### 第二步：找（定位节点）

- **函数：** `kth(k)`。
    
- **逻辑：** 根据子树大小（`sz`），像查字典一样找到当前排名第 `k` 的小朋友。**注意：** 找人的路上必须执行 `dn`（下传标记）。
    

### 第三步：转（区间提取）

- **动作：** 1. 将排名第 $L$ 的小朋友（实际是 $L-1$ 的前驱）Splay 到根节点。
    
    2. 将排名第 $R+2$ 的小朋友（实际是 $R+1$ 的后继）Splay 到根节点的右儿子。
    
- **结果：** 此时根节点的右儿子的左子树，就是我们要操作的闭区间 $[L, R]$。
    

### 第四步：贴（打上标记）

- **动作：** 给这棵子树的根节点贴上 `rev ^= 1` 的便签。
    
- **意义：** 告诉这个小团队：“你们全员待命，准备镜像转身。”
    

### 第五步：传（输出与标记下传）

- **动作：** 最后进行一次中序遍历（左-中-右）。
    
- **逻辑：** 每次遇到脑门上有条的小朋友，先让他“换手”并把条子传给下一代（`dn` 操作），然后再输出他的值。

## 3. 具体案例分析过程

假设序列是 `[1, 2, 3, 4, 5]`，我们要翻转区间 **`[2, 4]`**（即翻转 `2, 3, 4`）。

### 状态 0：初始

- 加上哨兵后的序列：`[0, 1, 2, 3, 4, 5, 6]`。
    
- 中序遍历顺序：`0-1-2-3-4-5-6`。
    

### 状态 1：执行 `L = kth(2)`（找左保镖）

- 第 2 个人是 `1`。
    
- 执行 `spl(1, 0)`，把 `1` 拽到根部。
    
- **此时：** 根是 `1`，它右边是 `2, 3, 4, 5, 6`。
    

### 状态 2：执行 `R = kth(6)`（找右保镖）

- 注意：原本第 4 个数是 `4`，加上一个前面的哨兵是第 5 个，我们要找它的后一个，所以是第 6 个，也就是 `5`。
    
- 执行 `spl(5, 1)`，把 `5` 拽到 `1` 的右儿子。
    

### 状态 3：区间提取完成

- 此时 `1` 是根，`5` 是右儿子。
    
- 夹在 `1` 和 `5` 中间的数（`2, 3, 4`）必然全部在 **`5` 的左子树**里。
    
- 我们给这棵左子树的根（假设是 `3`）贴上 `rev = 1`。
    

### 状态 4：下传与输出

- 当我们最后遍历到 `3` 时，触发 `dn(3)`：
    
    - `3` 把左右儿子互换（假设左边原本是 `2`，右边是 `4`，现在互换）。
        
    - 中序遍历读到这一段时，顺序变成了：`4-3-2`。
        
- **最终序列：** `0, 1, 4, 3, 2, 5, 6`。
    
- **去掉哨兵：** `1, 4, 3, 2, 5`。**翻转成功！**

## 4. 核心避坑指南（导师寄语）

1. **哨兵别忘了：** 如果没有 `0` 和 `n+1`，当你翻转 `1~5` 时，`kth(l)` 就会报错，因为你找不到 `1` 前面的那个人。
    
2. **先下传再操作：** 无论是 `kth` 找人、还是 `dfs` 输出，只要你打算往子节点走，必须先调用 `dn(u)`。否则你看到的小朋友可能还没来得及“转身”，顺序就是错的。
    
3. **1-indexed 的严谨性：** 在 Python 代码中，数组开 `N+1` 或 `N+5`，保证索引从 1 开始。在我们的逻辑里，节点的编号正好对应它的初始值，这样理解起来最简单。
