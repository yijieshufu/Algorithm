# 线段树动态开点（树状数组 + 离散化）

## 题目描述

### **1. 题目背景与目标**

你需要维护一个初始为空的可重正整数集合 $A$。该集合允许重复元素。你需要支持两种操作：**向集合中添加元素**以及**查询集合中符合特定范围要求的元素之和**。

### **2. 操作类型**

- **操作 1 (更新)**：输入 $x$ 和 $c$。
    
    - 在集合 $A$ 中加入 $x$ 个数值为 $c$ 的正整数。
        
- **操作 2 (查询)**：输入 $c$。
    
    - 计算集合 $A$ 中所有**小于或等于 $c$** 的元素之和，并输出结果。
        

---

### **3. 输入/输出格式**

- **输入**：
    
    - 第一行：一个正整数 $Q$，表示操作的总次数。
        
    - 接下来 $Q$ 行：每行第一个数为 `type`。
        
        - 若 `type = 1`，接着输入 $x_i, c_i$。
            
        - 若 `type = 2`，接着输入 $c_i$。
            
- **输出**：
    
    - 对于每一个 `type = 2` 的查询，输出一行对应的权值总和。
## 解题思路

### 1. 核心矛盾分析

- **空间问题**：由于 $c_i \le 10^9$，我们无法直接开辟一个数组 `count[10^9]` 来统计每个数字出现的次数。
    
- **时间问题**：$Q = 10^6$ 且时限为 3 秒。这意味着我们必须使用 $O(Q \log Q)$ 或 $O(Q \log M)$（$M$ 为不同权值的数量）复杂度的算法。普通的线性遍历会超时。
    
- **功能需求**：我们需要一个能够支持“单点更新”和“前缀和查询”的数据结构。
    

---

### 2. 主流方案：离散化 + 树状数组 (Fenwick Tree)

这是解决此类权值范围巨大但操作次数有限的问题最经典的方法。

#### **第一步：离散化 (Discretization)**

由于我们只关心出现在输入中的 $c_i$，而不关心 $10^9$ 空间中其他没有出现过的数字，我们可以：

1. 预先读取所有的操作，将所有的 $c_i$ 存入一个列表。
    
2. 对列表进行 **排序** 并 **去重**。
    
3. 用每个数字在排序后列表中的 **排名 (Rank)** 来代替数字本身。
    
    - 例如：若输入中出现的 $c_i$ 有 $\{10, 10^9, 500\}$，排序去重后为 $\{10, 500, 10^9\}$。
        
    - 映射关系：$10 \to 1, 500 \to 2, 10^9 \to 3$。
        

#### **第二步：维护树状数组**

使用映射后的排名作为索引，建立一个树状数组 `bit`：

- **操作 1 (添加)**：设 $c$ 对应的排名为 $idx$，则在树状数组的 $idx$ 位置增加值 $x \times c$。
    
- **操作 2 (查询)**：设查询的 $c$ 对应的排名为 $idx$，则在树状数组中查询前缀和 `query(idx)`。
    
    - _注意_：如果查询的 $c$ 在离散化列表中不存在，需要通过二分查找（如 `bisect_right`）找到小于等于 $c$ 的最大已知排名的位置。
        

---

### 3. 另一种方案：动态开点线段树

如果你不想预处理所有数据（即进行“在线”处理），可以使用动态开点线段树。

- **原理**：初始时不创建任何节点。只有当某个区间的权值被增加时，才动态地创建该区间对应的线段树节点。
    
- **优点**：不需要预处理，支持在线操作。
    
- **缺点**：在 $10^6$ 级别的数据下，Python 的对象分配开销非常大，容易导致内存或时间超限。
    

---

### 4. 针对 Python 环境的优化建议

考虑到 $10^6$ 的数据量对于 Python 3.7 来说非常吃力，建议采用以下技巧：

- **快速 I/O**：使用 `sys.stdin.read().split()` 一次性读取并配合迭代器。
    
- **树状数组优于线段树**：树状数组的常数更小，且可以使用简单的 `list` 实现，避免复杂的类定义。
    
- **避免冗余查找**：在离散化后，使用字典或二分查找快速定位排名。
    
- **1-Indexed**：树状数组天生适合 1-indexed 逻辑，这与你习惯的代码风格非常契合。
    

## 代码解释
### 1. 核心思想：离散化 (Discretization)

题目中数值 $c$ 的范围是 $1 \le c \le 10^9$，如果直接以 $c$ 作为索引建立数组，空间会崩溃。

- **做法**：代码首先通过 `all_vals.append(c)` 收集了所有在操作中出现的数值。
    
- **转换**：`uc = sorted(set(all_vals))` 对这些值进行去重和排序。
    
- **结果**：将巨大的数值范围映射到了一个连续的、较小的区间 $[1, \text{len}(uc)]$。例如，如果出现的数是 $\{10, 10^9, 500\}$，离散化后它们分别对应索引 $1, 3, 2$。
    

### 2. 数据结构：树状数组 (Fenwick Tree)

为了在 $O(\log N)$ 的时间内完成“单点更新”和“前缀和查询”，代码实现了经典树状数组。

- **`upd(i, d)`**：在离散化后的位置 $i$ 增加值 $d$（这里 $d = x \times c$）。它利用 `i += i & -i` 向上更新覆盖该节点的父节点。
    
- **`qry(i)`**：查询从索引 $1$ 到 $i$ 的权值总和。它利用 `i -= i & -i` 向下累加相关联的区间值。
    

---

### 3. 代码执行流程分析

#### **第一阶段：输入与准备**



```Python
it = iter(sys.stdin.read().split()) # 快速 I/O
...
for _ in range(q):
    # 存储操作并记录 c 的值，用于后续离散化
```

- 使用 `sys.stdin.read().split()` 配合迭代器 `it` 是 Python 处理百万级数据最快的方式之一。
    

#### **第二阶段：映射建立**


```Python

uc = sorted(set(all_vals))
n, tree = len(uc), [0] * (len(uc) + 1)
```

- `uc` 是有序的。后面使用 `bisect_left` 在 `uc` 中查找某个具体的 $c$ 的位置，这个位置（+1 后）就是树状数组中的索引。
    

#### **第三阶段：处理操作**

```Python
for op in ops:
    idx = bisect_left(uc, op[2] if op[0] == 1 else op[1]) + 1
    if op[0] == 1:
        upd(idx, op[1] * op[2]) # 加入 x 个 c，即总权值增加 x * c
    else:
        res.append(str(qry(idx))) # 查询 <= c 的总权值
```

- **操作 1**：集合增加了 $x$ 个 $c$，因此对应索引处的总和增加 $x \times c$。
    
- **操作 2**：通过 `qry(idx)` 获取树状数组中所有排名小于等于 $c$ 的元素权值总和。

### 4. 复杂度与性能

|**维度**|**复杂度**|**说明**|
|---|---|---|
|**时间复杂度**|$O(Q \log Q)$|排序 $O(Q \log Q)$，查询/更新 $Q \times (\log Q + \log Q)$。|
|**空间复杂度**|$O(Q)$|存储 `ops`、`uc` 和 `tree` 数组，最大占用约几百 MB。|
