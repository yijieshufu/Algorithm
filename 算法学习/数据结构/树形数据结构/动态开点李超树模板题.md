# 动态开点李超树模板题（斜率优化 / 凸壳）

## 题目描述

**核心场景：**

商店有 $N$ 个顾客和 $M$ 件商品。每件商品 $j$ 需要设定一个价格 $P_j$。

- **顾客属性：** 第 $i$ 个顾客有一个“积极性” $B_i$。
    
- **商品属性：** 第 $j$ 个商品有一个“价值” $C_j$。
    
- **购买条件：** 只有当 **顾客积极性 + 商品价值 $\ge$ 商品价格** 时，顾客才会购买。
    
    - 公式表达：$B_i + C_j \ge P_j$
        

**目标：**

对于每一件商品 $j$，找到一个最优价格 $P_j$，使得该商品的**总销售额最大化**。

- 总销售额 = $P_j \times (\text{购买该商品的人数})$。
    

**约束条件：**

- $N, M \le 2 \times 10^5$（数据量大，要求 $O(N \log N)$ 或 $O(N+M)$ 级别的算法）。
    
- $B_i, C_j \le 10^9$（需考虑大整数，Python 自动支持，但计算效率需注意）。
## 题型判断
### 题型分类
- **贪心/数学推导：** 确定最优价格的候选集。

- **[[../../../算法的思想/几何优化]] (Convex Hull Trick, CHT)：** 解决“多个线性函数求最值”的效率问题。
### 核心难点

- **暴力破解不可行：** 如果对每个商品（$M$个）都枚举所有可能的定价策略（$N$种），复杂度为 $O(M \times N)$。在 $2 \times 10^5$ 的规模下，计算量高达 $4 \times 10^{10}$，必然超时。
    
- **动态最优值：** 随着商品价值 $C_j$ 的变化，最优定价所对应的“购买人数”也会随之改变，这并非简单的线性关系。
## 解题思路
推导的核心在于：**对于某一件固定的商品 $j$，我们要决定卖给“多少人”最划算。**

---

### 第一步：拆分购买条件

题目给出的条件是：$B_i + C_j \ge P_j$。

我们将它变形，把属于**顾客**的变量和属于**商品**的变量分开：

$$
B_i \ge P_j - C_j
$$

为了方便分析，我们令 **$K_j = P_j - C_j$**。

- 这个 $K_j$ 可以理解为这件商品对顾客“积极性”的最低门槛。
    
- 只要顾客的 $B_i \ge K_j$，他就会买。
    

### 第二步：确定定价策略

既然我们要最大化总销售额，那么定价 $P_j$ 一定会恰好卡在某个顾客的“临界点”上。

如果我们希望有 **$k$** 个人购买，那么我们应该让这 $k$ 个 $B_i$ 最大的顾客都能买得起。

1. 将所有顾客的积极性 $B$ **从大到小排序**，记为 $B_1, B_2, \dots, B_N$。
    
2. 如果我们想让前 $k$ 个人购买，那么门槛 $K_j$ 最高只能设为 $B_k$。
    
3. 此时，根据公式 $P_j = K_j + C_j$，我们的定价就是：

    $$


P_j = B_k + C_j

$$
    

### 第三步：构建销售额函数

对于第 $j$ 件商品，如果目标客户数是 $k$，总销售额 $Sales$ 为：
$$

Sales = \text{价格} \times \text{人数}

$$
$$

f_j(k) = (B_k + C_j) \times k

$$
展开这个公式：
$$

f_j(k) = k \cdot C_j + k \cdot B_k

$$
### 第四步：转化为几何模型（重点）

观察上面的公式，对于每一件商品 $j$ 来说，$C_j$ 是已知的常量。我们要在众多的 $k$（从 $1$ 到 $N$）中选一个，让 $f_j(k)$ 最大。

我们可以把 $f_j(k)$ 看成是一个关于 $x$ 的**一次函数**，其中 $x = C_j$：
$$

y = m_k \cdot x + b_k

$$

- **斜率 $m_k = k$**（即购买人数，取值范围是 $1, 2, \dots, N$）
    
- **截距 $b_k = k \cdot B_k$**（即这 $k$ 个人的积极性贡献的保底收入）
    

---

### 结论

现在问题变成了：

1. 你有 $N$ 条直线，第 $k$ 条直线是 $y = k \cdot x + (k \cdot B_k)$。
    
2. 给定 $M$ 个询问（每个询问是一个 $C_j$），求这 $N$ 条直线在 $x = C_j$ 时的最大 $y$ 值。
    

**这就是典型的“李超线段树”或者“单调栈维护凸包”的应用场景。**

### 为什么这个推导很重要？

- **消除 $P_j$ 的不确定性：** 通过引入 $K_j$ 和排序后的 $B_k$，我们把一个“连续的定价问题”变成了一个“离散的选择人数问题”。
    
- **解耦变量：** 把 $B_i$ 和 $C_j$ 分开后，我们发现每一条直线的属性（斜率和截距）只跟顾客数据有关，而询问只跟商品数据有关。

## 代码解释

这段代码非常精彩，它完整地实现了我们刚才讨论的**凸包优化（CHT）**。

为了让你彻底理解，我把代码拆解成**预处理、构建凸壳、查询**三个核心模块来解释。

---

## 1. 预处理：生成“赛跑选手”（直线）

```Python
B[1:] = sorted(B[1:]) # 将顾客积极性从小到大排序
# 生成直线列表
lines = [(N - i + 1, B[i] * (N - i + 1)) for i in range(N, 0, -1)]
```

- **排序：** 排序后，$B[1]$ 是最小的，$B[N]$ 是最大的。
    
- **直线映射：** * 当 $i=N$ 时（最大的 $B$），只有 1 个人能买起。斜率 $k=1$，截距 $b = B[N] \times 1$。
    
    - 当 $i=1$ 时（最小的 $B$），全员 $N$ 人都能买起。斜率 $k=N$，截距 $b = B[1] \times N$。
        
    - **结果：** `lines` 里的直线是按**斜率 $k$ 从小到大**排列的（$1, 2, \dots, N$）。这为后续维护凸壳提供了便利。
        

---

## 2. 构建凸壳：剔除“酱油选手”

这是 `build_convex_hull` 函数的核心，也就是我们说的“维护上轮廓”。

```Python
while len(hull) >= 2:
    k1, b1, k2, b2 = hull[-2][0], hull[-2][1], hull[-1][0], hull[-1][1]
    # 交叉相乘判断交点位置，避免浮点运算
    if (b1 - b2) * (k - k2) >= (b2 - b) * (k2 - k1):
        hull.pop()
    else:
        break
```

- **逻辑原理：** 由于直线的斜率 $k$ 是单调递增的，新加入的直线 $L_{new}$ 最终一定会超越前面的直线。
    
    我们通过比较**交点**来判断：如果 $L_1$ 与 $L_2$ 的交点在 $L_2$ 与 $L_3$ 交点的右侧，说明 $L_2$ 在任何时候都不会成为“第一名”。
    
- **数学技巧：** 这里用了 `(b1 - b2) * (k - k2) >= (b2 - b) * (k2 - k1)`，这其实是交点公式 $x = \frac{b_2 - b_1}{k_1 - k_2}$ 的变形，**交叉相乘法**可以完美避开除法带来的浮点数精度误差，是算法竞赛中的标准写法。
    

---

## 3. 查询：寻找“当下的最强者”

```Python
def query_max(hull, x):
    l, r = 0, len(hull) - 1
    while l < r:
        mid = (l + r) // 2
        v1 = hull[mid][0] * x + hull[mid][1]
        v2 = hull[mid + 1][0] * x + hull[mid + 1][1]
        l, r = (mid + 1, r) if v1 <= v2 else (l, mid)
    return hull[l][0] * x + hull[l][1]
```

- **二分搜索：** 因为我们的直线是组成了一个“上凸壳”，随着 $x$（商品价值）的增加，最优直线的斜率也是单调变化的。
    
- **比较相邻直线：** 每次取中间的两条直线 $v_1$ 和 $v_2$：
    
    - 如果 $v_1 \le v_2$，说明最高点还在右边，我们往右找。
        
    - 否则，往左找。
        
- **效率：** 这种二分法让你在 $O(\log N)$ 的时间内就能在几万条直线中精准锁定最高的那一条。
