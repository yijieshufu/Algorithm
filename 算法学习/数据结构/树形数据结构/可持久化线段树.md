# 可持久化线段树（静态区间第 k 小）

## 题目描述

给定一个长度为 $N$ 的序列 $\{a\}$，共有 $Q$ 次询问。对于每次询问，给定三个整数 $l, r, k$，你需要求出序列在区间 $[l, r]$ 内的所有元素中，从小到大排序后的第 $k$ 个元素。

**特殊情况：** 如果询问中的 $k = 0$，请直接输出 $1$。

---

### 输入格式

1. **第一行：** 包含两个正整数 $N$ 和 $Q$，分别表示序列的长度和询问的次数。
    
2. **第二行：** 包含 $N$ 个整数，表示序列中的元素 $a_i$。
    
3. **接下来的 $Q$ 行：** 每行包含三个正整数 $l, r, k$，表示一次查询区间 $[l, r]$ 内第 $k$ 小值的操作。
## 题型

可持久化线段树（主席树）


## 解题思路

这道题是经典的**静态区间第 $k$ 小**问题。由于 $N, Q$ 均为 $10^5$ 级别，且 $a_i$ 范围高达 $10^9$，简单的排序或暴力扫描无法满足时间要求。


---

### 1. 数据预处理：离散化 (Discretization)

由于 $a_i$ 的数值范围（$0$ 到 $10^9$）远大于序列长度 $N$，我们无法直接以数值作为线段树的下标。

- **操作：** 将序列中所有出现的数值去重并排序，记录在数组 $v$ 中。
    
- **映射：** 将原序列 $a_i$ 替换为其在 $v$ 中的排名（从 $1$ 开始）。这样，所有数值都被压缩到了 $[1, N]$ 的范围内。
    

### 2. 核心算法：可持久化线段树 (Chairman Tree)

这道题的标准解法是使用**可持久化线段树**（俗称“主席树”）。其核心逻辑是将区间查询转化为**前缀信息**的差分。

#### A. 前缀和思想

假设我们为序列的每一个前缀 $[1, i]$ 都建立一棵权值线段树 $T_i$：

- $T_i$ 维护的是：在前 $i$ 个数中，落在某个数值范围内的元素个数。
    
- **差分特性：** 区间 $[l, r]$ 内的数值分布情况，可以通过 $T_r$ 与 $T_{l-1}$ 两棵树对应节点相减得到。即：`count(l, r, range) = count(1, r, range) - count(1, l-1, range)`。
    

#### B. 可持久化结构

如果为每个 $i$ 都建一棵完整的线段树，空间复杂度 $O(N^2)$ 会爆炸。

- **观察：** $T_i$ 相比于 $T_{i-1}$，只增加了一个数字 $a_i$。在权值线段树中，这只会影响一条从根到叶子的路径（长度为 $\log N$）。
    
- **优化：** $T_i$ 复用 $T_{i-1}$ 中未发生变化的节点，仅为改变的 $\log N$ 个节点开辟新空间。
    

### 3. 查询过程 (Tree Binary Search)

当查询区间 $[l, r]$ 的第 $k$ 小时：

1. 同时从 $T_r$ 和 $T_{l-1}$ 的根节点出发。
    
2. 计算左子树的贡献：`left_cnt = count(T_r.left) - count(T_{l-1}.left)`。
    
3. **判定：**
    
    - 如果 $k \le left_cnt$，说明第 $k$ 小数在左子树，递归进入左子树。
        
    - 如果 $k > left_cnt$，说明第 $k$ 小数在右子树，递归进入右子树，并更新 $k = k - left_cnt$。
        
4. 当到达叶子节点时，该节点对应的离散化原值即为答案。
    

---

### 4. 复杂度分析

- **时间复杂度：** 离散化 $O(N \log N)$ + 建树 $O(N \log N)$ + $Q$ 次查询 $O(Q \log N)$。总复杂度 $O((N+Q) \log N)$。
    
- **空间复杂度：** 离散化数组 $O(N)$ + 主席树节点 $O(N \log N)$。约需 $10^5 \times 20$ 左右个节点。
    

## 代码分析

### 静态内存分配
```Python
# N=10^5 每次修改新建 logN 个节点，加上初始建树，40 * N 是一个非常安全的空间范围。
MAX_N = 200005 * 40  

ls, rs, cnt = [0] * MAX_N, [0] * MAX_N, [0] * MAX_N
root, tot = [0] * 200005, 0
```

### 单点更新函数

```Python
def upd(prev, l, r, pos):
    """
    prev: 旧版本线段树中对应位置的节点编号
    l, r: 当前节点所表示的权值范围 [l, r]
    pos: 当前要插入的数值（离散化后的排名）
    """
    global tot
    tot += 1                # 申请一个全新的节点编号
    curr = tot              # curr 是当前新版本树中的节点索引
    
    # 【核心操作：路径拷贝】
    # 首先将旧节点 prev 的所有信息（左子树索引、右子树索引）拷贝给新节点 curr
    # 同时，新节点覆盖的范围增加了一个数，所以 cnt[curr] = 旧节点的计数 + 1
    ls[curr], rs[curr], cnt[curr] = ls[prev], rs[prev], cnt[prev] + 1
    
    # 递归终止条件：到达叶子节点（权值范围缩小到一个点）
    if l == r: 
        return curr
    
    mid = (l + r) // 2
    # 判断插入的数值 pos 在左半区还是右半区
    if pos <= mid:
        # 如果在左边，左子树需要新建（递归调用），右子树直接指向旧树的 rs[prev]
        ls[curr] = upd(ls[prev], l, mid, pos)
    else:
        # 如果在右边，右子树需要新建（递归调用），左子树直接指向旧树的 ls[prev]
        rs[curr] = upd(rs[prev], mid + 1, r, pos)
        
    return curr             # 返回当前新建节点的编号给父节点
```

### 区间第 $k$ 小查询

```Python
def qry(u, v, l, r, k):
    """
    u: 代表前缀 [1, L-1] 的线段树根节点编号
    v: 代表前缀 [1, R] 的线段树根节点编号
    l, r: 当前查询的权值（离散化排名）范围 [l, r]
    k: 要找的是该区间内排名第 k 的数
    """
    # 递归终点：当权值范围缩小到一个点时，说明找到了第 k 小的数对应的离散化索引
    if l == r: 
        return l
    
    mid = (l + r) // 2
    
    # 【核心差分逻辑】
    # cnt[ls[v]] 是前 R 个数中，落在左子树权值范围内的个数
    # cnt[ls[u]] 是前 L-1 个数中，落在左子树权值范围内的个数
    # 两者相减，得到的 num 就是原序列区间 [L, R] 内，落在左半部分权值范围内的元素总数
    num = cnt[ls[v]] - cnt[ls[u]]
    
    # 判定第 k 小在哪一侧：
    if k <= num:
        # 如果左侧的元素个数足够 k 个，说明第 k 小一定在左子树
        return qry(ls[u], ls[v], l, mid, k)
    else:
        # 如果左侧不足 k 个，说明第 k 小在右子树
        # 此时要在右子树找排名为 (k - num) 的数
        return qry(rs[u], rs[v], mid + 1, r, k - num)
```
