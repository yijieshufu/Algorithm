# 带修改的字符串哈希（BIT + Hash）

## 题目描述

给定一个长度为 $N$ 的字符串 $S$，字符集仅包含小写字母。你需要支持以下两种操作：

1. **修改操作**：给定位置 $x$ 和字符 $c$，将 $S[x]$ 修改为 $c$。
    
2. **查询操作**：给定四个正整数 $l_1, r_1, l_2, r_2$，询问子串 $S[l_1, r_1]$ 与子串 $S[l_2, r_2]$ 是否完全相同。

## 题型判断

动态字符串哈希

## 解题思路

### 1. 核心思路分析

#### A. 字符串哈希 (Rolling Hash)

我们将字符串看作一个 $P$ 进制的数。对于长度为 $n$ 的字符串 $S$，其哈希值通常定义为：

$$
Hash(S) = (\sum_{i=1}^{n} S[i] \cdot P^{n-i}) \pmod M
$$

其中 $P$ 是一个大于字符集大小的质数（如 131 或 13331），$M$ 是一个大质数（如 $10^9+7$ 或 $2^{61}-1$）。

对于任意子串 $S[l..r]$，如果我们能快速维护前缀哈希，就可以通过以下公式在 $O(1)$ 得到子串哈希：

$$
Hash(l, r) = (Prefix[r] - Prefix[l-1] \cdot P^{r-l+1}) \pmod M
$$

#### B. 动态维护：结合树状数组 (BIT)

由于存在修改操作，前缀哈希会失效。我们需要一种数据结构来维护这个“加权和”。

我们可以利用树状数组（Fenwick Tree）来存储每个字符的加权值：

$$
Val[i] = S[i] \cdot P^{i} \pmod M
$$

树状数组维护的是这个序列的区间和。

- **修改 $S[x]$ 为 $c$：**
    
    计算增量 $\Delta = (c - S[x]) \cdot P^{x} \pmod M$，在树状数组中更新位置 $x$。
    
- **查询 $Hash(l, r)$：**
    
    通过树状数组求得区间和 $Sum(l, r) = \sum_{i=l}^{r} S[i] \cdot P^{i} \pmod M$。
    
    为了比较两个长度不同的子串（或位置不同的子串），我们需要将它们的权重对齐。通常做法是将 $Sum(l, r)$ 除以 $P^{l}$，或者统一乘到足够大的幂次。

## 代码解析


这段代码实现了一个**基于树状数组（BIT）动态维护字符串哈希**的算法。它能够高效地处理单点修改和区间相等性查询。

以下是对代码的详细解析和注解：

---

### 1. 输入预处理与初始化

这部分建立了高效的 I/O 环境，并准备了哈希所需的常数。
```Python
import sys
# 读取所有输入并切分为迭代器，这种方式在处理大量数据（10^5 级别）时比 input() 快得多
it = iter(sys.stdin.read().split())
n, q = int(next(it)), int(next(it))
s_str = next(it)

mod = 1000000007 # 大质数，用于取模防止溢出并减少冲突
p = 131          # 进制基数，通常取 131 或 13331

# 1-indexed 数组：s[0] 填充 0，字符串字符存入 s[1...n]
s = [0] + [ord(s_str[i]) for i in range(n)]

# 预处理 P 的幂次数组 powP[i] = p^i % mod
powP = [1] * (n + 1)
for i in range(1, n + 1):
    powP[i] = powP[i - 1] * p % mod
```

---

### 2. 树状数组 (Fenwick Tree) 函数

由于字符串会发生修改，我们需要树状数组来维护前缀哈希和，从而在 $O(\log N)$ 内完成修改与区间求和。

```Python
# 单点更新：在位置 i 增加值 d
def upd(i, d, bit):
    while i <= n: 
        bit[i] = (bit[i] + d) % mod
        i += i & -i # 树状数组的核心：加上最低位的 1 (lowbit)

# 查询前缀和：返回 [1, i] 的哈希权重累加和
def qry(i, bit):
    r = 0
    while i > 0: 
        r = (r + bit[i]) % mod
        i -= i & -i # 减去最低位的 1
    return r
```

---

### 3. 构建初始哈希树

这里的哈希值定义为 $Hash(i) = S[i] \cdot P^{i-1}$。树状数组维护的是这个值的序列和。

```Python
bit = [0] * (n + 1)
for i in range(1, n + 1):
    # 将每个字符的加权值加入树状数组
    upd(i, s[i] * powP[i - 1] % mod, bit)
```

---

### 4. 核心逻辑：操作处理

这是代码最关键的部分，特别是查询时的**幂次对齐**逻辑。

```Python
for _ in range(q):
    op = int(next(it))
    
    if op == 1:
        # 修改操作：将位置 x 修改为字符 c
        x, c = int(next(it)), next(it)
        # 计算新旧字符的差值（加权后）
        d = (ord(c) - s[x]) * powP[x - 1] % mod
        upd(x, d, bit)
        s[x] = ord(c) # 更新原数组记录
        
    else:
        # 查询操作：比较 S[l1..r1] 和 S[l2..r2]
        l1, r1, l2, r2 = int(next(it)), int(next(it)), int(next(it)), int(next(it))
        
        # 1. 长度不等直接 No
        if r1 - l1 != r2 - l2:
            print("No")
            continue
            
        # 2. 获取两个区间的加权哈希和
        # h = sum(S[i] * P^{i-1}) for i in [l, r]
        h1 = (qry(r1, bit) - qry(l1 - 1, bit) + mod) % mod
        h2 = (qry(r2, bit) - qry(l2 - 1, bit) + mod) % mod
        
        # 3. 幂次对齐比较
        # 因为 h1 的项是从 P^{l1-1} 开始的，h2 是从 P^{l2-1} 开始的
        # 要比较它们，需要通过乘法将两者的起始幂次“拉平”到同一水平
        if l1 < l2:
            # 如果第一个区间在前，给 h1 补齐幂次差
            h1 = h1 * powP[l2 - l1] % mod
        else:
            # 如果第二个区间在前，给 h2 补齐幂次差
            h2 = h2 * powP[l1 - l2] % mod
            
        print("Yes" if h1 == h2 else "No")
```

---

### 算法要点分析

#### 为什么需要“对齐”？

树状数组里存的是 $\sum S[i] \cdot P^{i-1}$。

- 假设区间 1 是 `[1, 2]`，对应值是 $S[1] \cdot P^0 + S[2] \cdot P^1$。
    
- 假设区间 2 是 `[3, 4]`，对应值是 $S[3] \cdot P^2 + S[4] \cdot P^3$。
    

即使这两个子串内容完全一样（即 $S[1]=S[3], S[2]=S[4]$），由于 $P$ 的幂次不同，直接计算出的 `h1` 和 `h2` 也不相等。通过执行 `h1 = h1 * powP[3 - 1]`，`h1` 变成了：

$$
(S[1] \cdot P^0 + S[2] \cdot P^1) \cdot P^2 = S[1] \cdot P^2 + S[2] \cdot P^3
$$

此时 `h1` 就能与 `h2` 直接进行相等性判断了。

#### 复杂度

- **时间复杂度**：$O(N + Q \log N)$。预处理 $O(N)$，每次修改和查询都是 $O(\log N)$。
    
- **空间复杂度**：$O(N)$。主要开销在 `powP` 数组、`s` 数组和 `bit` 数组。
