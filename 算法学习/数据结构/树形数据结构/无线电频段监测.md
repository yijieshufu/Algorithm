# 题目描述

### 1. 题目详细描述

**背景设定：**

在一个巨大的数据仓库中，有编号为 $1$ 到 $N$ 的无线电频道。初始时，所有频道的信号值均为 $0$。

**操作要求：**

程序需要支持以下两种动态操作：

1. **修改操作 (指令 1)：** 格式为 `1 l r v`。将编号在闭区间 $[l, r]$ 内的所有频道的信号值增加或减少 $v$。
2. **查询操作 (指令 2)：** 格式为 `2 l r x`。统计编号在闭区间 $[l, r]$ 内，信号值大于或等于 $x$ 的频道数量。

**数据规模：**

- 操作次数 $Q \le 10^3$ (非常小)。
- 频道总数 $N \le 2 \times 10^6$ (非常大)。
- 修改值 $v$ 和阈值 $x$ 范围在 $[-10^4, 10^4]$。
- 范围 $l, r$ 最大可达 $2 \times 10^6$。

### 2. 题型判断

这道题属于 **“区间信息维护”** 类题目，具体细分如下：

- **核心考点：** **区间修改 (Range Update)** 与 **区间查询 (Range Query)**。
- **具体题型：** **离散化 / 坐标压缩 + 差分或线段树**。
- **难度评价：** 中等偏易。
  - 虽然 $N$ 很大，但由于 **$Q$ 非常小 ($1000$)**，这暗示了时间复杂度可以与 $Q$ 相关，而不需要完全取决于 $N$。

# 题目分析过程

针对这道题目，由于频道总数 $N$ 非常大（$2 \times 10^6$），而操作次数 $Q$ 却非常小（只有 $1000$），解决本题的关键在于**坐标离散化（Coordinate Compression）**。

下面是详细的分析过程：

### 1. 核心矛盾分析

- **空间/时间上的巨大差异：** 频道数量 $N = 2,000,000$，如果直接开一个数组 `a[N]` 并对区间 $[l, r]$ 进行暴力循环修改，每次修改的时间复杂度是 $O(N)$。在 $Q=1000$ 次操作下，总复杂度达到 $2 \times 10^9$，这在 Python 中（甚至 C++ 中）通常会超时。
    
- **离散化契机：** 虽然有 $200$ 万个频道，但操作只有 $1000$ 次。这意味着在整个坐标轴上，只有操作涉及到的端点（$l$ 和 $r$）及其附近，信号值才会发生变化。在两个相邻的操作端点之间，所有频道的信号值增长情况是**完全一致**的。

---

### 2. 算法步骤详解

#### 第一步：收集关键端点

我们需要找出所有可能改变信号状态的“分界点”。

对于每一个操作 `(l, r)`，信号值从 $l$ 开始变化，到 $r$ 结束。为了方便处理闭区间，我们可以记录每一个操作的起始点 $l$ 和结束点的下一个点 $r+1$。

- 将所有的 $l$ 和 $r+1$ 放入一个列表。
    
- 对列表进行**排序**并**去重**。
    
- 假设去重后的有序点集为 $P = \{p_1, p_2, \dots, p_m\}$。这些点将整个数轴切分成了若干个“基本区间”：$[p_1, p_2-1], [p_2, p_3-1], \dots$。

#### 第二步：维护基本区间

这些基本区间的数量最多只有 $2Q$ 个（约 $2000$ 个）。

- 我们可以开一个数组 `val`，`val[i]` 表示第 $i$ 个基本区间内所有频道的当前信号值。
    
- 由于同一基本区间内的频道信号值始终相同，我们只需要记录该区间的**长度** $len_i = p_{i+1} - p_i$。

#### 第三步：模拟操作

1. **修改 (1 l r v)：**
    
    遍历所有的基本区间。如果某个基本区间 $[p_i, p_{i+1}-1]$ 完全落在操作范围 $[l, r]$ 内，则将该区间的信号值 `val[i]` 增加 $v$。
    
2. **查询 (2 l r x)：**
    
    遍历所有的基本区间。如果某个基本区间 $[p_i, p_{i+1}-1]$ 完全落在查询范围 $[l, r]$ 内，且该区间的信号值 `val[i] >= x`，则将该区间的长度 $len_i$ 累加到结果中。

---

### 3. 复杂度分析

- **空间复杂度：** 存储离散化后的端点和区间值，规模为 $O(Q)$，即约 $2000$ 个元素的数组。
    
- **时间复杂度：** * 预处理端点：$O(Q \log Q)$。
    
    - 每次修改/查询：遍历 $O(Q)$ 个基本区间。
        
    - 总复杂度：$O(Q^2)$。
        
    - 代入数据：$1000^2 = 1,000,000$。这个量级对于 Python 3.7 来说非常轻松，可以在几十毫秒内完成。

---

# 代码分析

这段代码是针对题目“超大区间、小量操作”这一特性，采用**坐标离散化（Coordinate Compression）** 思想编写的。它巧妙地避开了 $N=2 \times 10^6$ 的内存限制，将复杂度压缩到了与操作数 $Q$ 相关。

下面我将代码分为五个功能模块，带你逐一拆解理解：

---

### 1. I/O 与预处理模块

这是按照你要求的模板进行的快速读入。

```Python
import sys
input = sys.stdin.read().split() # 一次性读取所有输入并按空格/换行切分
it = iter(input)                 # 创建迭代器，方便逐个取出数据
q = int(next(it))                # 获取操作总数 Q
# 将所有操作预存到列表 ops 中，避免多次调用 next(it) 导致逻辑混乱
ops = [(int(next(it)), int(next(it)), int(next(it)), int(next(it))) for _ in range(q)]
```

---

### 2. 离散化：端点收集模块

这是算法的核心。因为只有在 $l$ 或 $r+1$ 这些位置，信号值才会发生“突变”，所以我们只需要关注这些“关键点”。

```Python
pts = set()
for t, l, r, _ in ops:
    pts.add(l)      # 区间的左端点（开始变化的位置）
    pts.add(r + 1)  # 区间结束后的第一个点（恢复原状或进入下个阶段的位置）
p = sorted(pts)     # 将所有关键点去重并排序，形成一个“刻度尺”
m = len(p)          # 关键点的数量
```

---

### 3. 区间映射模块

通过排序后的点集 `p`，我们将原本连续的数轴切分成了 $m-1$ 个小区间。

- **第 1 个区间**：$[p[0], p[1]-1]$
    
- **第 2 个区间**：$[p[1], p[2]-1]$
    
- ... 依此类推。
    
```Python
v_arr = [0] * (m + 1)  # 1-indexed: 存储每个小区间的当前信号值
l_arr = [0] * (m + 1)  # 1-indexed: 存储每个小区间的真实长度（包含多少个频道）

for i in range(1, m):
    # p[i] - p[i-1] 计算出当前小区间包含的整数点个数
    # 例如 p[0]=1, p[1]=5，则区间 [1, 4] 长度为 5-1=4
    l_arr[i] = p[i] - p[i - 1] 
```

---

### 4. 模拟操作模块（修改与查询）

这一步是代码逻辑最重的地方。它遍历所有的操作，并对受影响的“小区间”进行处理。

```Python
for t, ql, qr, qv in ops:
    if t == 1:
        # 【修改操作】
        for j in range(1, m):
            # 检查第 j 个小区间 [p[j-1], p[j]-1] 是否被包含在修改范围 [ql, qr] 内
            if p[j - 1] >= ql and p[j] - 1 <= qr:
                v_arr[j] += qv # 如果在范围内，直接更新该区间统一的信号值
    else:
        # 【查询操作】
        # 使用生成器表达式统计符合条件的区间长度总和
        ans = sum(l_arr[j] for j in range(1, m) 
                  if p[j - 1] >= ql and p[j] - 1 <= qr # 区间在查询范围内
                  and v_arr[j] >= qv)                  # 信号值满足阈值 x (这里变量名是 qv)
        print(ans)
```

---

### 5. 算法效率总结

- **空间换时间**：我们没有开 $2,000,000$ 大小的数组，而是开了两个大小约为 $2,000$ ($2Q$) 的数组 `v_arr` 和 `l_arr`。
    
- **复杂度计算**：
    
    - **排序端点**：$O(Q \log Q)$。
        
    - **执行操作**：共有 $Q$ 个操作，每个操作都要遍历一遍 $m$ 个区间（$m \approx 2Q$）。
        
    - **总复杂度**：$O(Q^2)$。
        
    - **实际表现**：对于 $Q=1000$，运算量约为 $10^6$，在 Python 3.7 中运行速度极快。

---

## 代码

```python
import sys
input = sys.stdin.read().split()
it = iter(input)
q = int(next(it))
ops = [(int(next(it)), int(next(it)), int(next(it)), int(next(it))) for _ in range(q)]
pts = set()
for t, l, r, _ in ops:
    pts.add(l)
    pts.add(r + 1)
p = sorted(pts)
m = len(p)
v_arr = [0] * (m + 1)
l_arr = [0] * (m + 1)
for i in range(1, m):
    l_arr[i] = p[i] - p[i - 1]
for t, ql, qr, qv in ops:
    if t == 1:
        for j in range(1, m):
            if p[j - 1] >= ql and p[j] - 1 <= qr:
                v_arr[j] += qv
    else:
        ans = sum(l_arr[j] for j in range(1, m) if p[j - 1] >= ql and p[j] - 1 <= qr and v_arr[j] >= qv)
        print(ans)
```
