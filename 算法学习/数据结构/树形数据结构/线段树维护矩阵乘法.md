# 线段树 + 矩阵快速幂（区间斐波那契求和）

## 题目描述
### 1. 题目详细描述

**题目名称：** 斐波那契数组维护 (Fibonacci Array Maintenance)

**基本设定：**

- **斐波那契数列定义：** $f_1 = 1, f_2 = 1$，且对于 $i \ge 3$，$f_i = f_{i-1} + f_{i-2}$。
    
- **初始状态：** 给定一个长度为 $n$ 的数组 $a$，其中包含了 $n$ 个整数。
    

**操作类型（共 $q$ 次）：**

1. **区间修改 (1, l, r, x)：** 将数组 $a$ 中下标从 $l$ 到 $r$ 的所有元素 $a_i$ 都加上 $x$（即 $a_i = a_i + x$）。
    
2. **区间查询 (2, l, r)：** 计算 $\sum_{i=l}^r f_{a_i} \pmod{10^9 + 7}$ 的值。即：求数组指定区间内每个元素作为下标对应的斐波那契数之和。

## 题型判断
线段树 (Segment Tree) + 矩阵快速幂 (Matrix Exponentiation)

#### 为什么是线段树？

- 题目涉及到**区间修改**（Range Update）和**区间查询**（Range Query）。
    
- 数据规模 $n, q$ 均为 $10^5$，要求操作的复杂度在 $O(\log n)$ 级别。这是典型的线段树应用场景。
    

#### 为什么需要矩阵快速幂？

- 由于 $a_i$ 的值高达 $10^9$，直接预处理斐波那契数列是不可能的。
    
- 斐波那契数列具有矩阵递推性质：
    
    $$\begin{pmatrix} f_{n+1} \\ f_n \end{pmatrix} = \begin{pmatrix} 1 & 1 \\ 1 & 0 \end{pmatrix} \begin{pmatrix} f_n \\ f_{n-1} \end{pmatrix}$$
    
- 当执行区间加法 $a_i + x$ 时，对应的斐波那契向量变换为：
    
    $$\begin{pmatrix} f_{a_i+x+1} \\ f_{a_i+x} \end{pmatrix} = M^x \cdot \begin{pmatrix} f_{a_i+1} \\ f_{a_i} \end{pmatrix}$$
    
    其中 $M = \begin{pmatrix} 1 & 1 \\ 1 & 0 \end{pmatrix}$。
## 解题思路
### 一、 核心数学转化：从标量到向量

传统的线段树维护的是区间各项之和 $\sum a_i$。但这道题要求维护的是 $\sum f_{a_i}$。由于斐波那契数 $f_n$ 与 $n$ 之间是非线性关系，直接对 $a_i$ 进行加法操作无法直接推导出 $f_{a_i+x}$ 的和。

**1. 矩阵表示法**

我们引入斐波那契数列的矩阵递推公式。定义向量 $V_n = \begin{pmatrix} f_{n+1} \\ f_n \end{pmatrix}$，转移矩阵 $M = \begin{pmatrix} 1 & 1 \\ 1 & 0 \end{pmatrix}$。

则有：

$$V_n = M \cdot V_{n-1} = M^{n-1} \cdot V_1 = M^{n-1} \cdot \begin{pmatrix} 1 \\ 1 \end{pmatrix}$$

**2. 区间修改的线性化**

当 $a_i$ 增加 $x$ 时：

$$V_{a_i+x} = M^x \cdot V_{a_i}$$

利用矩阵乘法对加法的**分配律**：

$$\sum V_{a_i+x} = \sum (M^x \cdot V_{a_i}) = M^x \cdot \left( \sum V_{a_i} \right)$$

这意味着：**对原数组 $a$ 的区间加法，等价于对线段树维护的向量和进行一次矩阵乘法。**

## 代码的解析

### 1. 基础配置与 I/O

```Python
import sys
sys.setrecursionlimit(200000) # 增加递归深度，防止线段树递归过深导致崩溃

it = iter(sys.stdin.read().split()) # 快速读入：一次性读取并生成迭代器
mod = 1000000007
n, q = int(next(it)), int(next(it))
a_init = [0] + [int(next(it)) for _ in range(n)] # 1-indexed 数组
```

- **1-indexed**: `a_init` 前面补了一个 `[0]`，使得下标从 1 开始，符合竞赛题习惯。

### 2. 矩阵快速幂 (`mat_pow`)

这个函数用于计算转移矩阵 $M^k$，其中 $M = \begin{pmatrix} 1 & 1 \\ 1 & 0 \end{pmatrix}$。
```Python
def mat_pow(k):
    a, b, c, d = 1, 1, 1, 0 # 初始矩阵 M
    ra, rb, rc, rd = 1, 0, 0, 1 # 单位矩阵 I
    while k:
        if k & 1:
            # 结果矩阵 R = R * M
            ra, rb, rc, rd = (ra*a+rb*c)%mod, (ra*b+rb*d)%mod, (rc*a+rd*c)%mod, (rc*b+rd*d)%mod
        # 矩阵自乘 M = M * M
        a, b, c, d = (a*a+b*c)%mod, (a*b+b*d)%mod, (c*a+d*c)%mod, (c*b+d*d)%mod
        k >>= 1
    return ra, rb, rc, rd # 返回 2x2 矩阵的四个分量
```

### 3. 线段树存储结构

#### 1. 空间分配：为什么是 `4 * n + 1`？

线段树是一棵近似的完全二叉树。对于一个长度为 $n$ 的区间：

- 其叶子节点有 $n$ 个。
    
- 非叶子节点有 $n-1$ 个。
    
- 总节点数在最坏情况下（当 $n$ 不是 2 的幂时）可能接近 $4n$。
    
- **`+ 1`**：是因为代码采用了 **1-indexed**（下标从 1 开始），为了防止 `p << 1 | 1`（即 $2p+1$）在访问最后一个节点时越界，习惯性地多开一个空间。
#### 2. 向量存储：`t_v1` 与 `t_v2`

这两行维护的是线段树节点里的“值”，即我们之前提到的**求和向量**。
```Python
t_v1 = [0] * (4 * n + 1)  # 存储该区间所有向量的第一个分量之和 (Σ f_{a_i+1})
t_v2 = [0] * (4 * n + 1)  # 存储该区间所有向量的第二个分量之和 (Σ f_{a_i})
```

- **`t_v2`**：是你最关心的数组，因为它存储的就是题目要求的答案 $\sum f_{a_i} \pmod{10^9+7}$。
    
- **为什么要存两个？** 因为斐波那契的矩阵变换需要两个相邻的数才能推导下一项。你可以理解为每个线段树节点 $p$ 存了一个小矩阵：$\begin{pmatrix} t\_v1[p] \\ t\_v2[p] \end{pmatrix}$。
#### 3. 懒标记存储：`la, lb, lc, ld`

这四行维护的是线段树的 **Lazy Tags**。由于这里的更新操作是“区间加 $x$”，在矩阵空间里表现为“左乘一个 $2 \times 2$ 矩阵”，所以我们需要 4 个数组来存这个矩阵的四个位置。

设节点 $p$ 的懒标记矩阵为 $L_p = \begin{pmatrix} la[p] & lb[p] \\ lc[p] & ld[p] \end{pmatrix}$：
```Python
la = [1] * (4 * n + 1)  # 矩阵左上角元素 (Initial: 1)
lb = [0] * (4 * n + 1)  # 矩阵右上角元素 (Initial: 0)
lc = [0] * (4 * n + 1)  # 矩阵左下角元素 (Initial: 0)
ld = [1] * (4 * n + 1)  # 矩阵右下角元素 (Initial: 1)
```

- **初始值为 `[1, 0, 0, 1]`**：这非常有讲究！这四个数构成了一个**单位矩阵 $I$**：
    
    $$I = \begin{pmatrix} 1 & 0 \\ 0 & 1 \end{pmatrix}$$
    
- **原理**：在线段树中，如果一个区间没有待处理的更新，它的标记应该是“无影无踪”的。对于加法，标记初始是 $0$；对于乘法或矩阵变换，标记初始必须是**单位元**（即乘以它之后不改变原值）。
---

### 4 线段树的**核心操作部分**

它通过“懒标记（Lazy Propagation）”技术，将原本需要 $O(N)$ 的区间矩阵乘法压缩到了 $O(\log N)$。
#### 1. `apply`：更新节点并累加标记

这是最核心的函数。当一个矩阵作用于某个区间时，它必须同时更新**当前的值**和**该区间的标记**。

```Python
def apply(p, ma, mb, mc, md):

    # 矩阵 [ma,mb; mc,md] 左乘当前节点向量 (t_v1, t_v2)

    t_v1[p], t_v2[p] = (ma*t_v1[p]+mb*t_v2[p])%mod, (mc*t_v1[p]+md*t_v2[p])%mod

    # 懒标记合并：新矩阵左乘旧懒标记矩阵

    la[p], lb[p], lc[p], ld[p] = (ma*la[p]+mb*lc[p])%mod,(ma*lb[p]+mb*ld[p])%mod,(mc*la[p]+md*lc[p])%mod,(mc*lb[p]+md*ld[p])%mod
```

> **数学原理**：
> 
> - **向量更新**：利用矩阵乘法的分配律，$\sum (M \cdot V_i) = M \cdot (\sum V_i)$。
>     
> - **标记叠加**：如果一个区间先被要求增加 $x$（乘 $M^x$），再被要求增加 $y$（乘 $M^y$），那么总变换就是 $M^y \cdot M^x$。
>     

---

#### 2. `push_down`：下传标记

当我们要访问子区间时，必须先把当前区间堆积的“变换任务”下发给子节点。

```Python
def push_down(p):
    # 如果标记是单位矩阵 (1,0,0,1)，说明没有待处理的任务，直接返回
    if (la[p], lb[p], lc[p], ld[p]) == (1, 0, 0, 1): return
    
    l, r = p << 1, p << 1 | 1 # 快速定位左右子节点：2p 和 2p+1
    
    # 将当前的矩阵标记应用到左右子节点
    apply(l, la[p], lb[p], lc[p], ld[p])
    apply(r, la[p], lb[p], lc[p], ld[p])
    
    # 下传后，重置当前节点的标记为单位矩阵
    la[p], lb[p], lc[p], ld[p] = 1, 0, 0, 1
```

---

#### 3. `push_up`：向上合并结果

当子节点更新完成后，父节点需要重新计算自己的总和。

```Python
def push_up(p):
    # 父节点的向量和 = 左子节点向量和 + 右子节点向量和
    t_v1[p] = (t_v1[p<<1] + t_v1[p<<1|1]) % mod
    t_v2[p] = (t_v2[p<<1] + t_v2[p<<1|1]) % mod
```

---

#### 4. `build`：递归建树

在程序开始时，将原始数组转化为线段树结构。

```Python
def build(p, l, r):
    if l == r:
        # 叶子节点：计算 f_{a_i} 对应的矩阵分量
        # 这里 mat_pow(a_init[l]) 返回的是 M^{a_i}
        ma, mb, mc, md = mat_pow(a_init[l])
        t_v1[p], t_v2[p] = ma, mc # 存储向量 [f_{a_i+1}, f_{a_i}]
        return
    m = (l + r) // 2
    build(p << 1, l, m)      # 构建左子树
    build(p << 1 | 1, m + 1, r) # 构建右子树
    push_up(p)               # 向上合并
```

---

#### 5. `upd`：区间修改

修改区间 $[ql, qr]$，让该区间内每个数 $a_i$ 都加上 $x$。

```Python
def upd(p, l, r, ql, qr, ma, mb, mc, md):
    # 如果当前区间 [l, r] 完全被目标区间 [ql, qr] 覆盖
    if ql <= l and r <= qr:
        apply(p, ma, mb, mc, md) # 直接更新并打上标记，不再向下递归
        return
    push_down(p) # 否则，先下传已有标记
    m = (l + r) // 2
    if ql <= m: upd(p << 1, l, m, ql, qr, ma, mb, mc, md)
    if qr > m: upd(p << 1 | 1, m + 1, r, ql, qr, ma, mb, mc, md)
    push_up(p)   # 最后更新当前节点的求和值
```

---

#### 6. `qry`：区间查询

查询区间 $[ql, qr]$ 的 $\sum f_{a_i}$。

```Python
def qry(p, l, r, ql, qr):
    # 完全覆盖，直接返回该节点的和向量中的第二项 (t_v2)
    if ql <= l and r <= qr: return t_v2[p]
    
    push_down(p) # 下传标记，保证子节点数据是最新的
    m = (l + r) // 2
    res = 0
    if ql <= m: res = (res + qry(p << 1, l, m, ql, qr)) % mod
    if qr > m: res = (res + qry(p << 1 | 1, m + 1, r, ql, qr)) % mod
    return res
```

---

#### 逻辑总结图示

- **向下走 (`upd`, `qry`)**：必须调用 `push_down`，确保读取到的数据没有被“懒”掉。
    
- **向上走 (`upd`, `build`)**：必须调用 `push_up`，确保父节点反映了子节点的变化。
    
- **打住不走 (`apply`)**：当区间匹配时，直接修改并记录标记，这是 $O(\log N)$ 的精髓。
    

### 5. 主循环逻辑

for _ in range(q):
    op = int(next(it))
    if op == 1:
        l, r, x = int(next(it)), int(next(it)), int(next(it))
