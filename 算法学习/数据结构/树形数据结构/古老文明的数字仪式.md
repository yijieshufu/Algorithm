# 古老文明的数字仪式（前 k 个数中位数）

## 题目描述

在一个古老文明的背景下，你需要处理一个长度为 $N$ 的数字序列 $A$。祭司会进行 $M$ 次询问，每次询问给出一个正奇数 $k$，要求你输出该序列中**前 $k$ 个元素的中位数**。

- **输入：** 序列长度 $N$ 和询问次数 $M$；长度为 $N$ 的整数数组；$M$ 个表示前 $k$ 项的奇数。
    
- **输出：** 针对每个询问 $k$，输出前 $k$ 个元素排序后的中间值。
    
- **数据规模：** $N$ 最大为 $100,000$，数值 $A_i$ 最高达 $10^9$。这意味着直接对每个询问进行重新排序（$O(M \cdot N \log N)$）会超时，必须使用更高效的数据结构。
## 题型判断
**动态中位数 (Dynamic Median)** 或 **对顶堆 (Dual Heap)** 类型的问题
属于**基础数据结构** 应用题
## 解题思路

针对 $N=10^5$ 的规模，最理想的复杂度是 $O((N+M) \log N)$。

#### 核心策略：对顶堆 (Dual Heap)

这是解决动态中位数最常用的技巧。我们维护两个堆：

1. **大根堆 (Max-Heap)：** 存储前 $k$ 个数中较小的那一半元素。
    
2. **小根堆 (Min-Heap)：** 存储前 $k$ 个数中较大的那一半元素。
    

**维护规则：**

- 始终保持大根堆的大小等于或比小根堆多 $1$（因为 $k$ 是奇数，中位数恰好是大根堆的堆顶）。
    
- 当新元素加入时，先根据大小放入合适的堆，再通过弹出和插入操作平衡两个堆的大小。

#### 离线处理 (Offline Processing)

由于询问的 $k$ 不是递增的（例如样例中先问 $k=3$ 再问 $k=1$），为了效率，我们可以：

- 将所有询问按 $k$ 从小到大排序。
    
- 顺序遍历数组，当遍历到的索引达到某个询问的 $k$ 时，直接记录当前大根堆的堆顶作为答案。
    
- 最后按照原始询问顺序输出。

## 代码的理解

### 1. 对顶堆的数据结构 (L 与 R)

在 Python 中，`heapq` 默认是**小根堆**。为了实现“对顶”，代码巧妙地维护了两个集合：

- **`L` (左侧大根堆):** 存储前 $k$ 个数中**较小**的那一半。代码里用了 `-x` 来存取，这样原本最大的数会变成负值中最小的，从而实现了大根堆的功能。
    
- **`R` (右侧小根堆):** 存储前 $k$ 个数中**较大**的那一半。
    

**平衡逻辑 (`add(x)` 函数)：**

- **新成员入场：** 如果新数 $x$ 比左侧最大的数还小，就进 `L`；否则进 `R`。
    
- **动态调整：** 因为题目要求的 $k$ 始终是奇数，所以代码的目标是让 **`L` 的长度恰好比 `R` 多 1**。
    
    - 如果 `L` 太臃肿了，就把 `L` 的堆顶（左侧最大值）踢到 `R` 里去。
        
    - 如果 `R` 抢了地盘，就把 `R` 的堆顶（右侧最小值）拉回到 `L` 里。
        
- **结果：** 调整完后，`L` 的堆顶 `-L[0]` 永远是这 $k$ 个数的中位数。
    

---

### 2. 离线处理的巧妙之处 (`qs` 排序)

这是代码效率极高的关键。如果祭司先问第 100 个数的中位数，再问第 3 个数的，普通做法可能要推倒重来。
```Python
qs = sorted((int(next(it)), i) for i in range(1, m+1))
```

- **排序：** 代码把所有的询问按 $k$（序列长度）从小到大排了序。
    
- **单调增长：** 这样我们只需要维护一个指针 `idx`，从 $1$ 开始一直往后加数，**永不回头**。
    
- **还原顺序：** 为了最后能按祭司提问的原始顺序回答，代码在排序时带上了原始索引 `i`，并将答案存入 `ans[i]`。
    

---

### 3. 代码执行流追踪

假设序列是 `1 3 5 7 9 11 6`，祭司问 $k=3$ 和 $k=1$：

1. **准备：** `qs` 排序后变成 `[(1, 2), (3, 1)]`（先算 $k=1$ 的，再算 $k=3$ 的）。
    
2. **处理 $k=1$：** `idx` 移到 1，`add(a[1])`。此时 `L = [1]`, `R = []`。中位数是 `ans[2] = 1`。
    
3. **处理 $k=3$：** `idx` 继续移到 3。
    
    - 加入 `3`：`L` 变大，调整后 `L=[1], R=[3]`。
        
    - 加入 `5`：`add(5)` 进 `R`，调整后 `L=[3, 1], R=[5]`。
        
    - 此时 `L` 堆顶是 `3`，中位数 `ans[1] = 3`。
        
4. **输出：** 按照 `ans[1]`, `ans[2]` 的顺序打印，得到 `3 \n 1`。
    

---

### 总结

这段代码的时间复杂度是 $O(N \log N + M \log M)$，在 Python 5s 的运行限制下绰绰有余。
