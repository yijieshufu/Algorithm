# 区间第 k 小

## 题目描述

### 题目详细描述

- **核心任务**：给定一个长度为 $N$ 的静态数组 $a$。你需要处理 $Q$ 个询问。
    
- **询问格式**：每个询问给出三个参数 $(l, r, k)$。
    
- **目标**：在原始数组的下标区间 $[l, r]$ 中，找出所有元素排序后的**第 $k$ 小**的数值。
    
- **数据范围**：
    
    - $N, Q \le 2 \times 10^5$：这意味着算法的时间复杂度必须控制在 $O(Q \log N)$ 或 $O(Q \log^2 N)$ 左右，普通的 $O(N)$ 遍历查询会超时（$O(N \cdot Q)$）。
        
    - $-10^9 \le a_i \le 10^9$：数值范围很大，通常需要先进行**离散化**处理。
        
    - $1 \le k \le r-l+1$：保证 $k$ 在有效区间长度内。
### 2. 题型判断与核心考点

这道题属于典型的**中高级数据结构题**，具体细分如下：

#### **主要题型：静态区间查询 / 可持久化数据结构**

由于数组在查询过程中不会发生修改（静态），最主流且高效的解法是使用 **可持久化线段树（主席树）**。

#### **核心算法/技巧：**

1. **离散化 (Discretization)**：因为 $a_i$ 范围到了 $10^9$，但 $N$ 只有 $2 \times 10^5$，需要将这些大数值映射到 $[1, N]$ 的排名空间内。
    
2. **主席树 (Persistent Segment Tree)**：
    
    - 利用前缀和的思想，对数组的每一个前缀 $a[1 \dots i]$ 建立一棵线段树。
        
    - 为了节省空间，相邻的两棵树（$i-1$ 和 $i$）共用大部分节点，只新建一条路径。
        
    - 查询区间 $[l, r]$ 时，通过第 $r$ 棵树减去第 $l-1$ 棵树的结构，可以得到区间 $[l, r]$ 内数值出现次数的频次图。
        
3. **线段树二分**：在处理好的差分线段树上进行二分查找，根据左子树节点的计数决定向左还是向右递归，从而在 $O(\log N)$ 时间内锁定第 $k$ 小。
## 解题思路

针对这道题，最核心的难点在于：**如何在 $O(\log N)$ 的时间内定位区间 $[l, r]$ 内第 $k$ 小的数**。

普通的排序或暴力扫描由于 $Q$ 较大，必然超时。最标准的解法是使用 **可持久化线段树（Persistent Segment Tree）**，也被常称为 **主席树**。

以下是详细的解题分析过程：

---

### 1. 离散化 (Discretization)

题目中 $a_i$ 的范围高达 $\pm 10^9$，但元素个数 $N$ 只有 $2 \times 10^5$。

- **操作**：将所有出现的数值去重、排序。
    
- **目的**：把具体的数值映射到 $[1, m]$（$m \le N$）的排名空间内。线段树将基于这些“排名”构建，而不是基于原始数值。

### 2. 构建前缀线段树 (Prefix Segment Trees)

我们需要一种方式来表示区间 $[l, r]$ 内的数值分布。

- **核心思想**：如果能得到区间 $[1, r]$ 的数值分布和区间 $[1, l-1]$ 的数值分布，两者相减即可得到 $[l, r]$ 的分布。
    
- **结构**：
    
    - 对于数组的每一个前缀 $a[1 \dots i]$，都维护一棵线段树 $T_i$。
        
    - 线段树的每个节点存储：该值域范围内出现了多少个元素（`count`）。
        
    - $T_i$ 是在 $T_{i-1}$ 的基础上，增加了一个新元素 $a_i$ 后生成的。

### 3. 可持久化（空间优化）

如果直接建 $N$ 棵完整的线段树，空间复杂度 $O(N^2)$ 会直接导致内存溢出。

- **观察**：$T_i$ 相比于 $T_{i-1}$ 只改变了一个元素，对应在线段树上，只有从根节点到叶子节点的一条路径（长度为 $\log m$）发生了变化。
    
- **方案**：让 $T_i$ 的大部分节点直接指向 $T_{i-1}$ 的已有节点，只为发生变化的路径新建节点。
    
- **复杂度**：空间复杂度降至 $O(N \log N)$。

### 4. 区间查询 (Querying)

当查询区间 $[l, r]$ 的第 $k$ 小值时：

1. **差分计算**：利用 $T_r$ 和 $T_{l-1}$ 的对应节点进行相减，得到区间 $[l, r]$ 的内部计数。
    
2. **树上二分**：
    
    - 查看左子树的元素个数 `cnt_left = T[T[r].left].count - T[T[l-1].left].count`。
        
    - **如果 $k \le cnt\_left$**：说明第 $k$ 小的数一定在左半值域，进入左子树继续查询。
        
    - **如果 $k > cnt\_left$**：说明左半部分不够，第 $k$ 小在右半值域，进入右子树查询第 $k - cnt\_left$ 小的数。
        
3. **返回结果**：递归到叶子节点时，返回对应的离散化原始数值。

---

### 5. 复杂度总结

- **预处理（离散化）**：$O(N \log N)$
    
- **建树（$N$ 次插入）**：$O(N \log N)$
    
- **单次查询**：$O(\log N)$
    
- **总时空复杂度**：时间 $O((N+Q) \log N)$，空间 $O(N \log N)$。

## 代码解析
### 1. 初始化与内存预分配

```Python
MAX_N = 200005 * 40
ls, rs, cnt = [0] * MAX_N, [0] * MAX_N, [0] * MAX_N
root, tot = [0] * 200005, 0
```

预分配了巨大的列表 `ls`（左儿子）、`rs`（右儿子）和 `cnt`（当前节点范围内数字的个数）  
`root[i]` 存储的是在处理完原始数组前 $i$ 个元素后，对应的那棵线段树的**根节点编号**。  
每当我们需要创建一个新节点（无论是更新路径还是初始化）时，都会执行 `tot += 1`。

### 可持久化线段树（主席树）写操作

```python
def upd(prev, l, r, pos): # pos离散化后的排名
    global tot
    tot += 1           # 计数器加1，分配一个新的节点编号
    curr = tot         # 记录当前新节点的编号
    
    # 【关键】先复制旧节点的全部信息
    # ls:左儿子, rs:右儿子, cnt:该区间内数字个数
    ls[curr], rs[curr], cnt[curr] = ls[prev], rs[prev], cnt[prev] + 1
    
    if l == r:         # 到达叶子节点（值域区间缩小到了一个点）
        return curr    # 递归终点，返回当前节点编号
    
    mid = (l + r) // 2
    if pos <= mid:     # 待插入的数在左半边
        # 递归更新左子树，并将新生成的左儿子编号连在 curr 上
        ls[curr] = upd(ls[prev], l, mid, pos)
    else:              # 待插入的数在右半边
        # 递归更新右子树，并将新生成的右儿子编号连在 curr 上
        rs[curr] = upd(rs[prev], mid + 1, r, pos)
    
    return curr        # 将当前新节点的编号返回给上一层父节点
```

### **区间第 $k$ 小**问题的核心查询
```Python
#- `v` 代表前缀区间 $[1, R]$ 的数值分布。
#- `u` 代表前缀区间 $[1, L-1]$ 的数值分布
def qry(u, v, l, r, k): 
    if l == r: return l  # 终点：值域缩小到了一个确定的点，这就是我们要找的排名

    mid = (l + r) // 2
    # 计算区间 [L, R] 内，落在值域 [l, mid] 之间的元素个数
    num = cnt[ls[v]] - cnt[ls[u]]

    if k <= num:
        # 第 k 小在左半值域，去左子树找
        return qry(ls[u], ls[v], l, mid, k)
    else:
        # 第 k 小在右半值域，去右子树找，k 要减去左边已有的数量
        return qry(rs[u], rs[v], mid + 1, r, k - num)
```

### 离散化 (Discretization) —— 缩小处理范围

```Python
a = [0] + [int(next(it)) for _ in range(n)]
u = sorted(set(a[1:]))
rank = {v: i + 1 for i, v in enumerate(u)} # 数值 -> 排名
m = len(u)
```
rank：具体的数值对应的排名是多少
### 建立“版本链” —— 记录历史状态

```Python
for i in range(1, n + 1):
	#root[i-1] 是上一个版本的根节点编号
	# upd(...) 函数会基于上一个版本，只新建一条从根到叶子的路径，并返回新树的根节点编号
    root[i] = upd(root[i - 1], 1, m, rank[a[i]])
```
### 在线查询与结果还原
```Python
out = []
for _ in range(q):
# 1. root[r] 代表前缀 [1...r] 的数值分布情况。 
# 2. root[l-1] 代表前缀 [1...l-1] 的数值分布情况
    l, r, k = int(next(it)), int(next(it)), int(next(it))
```
