
## 1. 题目大意与题型判断

**题目描述：**

你面前有一排 $N$ 栋楼，每栋楼都有自己的高度。对于每一栋楼，你需要找到：

1. 它**左边**第一个比它高的楼的编号。
    
2. 它**右边**第一个比它高的楼的编号。
    
    如果找不到，就输出 -1。

**题型判断：**

这道题的关键词是“**找第一个比自己大的元素**”。在算法界，这种问题的标准解法叫作 **单调栈（Monotonic Stack）**。

---

## 2. 它解决了什么痛点？

如果我们用最笨的方法（暴力枚举）：

- 对于每一栋楼，都向左、向右挨个去数，直到碰到更高的楼为止。
    
- 如果所有楼都是降序排列的（比如 5, 4, 3, 2, 1），当你找最后一栋楼的左邻居时，你要数遍前面所有的楼。
    
- 在数据量达到 $7 \times 10^5$ 时，这种“挨个查”的方法会导致计算次数接近 $7 \times 10^5 \times 7 \times 10^5 \approx 5000$ 亿次，电脑会直接“罢工”（超时）。

**单调栈的痛点解决：** 它的效率极高，能保证每栋楼只进出“栈”一次，时间复杂度是 $O(N)$，处理这道题绰绰有余。

---

## 3. 核心思路：排队的小伙伴

我们用一个生活中的例子来理解**单调栈**：

想象楼房都在排队。我们维持一个**从大到小**的队伍（栈）。

1. **左边第一个更高：**
    
    - 当你（当前楼房）准备进入这个队伍时，如果你发现队伍末尾的人比你矮，那他永远挡不住你的视线，而且对于你后面的人来说，你比他更高、更近，所以他“没用了”，请他**出队**。
        
    - 直到你发现队伍末尾有一个人比你高，那么他就是你**左边第一个比你高**的人！
        
    - 如果你把前面的人都踢走了，队伍空了，说明你左边没人的高度能超过你，输出 -1。
        
2. **右边第一个更高：**
    
    - 逻辑是一样的，只不过我们要从右往左走一遍，或者在从左往右走的时候，当你把别人“踢出队”时，你就是那个人的**右边第一个更高**的人。

---

## 代码

```python
import sys
input = sys.stdin.read().split()
it = iter(input)

n = int(next(it))
h = [0] * (n + 1)
L = [-1] * (n + 1)
R = [-1] * (n + 1)
for i in range(1, n + 1):
    h[i] = int(next(it))

stack = []
for i in range(1, n + 1):
    while stack and h[stack[-1]] <= h[i]:
        stack.pop()
    if stack:
        L[i] = stack[-1]
    stack.append(i)

stack = []
for i in range(n, 0, -1):
    while stack and h[stack[-1]] <= h[i]:
        stack.pop()
    if stack:
        R[i] = stack[-1]
    stack.append(i)

print(*(L[1:]))
print(*(R[1:]))
```
