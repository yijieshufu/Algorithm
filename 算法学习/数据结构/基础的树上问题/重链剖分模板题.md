
# 换根树链剖分 (蓝桥杯省赛/国赛压轴考点)

## 一、 题目背景与普适模板库

- **题目描述**：给定一棵 $n$ 个节点的树，支持五种操作：动态换根、路径加值、子树加值、路径求和查询、子树求和查询。
    
- **输入条件**：$1 \le n, m \le 10^5$，初始有 $a_i$ 权值与树结构，后续有 $m$ 次操作。
    
- **所需积木**：【树链剖分 HLD】+【线段树 Segment Tree】。
    
- **通用积木代码**（绝对普适，原封不动）：
    
```Python
# 【模板积木 1】树链剖分 (HLD)
# 功能：建立重链剖分，维护 dfs 序、深度、父节点、重儿子、链顶
def hld_init(n, g, root):
    sz = [0] * (n + 1)
    dep = [0] * (n + 1)
    fa = [0] * (n + 1)
    son = [0] * (n + 1)
    
    stack = [(root, 0, 1)]
    order = []
    while stack:
        u, p, d = stack.pop()
        fa[u] = p
        dep[u] = d
        sz[u] = 1
        order.append(u)
        for v in g[u]:
            if v != p:
                stack.append((v, u, d + 1))
    
    for u in reversed(order):
        if fa[u]:
            sz[fa[u]] += sz[u]
            if son[fa[u]] == 0 or sz[u] > sz[son[fa[u]]]:
                son[fa[u]] = u

    top = [0] * (n + 1)
    dfn = [0] * (n + 1)
    rnk = [0] * (n + 1)
    cur_dfn = 0
    
    stack = [(root, root)]
    while stack:
        u, t = stack.pop()
        cur_dfn += 1
        dfn[u] = cur_dfn
        rnk[cur_dfn] = u
        top[u] = t
        
        # 先压轻儿子，再压重儿子，保证重儿子先出栈（DFN 连续）
        for v in g[u]:
            if v != fa[u] and v != son[u]:
                stack.append((v, v))
        if son[u]:
            stack.append((son[u], t))
            
    return dfn, rnk, top, fa, dep, sz

# 【模板积木 2】线段树 (Lazy Tag)
# 功能：区间加、区间求和
def build(node, l, r, rnk, a, tree):
    if l == r:
        tree[node] = a[rnk[l]]
        return
    mid = (l + r) // 2
    build(node * 2, l, mid, rnk, a, tree)
    build(node * 2 + 1, mid + 1, r, rnk, a, tree)
    tree[node] = tree[node * 2] + tree[node * 2 + 1]

def push_down(node, l, r, tree, lazy):
    if lazy[node] != 0:
        mid = (l + r) // 2
        lazy[node * 2] += lazy[node]
        tree[node * 2] += lazy[node] * (mid - l + 1)
        lazy[node * 2 + 1] += lazy[node]
        tree[node * 2 + 1] += lazy[node] * (r - mid)
        lazy[node] = 0

def update(node, l, r, ql, qr, val, tree, lazy):
    if ql <= l and r <= qr:
        tree[node] += val * (r - l + 1)
        lazy[node] += val
        return
    push_down(node, l, r, tree, lazy)
    mid = (l + r) // 2
    if ql <= mid: update(node * 2, l, mid, ql, qr, val, tree, lazy)
    if qr > mid: update(node * 2 + 1, mid + 1, r, ql, qr, val, tree, lazy)
    tree[node] = tree[node * 2] + tree[node * 2 + 1]

def query(node, l, r, ql, qr, tree, lazy):
    if ql <= l and r <= qr:
        return tree[node]
    push_down(node, l, r, tree, lazy)
    mid = (l + r) // 2
    res = 0
    if ql <= mid: res += query(node * 2, l, mid, ql, qr, tree, lazy)
    if qr > mid: res += query(node * 2 + 1, mid + 1, r, ql, qr, tree, lazy)
    return res
```

## 二、 积木说明书与拼装逻辑 (核心思路)

- **积木说明书**：
    
    - `hld_init`：入参为树的邻接表 `g` 和初始根。出参 `dfn` 记录时间戳，`rnk` 记录时间戳对应的节点，`top` 用于链上 $O(\log n)$ 跳跃。
        
    - `线段树系列`：标准带 Lazy 标记的线段树，所有操作必须显式传入 `tree` 和 `lazy` 数组，保证函数的纯净。
        
- **决策依据**：题目虽然动态换根，但**“任意两点的简单路径”在树的形态变化中是绝对不变的**。因此，底层的 HLD 始终以 1 为根进行静态构建即可。动态换根只影响**子树的界定范围**。
    
- **拼装与适配过程**：
    
    - 在 `solve()` 中，我们将换根操作仅仅记录为一个变量 `cur_root`。
        
    - **路径操作**：无视 `cur_root`，直接套用 HLD 的 LCA 跳跃逻辑。
        
    - **子树操作**：进行几何关系的转化（见下方代码注释），将动态根的情况转化为对静态 DFS 序数组的一个或两个区间的操作。
        
- **复杂度**：建树 $O(n)$，每次查询/修改 $O(\log^2 n)$，总时间复杂度 $O(m \log^2 n)$，空间复杂度 $O(n)$。

## 三、 我的盲点记录

1. **关于纯净函数的严格性**：线段树的 `tree` 和 `lazy` 必须作为参数传入，这不仅符合我们的搭积木规范，也能避免在复杂题目中多棵线段树导致的全局变量污染。
    
2. **不依靠 `son` 数组找特定儿子的绝妙技巧**：
    
    当我们需要找 $u$ 向着 $rt$ 方向的儿子 $v$ 时：利用 `top` 向上跳，如果最后停在与 $u$ 相同的重链上（即 `top[curr] == top[u]`），由于重链上的 DFS 序是连续的，且 $v$ 是 $u$ 的儿子，那么 $v$ 的 DFS 序**必然就是** `dfn[u] + 1`。直接取 `rnk[dfn[u] + 1]` 即可！完全不需要在模板外强行获取 `son` 数组。
