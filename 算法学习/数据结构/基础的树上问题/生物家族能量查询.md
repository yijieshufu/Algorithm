## 一、 题目背景与模板库

- **题目描述**：给定一棵 $n$ 个节点的树，每个节点有能量值。查询 $q$ 次，每次求某个节点及其所有子孙节点的能量总和。
    
- **输入条件**：$n, q$ 规模较大；给出直系祖先关系 $u \to v$；需要寻找根节点。
    
- **所需积木**：【邻接表建树】、【DFS 预处理子树信息】。
    

### 通用积木代码
```Python

import sys

# 【模板积木 1】邻接表建树 (带入度统计找根)
# n: 节点数, edges: [(u, v), ...]
def build_tree(n, edges):
    g = [[] for _ in range(n + 1)]
    deg = [0] * (n + 1)
    for u, v in edges:
        g[u].append(v)
        deg[v] += 1
    
    root = 1
    for i in range(1, n + 1):
        if deg[i] == 0:
            root = i
            break
    return g, root

# 【模板积木 2】DFS 预处理子树信息 (自底向上规约)
# u: 当前节点, g: 邻接表, val: 原始权值, res: 存储结果
def dfs_subtree(u, g, val, res):
    res[u] = val[u]
    for v in g[u]:
        dfs_subtree(v, g, val, res)
        # 核心：回溯时累加
        res[u] += res[v]
```

## 二、 积木说明书与拼装逻辑

- **积木说明书**：
    
    - `build_tree`: 入参是边集，出参是邻接表 `g` 和 树根 `root`。通过 `deg` 数组记录入度，**入度为 0 的点即为整棵树的祖宗**。
        
    - `dfs_subtree`: 采用深度优先遍历。入参是当前节点、树结构、节点权值。它维护的是 `res` 数组，执行完后 `res[i]` 即为以 $i$ 为根的子树总和。
        
- **决策依据**：看到“树上子树查询”且为“静态统计”（不涉及动态修改点值），最稳妥高效的方法就是 **DFS 一次性预处理**。
    
- **拼装过程**：
    
    1. 使用 `it` 读取所有数据。
        
    2. 调用 `build_tree` 建立父子指向关系。
        
    3. 调用 `dfs_subtree(root, ...)` 从根节点开始递归，利用回溯阶段将子孙的能量值“逐级上报”给祖先。
        
    4. 最终 `res` 数组就成了一个 $O(1)$ 的查询表。
        
- **复杂度**：时间复杂度 $O(n + q)$，空间复杂度 $O(n)$。

## 三、 我的盲点记录

1. **递归限制**：Python 处理深树必须手动开启 `sys.setrecursionlimit`，否则在 $n > 1000$ 时会崩溃。
    
2. **根节点识别**：题目未指定 1 号为根，必须通过入度判断。
    
3. **回溯逻辑**：`res[u] += res[v]` 必须写在 `dfs_subtree(v)` 之后，确保子节点已经算完了自己的总和再报给父节点。
