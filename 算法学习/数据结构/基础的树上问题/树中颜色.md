
## 一、 题目背景与模板库

- **题目描述**：给定一棵 $n$ 个节点的有根树（根为 1），每个节点带有颜色 $c_i$。求 $q$ 次询问中，以指定节点 $x$ 为根的子树内有多少种不同的颜色。
    
- **输入条件**：$1 \le n, q \le 10^5$，颜色种类极少（$1 \le c_i \le 100$）。
    
- **所需积木**：【树的建图（解耦版）】、【树形 DP 框架】、【Bitmask 集合统计】。

### 通用积木代码

```Python
# 【模板积木 1】树的建图 (解耦版)
# 入参：节点数 n，边列表 edges；出参：邻接表 g
def build_tree(n, edges):
    g = [[] for _ in range(n + 1)] # -> 强制 1-indexed
    for u, v in edges:
        g[u].append(v)
        g[v].append(u)
    return g

# 【模板积木 2】位计数 (统计集合大小)
def count_bits(mask):
    return bin(mask).count('1')
```

## 二、 积木说明书与拼装逻辑 (核心思路)

- **积木说明书**：
    
    - `build_tree`：负责将原始边数据转化为邻接表。
        
    - `dfs(u, p)`：入参为当前节点和父节点，通过递归回溯（后序遍历）汇总子树信息。
        
- **决策依据**：
    
    - **看到“子树查询”**：确定使用树形 DP 框架。
        
    - **看到“不同种类数量”且“种类范围极小（100）”**：放弃 `set`，改用位运算。因为 Python 的 `int` 是高精度的，`1 << 100` 可以完美充当一个高效的“位集合”。
        
- **拼装过程**：
    
    1. 使用极简 I/O 模板读取数据。
        
    2. 预收集所有边，调用 `build_tree` 积木。
        
    3. DFS 过程中，父节点的掩码等于“自身颜色掩码”与“所有子节点返回掩码”的**按位或（OR）**运算。
        
    4. 最后用 `count_bits` 积木解析结果。
        
- **复杂度**：时间复杂度 $O(n + q)$；空间复杂度 $O(n)$。

## 三、 我的盲点记录

- **建图习惯适配**：在 `solve()` 中需要先遍历 $n-1$ 次读取边，存入 `edges` 列表，再统一喂给建图积木。
    
- **递归深度红线**：由于 $n = 10^5$，必须手动通过 `sys.setrecursionlimit(200000)` 扩容，否则会引发 `RecursionError`。
    
- **高精度优势**：利用 Python 整数不限位的特性处理 100 位掩码，比在 C++ 中开 `std::bitset` 或手动分段处理要简洁得多。
