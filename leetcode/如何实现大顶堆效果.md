在 Python 中实现“大顶堆”主要有三种方式。由于 Python 官方的 `heapq` 库只实现了小顶堆，开发者通常会采用一些“机灵”的变通方法。

---

### 方法一：取负号（最常用、最推荐）

这是你在刚才的代码中看到的方法。逻辑非常简单：**如果你想让最大的数排在最前面，就把所有的数都乘以 $-1$。**

- **原理：** 如果 $5 < 10$，那么 $-5 > -10$。在小顶堆里，$-10$ 会排在 $-5$ 前面。
    
- **操作：** 存进去的时候取负，取出来的时候再取负回来。

```Python
import heapq

max_heap = []

# 1. 插入元素
nums = [1, 5, 2, 10]
for n in nums:
    heapq.heappush(max_heap, -n)  # 存入负数

# 此时 max_heap 内部结构（逻辑上）是 [-10, -5, -2, -1]

# 2. 查看堆顶（最大值）
print(-max_heap[0])  # 输出 10

# 3. 弹出堆顶
top = -heapq.heappop(max_heap)
print(top)  # 输出 10
```

---

### 方法二：包装对象（进阶版）

如果你处理的不是简单的数字，而是复杂的对象，或者你觉得取负号太琐碎，可以写一个包装类，**重写比较运算符**。

```Python
class MaxHeapItem:
    def __init__(self, val):
        self.val = val

    # 重写 "小于" 运算，让它在逻辑上变成 "大于"
    def __lt__(self, other):
        return self.val > other.val

# 使用方法
heap = []
heapq.heappush(heap, MaxHeapItem(10))
heapq.heappush(heap, MaxHeapItem(20))

print(heapq.heappop(heap).val) # 输出 20
```

---

### 方法三：使用内部函数 `_heapify_max`

其实 Python 的 `heapq` 库内部是写了大顶堆算法的，只是没有以公开接口的形式暴露（名字前面有下划线）。它可以一次性把列表变成大顶堆，但不支持后续的 `push` 操作。

```Python
import heapq

nums = [1, 5, 2, 10]
heapq._heapify_max(nums)  # 注意有下划线

print(nums[0])  # 输出 10
# 注意：没有 heapq._heappush_max 这种现成函数，所以这种方法灵活性较差
```

---

### 为什么官方不直接给一个 `MaxHeap`？

这是一个很有意思的设计哲学问题。Python 的开发者认为：

1. **保持简洁：** 只提供一套算法，通过取负号就能解决 99% 的需求。
    
2. **性能：** 为了多一套大顶堆逻辑而增加代码量不符合 Python 的风格。

### 总结

在算法面试或日常开发中，**方法一（取负号）** 是绝对的主流。它简单、高效、不需要额外的类定义。

> **小贴士：** 当你计算中位数时，那个 `(A[0] - B[0]) / 2.0` 里的**减号**，其实就是因为 `B` 里面存的是负数。$A[0] - (-B[0]) = A[0] + B[0]$。

既然你已经掌握了堆的实现，想不想试试用这个“双堆法”去解决另一道经典题：**“前 K 个高频元素”**？
