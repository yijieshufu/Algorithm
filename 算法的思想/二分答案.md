最核心的直觉是：**把一个“求最优解”的问题，转化成一系列“做判断题”的问题。**

通常我们写算法是想直接推导出答案，但二分答案反其道而行之——**我先猜一个答案，然后问电脑：“这个答案行不行？”**

---

## 1. 核心逻辑：从“直接求”到“做判断”

假设你在玩“猜价格”游戏。一个商品的价格在 $1$ 到 $1000$ 元之间。

- **常规思路（直接求）：** 盯着商品看半天，试图通过材质、品牌计算出它到底是 $567$ 元。这很难。
    
- **二分答案（做判断）：** 1. 你问：“价格比 $500$ 高吗？” 对方说：“高。”（排除 $1-500$）
    
    2. 你问：“价格比 $750$ 高吗？” 对方说：“低。”（排除 $751-1000$）
    
    3. 通过不断缩小范围，你最终锁定了 $567$。
    

在这个过程中，你并不需要知道怎么计算价格，你只需要一个 **“裁判”** 告诉你：**当前的猜测是“高了”还是“低了”。**

---

## 2. 必须满足的前提：单调性 (Monotonicity)

不是所有题目都能二分，只有满足**单调性**的题目才行。

> **单调性定义：** > 如果答案 $X$ 成立，那么比 $X$ 更“差”的答案（更小的最小值或更大的最大值）一定也成立；
> 
> 如果答案 $X$ 不成立，那么比 $X$ 更“优”的答案一定也不成立。

---

## 3. 二分答案的三大支柱

要写出一个二分答案的程序，你需要准备好这三个部分：

### ① 确定搜索范围 $[L, R]$

答案最小可能是多少？最大可能是多少？
### ② 编写 `check(mid)` 函数（核心）

这是你的“裁判”。它的任务是：**验证答案 `mid` 是否可行。**

- **注意：** 这个函数不需要找最优解，只需要给一个布尔值（True/False）。

### ③ 循环缩减区间
```Python
while L <= R:
    mid = (L + R) // 2
    if check(mid):
        ans = mid    # 记录当前最好的合法答案
        L = mid + 1  # 试图找一个更大的答案
    else:
        R = mid - 1  # mid 太大了，达不到，往小了找
```

---

# 代码
```Python
def binary_search(l, r, check):
    ans = l
    while l <= r:
        mid = (l + r) // 2
        if check(mid):
            ans, l = mid, mid + 1
        else:
            r = mid - 1
    return ans
```
