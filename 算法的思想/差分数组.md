
### 1. 核心直觉：从“点”到“面”的转化

假设你有一排树，你想让第 1 棵到第 3 棵树都长高 5 米。

- **常规做法（暴力）：** 你跑到第 1 棵树那加 5，跑到第 2 棵那加 5，跑到第 3 棵那加 5。如果区间很长，你会累死。
    
- **差分做法（下达指令）：**
    
    1. 你在第 1 棵树那插个牌子说：**“从这里开始，后面所有树都加 5 米！”**
        
    2. 你在第 4 棵树那插个牌子说：**“从这里开始，之前的加 5 米效果结束了，减去 5 米！”**
        

当你从左往右走时，只要手里拿着这些“指令”，就能实时算出当前树的高度。

---

### 2. 差分数组的数学原理

设原数组为 $h$，差分数组为 $d$。

- $d[i]$ 的定义是：第 $i$ 棵树与第 $i-1$ 棵树的高度差。
    
- 如果你想在区间 $[L, R]$ 整体加上 $v$：
    
    - 只需要让 $d[L] = d[L] + v$ （开始增益）
        
    - 让 $d[R+1] = d[R+1] - v$ （结束增益）
        

**如何还原高度？**

当前树受到的总加成 = **前面所有差分值的累加和**（前缀和）。

### 3. 结合你的代码深入理解

在你写的 `check(mid)` 函数中，差分数组的应用非常巧妙：

```Python
diff = [0] * (n + 2)  # 差分数组，记录“指令”
total_days = cur_add = 0
for i in range(1, n + 1):
    cur_add += diff[i] # 【关键】累加指令，得到当前位置 i 的实际魔法增量
    now_h = h[i] + cur_add # 初始高度 + 魔法累积 = 当前实际高度
    if now_h < mid:
        need = mid - now_h # 还差多少达到目标
        total_days += need
        # ... 略过判断 ...
        
        cur_add += need    # 【关键】因为从这里开始施法，所以当前位置立刻获得 need
        if i + k <= n:
            diff[i + k] -= need # 【核心】在 i+k 位置埋下“减法”指令，因为魔法范围只有 k
```

#### 为什么 `cur_add` 也要加 `need`？

在标准的差分应用中，我们通常是先填满 `diff` 数组，最后再统一求前缀和。

但在你的这道题里，我们是 **“边走边填”**。

当你发现第 $i$ 棵树不够高并施放魔法时，它产生的增益不仅影响 $i+1$ 往后的树，**它对当前的第 $i$ 棵树也立竿见影**。所以 `cur_add` 既要加上 `diff[i]`（之前埋下的指令），也要加上 `need`（刚刚下达的指令）。

`diff[i]` 负责记录 **“将来”** 的变化，而 `cur_add` 负责管理 **“现在”** 的总量

---

### 4. 总结：差分数组的威力

|**维度**|**普通数组修改**|**差分数组修改**|
|---|---|---|
|**单次修改复杂度**|$O(K)$（需要循环 $K$ 次）|$O(1)$（只需要改两处）|
|**查询当前值复杂度**|$O(1)$（直接读）|$O(1)$（通过 `cur_add` 随扫描随得）|
|**适用场景**|频繁查询，极少修改|**频繁区间修改，一次性或顺序查询**|

**一句话总结：**

差分数组就是 **“打标记”**。在起点打一个“开始加”的标记，在终点后面打一个“停止加”的标记。遍历时只要带着这些标记走，就能瞬时知道任何位置的状态。
