这段代码的核心逻辑在于使用**贡献法**（Contribution Method）来计算逆序对的总数。

在 $n!$ 个全排列中，我们不直接去数每个排列有多少逆序对，而是计算**每一对下标 $(i, j)$** 在所有排列中总共贡献了多少个逆序对。

---

### 1. 计算贡献系数 `w` 和 `g` (权重预处理)

这两行代码本质上是在处理“期望”和“概率”的分子部分。

- **`g` (非质数对权重)**：计算的是 $\frac{n!}{2} \pmod p$。
    
    - **原理**：在所有排列中，对于任意两个位置，前面的数大于后面的数的概率是 $50\%$。所以每一对“自由”的位置在 $n!$ 个排列中，贡献了 $\frac{n!}{2}$ 个逆序对。
        
    - **技巧**：代码通过 `if i != 2` 跳过数字 2，直接实现了除以 2 的模运算，避免了计算逆元。
        
- **`w` (混合对权重)**：计算的是 $\frac{n!}{s+1} \pmod p$。
    
    - **原理**：当一个非质数位置的数 $x$ 与 $s$ 个已经排好序的质数位置的数进行比较时，$x$ 在这 $s+1$ 个数中的排名有 $s+1$ 种等概率的情况。
        
    - **技巧**：同理，通过 `if i != s+1` 跳过该项，实现了除以 $(s+1)$。
        

---

### 2. 计算“非质数 vs 非质数”的贡献



```Python
ans = (ans + t * (t - 1) // 2 % p * g) % p
```

- **逻辑**：非质数位置的数在排序过程中**完全不动**。
    
- **计算**：
    
    1. 从 $t$ 个非质数下标中任选两个，组合数为 $\binom{t}{2} = \frac{t(t-1)}{2}$。
        
    2. 每一对在所有排列中贡献 $\frac{n!}{2}$（即变量 `g`）个逆序对。
        
    3. **总贡献** = $\text{组合数} \times g$。
        

---

### 3. 计算“非质数 vs 质数”的贡献 (倒序遍历)

这是代码最难理解但也最精妙的部分。

#### 变量含义：

- **`A`**：当前下标 $i$ **左边**的质数个数。
    
- **`B`**：当前下标 $i$ **右边**的质数个数。
    

#### 核心逻辑：

当我们遍历到一个**非质数下标**时，我们需要知道它和所有质数位置产生的逆序对总和。

假设质数位置上的数已经排好序为 $v_1 < v_2 < \dots < v_s$。非质数位置的数 $x$ 插入其中，有 $s+1$ 种排名可能：

1. **左侧贡献**：如果 $x$ 插入后，左边有 $A$ 个质数位置。若 $x$ 小于其中 $k$ 个质数，则产生 $k$ 个逆序对。在所有 $s+1$ 种排名中，左侧产生的逆序对总数刚好是等差数列 $0 + 1 + 2 + \dots + A = \frac{A(A+1)}{2}$。
    
2. **右侧贡献**：同理，如果右边有 $B$ 个质数位置，产生的逆序对总数是 $0 + 1 + 2 + \dots + B = \frac{B(B+1)}{2}$。
    

#### 代码实现：



```Python
# A*(A+1)/2 是该点与左侧质数产生的逆序对总权重
# B*(B+1)/2 是该点与右侧质数产生的逆序对总权重
contribution = (A * (A + 1) // 2 + B * (B + 1) // 2) % p
ans = (ans + contribution * w) % p
```

这里乘以 `w`（即 $\frac{n!}{s+1}$），就把这 $s+1$ 种情况的“总和”转化为了在 $n!$ 个排列下的**总贡献**。

---

### 总结：为什么质数 vs 质数不计算？

因为题目定义“质数排序”后，质数下标位置的数是**升序**的。对于任何两个质数下标 $i < j$，永远满足 $a_i < a_j$，**逆序对贡献永远为 0**。

**这份代码通过三个步骤完成了计算：**

1. **非质数 vs 非质数**：直接组合计算。
    
2. **非质数 vs 质数**：利用等差数列和 $w$ 系数动态计算。
    
3. **质数 vs 质数**：无贡献，不计算。
