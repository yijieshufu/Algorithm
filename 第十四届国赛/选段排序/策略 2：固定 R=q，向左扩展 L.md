```python
 # === 策略 2：固定 R=q，向左扩展 L ===
    # 重新初始化用于策略2的数据
    # cur_max: 记录当前扫描范围内的最大值 (也就是排序后 A[q] 的值)
    # s[0]: 小根堆堆顶，存的是这 k 个最大数里最小的那个
    #       也就是排序后 A[p] 的值 (第 k 大)

    # 需要重新填充 s 堆，因为上面的逻辑可能没动 s，但为了保险起见还是重置一下逻辑清晰
    # 其实 s 在初始化阶段已经填好了 [p, q] 的数，但我们需要从 p-1 往左走
    # 为了逻辑复用，我们这里直接使用 s，此时 s 中是从 p 到 q 的数
    
    # 此时 s 里面包含 [p, q] 的所有数。
    # cur_max 初始化为 [p, q] 中的最大值。
    # -g[0] 在上面循环前是初始区间的最大值? 不对，g 已经被修改了。
    # 让我们用稳妥的方式获取当前 [p, q] 的最大值：
    # 初始化时 g 存负数，但 g 已经被上面的循环改乱了。
    # 所以我们手动算一下初始的 cur_max
    cur_max = -1
    s = [] # 清空 s 重新来，防止脏数据
    for i in range(p, q + 1):
        heapq.heappush(s, a[i])
        cur_max = max(cur_max, a[i])
    
    # 计算初始状态（也就是只排 [p, q]）的一遍，虽然上面算过，但取 max 不影响
    ans = max(ans, cur_max - s[0])

    # 开始向左遍历 L，从 p-1 到 1
    for i in range(p - 1, 0, -1):
        val = a[i]
        
        # 1. 更新最大值 (A[q] 现在的角色)
        cur_max = max(cur_max, val)
        
        # 2. 更新堆，保持堆中只有 k 个最大的元素
        # 加入新元素
        heapq.heappush(s, val)
        # 弹出堆中最小的元素，留下的就是比较大的数
        heapq.heappop(s)
        
        # 3. 计算当前收益：(最大值) - (第 k 大)
        # s[0] 就是当前第 k 大的数 (也就是 A[p] 的位置)
        ans = max(ans, cur_max - s[0])
```

这部分代码是策略 1 的**完全镜像**（反向操作）。

如果说策略 1 是“往右探险，寻找第 $K$ 小”，那么策略 2 就是**“往左回溯，寻找第 $K$ 大”**。

我们需要先理清这个逻辑转换：

- **固定右端点 $R=q$**：意味着排序后的区间 $[L, q]$ 里，最右边的元素 $A_q$ 一定是该区间的**最大值**。
    
- **寻找 $A_p$**：因为 $p$ 和 $q$ 的距离固定，在排序后的区间里，$A_p$ 将会位于倒数第 $K$ 个位置（即**第 $K$ 大**的值），其中 $K = q - p + 1$。

---

### 1. 角色大反转

我们需要维护两个核心变量，它们角色的变化如下：

| **变量**   | **策略 1 (向右)**             | **策略 2 (向左)**                 |
| -------- | ------------------------- | ----------------------------- |
| **基准值**  | `cur_min` (地板，最小值)        | **`cur_max` (天花板，最大值)**       |
| **目标值**  | 第 $K$ 小 (用大根堆维护)          | **第 $K$ 大 (用小根堆维护)**          |
| **物理意义** | $A_p$ 是最小值，$A_q$ 是第 $K$ 小 | **$A_q$ 是最大值，$A_p$ 是第 $K$ 大** |

---

### 2. 为什么用小根堆 (`s`) 维护“第 K 大”？

这是一个经典的算法思维：

- 我们要保留“全场**最大**的 $K$ 个人”。
    
- 为了判断新来的人能不能进这个“精英俱乐部”，我们需要知道俱乐部里**最弱**的那个人是谁（如果新来的比最弱的强，就踢掉最弱的，换新来的）。
    
- **小根堆**的堆顶恰好就是堆里的“最小值”。
    
- 所以：**维护 Top K 大元素，要用大小为 K 的小根堆。** 堆顶就是这 $K$ 个大佬里的“守门员”（也就是第 $K$ 大）。

### 3. 代码逐行剧场式解读

假设 $p=3, q=5$，所以 $K=3$。

数组片段：`[... 100, 10, 5, 8 ...]`

初始区间 $[p, q]$ 是 `[10, 5, 8]`。

#### **第一幕：重置与初始化**

```Python
# 防止之前的逻辑污染变量，清空重来
cur_max = -1
s = [] 

# 把初始区间 [10, 5, 8] 放进去
for i in range(p, q + 1):
    heapq.heappush(s, a[i])
    cur_max = max(cur_max, a[i])
```

- **`s` 堆**：`[5, 8, 10]` (自动排序后)。堆顶 `s[0]` 是 **5**。
    
    - 这就是当前“最大的3个数”里的老幺（第3大）。
        
- **`cur_max`**：`10`。
    
- **当前 Ans**：`10 - 5 = 5`。

#### **第二幕：向左回溯 (i = p-1)**

现在我们向左扩展一个位置，新遇到的数字是 **`100`**。

当前区间变成了 `[100, 10, 5, 8]`。

**步骤 1：更新天花板 (cur_max)**

```Python
cur_max = max(cur_max, val)
# cur_max = max(10, 100) = 100
```

> **解读**：因为固定了右端点，排序后最右边一定是最大值。现在最大值变成了 100。这就是排序后 $A_q$ 的新值。

**步骤 2：更新精英俱乐部 (堆 s)**

我们要从 `[100, 10, 5, 8]` 里选出最大的 3 个。

Python

```python
heapq.heappush(s, val)
# 把 100 放进堆。堆现在有：5, 8, 10, 100 (4个数)
```

Python

```python
heapq.heappop(s)
# 踢掉堆里最小的一个。
# 5, 8, 10, 100 里面，5 最小。
# 踢掉 5！
```

> **解读**：`5` 太弱了，有了 `100` 的加入，`5` 已经被挤出了“前三名”。
> 
> **现在的堆**：`[8, 10, 100]` (堆顶 `s[0]` 变成了 **8**)。
> 
> **物理意义**：8 就是当前区间 `[100, 10, 5, 8]` 里的**第 3 大**。

**步骤 3：计算收益**

Python

```python
ans = max(ans, cur_max - s[0])
# 算式：100 - 8 = 92
```

> **收益大增！** 之前的收益是 5，现在变成了 92。
> 
> 为什么？因为我们引入了一个巨大的数 100，它拉高了 $A_q$ (最大值)，同时虽然 $A_p$ (第K大) 也变大了，但涨幅没最大值那么猛。

### 总结图示

| **步骤** | **动作**    | **数据池**           | **cur_max (Aq​)** | **s 堆 (Top K)**                          | **s[0] (Ap​)** | **收益**   |
| ------ | --------- | ----------------- | ----------------- | ---------------------------------------- | -------------- | -------- |
| **初始** | 仅 `[p,q]` | `[10, 5, 8]`      | 10                | `[5, 10, 8]`                             | 5              | 10-5=5   |
| **向左** | 遇到 `100`  | `[100, 10, 5, 8]` | **100**           | `[8, 10, 100]`<br><br>  <br><br>_(5被踢走)_ | **8**          | 100-8=92 |

这就是这段代码的全部奥秘：**一边向左吞噬数字，一边不断更新“当前范围最大值”和“当前范围第 K 大值”。**
