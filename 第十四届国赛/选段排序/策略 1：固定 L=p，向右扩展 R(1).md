```python
# === 策略 1：固定 L=p，向右扩展 R ===
    # 当前范围 [p, q]。
    # cur_min: 记录当前扫描过的范围内的最小值 (也就是排序后 A[p] 的值)
    # g[0]: 堆顶存的是这 k 个数里“负得最多”的，取反后就是这 k 个最小数里最大的那个
    #       也就是排序后 A[q] 的值 (第 k 小)
    
    # 初始化 cur_min 为 [p, q] 中的最小值。
    # 注意：s 堆里全是 [p, q] 的数，所以 s[0] 此时就是 [p, q] 的最小值
    cur_min = s[0] 
    
    # 初始答案：只排序 [p, q] 时的结果
    # -g[0] 是第 k 小的值 (A[q])， cur_min 是最小值 (A[p])
    ans = -g[0] - cur_min

    # 开始向右遍历 R，从 q+1 到 n
    for i in range(q + 1, n + 1):
        val = a[i]
        
        # 1. 更新最小值 (A[p] 现在的角色)
        cur_min = min(cur_min, val)
        
        # 2. 更新堆，保持堆中只有 k 个最小的元素
        # 我们把新元素加入大根堆
        heapq.heappush(g, -val)
        # 弹出堆中“最大”的元素（负得最少的），保证堆大小为 k，且留下的都是比较小的数
        heapq.heappop(g)
        
        # 3. 计算当前收益：(第 k 小) - (最小值)
        # -g[0] 就是当前第 k 小的数
        ans = max(ans, -g[0] - cur_min)
```


这段代码的核心在于 **“动态维护区间内的第 $K$ 小值”**。

为了让你彻底理解，我们把代码逻辑拆解成通俗的“场景剧”。

### 1. 场景设定

- **题目要求**：在一个变动的区间 $[p, R]$ 里排序。
    
- **你的目标**：计算 `排序后的 A[q]` - `排序后的 A[p]`。
    
- **物理意义**：
    
    - `排序后的 A[p]`：因为 $p$ 是区间的起点，排序后它一定是**整个区间的最小值**。
        
    - `排序后的 A[q]`：因为 $p$ 和 $q$ 的相对距离不变（差值为 $q-p$），所以它永远是这个区间里的**第 $K$ 小的数**（其中 $K = q - p + 1$）。
        

所以，这段代码的任务就是：**随着区间向右扩大，实时找出“最小值”和“第 $K$ 小的值”。**

---

### 2. 关键变量的角色

#### `cur_min` (地板)

- **角色**：它很简单，就是记录“我见过的所有数里最小的那个”。
    
- **对应**：排序后的 $A_p$。
    

#### `g` (VIP 俱乐部)

- **角色**：这是一个**容量固定为 $K$ 的大根堆**（虽然 Python 用负数模拟，逻辑上是大根堆）。
    
- **存的是谁**：它只存当前区间内**最小的 $K$ 个数**。
    
- **守门员 (`-g[0]`)**：堆顶元素是这 $K$ 个数里**最大的那个**。
    
    - 既然这 $K$ 个数是全场最小的 $K$ 个，那么其中最大的那个，自然就是全场的**第 $K$ 小**。
        
    - **对应**：排序后的 $A_q$。
        

---

### 3. 代码逐行图解

假设 $p=1, q=3$，所以 $K=3$。

我们当前的数组片段是 `[10, 5, 8, 2, 6]`。

#### **第一阶段：初始化**

代码运行前，我们先处理了 $[p, q]$ 也就是 `[10, 5, 8]`。

- **`s` 堆** (不用管它，它是辅助别的策略的)。
    
- **`g` 堆** (存负数)：`[-10, -5, -8]`。
    
- **`cur_min`**：`5` (这是 `10, 5, 8` 里的最小值)。
    
- **`-g[0]`**：当前堆里负得最多的是 `-10`，取反是 `10`。
    
    - **验证**：`[5, 8, 10]` 里，第3小确实是 10。
        
    - **当前 Ans**：$10 - 5 = 5$。
        

#### **第二阶段：循环开始 (向右扩展)**

现在循环到了 `i`，新进来的数字是 **`2`**。区间变成了 `[10, 5, 8, 2]`。

**步骤 1：更新最小值**

```Python
cur_min = min(cur_min, val) 
# cur_min = min(5, 2) = 2
```

> **解读**：现在区间的最小值变成了 2。这就是排序后的 $A_p$。

**步骤 2：更新堆 (最精彩的一步)**

我们的目标是：保留 `[10, 5, 8, 2]` 中最小的 3 个数。

```Python
heapq.heappush(g, -val) 
# g 变成了 [-10, -8, -5, -2] (注意 Python 堆会自动调整顺序)
# 此时堆里有 4 个数，太多了！我们只要 K=3 个。
```


```Python
heapq.heappop(g)
# 弹出堆顶（最小的负数，也就是绝对值最大的正数）。
# 堆里现在是 -10, -8, -5, -2。最小的是 -10。
# 于是 -10 被踢出去了！
```

> **为什么踢走 -10 (也就是原值 10)？**
> 
> 因为我们只想要“最小的 3 个数”。
> 
> 现在的候选人是 `10, 8, 5, 2`。显然 `10` 太大了，它挤不进“最小前三名”的榜单，所以把它淘汰。
> 
> **结果**：`g` 堆里剩下 `[-8, -5, -2]`。
> 
> **堆顶 `-g[0]`**：现在是 `-8`，取反是 **8**。
> 
> **验证**：`[10, 5, 8, 2]` 排序后是 `2, 5, 8, 10`。第 3 小确实是 **8**。

**步骤 3：计算收益**


```Python
ans = max(ans, -g[0] - cur_min)
# 算式：8 - 2 = 6
# 之前的 ans 是 5，现在更新为 6。
```

---

#### **再走一步 (加深理解)**

下一个数字是 **`6`**。区间 `[10, 5, 8, 2, 6]`。

1. **更新 min**：`cur_min = min(2, 6) = 2`。 (没变)
    
2. **更新堆**：
    
    - `g` 现有 `[-8, -5, -2]`。
        
    - 推入 `-6`。堆变 `[-8, -6, -5, -2]`。
        
    - 弹出最小负数（最大原值）。这里 `-8` 是最小的负数，踢走！
        
    - `g` 剩下 `[-6, -5, -2]`。
        
    - **解读**：在 `8, 6, 5, 2` 里，`8` 确实是最大的，被淘汰。留下了 `6, 5, 2` 作为“最小的三位”。
        
3. **计算**：
    
    - 堆顶现在是 `-6` (原值 6)。
        
    - `ans = 6 - 2 = 4`。
        
    - 比之前的 6 小，不更新。
        

---

### 总结

这段代码的灵魂在于：

1. **固定左端点** $\rightarrow$ 最小值 `cur_min` 只会越来越小（或不变），容易维护。
    
2. **维护第 K 小** $\rightarrow$ 利用大小为 $K$ 的大根堆。
    
    - 每进来一个新数，就和堆里的“大哥”（最大值）比。
        
    - 如果新数比“大哥”小，就把“大哥”踢走，新数上位。
        
    - 如果新数比“大哥”大，新数直接被踢走（代码里是先 push 再 pop，效果一样）。
        
    - 最终留在堆里的，永远是当前扫描范围内**最小的 $K$ 个数**。
        

你只需要记住：**`g` 堆就是那个“最小K个数”的俱乐部，而 `cur_min` 是地板。我们算的就是“俱乐部里的最大值”和“地板”的差。**
