
## 一、 题目深度分析

这是一道典型的**约束排列计数问题**。由于 $N \le 15$，且要求的是“总方案数”，我们的解题思路通常遵循以下路径：

### 1. 为什么暴力全排列（DFS）不行？

如果直接搜索所有可能的排列，复杂度是 $O(N!)$。

- 当 $N=15$ 时，$15! \approx 1.3 \times 10^{12}$。
    
- 即便有约束条件剪枝，在最坏情况下（全是关系 `0`）搜索空间依然巨大，普通 DFS 极易超时。
    

### 2. 为什么选择状态压缩 DP？

观察发现，当我们决定下一个位置填什么数时，只需要知道两个信息：

1. **哪些数字已经被用过了**（决定了哪些数字还能选）。
    
2. **上一个数字是多少**（决定了下一个数字必须是、或不能是某个特定的值）。
    

这两个信息构成了“状态”。我们可以用一个二进制整数 `mask` 来表示已选数字的集合（第 $i$ 位为 1 表示数字 $i$ 已选），用 `last` 表示最后一个数字。这种方法将复杂度降低到 $O(N^2 \cdot 2^N)$：

- 状态数：$N \cdot 2^N = 15 \cdot 2^{15} \approx 4.9 \times 10^5$。
    
- 这个规模对于 Python 在 1 秒内的运行非常友好。
    

### 3. 关系逻辑转换

- **关系 1 (连续)**：`nums[i+1]` **必须等于** `nums[i] + 1`。
    
- **关系 0 (不连续)**：`nums[i+1]` **绝对不能等于** `nums[i] + 1`。
    

> **注意**：关系 0 并不代表数字必须变小，只要不是恰好大 1 即可（比如从 3 跳到 5，或者从 3 跳到 2）。

## 二、 代码详细注解

基于你提供的 Python 3.8 兼容代码，我为你添加了逐行的逻辑注解：

```Python
import sys, functools

# 提高递归深度。Python 默认限制较小，对于 N=15 的递归树，建议调高以防溢出。
sys.setrecursionlimit(2000)

# 使用 sys.stdin.read().split() 一次性读取所有空格/换行分隔的整数
# data[0] 是 N，data[1:] 是那 N-1 个关系
data = list(map(int, sys.stdin.read().split()))
if not data:
    sys.exit(0)

n, rels = data[0], data[1:]

# @functools.lru_cache(None) 是 Python 3.8 实现记忆化搜索的“神技”
# 它会自动存储 (mask, last) 对应的结果，避免重复计算相同的子问题
@functools.lru_cache(None)
def solve(mask, last):
    # 通过计算二进制中 1 的个数，确定当前已经填到了第几个位置
    # 例如：mask 为 1011 (二进制)，count('1') 为 3，说明正准备填第 4 个数字
    idx = bin(mask).count('1')
    
    # 递归终点：当已填数字个数等于 N 时，说明找到了一种合法排列，返回 1 种方案
    if idx == n: 
        return 1
    
    # 题目核心逻辑：
    # 下一个“禁忌/目标”数字是当前最后一个数字 + 1
    nxt_target = last + 1
    
    # 根据输入的关系数组 rels[idx-1] 确定当前可选数字的范围：
    # 如果关系是 1：候选者【只能】是 nxt_target
    # 如果关系是 0：候选者可以是 1~N 中除了 nxt_target 以外的【任何】数字
    if rels[idx-1] == 1:
        cands = [nxt_target]
    else:
        cands = [i for i in range(1, n + 1) if i != nxt_target]
    
    # 统计所有合法的分支方案数：
    # 1. 候选数字 v 必须在 1~N 范围内
    # 2. 候选数字 v 必须还没被使用过 (mask & (1 << v-1) == 0)
    res = 0
    for v in cands:
        if 1 <= v <= n and not (mask & (1 << v-1)):
            # 递归调用：更新 mask（标记 v 已用），并将 v 作为新的 last
            res += solve(mask | (1 << v-1), v)
    
    return res

# 程序的入口点：
# 因为第一个数字没有任何前置关系限制，所以我们需要尝试 1 到 N 每一个数字作为开头
# sum() 将以每个数字开头的合法排列方案总数相加
if n == 0:
    print(0)
elif n == 1:
    print(1)
else:
    # 初始状态：mask 标记当前数字已用，last 为当前数字
    print(sum(solve(1 << i, i + 1) for i in range(n)))
```

## 三、 关键执行逻辑示例

假设 $N=3$，关系为 `[0, 1]`（第一个不连续，第二个连续）：

1. **第一层尝试**：
    
    - 选 `1` 开头：`solve(mask=001, last=1)`。
        
    - 接下来关系是 `0`：不能选 `1+1=2`。剩下可选 `3`。
        
    - 进入 `solve(mask=101, last=3)`。
        
    - 接下来关系是 `1`：必须选 `3+1=4`。但 `4 > N`，此路不通。
        
2. **另一条路**：
    
    - 选 `2` 开头：`solve(mask=010, last=2)`。
        
    - 关系 `0`：不能选 `3`。剩下可选 `1`。
        
    - 进入 `solve(mask=011, last=1)`。
        
    - 关系 `1`：必须选 `1+1=2`。但 `2` 已在 `mask` 中，此路不通。
        
3. **最终成功路径**：
    
    - 选 `3` 开头 $\to$ 选 `1` (不连续) $\to$ 选 `2` (连续)。满足条件！
