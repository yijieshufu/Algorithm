```python
    # 2. 树状数组 (BIT) 初始化
    # 离散化坐标：包含 A[i] 以及右侧映射值 (x+1)//2 和 (x+2)//2
    coords = sorted(set(A) | set((x+1)//2 for x in A) | set((x+2)//2 for x in A))
    rank = {x: i + 1 for i, x in enumerate(coords)}
    m = len(coords)
    bit = [0] * (m + 1)
    
    def add(i, d): # BIT 更新
        while i <= m: bit[i] += d; i += i & -i
            
    def get_kth(k): # BIT 查第k小
        idx, s = 0, 0
        for i in range(m.bit_length(), -1, -1):
            nxt = idx + (1 << i)
            if nxt <= m and s + bit[nxt] < k:
                idx, s = nxt, s + bit[nxt]
        return coords[idx]

    # 初始状态：假设所有点都在右侧 R，每个点贡献两个权重1 (解决精度问题)
    for x in A:
        add(rank[(x + 1) // 2], 1)
        add(rank[(x + 2) // 2], 1)
```


这段代码是整个解法中**最精妙、技术含量最高**的部分。它的主要任务是解决当“天然中位数”不满足条件时，如何在 $O(\log N)$ 的时间内找到一个**最优的 $x$**。

这里涉及到了三个核心概念：**离散化**、**树状数组（BIT）求第 $k$ 小**、以及**处理小数的数学技巧**。

我们逐块来拆解：

---

### 1. 为什么需要这一大坨逻辑？

当我们要找一个数 $x$，使得它到一堆数的距离之和最小，这个 $x$ 就是**中位数**。

但在本题的“约束情况”下，问题变成了：

我们选定一个 $x$，它同时影响两部分的代价：

1. **左边部分**：$A_i$ 都要变成 $x$。（权重为 1）
    
2. **右边部分**：$A_j$ 都要变成 $2x - 1$。（权重为 1）
    

为了把问题统一，我们把右边的公式变形：

$$
|A_j - (2x - 1)| = |2x - (A_j + 1)| = 2 \cdot |x - \frac{A_j + 1}{2}|
$$

**转化后的结论**：

我们在找一个 $x$，使得它：

- 到**左边元素** $A_i$ 的距离权重是 **1**。
    
- 到**右边元素** $\frac{A_j + 1}{2}$ 的距离权重是 **2**。
    

这就是一个**加权中位数问题**。

---

### 2. 代码详解

#### A. [[离散化坐标]] (`coords`, `rank`)


```Python
coords = sorted(set(A) | set((x+1)//2 for x in A) | set((x+2)//2 for x in A))
rank = {x: i + 1 for i, x in enumerate(coords)} # enumerate 会同时给你 索引(i) 和 值(x)
```

**为什么要离散化？**

- 数组 $A$ 中的数字可能很大（比如 $10^9$），直接用这些数字做数组下标（`bit[1000000000]`）内存会爆炸。
    
- 但数字的**个数**只有 $N$ 个（$10^5$）。我们只关心它们的大小关系。
    
- 所以我们把所有可能用到的数值收集起来，排序，给它们编号：最小的叫 1 号，第二小的叫 2 号……
    

**为什么是这三个集合？**

- `set(A)`：左边元素直接用原值 $A_i$，所以需要这些坐标。
    
- `set((x+1)//2)` 和 `set((x+2)//2)`：这是为了处理右边元素转化后的 $\frac{A_j + 1}{2}$（详见下文的“分裂技巧”）。
    

#### B. [[树状数组 (BIT) 的基础功能]]

```Python
m = len(coords)
bit = [0] * (m + 1)

def add(i, d): ...      # 单点更新：给第 i 个位置加上 d
```

这是一个标准的树状数组，用来维护 **“权重的个数”**。

- `add(rank[v], 1)` 意味着：数值 $v$ 出现了一次（或者说权重增加了 1）。
    

#### C. 树状数组查第 $k$ 小 (`get_kth`)

```Python
def get_kth(k): 
    idx, s = 0, 0
    # bit_length() 返回二进制位数，类似于 log2(m)
    for i in range(m.bit_length(), -1, -1):
        nxt = idx + (1 << i)
        if nxt <= m and s + bit[nxt] < k:
            idx, s = nxt, s + bit[nxt]
    return coords[idx]
```

这是一个高级写法，叫 **“[[树状数组上的倍增]]（Binary Lifting）”**。

- **普通做法**：二分查找答案 $x$，然后查 BIT 问“小于等于 $x$ 的权重有多少？”，复杂度 $O(\log^2 N)$。
    
- **倍增做法**：直接在 BIT 内部跳跃。类似于“在这个累加数组中，刚好累加到 $k$ 的那个位置在哪里？”。复杂度 **$O(\log N)$**。
    
- **作用**：快速找到加权中位数。
    

---

### 3. 最难懂的“分裂技巧” (核心)



```Python
# 初始状态：假设所有点都在右侧 R
for x in A:
    add(rank[(x + 1) // 2], 1)
    add(rank[(x + 2) // 2], 1)
```

**问题**：右边元素的“虚拟坐标”是 $\frac{A_j + 1}{2}$，权重是 **2**。

举例：如果 $A_j = 4$，坐标就是 $\frac{5}{2} = 2.5$。

**难点**：树状数组不支持小数下标 $2.5$。

**解决方案：权重的拆分（Splitting）**

我们不把“权重2”加在 $2.5$ 上，而是：

- 在 **2** 的位置加权重 **1**。
    
- 在 **3** 的位置加权重 **1**。
    
- 平均位置正好是 $2.5$，总权重正好是 2。
    

**数学验证**：

- **偶数情况** $x=4$：目标 $2.5$。
    
    - 代码：`(4+1)//2 = 2`，`(4+2)//2 = 3`。在 2 和 3 各加 1。✅
        
- **奇数情况** $x=5$：目标 $3.0$。
    
    - 代码：`(5+1)//2 = 3`，`(5+2)//2 = 3`。在 3 加了两次 1（即权重2）。✅
        

这行代码完美解决了 **“浮点数坐标”** 和 **“权重为2”** 两个问题，将它们转化为了整数域上的操作。

---

### 4. 总结：这段代码的“剧本”

1. **准备地图**：把所有可能用到的坐标（包括 $A_i$ 和那些拆分后的半值）都找出来，压缩成 $1, 2, 3 \dots m$。
    
2. **初始布局**：刚开始，假设分割点 $i=0$，意味着所有数字都在**右边**。
    
    - 所以我们要把所有数字都按照“右侧逻辑”（拆分成两个半值，各加权重1）加入树状数组。
        
3. **寻找中点**：`get_kth` 就像一个天平。如果总权重是 $W$，它能瞬间告诉我们：哪个数值 $x$ 能把左右两边的权重平衡住（即 $W/2$ 处的值）。
    

**一句话概括**：

这段代码建立了一个**支持“加权中位数”快速查询的数据结构**，并用这就把“浮点数坐标”和“双倍权重”巧妙地转换成了整数操作。
