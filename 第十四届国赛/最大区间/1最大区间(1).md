这道题目是一个非常经典的算法题，通常被称为 **“直方图中的最大矩形” (Largest Rectangle in Histogram)**。

通过你提供的代码和截图，我们可以看出解决这个问题的核心思路是使用了**单调栈 (Monotonic Stack)**。

以下是对题目思路的详细分析以及对代码的逐行详细注解。

---

### 1. [[题目思路]]分析

#### 核心问题

我们需要找到一段区间 $[L, R]$，使得这段区间的长度乘以区间内的最小值最大化。

公式：$\text{Res} = (R - L + 1) \times \min(A_L, \dots, A_R)$

#### 暴力解法 (Time Out)

如果我们枚举所有的 $L$ 和 $R$，时间复杂度是 $O(n^2)$。给定数据规模 $n \le 3 \times 10^5$，这肯定会超时。

#### 优化思路：以“某个数”为中心

与其枚举区间，不如**枚举每一个高度（每一个数 $A_i$）**，假设这个 $A_i$ 就是包含它的区间里的**最小值**。

为了让结果最大，我们需要让这个 $A_i$ 向左和向右尽可能延伸，直到遇到一个**比它小**的数为止。

- **向左找**：找到第一个比 $A_i$ 小的数的下标，记为 `left_bound`。
    
- **向右找**：找到第一个比 $A_i$ 小的数的下标，记为 `right_bound`。
    
- **计算面积**：此时以 $A_i$ 为最小值的最大区间长度为 `(right_bound - 1) - (left_bound + 1) + 1`，即 `right_bound - left_bound - 1`。

#### 为什么使用单调栈？

我们需要快速找到每个数“左边第一个比它小”和“右边第一个比它小”的位置。**单调递增栈**可以完美解决这个问题：

1. 栈中存储元素的下标。
    
2. 保持栈中对应的数值单调递增。
    
3. **关键时刻（踢出元素时）**：
    
    - 当我们要把一个新元素 `x` 入栈时，如果 `x` 比栈顶元素对应的值小，说明**栈顶元素的“右边界”找到了**（就是当前的 `x`）。
        
    - 此时将栈顶元素弹出，记为 `t`。
        
    - 弹出后，新的栈顶元素就是 `t` 的 **“左边界”**（因为栈是单调增的，新栈顶离 `t` 最近且比 `t` 小）。
        
    - 此时我们就可以计算以 `f[t]` 为高度的矩形面积了。

### 2. 详细代码注解

这段代码写得非常简洁且高效。使用了哨兵技巧（前后加0）来简化边界判断。

```Python
import sys

# 1. 使用 sys.stdin.readline 加速输入，应对大数据量
input = sys.stdin.readline

def solve():
    # 读取 n (有时候 n 在逻辑中用不到，但必须读这一行让指针下移)
    try:
        line = input()
        if not line: return
        n = int(line.strip())
    except ValueError:
        return

    # 2. 核心数据读取 + 哨兵技巧
    # [0] + ... + [0]：首尾加0，避免判断栈空和处理最后残留元素，极其重要
    # split() 默认按空格分割，map(int, ...) 转为整数
    f = [0] + list(map(int, input().split())) + [0]

    res = 0
    st = [] # 单调栈，存下标

    # 3. 遍历每个元素
    # i 是当前下标 (充当右边界)，h 是当前高度
    for i, h in enumerate(f): # enumerate(f)把一个列表（List）里的“下标”和“数值”同时拿出来
        
        # 当当前高度 h 小于 栈顶高度时 -> 破坏了单调增 -> 结算时刻
        while st and h < f[st[-1]]:
            # 弹出栈顶，这个 t 就是我们要算面积的那根柱子（以此为高）
            t = st.pop()
            
            # 计算宽度
            # 右边界是 i (不包含)，左边界是新的栈顶 st[-1] (不包含)
            # 宽度公式 = 右下标 - 左下标 - 1
            width = i - st[-1] - 1
            
            # 更新最大面积
            res = max(res, width * f[t])
        
        # 进栈
        st.append(i)

    print(res)

if __name__ == '__main__':
    solve()
```

### 3. 图解示例 (Input: 1 1 3 3 1)

加上哨兵后数组为：`[0, 1, 1, 3, 3, 1, 0]`。下标从 0 到 6。

1. **Idx 0 (Val 0)**: 入栈。Stack: `[0]`
    
2. **Idx 1 (Val 1)**: `1 > 0`，入栈。Stack: `[0, 1]`
    
3. **Idx 2 (Val 1)**: `1` 不小于 `f[1](1)` (相等由逻辑决定，通常相等不弹或者弹，这里代码是 strictly `<` 才弹，所以相等继续入栈)。Stack: `[0, 1, 2]`
    
4. **Idx 3 (Val 3)**: `3 > 1`，入栈。Stack: `[0, 1, 2, 3]`
    
5. **Idx 4 (Val 3)**: `3` 不小于 `3`，入栈。Stack: `[0, 1, 2, 3, 4]`
    
6. **Idx 5 (Val 1)**: `1 < f[4](3)`。**触发 While 循环**！
    
    - **Pop 4**: 高度 3。左边界是 `st[-1]+1` 即 `3+1=4`。右边界是 `R-1` 即 `5-1=4`。宽度 `4-4+1=1`。面积 $1 \times 3 = 3$。Stack: `[0, 1, 2, 3]`
        
    - **Pop 3**: 此时 Top 是 3 (Val 3)。`1 < 3`。Pop 3。高度 3。左边界 `2+1=3`。右边界 `4`。宽度 `4-3+1=2`。面积 $2 \times 3 = \mathbf{6}$。(更新最大值)
        
    - **Pop 2**: 此时 Top 是 2 (Val 1)。`1 < 1` 为 False。停止 Pop。
        
    - **Push 5**: Stack: `[0, 1, 2, 5]`
        
7. **Idx 6 (Val 0)**: 遇到末尾哨兵，**强制清空栈**。
    
    - 会依次弹出 5, 2, 1... 计算所有可能的矩形。
        
    - 比如弹出 5 (Val 1): 左边界 `2+1=3`，右边界 `5`，宽度 `3`，面积 $3 \times 1 = 3$。
        
    - ...以此类推。

### 总结

这道题的核心在于理解：**对于任意一个柱子，如果以它为矩形的高度，那么矩形的宽度取决于它左右两边什么时候出现“比它矮”的柱子。** 单调栈是解决这类“左右边界”问题的最佳工具。
