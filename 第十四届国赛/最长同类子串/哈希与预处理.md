```python
# 2. 哈希与预处理
MOD, BASE = (1 << 61) - 1, 13131
limit = max(n, m) + 1
P = [1] * limit
for i in range(1, limit): P[i] = (P[i-1] * BASE) % MOD

def get_info(arr, L):
    lst, nxt = [-1] * L, [-1] * L
    pos = [-1] * 26
    for i in range(L):
        p = pos[arr[i]]
        if p != -1:
            lst[i] = p
            nxt[p] = i
        pos[arr[i]] = i
    return lst, nxt

s_lst, s_nxt = get_info(S, n)
t_lst, t_nxt = get_info(T, m)
```


这段代码是实现**同类串（Isomorphic Substrings）** 高效匹配的核心预处理部分。它的作用是为后续的**滚动哈希（Rolling Hash）** 提供数学基础和位置索引，使得我们能在 $O(1)$ 时间内更新窗口特征。

以下是代码逻辑的拆解说明：

### 1. 哈希常量与幂次预处理
```Python
MOD, BASE = (1 << 61) - 1, 13131
limit = max(n, m) + 1
P = [1] * limit
for i in range(1, limit): P[i] = (P[i-1] * BASE) % MOD
```

- **`MOD` (梅森素数)**：使用了 $2^{61}-1$。这是一个非常大的质数，在 Python 中处理速度快且冲突概率（Hash Collision）极低，能有效防止针对 $2^{64}$ 溢出的哈希攻击。
    
- **`BASE` (底数)**：多项式哈希的底数，选一个较大的质数（如 13131）可以增加随机性。
    
- **[[`P` 数组 (幂次表)]]**：预先计算 $BASE^0, BASE^1, \dots, BASE^N \pmod{MOD}$。
    
    - **作用**：在滚动哈希中，如果我们想从哈希值中删除窗口最左侧（高位）的字符，或者定位某个特定位置的贡献，需要直接利用 $BASE$ 的幂。
        

---

### 2. 相对位置索引预处理 (`get_info`)

这是处理“同类串”最巧妙的地方。同类串的本质不是字符是什么，而是**字符出现的模式**。

```Python
def get_info(arr, L):
    lst, nxt = [-1] * L, [-1] * L
    pos = [-1] * 26  # 记录 26 个小写字母最后出现的位置
    for i in range(L):
        p = pos[arr[i]]
        if p != -1:
            lst[i] = p    # lst[i]: 当前字符上一次出现的位置
            nxt[p] = i    # nxt[p]: p 位置字符下一次出现的位置
        pos[arr[i]] = i   # 更新该字符最后出现的位置为当前索引 i
    return lst, nxt
```

该函数为字符串中的每个位置生成了两个重要的“指南针”：

- **`lst[i]` (Last Appearance)**：
    
    - 它记录了位置 $i$ 的字符，**上一次**在字符串中出现是在哪个索引。
        
    - **用途**：用于判断在当前长度为 $k$ 的窗口内，位置 $i$ 的字符是否是“第一次”出现。如果 `i - lst[i] >= k`，说明上一次出现已经在窗口外了，此时它在窗口内被视为“首位字符”（特征值为 $0$）。
        
- **`nxt[p]` (Next Appearance)**：
    
    - 它记录了位置 $p$ 的字符，**下一次**会在哪个索引出现。
        
    - **用途**：**这是滚动哈希修正的关键**。当窗口向右滑动、左端点 $j$ 移出时，如果该字符在窗口内还有下一次出现（即 `nxt[j] < i`），那么这个 `nxt[j]` 位置的特征值会从原来的 `nxt[j] - j` 突变为 `0`（因为前驱没了，它变成了窗口内的老大）。我们需要利用 `nxt` 数组快速定位并减去这个差值贡献。
        

---

### 3. 核心逻辑总结

这段代码建立了一个**双向链表**结构的索引：

|**数组**|**记录内容**|**解决的问题**|
|---|---|---|
|**P**|$BASE$ 的幂次|实现 $O(1)$ 的哈希加减运算|
|**lst**|前驱位置|确定当前字符相对于窗口左边界的“身份”|
|**nxt**|后继位置|当窗口左移时，快速修正下一个相同字符的哈希贡献|

**形象比喻：**

如果字符串是 `a b a`，`get_info` 就像是给每个字母发了一个纸条。第一个 `a` 的纸条写着“我后面那个 `a` 在索引 2”；第二个 `a` 的纸条写着“我前面那个 `a` 在索引 0”。

当窗口滑过第一个 `a` 时，程序通过纸条（`nxt`）立刻知道索引 2 的 `a` 现在变成了它那个家族在窗口里的唯一代表，从而修正哈希值。

这就是为什么这份代码能处理 $10^5$ 数据量而不超时的原因：所有的判断和修正都通过这些预处理数组变成了常数时间的操作。
