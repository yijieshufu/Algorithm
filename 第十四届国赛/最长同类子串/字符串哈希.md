学习字符串哈希（String Hashing）是处理字符串匹配、重复子串等问题的“核武器”。它的核心思想很简单：**把一个字符串转换成一个唯一的数字**。

只要数字（哈希值）相同，我们就认为字符串相同。

---

### 1. 核心原理：进制转换法

我们将字符串看作一个 **$P$ 进制的数字**，然后对一个大质数 $M$ 取模。

- **公式**：对于字符串 $S = "abc"$，其哈希值为：

    $$

H = (ord('a') \cdot P^2 + ord('b') \cdot P^1 + ord('c') \cdot P^0) \pmod M

$$
    
- **常用参数**：
    
    - $P$：通常取 `131` 或 `13331`（经验值，冲突概率低）。
        
    - $M$：通常取 `2^64`（Python 会自动处理大整数）或者 `(1 << 61) - 1`（一个很大的素数）。
        

---

### 2.[滑动窗口](滑动窗口的公式理解.md)：$O(1)$ 时间更新哈希

这是哈希最强大的地方。当窗口向右滑动 1 位时，我们不需要重新计算整个字符串的哈希，只需要“踢出左边，加入右边”。

**操作步骤**：

假设窗口长度为 $L$，当前哈希值为 $H_{old}$，即将进入的字符是 $S[right]$，即将离开的是 $S[left]$。

1. **左边踢出**：减去 $S[left] \cdot P^{L-1}$。
    
2. **整体左移**：将剩下的值乘以 $P$（相当于所有位权加 1）。
    
3. **右边加入**：加上 $S[right]$。
    

**更新公式**：

$$

H_{new} = ((H_{old} - S[left] \cdot P^{L-1}) \cdot P + S[right]) \pmod M

$$

### 3. 代码实战：寻找重复子串

这是最基础的练习：判断字符串中是否存在两个相同的长度为 $k$ 的子串。

```Python
import sys

def check(s, k):
    # 基础参数
    P, MOD = 131, (1 << 61) - 1
    n = len(s)
    
    # 1. 预计算 P 的 k-1 次幂，用于踢出左侧字符
    Pk_1 = pow(P, k - 1, MOD)
    
    # 2. 计算第一个窗口（0 到 k-1）的哈希值
    curr_h = 0
    for i in range(k):
        curr_h = (curr_h * P + ord(s[i])) % MOD
    
    seen = {curr_h}
    
    # 3. 开始滑动窗口
    for i in range(k, n):
        # 踢出 s[i-k]，加入 s[i]
        left_char_val = ord(s[i-k])
        curr_h = (curr_h - left_char_val * Pk_1) % MOD # 减去左边
        curr_h = (curr_h * P + ord(s[i])) % MOD        # 加上右边
        
        if curr_h in seen:
            return True # 发现重复
        seen.add(curr_h)
    
    return False

# 测试
s = "banana"
print(check(s, 2)) # True, "an" 重复了
print(check(s, 3)) # True
```

### 4. 避坑指南

- **负数取模**：在 Python 中，`%` 运算符会自动处理负数（如 `-5 % 3 = 1`），但在 C++ 等语言中需要手动加上 `MOD`。
    
- **哈希冲突**：虽然 `(1 << 61) - 1` 非常安全，但在极少数竞赛题中（抗哈希测试用例），可能需要**双哈希**（用两套不同的 $P$ 和 $MOD$ 计算，只有两个哈希都相等才算相等）。
    
- **预处理幂次**：在循环中频繁调用 `pow()` 会变慢，通常提前用数组把 $P^0, P^1, \dots, P^n$ 存起来。
