既然咱们要从零开始，那就把这段代码想象成是在**为一个超级复杂的“找茬游戏”做准备工作**。

这段预处理代码主要做了两件事：**定规矩（哈希参数）** 和 **查户口（字符位置预记录）**。

---

### 第一块：定规矩（哈希参数与幂次数组）

```Python
MOD, BASE = (1 << 61) - 1, 13131
limit = max(n, m) + 1
P = [1] * limit
for i in range(1, limit): P[i] = (P[i-1] * BASE) % MOD
```

1. **`MOD` 和 `BASE`**：这就是我们之前说的“指纹算法”的参数。
    
    - `BASE = 13131`：相当于我们要把字符串看成一个“13131进制”的超级大数字。
        
    - `MOD = (1 << 61) - 1`：这是一个巨大的素数。因为数字太大会让电脑跑不动，所以我们要给它定一个“天花板”，超过这个数就取余。
        
2. **`P` 数组（重点）**：
    
    - 我们在算哈希时，经常要用到 $BASE^1, BASE^2, BASE^3 \dots$。
        
    - 如果每次都在循环里临时算（用 `pow` 函数），速度会非常慢。
        
    - **代码用意**：提前把这些“幂”算好，存在 `P` 数组里。以后想要 $BASE^{10}$，直接拿 `P[10]` 就行，这在算法里叫**空间换时间**。

---

### 第二块：查户口（`get_info` 函数）

这是最关键的逻辑。因为我们要找的是“同构子串”（比如 `aba` 和 `cdc` 结构一样），所以我们不能记字符本身，要记**字符出现的相对位置**。

```Python
def get_info(arr, L):
    lst, nxt = [-1] * L, [-1] * L # 准备两个空表格
    pos = [-1] * 26              # 准备一个“临时记录本”，记录 26 个字母最后出现的位置
    for i in range(L):
        p = pos[arr[i]]         # 去记录本查：这个字母上次出现在哪？
        if p != -1:
            lst[i] = p          # lst 记录：我的上一个分身在位置 p
            nxt[p] = i          # nxt 记录：位置 p 的下一个分身在位置 i
        pos[arr[i]] = i         # 更新记录本：这个字母最后一次出现在位置 i
    return lst, nxt
```

#### 为什么要记 `lst`（前世）和 `nxt`（今生）？

- **`lst[i]`**：当我们计算当前字符的哈希值时，我们要看它距离“前世”有多远。
    
- **`nxt[p]`**：当窗口向右滑动，把左边的字符“踢出去”时，我们需要通过 `nxt` 瞬间找到它的“下一任分身”，并告诉这个分身：“由于老大走了，你现在变成窗口里该字符的第一个分身了，你的特征值要变回 0 喔！”

---

### 第三块：实战演练（以 `banana` 为例）

假设我们处理字符串 `banana`，看 `get_info` 是怎么填表的：

| **下标 i** | **字符** | **pos 记录本变化** | **lst[i] (前世)** | **nxt (今生更新)** |
| -------- | ------ | ------------- | --------------- | -------------- |
| 0        | **b**  | `pos['b']=0`  | -1              |                |
| 1        | **a**  | `pos['a']=1`  | -1              |                |
| 2        | **n**  | `pos['n']=2`  | -1              |                |
| 3        | **a**  | `pos['a']=3`  | **1** (上一个a在1)  | `nxt[1] = 3`   |
| 4        | **n**  | `pos['n']=4`  | **2** (上一个n在2)  | `nxt[2] = 4`   |
| 5        | **a**  | `pos['a']=5`  | **3** (上一个a在3)  | `nxt[3] = 5`   |

**最后得到的 `lst` 数组：** `[-1, -1, -1, 1, 2, 3]`

**最后得到的 `nxt` 数组：** `[-1, 3, 4, 5, -1, -1]`

---

### 总结：这段代码到底准备了什么？

1. **`P` 数组**：一个快捷查询表，让你能瞬间知道“位权”是多少。
    
2. **`s_lst`**：告诉你每个字符的“上一次出现位置”。有了它，我们就能算出“距离”，从而生成同构特征。
    
3. **`s_nxt`**：告诉你每个字符的“下一次出现位置”。它是滑动窗口的“眼线”，负责在窗口滑动时通知下一个字符更新状态。

**教练叮嘱：**

小白最容易卡在 `nxt[p] = i` 这一行。你要记住：我们在处理当前位置 `i` 的时候，顺便回头给 `p` 位置（它的前任）留下一个线索，告诉 `p` 它的后继者是谁。
