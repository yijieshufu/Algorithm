```python
import sys

# 1. 快速读入并转换为数字序列
data = sys.stdin.read().split()
if not data: sys.exit(0)
S_str, T_str = data[0], data[1]
n, m = len(S_str), len(T_str)
S = [ord(c) - 97 for c in S_str]
T = [ord(c) - 97 for c in T_str]

# 2. 哈希与预处理
MOD, BASE = (1 << 61) - 1, 13131
limit = max(n, m) + 1
P = [1] * limit
for i in range(1, limit): P[i] = (P[i-1] * BASE) % MOD

def get_info(arr, L):
    lst, nxt = [-1] * L, [-1] * L
    pos = [-1] * 26
    for i in range(L):
        p = pos[arr[i]]
        if p != -1:
            lst[i] = p
            nxt[p] = i
        pos[arr[i]] = i
    return lst, nxt

s_lst, s_nxt = get_info(S, n)
t_lst, t_nxt = get_info(T, m)

# 3. 二分答案 + 滚动哈希校验
low, high, ans = 1, min(n, m), 0
while low <= high:
    k = (low + high) // 2
    found = False
    hashes = set()
    
    # 提取 S 的所有长度为 k 的同类串哈希
    h = 0
    for i in range(k):
        v = (i - s_lst[i]) if s_lst[i] >= 0 else 0
        h = (h * BASE + v) % MOD
    hashes.add(h)
    
    for i in range(k, n):
        j = i - k
        h = (h * BASE) % MOD # 移出 j (D[j]=0) 并整体左移
        nx = s_nxt[j]
        if nx != -1 and nx < i: # 修正 nx：从距离 j 变成距离“无”
            h = (h - (nx - j) * P[i - nx]) % MOD
        v = (i - s_lst[i]) if s_lst[i] > j else 0 # 移入 i
        h = (h + v) % MOD
        hashes.add(h)
    
    # 校验 T 中是否存在相同的哈希
    h = 0
    for i in range(k):
        v = (i - t_lst[i]) if t_lst[i] >= 0 else 0
        h = (h * BASE + v) % MOD
    if h in hashes:
        found = True
    else:
        for i in range(k, m):
            j = i - k
            h = (h * BASE) % MOD
            nx = t_nxt[j]
            if nx != -1 and nx < i:
                h = (h - (nx - j) * P[i - nx]) % MOD
            v = (i - t_lst[i]) if t_lst[i] > j else 0
            h = (h + v) % MOD
            if h in hashes:
                found = True; break
                
    if found:
        ans = k
        low = k + 1
    else:
        high = k - 1

sys.stdout.write(str(ans) + '\n')
```

[[2. 哈希与预处理]]
[[3. 二分答案 + 滚动哈希校验]]
这道题是蓝桥杯中典型的**字符串结构化匹配**问题，结合了**二分查找**与**进阶滚动哈希**技术。以下是详细的思路分析：

---

### 1. 核心概念：何为“同类串”？

题目定义的“同类串”（Isomorphic Strings）本质上是要求两个字符串的**字符分布模式**一致。

- **直观理解**：如果能通过建立一种字符一一对应的映射关系，将串 $A$ 转换为串 $B$，则它们是同类串。
    
- **数学转换**：为了方便计算机处理，我们将字符转换为**相对距离序列**。
    
    > 例如：对于子串 `aabab`
    > 
    > 1. 第 0 位 'a'：该窗口内第一次出现 $\rightarrow$ 记为 **0**
    >     
    > 2. 第 1 位 'a'：距离上一个 'a' 是 1 $\rightarrow$ 记为 **1**
    >     
    > 3. 第 2 位 'b'：该窗口内第一次出现 $\rightarrow$ 记为 **0**
    >     
    > 4. 第 3 位 'a'：距离上一个 'a' 是 2 $\rightarrow$ 记为 **2**
    >     
    > 5. 第 4 位 'b'：距离上一个 'b' 是 2 $\rightarrow$ 记为 **2**
    >     
    >     得到序列：`[0, 1, 0, 2, 2]`。任何同类串在该转换下生成的序列都是相同的（如 `xxkxk` 也会生成 `[0, 1, 0, 2, 2]`）。
    >     
    

---

### 2. 算法模型

由于最大长度 $k$ 具有单调性（如果存在长度为 10 的同类子串，必然存在长度为 9 的），我们采用 **二分答案 + 滚动哈希** 的组合。

#### A. 二分查找（Binary Search）

- **范围**：$low = 1, high = \min(|S|, |T|)$。
    
- **目标**：在 $O(\log N)$ 次尝试中找到最大的 $k$，使得 `check(k)` 为真。
    

#### B. 结构化滚动哈希（Rolling Hash with Structural Change）

这是本题最难的点。在普通哈希中，滑动窗口只需要“加一个、减一个”。但在同类串判定中，窗口滑动会导致内部元素的**相对距离值发生突变**。

1. **右端移入 ($i$)**：
    
    计算 $S[i]$ 与上一次出现位置的距离。如果上一次位置在当前窗口 $[j, i]$ 之外，则记为 $0$；否则记为实际距离。
    
2. **左端移出 ($j$)**：
    
    左端点 $j$ 在它所在的窗口里永远是“第一次出现”（即值总是 $0$），所以移除它只需要将哈希值整体去掉权重。
    
3. **关键：后继修正**：
    
    假设 $S[j]$ 这个字符在窗口内还有下一次出现，位置在 $nx$。
    
    - 在旧窗口里，$nx$ 的值是 $nx - j$。
        
    - 当 $j$ 移出后，在新窗口里，$nx$ 变成了该字符的“第一次出现”，其值必须变为 $0$。
        
    - **哈希补偿**：我们需要在 $O(1)$ 时间内从哈希值中减去原本属于 $nx$ 的贡献：

        $$


(nx - j) \times \text{BASE}^{\text{距离末尾的长度}}

$$
        

---

### 3. 复杂度与优化细节

|**维度**|**方案**|**原因**|
|---|---|---|
|**时间复杂度**|$O(N \log N)$|预处理 $O(N)$，二分 $\log N$ 次，每次哈希遍历 $O(N)$。|
|**空间复杂度**|$O(N)$|存储 `lst`, `nxt` 数组及幂次数组。|
|**哈希抗碰撞**|**梅森素数 $2^{61}-1$**|Python 处理大整数非常高效，该模数能极大降低冲突率。|
|**预处理**|`lst` & `nxt` 数组|提前记录每个位置的前驱和后继，保证 `check` 内部 $O(1)$ 更新。|

---

### 4. 总结

解决这类问题的通用套路是：

1. **特征化**：将抽象的“同类”属性转化为具体的数值序列（相对距离）。
    
2. **动态维护**：思考窗口滑动时，哪些位置的值会因为边界变化而改变（本题中是 `nxt[j]` 的值从 $nx-j$ 变为 $0$）。
    
3. **哈希加速**：利用多项式哈希将 $O(K)$ 的序列比较降至 $O(1)$。
