# 汉字田

```python
import math  

def solve_exactly_two():  
    # --- 第一部分：搭建舞台（初始化坐标点） ---    # 我们要在坐标系里画一个 3x3 的点阵  
    # 生成的 points 会是: [(0,0), (0,1), (0,2), (1,0), (1,1) ... (2,2)]  
    points = []  
    for x in range(3):  
        for y in range(3):  
            points.append((x, y))  

    # --- 第二部分：准备账本（存储直线信息） ---    # 我们用一个字典（Dictionary）来记录每一条被发现的直线  
    # key (键): 直线的唯一身份证号 (a, b, c)    # value (值): 一个集合 (set)，里面装着这条直线经过的所有点  
    line_to_points = {}  

    n = len(points)  # n 就是 9  
    # --- 第三部分：地毯式搜索（两两连线） ---    # 外层循环选第一个点，内层循环选第二个点  
    for i in range(n):  
        for j in range(i + 1, n):  
            p1, p2 = points[i], points[j]  
            # print(f"当前节点p1:{p1}和p2:{p2}")  

            # 【核心数学逻辑：直线的一般方程 Ax + By + C = 0】  
            # 我们通过 p1(x1, y1) 和 p2(x2, y2) 算出这条直线的 A, B, C 系数  
            a = p1[1] - p2[1]  # A = y1 - y2  
            b = p2[0] - p1[0]  # B = x2 - x1  
            c = p1[0] * p2[1] - p2[0] * p1[1]  # C = x1y2 - x2y1  

            # 【标准化过程：确保身份证号唯一】  
            # 为什么要这步？因为 2x + 2y + 2 = 0 和 x + y + 1 = 0 其实是同一条线  
            # 我们求出 A, B, C 的最大公约数，然后全体缩小（约分）  
            common = math.gcd(a, math.gcd(b, c))  
            a, b, c = a // common, b // common, c // common  

            # 【去镜像：确保方向统一】  
            # 规定身份证号的首位必须是正数。比如让 (-1, 1, 0) 变成 (1, -1, 0)            
            if a < 0 or (a == 0 and b < 0):  
                a, b, c = -a, -b, -c  

            # 最终确定的“身份证号”  
            line_id = (a, b, c)  

            # 【入账：把点扔进对应的桶里】  
            # 如果这条线是第一次发现，先在账本上开个新页（创建一个空集合）  
            if line_id not in line_to_points:  
                line_to_points[line_id] = set()  

            # 把这两个点登记在这条线上  
            # 集合(set)会自动去重，所以同一个点无论被登记几次，都只算一个点  
            line_to_points[line_id].add(p1)  
            line_to_points[line_id].add(p2)  

    # --- 第四部分：筛选答案（只要恰好有两个点的线） ---    
    count = 0  
    # 遍历账本里的每一条直线  
    for line in line_to_points:  
        # line_to_points[line] 拿到的是这条线上所有点的集合  
        # 我们用 len() 数数集合里有几个点  
        if len(line_to_points[line]) == 2:  
            count += 1  # 只有恰好两个点的线，才算入最终成绩  

    return count  


# 执行并打印  
print(f"恰好经过两个点的直线共有: {solve_exactly_two()}")
```

我们要解决的问题其实是一个“**连连看**”的游戏：

> 面前有 9 个点（3排3列），拿出一支无限长的笔划直线。什么样的直线才算数？**必须刚好穿过两个点，多一个不行，少一个也不行。**

下面是小白视角的“四步拆解法”：
### 第一步：给点“取名字”（坐标化）

在电脑眼里，它不认识什么“左上角”、“中间”。我们需要给每个点一个编号。 想象一个直角坐标系：

- 左下角的点叫 **(0, 0)**
    
- 中心点叫 **(1, 1)**
    
- 右上角的点叫 **(2, 2)**

这样 9 个点就变成了 9 组数字。这就是代码里 `points = []` 那一段在做的事情。

### 第二步：找出所有可能的“路线”

电脑很笨，但它跑得快。它会用**穷举法**：

1. 它先选点 A，再选点 B，连成一条线。
    
2. 它会把 9 个点里所有两两组合的可能性都试一遍。
    
3. 数学公式告诉我们，9 个点两两组合共有 **36 种**组合。

**但是！** 这 36 种组合里，有很多其实是**重复**的，或者**不符合要求**的。

### 第三步：识破“三点一线”的诡计（核心难点）

这是这道题最容易掉进去的陷阱。

如果你连接了 **(0,0)** 和 **(1,1)**，这条线延长出去，**刚好也会经过 (2,2)**。

- 在你的笔下，这虽然是同一个动作划出的一条线，但它穿过了 **3 个点**。
    
- 题目要求：**恰好**经过两个点。
    
- 所以，像**三条横线、三条竖线、两条长对角线**，这 8 条线都要被无情地**踢走**。

**代码是怎么识破它们的？**

代码为每一条线算了一个“身份证号”（即 $Ax + By + C = 0$ 的系数）。它发现，当它连接 (0,0) 和 (1,1) 时，身份证号是某个值；连接 (0,0) 和 (2,2) 时，化简后的身份证号**竟然一模一样**。

于是代码会把这些点都扔进同一个“桶”里。最后它一数：**“嘿！这个桶里有 3 个点，不合格，扔掉！”**

### 第四步：数数最后剩下了谁？

剔除了那 8 条经过 3 个点的“长线”后，剩下的就是我们要的“合格线”了：

1. **“切角线” (4条)**：  
    ![[../附件/Pasted image 20260125205044.png]]  
    只连接了侧边中点和顶点。比如连接 $(0,1)$ 和 $(1,2)$。这条线就像把田字格的一个角切掉了一小块，它绝对不会碰到第三个点。
    
2. **“斜跳线/马步线” (8条)**：  
    ![[../附件/Pasted image 20260125205153.png]]  
    想象国际象棋里的“马”或者中国象棋里的“马”，它们走“日”字格。比如从 $(0,0)$ 连到 $(1,2)$。这种线斜率很特殊，在 $3 \times 3$ 的格子里，它只会精准地踩中两个点。

**最后相加：$4 + 8 = 12$ 条。**
### 总结：代码在干什么？

这段代码其实就是一个**超级细心的管理员**：

1. **登记**：把 9 个点的坐标记下来。
    
2. **连线**：把所有两两组合都试一遍，算出每条线的“身份证”。
    
3. **分类**：把属于同一条直线的点放在同一个桶里。
    
4. **筛选**：最后看哪个桶里**恰好只有 2 个点**，就计数一次。
