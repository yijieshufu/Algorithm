# 分糖果

### 1. 核心概念：什么是“字典序”？

首先要明确一个核心规则：**在字典序里，首字母决定一切**。

- `b` 比 `aaaaa` 大。（因为 `b` 在字母表里排在 `a` 后面，不管 `a` 后面跟多少个字母，只要第一个字母输了，整个字符串就输了）。
    
- 只有当首字母一样时，我们才比较第二个字母。

**目标**：我们要把 $n$ 个糖果分给 $x$ 个同学，分完后，大家手里都有一个字符串。我们要让 **“手里字符串最大的那个同学”**，他的字符串尽可能小。
### 2. 策略第一步：必须排序

既然我们想要大家手里的字符串尽可能小，那我们肯定希望把小的字母（比如 `a`）放在字符串的**开头**。 如果把 `z` 放在开头，那这个字符串肯定很大。

所以，拿到题目后的第一反应是：**把所有糖果从小到大排好序**。 比如输入 `caabdc`，我们要先变成 `a, a, b, c, c, d`。
### 3. 策略第二步：确定“首字母”

大家都要分到糖果，且每人至少一个。为了让最大的那个尽可能小，最好的策略是**把最小的 $x$ 个字母，分别分给 $x$ 个同学作为“首字母”**。

如果不这样做（比如把最小的两个 `a` 都给了第一个人），那后面的人就只能拿 `b` 或者 `c` 开头，那个人的字符串瞬间就变大了，这不划算。

假设有 3 个同学，糖果是 `a, a, c, d, e, f`。

- **同学1** 拿第1个：`a`
    
- **同学2** 拿第2个：`a`
    
- **同学3** 拿第3个：`c`

这时候，基本的格局就定下来了。接下来的问题是：**剩下的糖果 `d, e, f` 给谁？** 这就是代码中三个 `if-else` 分支的来源。
### 4. 深入场景分析（对应代码的三个分支）

#### 场景一：起跑线就不公平（对应代码 `if s[0] != s[x-1]`）

假设：2 个同学（x=2），糖果是 `a, c, d, e`。

- 同学1 拿 `a`。
    
- 同学2 拿 `c`。

**分析**： 不管剩下的 `d, e` 给谁，同学2 的 `c` 开头已经是最大的了（因为 `c > a`）。

- 如果你把 `d` 给同学2，他变成 `cd`，比 `c` 更大，雪上加霜。
    
- 如果你把 `d` 给同学1，他变成 `ad`。注意！即使同学1 拿了 `adde...` 一大堆，只要它是 `a` 开头，它永远比 `c` 小。  
**结论**：

既然同学2 已经是“注定最大”的那个倒霉蛋了，为了不让他更倒霉，我们把**剩下所有的累赘**全扔给同学1（反正同学1怎么加都比同学2小）。

那么最终整个局面的最大值，就是同学2 手里那个原本的字符：**`c`**。

> **代码解释**：`print(s[x-1])` —— 直接输出第 $x$ 个同学拿到的那个首字母。

#### 场景二：起跑线相同，且剩下的糖果也全都一样（对应代码 `elif ... s[x] == s[-1]`）

假设：2 个同学，糖果是 `a, a, b, b, b, b`。

- 同学1 拿 `a`。
    
- 同学2 拿 `a`。
    
- 剩下：`b, b, b, b`。

**分析**： 现在两人一样大。剩下的全是 `b`。

- 如果全给同学1：`abbbb` vs `a`。最大值是 `abbbb`。
    
- 如果平分：`abb` vs `abb`。最大值是 `abb`。

很明显，`abb` 比 `abbbb` 要小（字典序里，前缀相同，越短越小）。

**结论**： 如果剩下的牌全是一样的，那就**均匀分摊**给所有人，大家一起承担，这样每个人的长度最短，字典序最小。

> **代码解释**：`print(''.join(s[: :x]))` —— 利用步长切片，把剩下的相同字符均匀发给每个人。

#### 场景三：起跑线相同，但剩下的糖果不一样（最难理解的一点，对应 `else`）

假设：2 个同学，糖果是 `a, a, b, c`。

- 同学1 拿 `a`。
    
- 同学2 拿 `a`。
    
- 剩下：`b, c`。

**分析**： 此时两人都是 `a` 开头，谁拿到了 `b` 或者 `c` 决定了胜负。

- **方案 A（一人一个）**：同学1 拿 `b` -> `ab`；同学2 拿 `c` -> `ac`。 结果：最大值是 **`ac`**。
    
- **方案 B（全部给一个人）**：同学1 拿 `b` 和 `c` -> `abc`；同学2 啥也不拿 -> `a`。 结果：最大值是 **`abc`**。

**关键比较**：`ac` 和 `abc` 谁大？ 比较第二位：`c` 比 `b` 大。所以 **`ac` > `abc`**。

**结论**：

为了避免出现 `ac` 这种“开头虽小但第二位很大”的情况，最好的办法是**把剩下所有的糖果（包括大的小的）全部塞给其中一个人**。

这样，这个人的字符串虽然很长（比如 `abc`），但他的第二位是 `b`（剩下里最小的），而如果把 `c` 分给别人，别人的第二位就是 `c`，那反而会让整体的最大值变大。

> **代码解释**：`print(''.join(s[x-1:]))` —— 从第 $x$ 个字符（也就是第一个同学拿到的那个首字母）开始，把后面所有的字符全拼起来给他。

### 总结流程图

1. **分首字母**：每人拿一个，这决定了基准。
    
2. **看最后一个首字母 `s[x-1]`**：
    
    - 如果它比第一个首字母大（场景一）：那最大值就是它自己。**结束**。
        
    - 如果它和第一个一样（大家起跑线相同）：看剩下的糖果。
        
        - 剩下全是同一种（场景二）：大家平分。**结束**。
            
        - 剩下有大有小（场景三）：为了保住“第二位”尽可能小，全堆给一个人。**结束**。

```python
import sys  
  
# n: 糖果数量, x: 同学数量  
n, x = map(int, sys.stdin.readline().split())  
# 将糖果按字典序排序，方便从小到大分配  
s = sorted(sys.stdin.readline().strip())  
  
# 情况 1：前 x 个字符不完全相同  
# 说明分配完第一轮后，第 x 个同学拿到的字符已经是目前最大的。  
# 为了不让它更大，后续所有糖果都塞给拿最小字符的同学。  
if s[0] != s[x-1]:  
    print(s[x-1])  
  
# 情况 2：前 x 个字符相同，且从第 x+1 个到最后一个字符也都相同  
# 这种情况下，大家平分剩下的糖果是最优的。  
# s[::x] 表示从第 0 个开始，每隔 x 个取一个，相当于把所有糖果平分。  
elif x < n and s[x] == s[-1]:  
    print("".join(s[::x]))  
  
# 情况 3：前 x 个字符相同，但后面的字符有变化  
# 把剩下的所有糖果都给第一个同学，形成的字符串字典序反而更小。  
# 例如 aaa + bcd，给一个人是 abcd，分给三个人最大可能是 ad。  
else:  
    # 从索引 x-1 开始到最后的所有字符归一个人  
    print("".join(s[x-1:]))
```
