# 486. 预测赢家 (Predict the Winner)

```Python
from functools import lru_cache
from typing import List

class Solution:
    def predictTheWinner(self, nums: List[int]) -> bool:
        n = len(nums)
        
        @lru_cache(None)
        def dfs(i: int, j: int) -> int:
            # 基础情况：区间只剩一个数
            if i == j:
                return nums[i]
            
            # 核心博弈方程：
            # 当前玩家选择左端点 nums[i]，则领先分数为 nums[i] 减去对手在剩余区间的领先分
            # 当前玩家选择右端点 nums[j]，则领先分数为 nums[j] 减去对手在剩余区间的领先分
            return max(nums[i] - dfs(i + 1, j), nums[j] - dfs(i, j - 1))
        
        # 先手分差 >= 0 则获胜
        return dfs(0, n - 1) >= 0
```
### 1. 算法判别：区间 DP + 博弈论

这道题是典型的**零和博弈**。两名玩家轮流决策，且目标都是最大化自己的收益。由于决策范围是一个连续的数组区间 $[i, j]$，且大区间的决策依赖于小区间的子结果，因此它属于**区间动态规划**的范畴。

### 2. 核心战术：净胜分（Lead Score）思维

很多初学者会尝试记录 Alice 和 Bob 的各自得分，这会把状态变成三维甚至四维。这份代码高明在只记录**“分差”**。

- **状态定义**：$dfs(i, j)$ 表示在数组区间为 $[i, j]$ 时，**当前行动者**相对于**对手**能领先的最大分数。
    
- **博弈逻辑**：我现在的得分 - 对方在剩下残局里的领跑分数。

---

### 3. 数学推导与代码映射

我们将递归过程形式化为如下状态转移方程：

$$
f(i, j) = \max(nums[i] - f(i+1, j), \quad nums[j] - f(i, j-1))
$$

- **选择左端点 `nums[i]`**：我拿走 $nums[i]$ 分，剩下的区间 $[i+1, j]$ 轮到对方。对方在这个残局中能领先我 $dfs(i+1, j)$ 分。所以我最终的领跑分数是 $nums[i] - dfs(i+1, j)$。
    
- **选择右端点 `nums[j]`**：同理，领跑分数为 $nums[j] - dfs(i, j-1)$。
    
- **最优决策**：因为玩家都是顶级聪明的，所以必然会在这两项中选一个最大的结果返回。

---

### 4. 代码细节拆解

- **`@lru_cache(None)`**：这是 Python 算法竞赛的“神技”。如果不加这行，这是一个 $O(2^n)$ 的指数级递归，蓝桥杯必 TLE。加了它，系统会自动记忆化，将复杂度降为 $O(n^2)$。
    
- **递归边界**：`if i == j: return nums[i]`。当区间只剩一个数，当前玩家直接拿走，他比对手多得的分数就是这个数本身。
    
- **判定条件**：`dfs(0, n - 1) >= 0`。最终得到的净胜分如果是正数或零，说明先手（Alice）的总分大于等于对方。

---

### 5. 复杂度与性能

| **维度**     | **分析**                            | **结论**   |
| ---------- | --------------------------------- | -------- |
| **时间复杂度**  | 状态总数为 $n \times n$，每个状态只算一次       | $O(n^2)$ |
| **空间复杂度**  | 记忆化哈希表存储了 $n^2$ 个结果               | $O(n^2)$ |
| **蓝桥杯适用性** | Python 3.8 环境下，$n \le 500$ 均可轻松秒杀 | 极高       |

### 教练的小贴士

在蓝桥杯的 Python 组中，这种 **DFS + `lru_cache`** 的写法比手写二维 DP 表（迭代法）更不容易出错，尤其是处理复杂的区间边界时。但请务必记住：如果 $n$ 特别大（如超过 1000），需要手动增加递归深度限制：`sys.setrecursionlimit(2000)`。
