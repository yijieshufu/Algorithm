# 1140.石子游戏 II
```Python
from typing import List
from functools import lru_cache

class Solution:
    def stoneGameII(self, piles: List[int]) -> int:
        n = len(piles)
        # 1. 预处理后缀和：s[i] 表示从 piles[i] 到 piles[n-1] 的总和
        s = [0] * (n + 1)
        for i in range(n - 1, -1, -1):
            s[i] = s[i + 1] + piles[i]

        # 2. 记忆化搜索
        @lru_cache(None)
        def dfs(i, M):
            # 剪枝：如果剩下的石子可以一次性全部拿走
            if i + 2 * M >= n:
                return s[i]
            
            # 状态转移：尝试拿走 x 堆，1 <= x <= 2M
            # 目标：使 (当前剩余总和 - 对手下一步能拿的最大值) 最大化
            return s[i] - min(dfs(i + x, max(M, x)) for x in range(1, 2 * M + 1))

        return dfs(0, 1)
```

## 一、 题目到底在问什么？

很多同学会被 $M$ 和 $X$ 的变化搞晕。其实核心就三点：

1. **目标：** Alice 要拿**尽可能多**的石子。注意，这不是判断胜负，而是**最大化得分**。
    
2. **博弈规则：** 你拿走 $X$ 堆，下一轮对方的上限就会变成 $2 \cdot \max(M, X)$。
    
    - _潜台词：_ 如果你为了眼前利益一次性拿太多（$X$ 很大），虽然现在爽了，但对方下一轮能拿的范围会剧增，可能反手把剩下的全收了。
        
3. **最优策略：** Alice 和 Bob 都是“顶级聪明”的。这意味着：**当你做选择时，必须假设对方在下一回合也会做出让你最难受（即他自己得分最高）的选择。**

---

## 二、 我们要如何解题？

### 1. 核心思想：转守为攻

在博弈类题目中，有一个[[../../../leetcode/万能公式]]：

> **我方当前能拿到的最大值 = 剩余石子总数 - 对手在下一回合能拿到的最大值**

这个公式非常巧妙，它把“两个人的博弈”转化成了“同一个状态的最优解”。

### 2. 状态定义

我们需要一个函数 `dfs(i, M)` 来表示：当石子还剩下第 $i$ 堆到最后一堆，且当前的限制参数为 $M$ 时，**当前行动的人**能拿到的最大石子数。

### 3. 贪心还是 DP？

这道题**不能贪心**。因为现在的最优选择可能导致未来崩盘。我们需要枚举所有可能的 $X$（从 $1$ 到 $2M$），计算每种方案下我方的最终得分，最后取最大值。由于存在大量重复计算，必须使用**记忆化搜索**。

## 三、 最优解代码实现 (Python 3.8.6)

```Python
from typing import List

from functools import lru_cache
class Solution:
    def stoneGameII(self, piles: List[int]) -> int:
        n = len(piles)
        # 1. 预处理后缀和：s[i] 表示从 piles[i] 到 piles[n-1] 的总和
        s = [0] * (n + 1)
        for i in range(n - 1, -1, -1):
            s[i] = s[i + 1] + piles[i]
        # 2. 记忆化搜索
        @lru_cache(None)
        def dfs(i, M):
            # 剪枝：如果剩下的石子可以一次性全部拿走
            if i + 2 * M >= n:
                return s[i]
            # 状态转移：尝试拿走 x 堆，1 <= x <= 2M
            # 目标：使 (当前剩余总和 - 对手下一步能拿的最大值) 最大化
            return s[i] - min(dfs(i + x, max(M, x)) for x in range(1, 2 * M + 1))
        return dfs(0, 1)
```

## 四、 教练笔记：为什么要这么写？

1. **[[../../../leetcode/后缀和（Suffix Sum）]]：**
    
    如果不预处理后缀和，你每次在计算“剩余石子”时都要用 `sum(piles[i:])`，这是一个 $O(N)$ 操作。预处理后变 $O(1)$，直接决定了你的程序会不会 TLE（超时）。
    
2. **`lru_cache(None)`：**
    
    这是 Python 的“自动记事本”。如果不加它，这是一个指数级复杂度的递归；加了它，就变成了 $O(N^3)$ 的动态规划，足以通过 100 堆的数据量。
    
3. **循环范围：**
    
    `range(1, 2 * M + 1)` 严格遵循题目 $1 \le X \le 2M$ 的限制。
