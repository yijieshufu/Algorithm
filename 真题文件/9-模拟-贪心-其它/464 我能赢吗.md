# 464 我能赢吗

这道题是博弈论 DP 中的**分水岭**。如果说之前的“预测赢家”是入门，那这道题引入了算法竞赛中极其重要的技巧——**状态压缩（State Compression）**。

在蓝桥杯中，这类“不能重复使用数字”的博弈题，几乎百分之百要用到位运算来优化空间。

---

## 1. [[题目逻辑分析]]

### 核心难点

与“石子游戏”不同，这道题的数字**不能重复使用**。

- 如果你选了 10，下一个玩家就只能从剩下的数字里挑。
    
- 我们不仅要记录当前的累加和，还要记录**哪些数字已经被用掉了**。

### 状态定义：[[为什么需要“状压”？]]

数字范围最大是 20。如果用一个列表记录每个数字是否被选过，递归的状态会非常难以存储。

由于 $maxChoosableInteger \le 20$，我们可以用一个 **20 位的二进制数**（整数）来表示数字的使用状态：

- 第 $i$ 位为 `1`：表示数字 $i+1$ 已被使用。
    
- 第 $i$ 位为 `0`：表示数字 $i+1$ 尚可使用。

### 必胜态判定

当前玩家在状态 `mask` 下能赢，只需满足以下任一条件：

1. **绝杀**：存在一个可选数字 $x$，使得 $x \ge$ 当前剩余所需总和。
    
2. **转嫁压力**：存在一个可选数字 $x$，使得我方选完后，对方面对新状态 `mask | (1 << (x-1))` 时**无法获胜**。

## 2. Python 3.8.6 核心代码
```Python
from functools import lru_cache

class Solution:
    def canIWin(self, maxChoosableInteger: int, desiredTotal: int) -> bool:
        # 边界情况 1：如果所有数字加起来都够不到 desiredTotal，谁都赢不了
        if (1 + maxChoosableInteger) * maxChoosableInteger // 2 < desiredTotal:
            return False
        # 边界情况 2：如果最大的数字直接超过目标，先手秒胜
        if maxChoosableInteger >= desiredTotal:
            return True

        # mask: 二进制位表示数字使用情况
        # current_total: 距离目标的剩余数值
        @lru_cache(None)
        def dfs(mask, current_total):
            # 尝试每一个数字 i
            for i in range(maxChoosableInteger):
                # 检查数字 i+1 是否被使用过 (检查 mask 的第 i 位是否为 0)
                if not (mask & (1 << i)):
                    # 如果选了这个数直接达到目标，或者选完后对方必输
                    if current_total <= i + 1 or not dfs(mask | (1 << i), current_total - (i + 1)):
                        return True
            return False

        # 初始状态：mask 为 0（都没选），剩余总和为 desiredTotal
        return dfs(0, desiredTotal)
```

## 3. 深度解析：状压博弈的精髓

### ① 为什么状态里只用了 `mask`？

你会发现 `lru_cache` 只需要记录 `mask` 就能正确运行。

- **原因**：因为 `mask` 已经唯一确定了哪些数字被用了。既然数字确定了，它们的总和也就确定了。
    
- **效果**：极大地减少了记忆化搜索的存储空间，从 `(mask, current_total)` 的二维状态降为 `mask` 一维。

### ② 位运算技巧（竞赛必备）

- **检查第 $i$ 位是否为 0**：`not (mask & (1 << i))`
    
- **将第 $i$ 位设为 1**：`mask | (1 << i)`

### ③ 剪枝策略

- **总和检查**：代码开头的 `(1 + n) * n // 2 < target` 是极其重要的剪枝。在蓝桥杯这种按测试点给分的比赛中，漏掉这个判别可能会导致部分超大数据集超时。

---

## 4. 算法教练的总结

这道题是“逻辑”与“计算机底层存储”的结合。

1. **博弈逻辑**：依然是 `if not dfs(下个状态): return True`。
    
2. **存储技巧**：用 `int` 模拟 `set` 或 `list`，实现 $O(1)$ 时间的读写。

**对比思考**：

- **预测赢家**：操作的是数组两端，状态是区间 $[i, j]$。
    
- **石子游戏**：操作的是连续堆，状态是索引 $i$ 和 $M$。
    
- **我能赢吗**：操作的是无序集合，状态是**位掩码 (Bitmask)**。
