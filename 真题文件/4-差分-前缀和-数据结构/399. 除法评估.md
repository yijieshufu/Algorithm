使用**带权并查集 (Weighted Union-Find)** 是解决此类“关系传递”问题的进阶方案。相比 Floyd 算法的 $O(N^3)$，并查集在处理大规模变量时效率更高，时间复杂度接近 $O(N+Q)$。

### 1. 题目考察类型

- **数据结构**：带权并查集。
    
- **核心逻辑**：维护节点与其根节点之间的**倍数关系**。

### 2. 最优解思路（带权并查集）

在普通的并查集中，我们只关心节点是否连通。而在本题中，我们需要维护一个权值数组 `w`，使得 `w[x]` 代表变量 $x$ 与其父节点 `parent[x]` 的比例，即：

$$
w[x] = \frac{x}{parent[x]}
$$

1. **查询（Find）与路径压缩**：
    
    - 当我们递归寻找根节点时，利用路径压缩将 $x$ 直接指向根节点 $root$。
        
    - 更新权值：$\frac{x}{root} = \frac{x}{parent[x]} \times \frac{parent[x]}{root}$。
        
2. **合并（Union）**：
    
    - 已知 $a/b = v$，若 $a$ 的根为 $ra$，$b$ 的根为 $rb$。
        
    - 我们需要建立 $ra \to rb$ 的关系。
        
    - 根据 $\frac{a}{ra} = w_a$ 和 $\frac{b}{rb} = w_b$，推导出 $\frac{ra}{rb} = \frac{a/w_a}{b/w_b} = \frac{a}{b} \times \frac{w_b}{w_a} = v \times \frac{w_b}{w_a}$。
        
3. **计算结果**：
    
    - 若 $C$ 和 $D$ 不在同一个集合，或变量未出现，返回 $-1.0$。
        
    - 若在同一集合，结果为 $\frac{C/root}{D/root} = \frac{w_C}{w_D}$。

### 3. Python 3.8.6 代码实现

```Python
class Solution:
    def calcEquation(self, equations: List[List[str]], values: List[float], queries: List[List[str]]) -> List[float]:
        parent = {}
        weight = {} # weight[x] = x / parent[x]

        def find(i):
            if i not in parent:
                parent[i] = i
                weight[i] = 1.0
                return i
            if parent[i] == i:
                return i
            
            # 路径压缩：先递归找到根，再更新当前节点的权值
            root = find(parent[i])
            weight[i] *= weight[parent[i]] # x/root = (x/p) * (p/root)
            parent[i] = root
            return root

        def union(i, j, v):
            root_i, root_j = find(i), find(j)
            if root_i != root_j:
                parent[root_i] = root_j
                # 更新根节点之间的权重：root_i / root_j
                weight[root_i] = v * weight[j] / weight[i]

        # 1. 构建并查集
        for (a, b), v in zip(equations, values): #zip 配对
            union(a, b, v)

        # 2. 处理查询
        res = []
        nodes = parent.keys()
        for c, d in queries:
            if c not in nodes or d not in nodes:
                res.append(-1.0)
            else:
                root_c, root_d = find(c), find(d)
                if root_c != root_d:
                    res.append(-1.0)
                else:
                    # c/d = (c/root) / (d/root)
                    res.append(weight[c] / weight[d])
        return res
```

### 复杂度分析

- **时间复杂度**：$O((N+Q) \alpha(N))$，其中 $\alpha$ 是反阿克曼函数。在实际竞赛中，这几乎可以看作是**线性时间**。
    
- **空间复杂度**：$O(N)$，仅需哈希表存储父节点和权重。

### 教练点评

- **优势**：并查集写法在处理**增量数据**（即边一条条加进来，同时进行查询）时非常高效。
    
- **易错点**：路径压缩时的权值更新 `weight[i] *= weight[parent[i]]` 必须在 `parent[i] = root` 之前利用旧的父节点权值。
