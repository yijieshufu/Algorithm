这道题是经典的**并查集（Disjoint Set Union, DSU）** 应用题。

### 1. 考察算法类型

- **核心算法**：**并查集**。
    
- **理由**：等式 `a==b` 具有传递性、对称性和自反性，这符合图论中“连通分量”或集合中“等价类”的特征。我们需要处理变量之间的连通关系，并验证不连通（不等式）的约束是否被违反。

### 2. 最优解思路

1. **初始化**：创建一个长度为 26 的数组 `p`（代表 'a'-'z'），初始时每个元素的父节点指向自己。
    
2. **处理等式（第一遍历）**：遍历所有 `==` 的方程，将等号两边的变量在并查集中进行 `union`（合并）。这会将所有相等的变量归入同一个集合。
    
3. **校验不等式（第二遍历）**：遍历所有 `!=` 的方程。对于每一对变量，利用 `find` 操作检查它们的根节点是否相同：
    
    - 如果根节点相同，说明它们在之前的等式逻辑中必须相等，这与当前的 `!=` 矛盾，立即返回 `False`。
        
4. **得出结论**：如果所有不等式都校验通过，返回 `True`。

**复杂度分析：**

- **时间复杂度**：$O(N + C \alpha(C))$，其中 $N$ 是方程数量，$C$ 是字符集大小（26），$\alpha$ 是反阿克曼函数（近乎常数）。
    
- **空间复杂度**：$O(C)$，仅需存储长度为 26 的父节点数组。

---

### 3. Python 3.8.6 代码实现（LeetCode 兼容）
```Python
class Solution:
    def equationsPossible(self, equations: list) -> bool:
        parent = list(range(26))
        
        # 并查集查找（路径压缩优化）
        def find(i):
            if parent[i] == i:
                return i
            parent[i] = find(parent[i])
            return parent[i]

        # 第一遍：处理所有等式
        for eq in equations:
            if eq[1] == '=':
                root1 = find(ord(eq[0]) - 97)
                root2 = find(ord(eq[3]) - 97)
                if root1 != root2:
                    parent[root1] = root2

        # 第二遍：校验所有不等式
        for eq in equations:
            if eq[1] == '!':
                if find(ord(eq[0]) - 97) == find(ord(eq[3]) - 97):
                    return False
        return True
```

### 💡 算法教练笔记

- **路径压缩**：代码中 `parent[i] = find(parent[i])` 是极致优化的关键，它能让树的高度趋近于常数。
    
- **为什么分两步？** 因为等式关系是“基础事实”，必须先构建完所有的相等关系网，才能判断不等式是否成立。如果边遍历边处理不等式，会丢失传递性带来的冲突。
    
- **字符转换**：`ord(c) - 97` 是处理小写字母到 0-25 索引转换的标准做法，效率高于字典。
