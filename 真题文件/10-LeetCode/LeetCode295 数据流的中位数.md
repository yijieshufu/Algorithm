[295. 数据流的中位数 - 力扣（LeetCode）](https://leetcode.cn/problems/find-median-from-data-stream/)

这道题是经典的“数据流中位数”问题。要在不断变化的数据中快速找到中位数，最直观的方法是排序，但每次插入都排序效率太低（$O(N \log N)$）。

这套代码采用了**双堆（Two Heaps）** 的策略，将时间复杂度降低到了插入 $O(\log N)$、查找 $O(1)$。

---

## 1. 核心思路：把数据分成两半

我们可以想象把所有的数字按从小到大排成一排，然后从中间切开：

- **左半部分（较小的一半）：** 我们用一个 **大顶堆 (Max-Heap)** 来存。因为我们只需要知道这半部分里“最大的那个数”是多少。
    
- **右半部分（较大的一半）：** 我们用一个 **小顶堆 (Min-Heap)** 来存。因为我们只需要知道这半部分里“最小的那个数”是多少。

### 为什么这样做？

- 如果总数是**奇数**，中位数就是这两个堆中元素较多的那个堆的堆顶。
    
- 如果总数是**偶数**，中位数就是这两个堆顶元素的平均值。

---

## 2. 代码详细注解

Python 的 `heapq` 库默认只提供**小顶堆**。为了实现**大顶堆**，代码巧妙地将数值**取负号**存入，这样数值最大的数取负后就变成了最小的，从而达到了大顶堆的效果。
```Python
from heapq import *

class MedianFinder:
    def __init__(self):
        # A 是小顶堆，保存较大的一半元素
        self.A = [] 
        # B 是大顶堆（存负数），保存较小的一半元素
        self.B = [] 

    def addNum(self, num: int) -> None:
        """
        核心逻辑：保持两个堆的平衡。
        约定：如果总数是奇数，让 A 的元素比 B 多一个。
        """
        # 情况 1：当 A 和 B 元素数量不同时（此时 A 比 B 多 1 个）
        # 目标：向 B 中添加一个元素，使两者平衡。
        if len(self.A) != len(self.B):
            # 先把新数 num 放入 A（小顶堆）
            heappush(self.A, num)
            # 再把 A 中最小的弹出，放入 B（大顶堆）
            # 这样保证了 B 里的数永远比 A 里的数小
            heappush(self.B, -heappop(self.A))
            
        # 情况 2：当 A 和 B 元素数量相同时
        # 目标：向 A 中添加一个元素。
        else:
            # 先把新数 num 放入 B（大顶堆，存负数）
            heappush(self.B, -num)
            # 再把 B 中最大的弹出，放入 A（小顶堆）
            # 这样保证了 A 里的数永远比 B 里的数大
            heappush(self.A, -heappop(self.B))

    def findMedian(self) -> float:
        """
        根据堆的元素个数直接计算中位数
        """
        # 如果数量不等，说明总数是奇数，A 的堆顶就是中位数
        if len(self.A) != len(self.B):
            return float(self.A[0])
        # 如果数量相等，说明总数是偶数，取两个堆顶的平均值
        else:
            # 注意：self.B[0] 是负数，所以 (A[0] - B[0]) 实际上是 A[0] + |B[0]|
            return (self.A[0] - self.B[0]) / 2.0
```

## 3. 关键动作解析：为什么要“先推入再弹出”？

你可能会问：`addNum` 时，为什么不直接根据数值大小决定放进 A 还是 B？

**原因：** 我们不知道新来的 `num` 到底属于“较小的一半”还是“较大的一半”。

- 在 `else` 分支中，我们想往 `A` 加元素。但 `num` 可能很小，直接放进 `A` 会破坏 `A` 存储“较大一半”的规则。
    
- 所以，我们**先把它扔进 B 排个序**，把 B 里最大的那个（也就是最靠近中间的那个数）踢出来给 A。这样 A 拿到的数一定是经过筛选的，保证了数据的有序性。

---

## 4. 复杂度分析

| **操作**         | **时间复杂度**   | **说明**            |
| -------------- | ----------- | ----------------- |
| **addNum**     | $O(\log N)$ | 堆的插入和弹出操作均为对数级别。  |
| **findMedian** | $O(1)$      | 直接访问堆顶元素（数组下标 0）。 |
| **空间复杂度**      | $O(N)$      | 需要存储所有流进来的数字。     |

**总结：** 这种双堆写法非常优雅，它规避了频繁排序的开销，利用堆的特性动态维护了数据的“中间地带”。
