这道题是典型的**逆序对（Inversion Pair）** 问题的变种。在算法竞赛（如蓝桥杯）中，这属于“动态维护区间信息”的范畴。

### 1. 题目考察类型

- **核心算法**：**树状数组 (Binary Indexed Tree, BIT)** 或 **归并排序 (Merge Sort)** 变形。
    
- **辅助技巧**：**离散化 (Discretization)**。由于原数组元素范围可能较大，而我们只关心相对大小，离散化可以将空间复杂度压缩至 $O(n)$。

### 2. 最优解思路

我们采用 **离散化 + 树状数组** 的方案，这种方法逻辑清晰，代码比归并排序更简洁：

1. **离散化**：将 `nums` 中的所有数值去重并排序，映射到 $1 \dots K$ 的排名（Rank）。例如 `[5, 2, 6, 1]` 映射为排名 `[3, 2, 4, 1]`。
    
2. **倒序遍历**：从右向左扫描数组。
    
    - **查询**：在树状数组中查询当前元素排名 $R$ 之前的频率前缀和（即已遍历过的、排名比当前小的元素个数）。
        
    - **更新**：将当前元素的排名 $R$ 在树状数组中的计数值 $+1$。
        
3. **复杂度**：
    
    - 时间复杂度：$O(n \log n)$，排序 $O(n \log n)$，遍历 $n$ 次，每次树状数组操作 $O(\log n)$。
        
    - 空间复杂度：$O(n)$。

---

### 3. Python 3.8.6 代码实现

```Python
class Solution:
    def countSmaller(self, nums: List[int]) -> List[int]:
        if not nums: return []
        
        # 离散化：得到每个数字在有序去重数组中的排名（从1开始）
        rank = {v: i + 1 for i, v in enumerate(sorted(set(nums)))}
        size = len(rank)
        tree = [0] * (size + 1)
        
        # 树状数组更新：单点增加
        def update(i):
            while i <= size:
                tree[i] += 1
                i += i & (-i)
        
        # 树状数组查询：前缀和
        def query(i):
            s = 0
            while i > 0:
                s += tree[i]
                i -= i & (-i)
            return s
            
        ans = []
        # 从右往左遍历，确保查询到的都是“右侧”的元素
        for x in reversed(nums):
            r = rank[x]
            # 查询排名比当前数小的元素累积个数
            ans.append(query(r - 1))
            # 将当前数插入树状数组
            update(r)
            
        return ans[::-1]
```

### 4. 教练笔记（针对蓝桥杯）

- **为什么要逆序遍历？** 因为题目要求“右侧小于当前元素的个数”。如果我们从右往左走，当我们处理到第 $i$ 个数时，树状数组里已经存进去的正好就是 $i$ 后面（右侧）的所有数。
    
- **树状数组 vs 归并排序**：虽然归并排序也能做，但在比赛中树状数组的逻辑更不容易写挂，尤其是处理重复元素时，树状数组只需要查询 `query(r - 1)` 即可完美避开等于当前值的元素。
    
- **性能提醒**：Python 的 `reversed()` 和 `append()` 效率很高，最后的 `[::-1]` 也是 $O(n)$ 的，整体效率在 $10^5$ 数据量下非常稳。
