这道题目是算法竞赛中的经典模型，考察的是**逆序对（Inversion Pair）** 的统计。

### 1. 考察算法类型

- **核心算法**：**分治法（归并排序变形）** 或 **树状数组（Binary Indexed Tree, BIT）**。
    
- **复杂度要求**：数组长度可达 50,000，必须达到 $O(n \log n)$ 的时间复杂度。

### 2. 最优解思路

虽然归并排序是教科书式的解法，但在 Python 中，利用**树状数组 + 离散化**通常代码更简洁，且不容易在递归深度上出问题。

**解题逻辑：**

1. **离散化**：由于原数组的值域可能很大，我们先将数字映射到它们的“排名”（Rank）。
    
2. **反向遍历**：从后往前遍历数组。
    
3. **查询与更新**：
    
    - 对于当前数字，在树状数组中查询比它“排名”小的数字已经出现了多少个，累加到结果中。
        
    - 将当前数字的排名更新到树状数组中（计数加 1）。

---

### 3. Python 3.8.6 代码实现（LeetCode 模式）
```Python
class Solution:
    def reversePairs(self, record: List[int]) -> int:
        if not record: return 0
        
        # 离散化：将值映射为 1 到 m 的排名
        ranks = {v: i + 1 for i, v in enumerate(sorted(list(set(record))))}
        m = len(ranks)
        bit = [0] * (m + 1)
        
        def update(i):
            while i <= m:
                bit[i] += 1
                i += i & (-i)
        
        def query(i):
            s = 0
            while i > 0:
                s += bit[i]
                i -= i & (-i)
            return s
        
        ans = 0
        # 从后往前扫，查询比当前数小的数有多少个
        for x in reversed(record):
            r = ranks[x]
            ans += query(r - 1)
            update(r)
            
        return ans
```

### 4. 为什么这么做？

- **时间复杂度**：离散化排序 $O(n \log n)$，遍历数组 $n$ 次，每次树状数组操作 $O(\log n)$，总计 $O(n \log n)$。
    
- **空间复杂度**：需要辅助数组存储树状数组和映射表，空间复杂度 $O(n)$。
    
- **优化点**：直接使用 Python 内置的 `sorted` 和 `set` 进行离散化效率极高。在 Python 中，通过 `i & (-i)` 快速获取最低位 1（lowbit）是处理前缀和问题的利器。
